# 竞赛中的数学题识别与策略

> Made by Aki.
> 最后更新于 2026.04.30

读完前面 99 篇文章，工具都有了。但在赛场上，面对一道新题——你怎么知道它是数学题？怎么决定用什么工具？投入多少时间？这篇总结在「知识」之上加上「策略」。

---

## 一、快速识别：这是数学题吗？

以下信号出现 $2$ 个以上，大概率是数学题：

| 信号 | 对应板块 |
|------|----------|
| 答案需要对 $10^9+7$ 或 $998244353$ 取模 | 全局（组合数、期望通常要求取模） |
| 输入只有 $1\sim 3$ 个整数，没有数组 | 数论、组合数学 |
| 题干出现「$\gcd$」「素数」「互质」「同余」 | 数论 |
| 题干出现「排列」「组合」「方案数」「至少/恰好」 | 组合数学 |
| 题干出现「期望」「概率」「随机」 | 概率与期望 |
| 题干出现「先手」「后手」「必胜」「必败」 | 博弈论 |
| 题干出现「坐标」「多边形」「凸包」「距离」 | 计算几何 |
| 数据范围中 $n \le 10^9$ 但时间限制 $1$s | 数论（需要 $O(\sqrt{n})$ 或 $O(\log n)$） |
| 数据范围很小（$n \le 20$）但看起来是计数 | 状态压缩 + 组合/容斥 |

## 二、按信号定位工具

| 你看到 | 立刻想到 |
|--------|----------|
| $a^b \bmod m$，$b$ 极大 | 快速幂 + 扩展欧拉定理降幂（#7, #10） |
| $\binom{n}{k} \bmod p$ | 阶乘+逆元 或 Lucas 定理（#14） |
| $ax \equiv b \pmod{m}$ | exgcd → 线性同余方程（#8, #12） |
| 多个 $x \equiv r_i \pmod{m_i}$ | CRT / exCRT（#13） |
| $\gcd$ 求和 | 莫比乌斯反演 + 整除分块（#21, #27） |
| 「恰好 $k$ 个」vs「至少 $k$ 个」 | 二项式反演（#36）|
| 括号匹配 / 出栈序列 | 卡特兰数（#37）|
| 将 $n$ 分成 $k$ 组 | 第二类斯特林数（#38）|
| 等幂求和 $\sum i^k$ | 伯努利数 或 斯特林+下降幂（#41, #39）|
| 状态转移有环 | 高斯消元 或 期望DP自环消去（#67, #75）|
| 所有事件都发生的期望时间 | Min-Max 容斥（#72）|
| Nim 游戏变种 | SG 函数 + mex（#56）|
| 凸包 / 最近点对 / 半平面交 | 计算几何对应算法（#89-#94）|
| 线性递推，数据大到 $10^{18}$ | 矩阵快速幂 或 BM+Kitamasa（#7, #78, #79）|
| 异或最大值 / 第 k 小异或和 | 线性基（#77）|

## 三、时间分配策略

在 $5$ 小时的 ICPC 区域赛中：

| 阶段 | 时间 | 策略 |
|------|------|------|
| 开场 30 分钟 | 签到期 | 先做输入输出简单、数据范围小的题。数学签到题通常是裸的快速幂、GCD、组合数公式 |
| 中期 2 小时 | 中等难度 | 数论分块、莫反、BSGS、CRT 这些有标准模板的中档题 |
| 后期 1.5 小时 | 难题攻坚 | 杜教筛、min25、LGV、exLucas 等 |
| 最后 1 小时 | 查漏补缺 | 对拍已有代码、检查边界和溢出 |

**关键原则**：中档数学题投入 $30$ 分钟没有进展就跳——数学题的特点是「要么会要么不会」，不像模拟题可以逐步 debug。

---

## 四、打表——数学题的重武器

当推导陷入僵局时，直接写暴力跑 $n=1$ 到 $30$，把结果序列扔进 [OEIS](https://oeis.org/) 或者自己观察模式（线性递推？周期？分段？）。ACM 比赛中无需证明，猜对即可。

---

## 五、对拍——数学题的保险丝

数学公式推导容易出错。写两份代码：一份是 $O(n)$ 的暴力（用于小数据验证），一份是 $O(\log n)$ 的公式算法。用脚本对拍 $n=1$ 到 $1000$，确认一致后再提交。

```bash
# 简易对拍脚本
for i in $(seq 1 1000); do
    ./brute $i > out1
    ./solve $i > out2
    diff out1 out2 || break
done
```
