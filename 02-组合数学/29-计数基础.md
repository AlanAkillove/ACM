# 计数基础：加法原理与乘法原理

> Made by Aki.
> 最后更新于 2026.04.30

从本篇开始，我们进入组合数学板块。如果说数论研究的是「整数长什么样」，组合数学研究的就是「怎么数数」——当你面对一个庞大到无法逐个枚举的集合时，如何用数学公式精确算出它的大小。

---

## 一、引言

ACM 竞赛中有一个极其常见的场景：答案是一个巨大的整数，需要对 $10^9+7$ 取模输出。

比如：「$n$ 个不同元素排成一排，有多少种排列方式？」答案 $n!$ 在 $n=20$ 时就超过了 `long long` 的范围。你不会真的写一段代码去生成所有排列然后 `ans++`——你需要一个**公式**，把计数过程从 $O(n!)$ 的枚举压缩到 $O(1)$ 的公式求值。

这就是组合计数的核心任务。而它最底层的两条地基，就是本篇要讲的**加法原理**和**乘法原理**。

---

## 二、前置知识

无特殊前置知识。只需理解基本的整数乘法和阶乘含义。

## 三、核心推导

### 3.1 两条原理——直觉先行

我们先不急着给公式，先看两个生活中最自然的「数数」场景。

**场景一（分类）**：你从北京去上海，飞机有 $5$ 个航班，高铁有 $10$ 趟，自驾有 $1$ 条路线。请问有多少种**不同的交通工具选择**？

你会很自然地说：$5 + 10 + 1 = 16$。你不会说 $5 \times 10 \times 1$，因为这三者是互斥的——你只能选一种交通工具。

这就是**加法原理**：若干互斥方案，总数等于各方案数之和。

**场景二（分步）**：你的密码锁有 $4$ 位，每位可以选 $0\sim9$。请问有多少种密码？

你会说 $10 \times 10 \times 10 \times 10 = 10000$。你不会说 $10+10+10+10$，因为每一步与上一步无关——每一步的选择数是独立乘上去的。

这就是**乘法原理**：若干独立步骤，总数等于各步骤选择数之积。

❓ **加法 vs 乘法，怎么区分？** 一个简单的判据：如果你描述问题时用了「或者」，大概率是加法（分类）；如果用了「然后」，大概率是乘法（分步）。「坐飞机**或者**高铁」= 加法。「先选第一位**然后**选第二位」= 乘法。

---

### 3.2 排列数 $P(n,k)$——从 $n$ 个中**有序地**挑 $k$ 个

> 问题：$5$ 匹马赛跑，冠军、亚军、季军分别是谁？有多少种可能的排名？

**推导**：冠军有 $5$ 种选择。冠军确定后，亚军只剩 $4$ 种选择（同一个人不能既冠军又亚军）。冠亚军都确定后，季军只剩 $3$ 种。

由乘法原理，总数为 $5 \times 4 \times 3 = 60$。

一般化：从 $n$ 个不同元素中有序选出 $k$ 个：

$$
\boxed{P(n, k) = n \times (n-1) \times \cdots \times (n-k+1) = \frac{n!}{(n-k)!}}
$$

❓ **为什么是 $n-k+1$ 而不是 $n-k$？** 我们来数：第 $1$ 个选有 $n$ 种，第 $2$ 个有 $n-1$ 种，...，第 $k$ 个有多少种？前面已经选走了 $k-1$ 个元素，还剩下 $n-(k-1) = n-k+1$ 个可选。所以最后一项是 $n-k+1$。

**验证**：$n=5, k=3$ 时 $P(5,3)=5 \times 4 \times 3 = 60$。如果用公式 $\frac{5!}{(5-3)!} = \frac{120}{2} = 60$。一致。

**特殊情况**：当 $k=n$ 时，$P(n,n) = n!$——这就是**全排列**。

---

### 3.3 组合数 $\binom{n}{k}$——从 $n$ 个中**无序地**挑 $k$ 个

> 问题：$5$ 个人中选 $3$ 人组成一个队伍（队内不分角色），有多少种选法？

注意与赛马排名问题的区别：这里没有「冠军/亚军/季军」的区别——$\{A,B,C\}$ 和 $\{B,A,C\}$ 是同一支队伍。

**推导**：先按有序排列来算：$P(5,3) = 5 \times 4 \times 3 = 60$ 种。但每个队伍（如 $\{A,B,C\}$）在有序排列中被算了 $3! = 6$ 次（ABC、ACB、BAC、BCA、CAB、CBA，都是这三人）。所以真正的组合数为 $60 / 6 = 10$。

一般化：

$$
\boxed{\binom{n}{k} = \frac{P(n,k)}{k!} = \frac{n!}{k!(n-k)!}}
$$

❓ **组合数和排列数的关系可以用一句话说清吗？** 排列 = 先选出 $k$ 个（组合），再把它们排好序（$k!$ 种排列）。所以 $P(n,k) = \binom{n}{k} \times k!$。反过来就是 $\binom{n}{k} = P(n,k) / k!$。

---

### 3.4 Pascal 三角形——组合数的递推结构

把 $\binom{n}{k}$ 按 $n$ 和 $k$ 排列成三角形（$n$ 行，$k$ 列），得到 Pascal 三角形：

```
n=0:           1
n=1:         1   1
n=2:       1   2   1
n=3:     1   3   3   1
n=4:   1   4   6   4   1
n=5: 1   5   10  10  5   1
```

这里有一个著名的递推关系：

$$
\boxed{\binom{n}{k} = \binom{n-1}{k-1} + \binom{n-1}{k}}
$$

**组合解释**：考虑 $n$ 个元素中的第一个元素 $A$。要么 $A$ 被选入组合（则从剩下的 $n-1$ 个中选 $k-1$ 个），要么 $A$ 不被选入（则从剩下的 $n-1$ 个中选 $k$ 个）。两者互斥，由加法原理得和。

这就是为什么 Pascal 三角形中每个数等于它上方两个数之和。

**对称性**：$\binom{n}{k} = \binom{n}{n-k}$。「选入的 $k$ 个」和「排除的 $n-k$ 个」是一一对应的，所以方案数相等。

---

### 3.5 二项式定理——组合数在代数中的面孔

二项式定理将 $\binom{n}{k}$ 解释为代数展开中的系数：

$$
\boxed{(x + y)^n = \sum_{k=0}^{n} \binom{n}{k} x^{n-k} y^k}
$$

**推导**：$(x+y)^n$ 展开时，从 $n$ 个因子中各选 $x$ 或 $y$ 相乘。选了 $k$ 个 $y$（其余 $n-k$ 个选 $x$）的方案数就是 $\binom{n}{k}$，贡献的项是 $x^{n-k} y^k$。所有 $k$ 的方案求和即为左边。

**两个重要推论**：

令 $x=y=1$：$\sum_{k=0}^{n} \binom{n}{k} = 2^n$——即 $n$ 个元素有 $2^n$ 个子集（每个元素选或不选）。

令 $x=1, y=-1$：$\sum_{k=0}^{n} (-1)^k \binom{n}{k} = 0 \;(n \ge 1)$——正负交替的二项式系数和为零。

## 四、代码实现

```cpp
using ll = long long;

// 排列数 P(n, k) — 逐项相乘，避免先算大阶乘后除
ll P(ll n, ll k) {
    ll ans = 1;
    for (ll i = 0; i < k; i++)
        ans *= (n - i);
    return ans;
}

// 组合数 C(n, k) — 用乘法形式 + 对称性优化
ll C(ll n, ll k) {
    if (k > n - k) k = n - k;         // 对称性：取较小的 k 减少循环
    if (k == 0) return 1;
    ll ans = 1;
    for (ll i = 1; i <= k; i++)
        ans = ans * (n - i + 1) / i;  // 边乘边除，防溢出
    return ans;
}
```

**说明**：`ans = ans * (n - i + 1) / i` 中，$i$ 个连续整数的乘积必被 $i!$ 整除，所以中间结果不会出现分数。这是组合数递推 $C(n,k) = C(n,k-1) \cdot (n-k+1) / k$ 的迭代实现。

---

## 五、复杂度与正确性分析

上述代码时间复杂度 $O(k)$，空间 $O(1)$。$n \le 60$ 时结果在 `long long` 范围内（$\binom{60}{30} \approx 1.18 \times 10^{17}$）。对于更大的 $n$，需要模运算——参见数论板块的第 14 篇（组合数取模）。

正确性由 3.2 和 3.3 节的推导保证。Pascal 递推关系在 3.4 节有两套等价证明（组合解释 + 代数展开），互为印证。

## 六、典型应用场景

排列组合是组合数学中最基础的操作。以下场景贯穿整个竞赛组合数学：

1. **全排列**：$n$ 个不同元素排成一排 = $n!$。是所有「有序排列」问题的原型。
2. **组合选择**：$n$ 个中选 $k$ 个 = $\binom{n}{k}$。是所有「无序选择」问题的原型。
3. **二项式系数**：$\binom{n}{k}$ 在展开 $(x+y)^n$ 时作为系数出现。在生成函数和多项式相关题目中无处不在。
4. **子集计数**：$n$ 个元素的子集总数为 $2^n$，大小为 $k$ 的子集数为 $\binom{n}{k}$。

---

## 七、小结

两条原理定义了组合计数的语法：加法原理（分类，用「或」连接）→ 加；乘法原理（分步，用「然后」连接）→ 乘。

排列数 $P(n,k)$ = 有序选 $k$ 个 = $n!/(n-k)!$。组合数 $\binom{n}{k}$ = 无序选 $k$ 个 = $P(n,k)/k! = n!/k!(n-k)!$。Pascal 三角形给出递推关系 $\binom{n}{k} = \binom{n-1}{k-1} + \binom{n-1}{k}$，二项式定理赋予 $\binom{n}{k}$ 以代数系数的身份。
