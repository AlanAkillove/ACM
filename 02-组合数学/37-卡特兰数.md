# 卡特兰数

> Made by Aki.
> 最后更新于 2026.04.30

卡特兰数可能是竞赛中出现频率最高的特殊数列。括号匹配、出栈序列、二叉树计数、凸多边形三角剖分——这些看似毫无关联的问题，答案共享同一个数列：$1, 1, 2, 5, 14, 42, 132, 429, 1430, \dots$

识别出问题是卡特兰数，然后用 $O(n)$ 递推或 $O(1)$ 通项求第 $n$ 项——这比现场推导快得多。

---

## 一、从括号匹配出发——最直观的引出方式

> $n$ 对括号，有多少种合法的括号序列？

$n=1$：`()` —— $1$ 种。

$n=2$：`(())` 和 `()()` —— $2$ 种。

$n=3$：`((()))`, `(()())`, `(())()`, `()(())`, `()()()` —— $5$ 种。

这个数列 $1, 2, 5, 14, 42, 132, \dots$ 就是卡特兰数。但括号匹配只是它的一面——下面会看到它至少有六张面孔。

## 二、闭公式与递推

### 2.1 闭公式

$$
\boxed{C_n = \frac{1}{n+1} \binom{2n}{n}}
$$

这是直接用组合数求第 $n$ 个卡特兰数的公式（$C_0 = 1$）。

**推导思路（Dyck 路径 + 反射原理）**：在 $n \times n$ 的网格中，从 $(0,0)$ 走到 $(n,n)$，只允许向右和向上，且路径不能穿过对角线（$y = x$）。

总路径数：$\binom{2n}{n}$（$2n$ 步中选 $n$ 步向上）。

违规路径数：将第一条违规边之后的路径沿 $y = x+1$ 反射，得到一条从 $(0,0)$ 到 $(n-1, n+1)$ 的路径，数量为 $\binom{2n}{n+1}$（选 $n-1$ 步向右或 $n+1$ 步向上）。

$$
C_n = \binom{2n}{n} - \binom{2n}{n+1} = \frac{1}{n+1} \binom{2n}{n}
$$

### 2.2 递推公式

$$
\boxed{C_{n+1} = \sum_{i=0}^{n} C_i \cdot C_{n-i}}
$$

**组合解释**：考虑第一对括号。它包围了 $i$ 对完整的子括号，外部有 $n-1-i$ 对。两部分都是合法的括号序列，方案数分别为 $C_i$ 和 $C_{n-1-i}$。所有 $i$ 求和即得。

### 2.3 竞赛实用递推

$$
\boxed{C_{n+1} = C_n \cdot \frac{4n+2}{n+2}}
$$

这个从闭公式直接推得：$C_{n+1}/C_n = \frac{4n+2}{n+2}$。在需要逐项递推模素数 $p$ 时非常实用。

❓ **六种模型（括号、出栈、二叉树、Dyck 路径、三角剖分、满二叉树）为什么答案都一样？** 因为它们之间存在一一对应的双射。例如：将左括号看作「入栈」、右括号看作「出栈」，括号匹配序列就对应出栈序列。将括号匹配画成折线图，向上走 = 左括号、向下走 = 右括号，得到的就是 Dyck 路径。这些双射本质上都描述了同一类「平衡约束下的计数问题」——这就是卡特兰数的深层统一性。

## 三、六种经典模型速查

| # | 模型 | 等价描述 | 第 $n$ 个卡特兰数代表 |
|---|------|----------|---------------------|
| 1 | 括号匹配 | $n$ 对合法括号序列 | $C_n$ |
| 2 | 出栈序列 | $n$ 个元素的出栈序列数 | $C_n$ |
| 3 | 二叉树 | $n$ 个结点的不同二叉树 | $C_n$ |
| 4 | Dyck 路径 | $2n$ 步上坡/下坡不穿 $x$ 轴 | $C_n$ |
| 5 | 三角剖分 | $(n+2)$ 边形的三角剖分 | $C_n$ |
| 6 | 满二叉树 | $n$ 个内部结点的满二叉树数 | $C_n$ |

## 四、代码实现

```cpp
// 递推：C[n] = C[n-1] * (4n-2) / (n+1)
vector<ll> catalan(int n, ll p) {
    vector<ll> C(n + 1);
    C[0] = 1;
    for (int i = 1; i <= n; i++)
        C[i] = C[i-1] * (4*i - 2) % p * qpow(i + 1, p - 2, p) % p;
    return C;
}

// 直接公式（需要阶乘+逆元）
ll catalan_direct(int n, ll p) {
    return C(2*n, n) * qpow(n + 1, p - 2, p) % p;
}
```
