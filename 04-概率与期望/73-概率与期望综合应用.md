## 1. 真题综合分析

### 1.1 🏆 CF 1265E - Beautiful Mirrors

**题目**：有 $n$ 面镜子，对镜子 $i$ 提问时有概率 $p_i/100$ 回答"美丽"，概率 $(1 - p_i/100)$ 回答"不美丽"。从镜子 $1$ 开始，每天提问一面镜子：
- 如果回答"美丽"，则第二天问下一面镜子
- 如果回答"不美丽"，则第二天**回到镜子 $1$ 重新开始**

求问到镜子 $n$ 且得到"美丽"回答的期望天数。

**分析**：

这是期望 DP 与概率论的经典结合。设 $E_i$ 为从镜子 $i$ 出发，到达最终状态（求得镜子 $n$ 的"美丽"回答）的期望天数。

对镜子 $i$（$1 \le i < n$）：

$$E_i = 1 + p_i \cdot E_{i+1} + (1-p_i) \cdot E_1$$

对镜子 $n$：

$$E_n = 1 + p_n \cdot 0 + (1-p_n) \cdot E_1 = 1 + (1-p_n) \cdot E_1$$

**解释**：从镜子 $i$ 出发，花 1 天提问。以概率 $p_i$ 得到"美丽"（进入下一个状态），以概率 $1-p_i$ 得到"不美丽"（回到镜子 1）。

这是一个线性方程组，但它的结构很特别——每个方程都涉及 $E_1$。

从 $i = n$ 开始反向代入：

$$E_n = 1 + (1-p_n)E_1$$

$$E_{n-1} = 1 + p_{n-1}E_n + (1-p_{n-1})E_1 = 1 + p_{n-1}[1 + (1-p_n)E_1] + (1-p_{n-1})E_1$$

整理出 $E_i$ 关于 $E_1$ 的线性形式：$E_i = A_i + B_i \cdot E_1$。

然后利用 $E_1$ 自身的方程（因为是同一个人）：

$$E_1 = 1 + p_1E_2 + (1-p_1)E_1$$

代入 $E_2 = A_2 + B_2E_1$：

$$E_1 = 1 + p_1(A_2 + B_2E_1) + (1-p_1)E_1$$

解得 $E_1$ 后即可得到所有 $E_i$。

**更优雅的方法**：

设 $E_1$ 为未知数，递推求出每个 $E_i$ 关于 $E_1$ 的表达式，然后利用 $E_1$ 自身的方程解出 $E_1$。

令 $f_i = E_i - E_1$（偏离 $E_1$ 的相对期望）：

$$E_i = E_1 + f_i$$

代入方程（$i < n$）：

$$E_1 + f_i = 1 + p_i(E_1 + f_{i+1}) + (1-p_i)E_1 = 1 + p_i f_{i+1} + E_1$$

所以：

$$f_i = 1 + p_i f_{i+1}$$

这不需要高斯消元！可以轻松递推。

边界：$i = n$：

$$E_1 + f_n = 1 + (1-p_n)E_1$$

$$f_n = 1 - p_n E_1$$

但 $f_n$ 也可以由递推得到：$f_n = 1 + p_n \cdot 0$（注意 $f_{n+1} = 0$ 因为超出范围）。

所以 $f_n = 1$，代入得 $1 = 1 - p_n E_1$，即 $E_1 = 0$？——不对，我们搞混了边界条件。

重新整理。直接正向递推：

从 $i = 1$ 开始，根据方程：

$$E_i - (1-p_i)E_1 = 1 + p_i E_{i+1}$$

我们可以把所有 $E_i$ 表达成 $E_1$ 的线性形式。令 $E_i = a_i \cdot E_1 + b_i$，则：

$$a_n \cdot E_1 + b_n = 1 + (1-p_n)E_1 \implies a_n = 1-p_n, b_n = 1$$

对于 $i < n$：

$$a_i E_1 + b_i = 1 + p_i(a_{i+1}E_1 + b_{i+1}) + (1-p_i)E_1 = 1 + [p_i a_{i+1} + (1-p_i)]E_1 + p_i b_{i+1}$$

所以：

$$\begin{cases}
a_i = p_i a_{i+1} + (1-p_i) \\
b_i = 1 + p_i b_{i+1}
\end{cases}$$

反向递推求出所有 $a_i, b_i$。然后利用 $E_1$ 自身的方程：

$$E_1 = a_1 E_1 + b_1$$

$$(1 - a_1)E_1 = b_1$$

$$E_1 = \frac{b_1}{1 - a_1}$$

**手算小例子**：

$n = 2, p_1 = 0.5, p_2 = 0.5$

$$a_2 = 1 - 0.5 = 0.5, b_2 = 1$$
$$a_1 = 0.5 \times 0.5 + 0.5 = 0.75, b_1 = 1 + 0.5 \times 1 = 1.5$$
$$E_1 = \frac{1.5}{1 - 0.75} = \frac{1.5}{0.25} = 6$$

所以期望需要 6 天。可以验证：有时反复失败回到起点，期望 6 天是合理的。

❓ **这个解法巧妙在哪里？**

> 它利用了两个关键洞察：(1) 所有 $E_i$ 都可以表示为 $E_1$ 的线性函数；(2) $f_i = E_i - E_1$ 的递推消去了 $E_1$ 项，变成了简单的形式。这比直接解 $n$ 元方程组高效得多（$O(n)$ vs $O(n^3)$）。

**代码实现**：

```cpp
#include <bits/stdc++.h>
using ll = long long;
const int MOD = 998244353;

ll mod_pow(ll a, ll b) {
    ll res = 1;
    while (b) {
        if (b & 1) res = res * a % MOD;
        a = a * a % MOD;
        b >>= 1;
    }
    return res;
}

ll inv(ll x) {
    return mod_pow(x, MOD - 2);
}

int main() {
    int n;
    std::cin >> n;
    
    std::vector<ll> p(n);
    for (int i = 0; i < n; ++i) {
        std::cin >> p[i];
        // p[i] = pi / 100, 在模意义下处理
        p[i] = p[i] * inv(100) % MOD;
    }
    
    // 反向递推 a_i, b_i
    ll a = 1 - p[n-1];  // a_n
    ll b = 1;            // b_n
    a = (a % MOD + MOD) % MOD;
    
    for (int i = n - 2; i >= 0; --i) {
        // a_i = p_i * a_{i+1} + (1 - p_i)
        a = (p[i] * a % MOD + (1 - p[i] + MOD) % MOD) % MOD;
        // b_i = 1 + p_i * b_{i+1}
        b = (1 + p[i] * b % MOD) % MOD;
    }
    
    // E_1 = b / (1 - a)
    ll ans = b * inv((1 - a + MOD) % MOD) % MOD;
    std::cout << ans << std::endl;
    
    return 0;
}
```

### 1.2 综合问题的解题策略

许多竞赛问题需要多种工具的组合。这里总结一些常见的组合模式：

**模式 1：期望 DP + 马尔可夫链 + 高斯消元**

适用于状态图有环、需要求解线性方程组的问题。先分析状态，列写期望方程，再用高斯消元或特殊结构优化。

**模式 2：Min-Max 容斥 + SOS DP**

适用于"所有事件都发生"的期望问题，且子集和可以用 SOS DP 加速（如 $n \le 20$ 的按位或）。

**模式 3：概率生成函数 + 微积分**

适用于求整值随机变量的分布或高阶矩，特别是涉及独立随机变量之和或复合分布时。

**模式 4：鞅 + 停时定理**

适用于看似复杂、难以直接列方程的随机过程。下面重点介绍。

## 2. 鞅与停时定理简介

鞅 (Martingale) 是概率论中一个深刻的概念，它描述了一种"公平博弈"——已知过去所有信息，对未来期望的最佳预测就是当前值。

### 2.1 鞅的定义

一个随机过程 $\{X_n\}_{n \ge 0}$ 称为关于另一个随机过程 $\{Y_n\}$ 的**鞅**，如果：

1. $\mathbb{E}[|X_n|] < \infty$（可积）
2. $\mathbb{E}[X_{n+1} \mid Y_0, Y_1, \dots, Y_n] = X_n$

直观理解：给定过去所有信息，下一步的期望值就是当前值。**公平赌博**就是鞅——输赢期望为零。

**例子**：
- 简单随机游走：$X_n = \sum_{i=1}^n \xi_i$，其中 $\xi_i$ 是独立公平的 $\pm 1$ 随机变量。则 $\{X_n\}$ 是鞅。
- 赌徒的资产在公平赌博中是一个鞅。

### 2.2 停时 (Stopping Time)

**停时** $\tau$ 是一个随机变量（取非负整数值），满足事件 $\{\tau \le n\}$ 仅由 $Y_0, Y_1, \dots, Y_n$ 决定。

直观理解：你决定"什么时候停止"的决定只能基于当前已知信息，不能预知未来。

### 2.3 停时定理 (Optional Stopping Theorem)

**停时定理**：如果 $\tau$ 是一个停时，且满足以下任意一个条件：

1. $\tau$ 几乎必然有界（$\tau \le C$ 几乎必然成立）
2. $\mathbb{E}[\tau] < \infty$ 且 $|X_{n+1} - X_n|$ 有界
3. $\{X_n\}$ 一致可积

则：

$$\mathbb{E}[X_\tau] = \mathbb{E}[X_0]$$

即**停时的期望等于初始值**。

### 2.4 经典应用："ABRACADABRA" 期望打字次数

**问题**：一只猴子随机敲击键盘，每次等概率打出 26 个字母之一。求猴子首次打出"ABRACADABRA"所需次数的期望。

这是一个经典的 WF 级别问题，可以用鞅的停时定理优雅求解。

**解法思路**：

考虑一个"公平赌博"场景：在每个时刻 $t$，新来一个赌徒，押注 1 元赌第 $t$ 个字母是 A（赔率 26:1）。如果赢了，将所有奖金（26 元）押注下一个字母是 B；如果又赢了，继续押注下一个字母，以此类推——直到输掉或完整匹配出"ABRACADABRA"。

每个赌徒的"净收益"过程是一个鞅。

关键观察：当"ABRACADABRA"首次出现时：
- 最后一个赌徒（在正确位置开始下注的）赢了全部 11 轮，获得 $26^{11}$ 元
- 还有一些赌徒部分获胜（他们的模式与目标模式的后缀匹配）

ABRACADABRA 的**部分匹配情况**：
- "ABRA" 是它的前缀，同时也出现在后缀中（后缀 4 个字符匹配前缀 4 个）
- 更具体地，前缀与后缀的匹配长度为：0（空串完全匹配），1（A），4（ABRA），11（整个串）

当模式首次出现时，除了最后那个大赢家，还有在位置偏移为 1 和 4 处开始的赌徒也有部分奖金。

总奖金额 = $26^{11} + 26^4 + 26^1$

因为是公平博弈，初始总投入的期望 = 最终总获利的期望。初始时每个时刻来一个赌徒押注 1 元，到停时 $\tau$ 时总押注为 $\tau$ 元。所以：

$$\mathbb{E}[\tau] = 26^{11} + 26^4 + 26^1$$

**惊人的简洁结果！**

对于一般模式 $S_1S_2\dots S_k$：

$$\mathbb{E}[T] = \sum_{i=1}^k [S_1\dots S_i \text{ 是 } S \text{ 的一个 border}] \cdot 26^i$$

其中 border 表示既是前缀又是后缀的字符串。

这个公式可以推广到任意字母表大小 $m$：

$$\mathbb{E}[T] = \sum_{i: \text{border of length }i} m^i$$

❓ **为什么鞅的方法能得出这么简洁的结果？**

> 鞅停时定理的核心思想是**构造一个公平博弈，使得停时恰好对应某个事件的发生**。在这个问题中，每个新来的赌徒押注 1 元，而系统的总资产是一个鞅。停时定理告诉我们"公平博弈在停时的期望收益为 0"，由此推出生成了 $\mathbb{E}[\tau]$ 的方程。这种"人为构造鞅 + 停时定理"的手法在 WF 级别的难题中极为强大。

### 2.5 "ABRACADABRA" 的代码实现

```cpp
#include <bits/stdc++.h>
using ll = long long;

// 计算模式串的 border（前缀函数 / KMP 的失配数组）
std::vector<int> compute_borders(const std::string& pattern) {
    int m = pattern.size();
    std::vector<int> pi(m, 0);
    for (int i = 1; i < m; ++i) {
        int j = pi[i - 1];
        while (j > 0 && pattern[i] != pattern[j]) {
            j = pi[j - 1];
        }
        if (pattern[i] == pattern[j]) {
            ++j;
        }
        pi[i] = j;
    }
    
    // 提取所有 border 长度
    std::vector<int> borders;
    int len = m;
    while (len > 0) {
        borders.push_back(len);
        len = pi[len - 1];
    }
    // borders 包含 {m, pi[m-1], pi[pi[m-1]-1], ...}
    return borders;
}

// 计算期望打字次数
// 字母表大小为 alphabet_size
ll expected_typing_time(const std::string& pattern, int alphabet_size) {
    auto borders = compute_borders(pattern);
    ll ans = 0;
    ll power = 1;
    // 注意：border 长度是递增的（从短的到长的），这里我们反转
    std::sort(borders.begin(), borders.end());
    for (int len : borders) {
        // 计算 alphabet_size^len
        power = 1;
        for (int i = 0; i < len; ++i) {
            power *= alphabet_size;
        }
        ans += power;
    }
    return ans;
}

int main() {
    std::string pattern = "ABRACADABRA";
    int alphabet = 26;
    
    ll ans = expected_typing_time(pattern, alphabet);
    std::cout << "期望打字次数: " << ans << std::endl;
    // 输出: 26^11 + 26^4 + 26^1
    
    // 验证另一个模式
    std::string pattern2 = "AAA";
    ll ans2 = expected_typing_time(pattern2, alphabet);
    std::cout << "模式 AAA 的期望次数: " << ans2 << std::endl;
    // 输出的 border 有 3 个: 1, 2, 3
    // 期望 = 26^1 + 26^2 + 26^3
    
    return 0;
}
```

### 解题思路速查

面对一道概率/期望题，建议按以下顺序思考：

1. **状态定义**：状态需要包含哪些信息？能否用马尔可夫性简化？
2. **边界条件**：哪些状态是吸收态/目标态/失败态？
3. **列写方程**：
   - 期望问题：$\mathbb{E}[i] = \text{一步代价} + \sum P_{ij} \cdot \mathbb{E}[j]$
   - 概率问题：$P[i] = \sum P_{ij} \cdot P[j]$
4. **求解方法**：
   - DAG：直接拓扑序反向递推（#67、#68）
   - 自环：移项消去（#67）
   - 一般环：高斯消元（#71）
   - 吸收链：标准型 + 基本矩阵（#69）
5. **高级技巧**：
   - 需要分布或高阶矩：PGF（#70）
   - 求"所有事件都发生"：Min-Max 容斥（#72）
   - 复杂公平博弈：鞅 + 停时定理（#73）
   - 复杂随机过程：尝试分解为独立部分 + 期望线性性质（#65）
