> Made by Aki.

## 引言

有一个 $n$ 个顶点的有向图，从顶点 $1$ 出发，每一步等概率选择当前顶点的一条出边移动。问到达顶点 $n$ 的概率是多少？（保证从任意顶点出发最终都能到达 $n$。）

这是一个典型的**概率 DP** 问题。和期望 DP 不同，概率 DP 不关心"期望步数"，而是关心"某个事件发生的概率是多少"。

概率 DP 的核心思想：**利用全概率公式，从边界条件出发，沿着转移的方向递推（或逆向求解）概率**。

---

## 1. 概率 DP 的基本范式

### 1.1 核心方程

概率 DP 的转移方程通常基于**全概率公式**：

设 $P[i]$ 为从状态 $i$ 出发，目标事件发生的概率。从 $i$ 出发，以概率 $p_{i,j}$ 转移到 $j$，则：

$$P[i] = \sum_{j} p_{i,j} \cdot P[j]$$

与期望 DP 对比：
- 期望 DP：$\mathbb{E}[i] = 1 + \sum_j p_{i,j} \cdot \mathbb{E}[j]$
- 概率 DP：$P[i] = \sum_j p_{i,j} \cdot P[j]$

**关键区别**：概率 DP 没有那个"1"（因为不计数步数），而且是直接对概率求和而不是期望。

### 1.2 边界条件

概率 DP 的边界条件通常是已知的：
- $P[目标] = 1$（已经在目标状态）
- $P[吸收失败状态] = 0$（已经确定失败）

---

## 2. 从终态倒推

和期望 DP 类似，概率 DP 也常常从边界条件出发逆向计算。

### 2.1 手算小例子

**题目**：从起点 $S$ 出发，每次抛一枚公平硬币：
- 正面（概率 1/2）：到 $A$
- 反面（概率 1/2）：到 $B$

从 $A$ 出发，再抛一次：
- 正面（概率 1/2）：到 $T$（目标，获胜）
- 反面（概率 1/2）：回到 $S$

从 $B$ 出发：
- 正面（概率 1/2）：回到 $S$
- 反面（概率 1/2）：失败（输掉游戏）

求从 $S$ 出发最终获胜的概率。

**分析**：

设 $P_S, P_A, P_B$ 分别为从 $S, A, B$ 出发获胜的概率。

边界：
- $P_T = 1$（到达目标即获胜）
- $P_{失败} = 0$（失败即输）

转移方程：

$$
\begin{aligned}
P_A &= \frac{1}{2} \cdot P_T + \frac{1}{2} \cdot P_S = \frac{1}{2} \cdot 1 + \frac{1}{2} \cdot P_S = \frac{1}{2} + \frac{1}{2}P_S \\
P_B &= \frac{1}{2} \cdot P_S + \frac{1}{2} \cdot 0 = \frac{1}{2} P_S \\
P_S &= \frac{1}{2} \cdot P_A + \frac{1}{2} \cdot P_B
\end{aligned}
$$

代入消去：

$$
\begin{aligned}
P_S &= \frac{1}{2} \left(\frac{1}{2} + \frac{1}{2}P_S\right) + \frac{1}{2} \left(\frac{1}{2} P_S\right) \\
    &= \frac{1}{4} + \frac{1}{4}P_S + \frac{1}{4}P_S \\
    &= \frac{1}{4} + \frac{1}{2}P_S
\end{aligned}
$$

所以：

$$P_S - \frac{1}{2}P_S = \frac{1}{4}$$

$$P_S = \frac{1}{2}$$

从起点出发，获胜概率是 1/2。

❓ **为什么方程可以写成这种"循环"的形式？**

> 这正是概率 DP 的核心——从当前状态出发，经过一步转移后到达新状态，然后从新状态继续这个过程。全概率公式告诉我们，$P[S]$ 等于所有"一步转移的概率 $\times$ 从新状态获胜的概率"之和。即使出现自环（回到自身）或循环，方程仍然成立，因为这是基于事件条件概率的精确分解。

## 3. DAG 上的概率 DP

如果状态转移图是 DAG（有向无环图），我们可以从目标状态开始，按拓扑序反向计算概率。

### 3.1 DAG 上的概率 DP 框架

```cpp
#include <bits/stdc++.h>
using ll = long long;

// DAG 上的概率 DP
// 图 G 的邻接表表示，从 1 到 n，求到达 n 的概率
double probability_on_dag(int n, const std::vector<std::vector<int>>& G) {
    // dp[i] = 从 i 出发到达 n 的概率
    std::vector<double> dp(n + 1, 0.0);
    dp[n] = 1.0;  // 已经在终点
    
    // 按拓扑序逆序处理（从 n 向 1 遍历）
    // 如果图是 DAG 且节点按拓扑序编号，可以直接从 n-1 到 1
    for (int u = n - 1; u >= 1; --u) {
        if (G[u].empty()) continue;  // 没有出边，无法到达 n
        double sum = 0.0;
        for (int v : G[u]) {
            sum += dp[v];
        }
        dp[u] = sum / G[u].size();  // 等概率选择出边
    }
    
    return dp[1];
}
```

### 3.2 示例：掷硬币至连续正面

**题目**：不断掷一枚公平硬币，问在出现"正反正"（HTH）之前先出现"正正"（HH）的概率。

这是一个经典的**模式匹配问题**，可以用概率 DP 和自动机（KMP 或状态机）来解决。

**状态定义**：

定义基于当前已经匹配的"后缀"长度。我们从空串开始，观察当前末尾匹配了哪个目标模式的前缀，然后根据掷币结果定义转移。

实际上，这种模式竞争问题更适合用**马尔可夫链**或**赌徒破产型**分析，我们会在 #69 中详细讨论吸收马尔可夫链。这里先给出一个更简单的例子。

### 3.3 手算小例子：简单游戏

**题目**：A 和 B 轮流投篮，A 命中率 $p$，B 命中率 $q$。A 先投，谁先命中谁获胜。求 A 获胜的概率。

**分析**：

设 $P_A$ 为 A 先投时 A 获胜的概率。

第一轮：
- A 命中（概率 $p$）：A 获胜
- A 未命中（概率 $1-p$）：轮到 B

B 投时，把状态看作"B 先投时 A 获胜的概率"——但这和"从 B 开始"是不同的状态。我们需要定义 $P_B$ 为 B 先投时，A 获胜的概率。

$$
\begin{aligned}
P_A &= p \cdot 1 + (1-p) \cdot P_B \\
P_B &= q \cdot 0 + (1-q) \cdot P_A
\end{aligned}
$$

解释：B 先投时，如果 B 命中（概率 $q$），A 输（概率 0）；如果 B 没中（概率 $1-q$），回到 A 先投的状态（概率 $P_A$）。

代入消去：

$$P_A = p + (1-p)(1-q)P_A$$

$$P_A - (1-p)(1-q)P_A = p$$

$$P_A \cdot [1 - (1-p)(1-q)] = p$$

$$P_A = \frac{p}{p + q - pq}$$

**验算**：若 $p = q = 0.5$：

$$P_A = \frac{0.5}{0.5 + 0.5 - 0.25} = \frac{0.5}{0.75} = \frac{2}{3}$$

A 先投，获胜概率 2/3，很合理——先手优势明显。

若 $p = 1$：$P_A = 1$（A 百发百中，必胜）。若 $p = 0$：$P_A = 0$（A 完全投不中，必输）。

❓ **这个推导中为什么不考虑"都投不中，无限循环"的情况？**

> 我们在方程中已经包含这个情况了！当 A 和 B 都投不中时，状态回到了 $P_A$，这通过 $(1-p)(1-q)P_A$ 这一项来处理。方程自动处理了无限循环——这就是全概率公式处理无限过程的美妙之处。

## 4. 有环概率 DP

当状态图中有环时，概率 DP 的方程会形成线性方程组。这与期望 DP 中的情况类似。

### 4.1 自环处理

自环的处理和期望 DP 类似：

$$P[i] = p_{i,i} \cdot P[i] + \sum_{j \neq i} p_{i,j} \cdot P[j]$$

$$P[i] \cdot (1 - p_{i,i}) = \sum_{j \neq i} p_{i,j} \cdot P[j]$$

$$P[i] = \frac{\sum_{j \neq i} p_{i,j} \cdot P[j]}{1 - p_{i,i}}$$

### 4.2 一般有向图

对于一般有向图，我们需要对每个状态列出方程，然后解线性方程组。这通常用高斯消元实现。

**通用框架**：

对每个顶点 $i$，设 $P[i]$ 为从 $i$ 出发到达目标 $T$ 的概率。如果从 $i$ 出发有 $deg_i$ 条出边，$v_{i,1}, \dots, v_{i,deg_i}$ 是出边指向的顶点（可能有重复），则：

$$P[i] = \frac{1}{deg_i} \sum_{k=1}^{deg_i} P[v_{i,k}]$$

整理成标准形式：

$$deg_i \cdot P[i] - \sum_{k=1}^{deg_i} P[v_{i,k}] = 0$$

加上边界条件 $P[T] = 1$，得到 $n$ 个方程、$n$ 个未知数，用高斯消元求解。

## 5. 概率 DP 与期望 DP 的区别对比

| 特性 | 期望 DP | 概率 DP |
|------|---------|---------|
| 求值 | 期望步数/得分等 | 事件发生概率 |
| 转移公式 | $\mathbb{E}[i]=1+\sum p_{i,j}\mathbb{E}[j]$ | $P[i]=\sum p_{i,j}P[j]$ |
| 有"1"项 | 是的（计步数） | 没有 |
| 边界 | 目标为 0 | 目标为 1，失败为 0 |
| 求解 | 自环消去/高斯消元/递推 | 自环消去/高斯消元/递推 |
| 适用范围 | "需要多少次""平均得分" | "获胜概率""到达概率" |

很多概率问题同时需要两者——比如"求获胜的期望步数"就需要先用概率 DP 求获胜概率，再用期望 DP（条件期望）求步数。

## 6. 两个进阶例子

### 6.1 掷骰子游戏

**题目**：掷一个公平的六面骰子。如果掷出 6，游戏结束并获胜；如果掷出 1，游戏结束但失败；如果掷出 2-5，继续下一轮。求获胜概率。

**解答**：

这个太简单了——相当于每次有 1/6 获胜、1/6 失败、4/6 继续。

设 $P$ 为获胜概率：

$$P = \frac{1}{6} \cdot 1 + \frac{1}{6} \cdot 0 + \frac{4}{6} \cdot P$$

$$P = \frac{1}{6} + \frac{2}{3} P$$

$$P - \frac{2}{3}P = \frac{1}{6}$$

$$\frac{1}{3}P = \frac{1}{6}$$

$$P = \frac{1}{2}$$

**更直观的理解**：获胜和失败的概率相当——无论你掷多少次，最终以 6 结束 vs 以 1 结束，在公平骰子下各占一半。

### 6.2 随机图中两点的连通概率

**题目**：有一个 $n$ 个节点的图，初始没有边。每次以概率 $p$ 在随机一对顶点之间加一条边（可能重复）。问经过 $m$ 次操作后，顶点 $1$ 和 $n$ 连通的概率。

这个问题的难度取决于 $n$ 和 $m$ 的大小，不过对于小 $n$ 我们可以用状态压缩 DP 和概率 DP 结合来解。

状态可以用节点的连通分量来表示（$n \le 15$ 时可以用并查集编码为整数状态），然后对每个状态计算 $m$ 步后的概率分布。

这实际上是概率 DP 和状态压缩的结合，在竞赛中很常见。

## 7. 代码实现：概率 DP 模板

以下是两个完整代码示例。

### 7.1 投篮游戏（前面手算的例子）

```cpp
#include <bits/stdc++.h>
using ll = long long;

// A 和 B 轮流投篮，A 先投
// p: A 的命中率, q: B 的命中率
// 返回 A 获胜概率
double shooting_game(double p, double q) {
    // 解方程:
    // PA = p + (1-p)*PB
    // PB = (1-q)*PA
    // => PA = p + (1-p)(1-q)*PA
    // => PA * (1 - (1-p)(1-q)) = p
    // => PA = p / (p + q - pq)
    
    double denom = p + q - p * q;
    if (denom == 0) return 0.0;  // p = q = 0
    return p / denom;
}

int main() {
    double p = 0.5, q = 0.5;
    double prob = shooting_game(p, q);
    std::cout << "A先投，A获胜概率: " << prob << std::endl;
    
    // 验证不同参数
    std::cout << "\n不同参数下的结果:" << std::endl;
    std::cout << "p=0.3, q=0.5: " << shooting_game(0.3, 0.5) << std::endl;
    std::cout << "p=0.8, q=0.3: " << shooting_game(0.8, 0.3) << std::endl;
    std::cout << "p=0.6, q=0.6: " << shooting_game(0.6, 0.6) << std::endl;
    
    return 0;
}
```

### 7.2 一般有向图概率 DP（高斯消元版）

当图中有环时，需要用高斯消元求解线性方程组。

```cpp
#include <bits/stdc++.h>
using ll = long long;
using Matrix = std::vector<std::vector<double>>;
using Vector = std::vector<double>;

// 高斯消元求解线性方程组 Ax = b
// A 是 n x n 矩阵，b 是 n 维向量
// 返回 x (n 维向量)
Vector gauss(Matrix A, Vector b) {
    int n = A.size();
    // 构造增广矩阵
    for (int i = 0; i < n; ++i) {
        A[i].push_back(b[i]);
    }
    
    for (int col = 0; col < n; ++col) {
        // 寻找主元
        int pivot = col;
        for (int row = col + 1; row < n; ++row) {
            if (std::abs(A[row][col]) > std::abs(A[pivot][col])) {
                pivot = row;
            }
        }
        std::swap(A[col], A[pivot]);
        
        if (std::abs(A[col][col]) < 1e-10) continue;  // 无解或多解
        
        // 归一化
        double div = A[col][col];
        for (int j = col; j <= n; ++j) {
            A[col][j] /= div;
        }
        
        // 消去其他行
        for (int row = 0; row < n; ++row) {
            if (row == col) continue;
            double factor = A[row][col];
            for (int j = col; j <= n; ++j) {
                A[row][j] -= factor * A[col][j];
            }
        }
    }
    
    Vector x(n);
    for (int i = 0; i < n; ++i) {
        x[i] = A[i][n];
    }
    return x;
}

// 一般有向图上的概率 DP（可能含环）
// n: 顶点数，G: 邻接表
// 返回从每个顶点出发到达终点 n 的概率
Vector probability_on_graph(int n, const std::vector<std::vector<int>>& G) {
    // 方程: P[i] = (1/deg[i]) * sum_{v in G[i]} P[v]
    // 整理: deg[i] * P[i] - sum P[v] = 0
    // 边界: P[n] = 1
    
    Matrix A(n, Vector(n, 0.0));
    Vector b(n, 0.0);
    
    for (int i = 0; i < n; ++i) {
        int deg = G[i].size();
        if (deg == 0) {
            // 没有出边：如果 i != n-1，则到不了终点
            if (i == n - 1) {
                A[i][i] = 1.0;
                b[i] = 1.0;
            } else {
                A[i][i] = 1.0;
                b[i] = 0.0;
            }
        } else if (i == n - 1) {
            // 终点
            A[i][i] = 1.0;
            b[i] = 1.0;
        } else {
            A[i][i] = deg;
            for (int v : G[i]) {
                A[i][v] -= 1.0;  // 注意：移项后系数为负
            }
            b[i] = 0.0;
        }
    }
    
    // 注意：我们的顶点编号是 0-based（0 对应 1）
    // 方程形式: deg * P[i] - sum P[next] = 0
    // 但 deg * P[i] - sum P[next] = 0 把 P[i] 系数放到了对角线上
    
    return gauss(A, b);
}

int main() {
    // 示例图：1->2, 1->3, 2->3, 3->3(自环)
    // 0(1) -> 1(2), 2(3)
    // 1(2) -> 2(3)
    // 2(3) -> 2(3) 自环
    int n = 3;
    std::vector<std::vector<int>> G(n);
    G[0] = {1, 2};  // 从顶点1可以到2或3
    G[1] = {2};     // 从顶点2只能到3
    G[2] = {2};     // 从顶点3只能到自身
    
    Vector P = probability_on_graph(n, G);
    
    std::cout << "从各顶点出发到达终点3的概率:" << std::endl;
    for (int i = 0; i < n; ++i) {
        std::cout << "  顶点 " << (i + 1) << ": " << P[i] << std::endl;
    }
    
    return 0;
}
```

## 8. 竞赛中的概率 DP 题型

### 8.1 回合制游戏

这类问题中，两个或多个玩家轮流操作，每步操作有随机结果，求某个玩家获胜的概率。如前面的投篮游戏。

### 8.2 随机过程的终止概率

给定一个随机过程（如随机游走、马尔可夫链），求进入某个吸收态的概率。如赌徒破产问题。

### 8.3 概率 + 状态压缩

当 $n$ 较小时（$n \le 15$），可以用位运算表示状态子集，在状态空间上做概率 DP。这类问题常和**Min-Max 容斥**（#72）有关。

### 8.4 概率图模型

配合马尔可夫链（#71），处理更复杂的概率转移。

## 10. 小结

概率 DP 和期望 DP 一脉相承——都基于全概率公式，都用状态转移来建模随机过程。区别在于：

- 概率 DP 求的是**某个事件发生的概率**，方程中**没有 "+1"**
- 边界条件通常是 $0$（失败）或 $1$（成功）
- 有自环时通过移项消去，更复杂的环则用高斯消元

关键技巧：

- **从终态倒推**：在 DAG 上很自然地逆序计算
- **正确识别状态**：状态要能捕捉所有影响未来结果的信息（马尔可夫性）
- **方程的一致性**：每个状态必须有一个方程，方程数等于状态数

掌握了概率 DP 和期望 DP，你已经覆盖了竞赛中 80% 的概率题。后续的文章将在这些基础上，引入更强大的工具——马尔可夫链、生成函数、Min-Max 容斥——来处理更复杂的问题。
