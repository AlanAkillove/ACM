# 期望的线性性质

> Made by Aki.
> 最后更新于 2026.04.30

## 引言

有 $n$ 个灯泡，初始全部关闭。每次操作等概率随机选择一个灯泡翻转它的状态（开 $\to$ 关，关 $\to$ 开）。问期望多少次操作后，所有灯泡都打开？

如果你试图直接分析"所有灯泡都打开"这个事件发生的概率分布，会发现状态空间巨大（$2^n$ 种状态），几乎无从下手。但如果我们换一个角度——把问题拆解到每个灯泡上，事情就会变得异常简单。

这就是**期望的线性性质**的魔力。它可能是竞赛概率题中最常用、最重要的一条性质。

---

## 1. 期望的定义

### 1.1 离散随机变量的期望

设 $X$ 是一个离散随机变量，取值为 $x_1, x_2, \dots$，对应的概率为 $p_1, p_2, \dots$，则 $X$ 的**期望 (Expectation)** 为：

$$\mathbb{E}[X] = \sum_{i} x_i \cdot p_i$$

> **直觉**：期望就是"概率加权平均"。如果某个值出现概率大，它对期望的贡献就大。

### 1.2 一个简单例子

掷一枚公平的六面骰子，点数 $X$ 的期望：

$$\mathbb{E}[X] = 1 \cdot \frac{1}{6} + 2 \cdot \frac{1}{6} + 3 \cdot \frac{1}{6} + 4 \cdot \frac{1}{6} + 5 \cdot \frac{1}{6} + 6 \cdot \frac{1}{6} = 3.5$$

3.5 并不是骰子的实际可能点数，但它在平均意义上描述了"大约能掷出多少点"。

❓ **期望值不是实际值，那它有什么用？**

> 期望告诉我们的是**长期平均**。掷一次骰子可能得到 1 也可能得到 6，但掷 10000 次的平均结果会非常接近 3.5。在竞赛中，我们关心的往往是"平均需要多少次""平均得分是多少"这类问题。

## 2. 期望的线性性质

### 2.1 核心定理

对于任意随机变量 $X, Y$（不需要独立！）和任意常数 $a, b$：

$$\mathbb{E}[X + Y] = \mathbb{E}[X] + \mathbb{E}[Y]$$

$$\mathbb{E}[aX + b] = a\mathbb{E}[X] + b$$

**这是概率论中最强大的一条性质**。加法线性不要求变量独立，这意味着我们可以把复杂问题拆解成若干简单部分，分别计算期望再求和。

### 2.2 证明

从定义出发：

$$
\begin{aligned}
\mathbb{E}[X + Y] &= \sum_x \sum_y (x + y) \cdot P(X = x, Y = y) \\
&= \sum_x \sum_y x \cdot P(X = x, Y = y) + \sum_x \sum_y y \cdot P(X = x, Y = y) \\
&= \sum_x x \sum_y P(X = x, Y = y) + \sum_y y \sum_x P(X = x, Y = y) \\
&= \sum_x x \cdot P(X = x) + \sum_y y \cdot P(Y = y) \\
&= \mathbb{E}[X] + \mathbb{E}[Y]
\end{aligned}
$$

注意到证明中**从未要求** $X$ 和 $Y$ 独立——因为我们把所有联合概率加起来了。这就是线性性质不需要独立性的原因。

## 3. 指示器随机变量

### 3.1 定义

**指示器随机变量 (Indicator Random Variable)** 是一种特殊变量，表示某个事件是否发生：

$$
I_A = \begin{cases}
1, & \text{如果事件 } A \text{ 发生}\\
0, & \text{如果事件 } A \text{ 未发生}
\end{cases}
$$

它有极其简洁的期望：

$$\mathbb{E}[I_A] = 1 \cdot P(A) + 0 \cdot P(A^c) = P(A)$$

> 这就是指示器的美妙之处——**期望 = 概率**。

### 3.2 核心技巧：把计数变量拆成指示器

许多计数随机变量 $X$（如"成功的次数""发生的数量"）可以写成若干指示器的和：

$$X = I_1 + I_2 + \dots + I_n$$

由线性性质：

$$\mathbb{E}[X] = \sum_{i=1}^n \mathbb{E}[I_i] = \sum_{i=1}^n P(I_i = 1)$$

**这就是指示器技巧**——把求期望转化为求概率的和。

## 4. 手算小例子

### 例 1：灯泡问题的解答

回到开头的灯泡问题。$n$ 个灯泡，初始关闭，每次随机选一个翻转，求期望多少次后全部打开。

**分析**：

设 $X$ 为全部打开所需次数。直接分析 $X$ 很难。

我们定义 $I_i$ 为"第 $i$ 个灯泡被翻转的次数是奇数"——因为初始关闭，每个灯泡只有在被翻转奇数次时才开启。

等等，这并没有把问题简化太多。这个问题其实更适合用「期望 DP」或"优惠券收集问题"的思路来解决。

我们换一个更简单的例子来展示指示器的威力。

### 例 2：随机排列中的固定点

将 $n$ 个元素随机排列，求有多少个元素在排列后位置不变（即固定点，Fixed Point）的期望数量。

**解答**：

令 $X$ 为固定点的数量。对每个位置 $i$，定义 $I_i$ 为"第 $i$ 个元素在位置 $i$"的指示器：

$$X = \sum_{i=1}^n I_i$$

由于排列是均匀随机的，第 $i$ 个元素恰好到位置 $i$ 的概率是 $\frac{1}{n}$：

$$\mathbb{E}[I_i] = P(\text{元素 } i \text{ 在位置 } i) = \frac{1}{n}$$

所以：

$$\mathbb{E}[X] = \sum_{i=1}^n \frac{1}{n} = 1$$

**惊人的结果**：不管 $n$ 是多少，随机排列中固定点的期望数量总是 1！

> 这里的"固定点"之间并不独立——如果元素 1 在位置 1，那么元素 2 在位置 2 的概率就变成了 $\frac{1}{n-1}$。但我们不需要处理这种依赖关系，因为线性性质根本不要求独立性！

### 例 3：掷骰子到出现所有点数

掷一枚公平的六面骰子，问期望多少次后所有点数都至少出现过一次？

这是经典的**优惠券收集问题 (Coupon Collector Problem)**。用指示器技巧来解决：

**解答**：

设 $X$ 为需要掷的次数。将整个过程分成 $n=6$ 个阶段：
- 阶段 1：收集到第 1 个新点数
- 阶段 2：收集到第 2 个新点数
- ...
- 阶段 6：收集到第 6 个新点数

令 $T_i$ 为"已经有 $i-1$ 个不同点数后，再得到一个新点数所需次数"。则 $X = \sum_{i=1}^6 T_i$。

当我们已经有 $i-1$ 个不同点数时，得到一个新点数的概率是 $p_i = \frac{6-(i-1)}{6} = \frac{7-i}{6}$。

$T_i$ 服从几何分布 $\text{Geom}(p_i)$，因此：

$$\mathbb{E}[T_i] = \frac{1}{p_i} = \frac{6}{7-i}$$

由线性性质：

$$
\mathbb{E}[X] = \sum_{i=1}^6 \frac{6}{7-i} = 6 \left( \frac{1}{6} + \frac{1}{5} + \frac{1}{4} + \frac{1}{3} + \frac{1}{2} + \frac{1}{1} \right) \approx 14.7
$$

所以平均需要约 14.7 次才能掷出所有 6 种点数。

❓ **为什么 $T_i$ 的期望可以直接相加？$T_i$ 之间不是相互依赖的吗？**

> 是的，$T_i$ 之间是依赖的——比如阶段 2 的长度取决于阶段 1 收集到了哪个点数。但期望的线性性质**不要求独立性**，所以 $\mathbb{E}[\sum T_i] = \sum \mathbb{E}[T_i]$ 依然成立。这是线性性质最大的优势。

## 5. 竞赛中的经典套路

### 5.1 分解为独立贡献

这是最常用的模式：将总代价分解为每个元素/每次操作/每个位置的独立贡献，对每个部分计算期望（通常是概率），然后求和。

```cpp
// n 个元素，每个元素有独立的概率 p_i 被选中
// 计算被选中元素数量的期望
#include <bits/stdc++.h>
using ll = long long;

double expected_selected_count(const std::vector<double>& prob) {
    double ans = 0.0;
    for (double p : prob) {
        ans += p;  // 指示器期望 = 概率
    }
    return ans;
}
```

### 5.2 随机排列 / 随机选择

在随机排列或随机子集问题中，计算某个量的期望时，通常利用**对称性**——每个位置/每个元素的概率相同。

### 5.3 线性性 + 全期望公式

当期望无法直接拆成独立指示器时，可以先用全期望公式（条件期望）建立递推，再运用线性性质：

$$\mathbb{E}[X] = \sum_{y} \mathbb{E}[X \mid Y = y] \cdot P(Y = y)$$

这是期望 DP 的基础，将在 #67 中详细展开。

### 5.4 典型应用：随机序列的逆序对期望

**问题**：随机生成一个 $n$ 个元素的排列，求逆序对数的期望。

**解答**：

令 $X$ 为逆序对数。对于每对 $(i, j)$（$i < j$），定义指示器 $I_{i,j}$ 表示"$a_i > a_j$"：

$$X = \sum_{1 \le i < j \le n} I_{i,j}$$

由于排列均匀随机，$a_i$ 和 $a_j$ 谁大的概率各半：

$$\mathbb{E}[I_{i,j}] = P(a_i > a_j) = \frac{1}{2}$$

因此：

$$\mathbb{E}[X] = \binom{n}{2} \cdot \frac{1}{2} = \frac{n(n-1)}{4}$$

## 6. 代码实现：用蒙特卡洛验证

下面是验证随机排列固定点期望为 1 的模拟代码：

```cpp
#include <bits/stdc++.h>
using ll = long long;

int count_fixed_points(const std::vector<int>& perm) {
    int cnt = 0;
    for (int i = 0; i < (int)perm.size(); ++i) {
        if (perm[i] == i + 1) cnt++;
    }
    return cnt;
}

int main() {
    int n = 10;
    int trials = 100000;
    
    std::random_device rd;
    std::mt19937 gen(rd());
    
    std::vector<int> perm(n);
    std::iota(perm.begin(), perm.end(), 1);
    
    double total = 0;
    for (int t = 0; t < trials; ++t) {
        std::shuffle(perm.begin(), perm.end(), gen);
        total += count_fixed_points(perm);
    }
    
    std::cout << "Average fixed points: " << total / trials << std::endl;
    std::cout << "Expected (theory): 1" << std::endl;
    
    return 0;
}
```

运行结果应当非常接近 1。

## 7. 期望的非线性运算

要特别小心的是——**线性性质只对加法和数乘成立**。对于非线性运算：

- $\mathbb{E}[X \cdot Y] \neq \mathbb{E}[X] \cdot \mathbb{E}[Y]$（除非 $X, Y$ 独立）
- $\mathbb{E}[1/X] \neq 1 / \mathbb{E}[X]$
- $\mathbb{E}[X^2] \neq (\mathbb{E}[X])^2$

方差就是一个典型的例子：$\text{Var}(X) = \mathbb{E}[X^2] - (\mathbb{E}[X])^2$，需要我们计算 $\mathbb{E}[X^2]$。

## 9. 小结

本文的核心内容：

- **期望的线性性质** $\mathbb{E}[X+Y] = \mathbb{E}[X] + \mathbb{E}[Y]$ 是概率题最重要的工具，不要求独立性。
- **指示器随机变量**技巧：把计数变量拆成 0-1 指示器之和，将期望转化为概率求和。
- **优惠券收集问题**展示了几何分布和线性性质结合使用的经典分析模式。

掌握了这些工具，你已经可以解决大量竞赛概率题了。接下来我们会看到，期望 DP 和概率 DP 正是建立在这些基础之上的自然延伸——线性性质和全期望公式一起构成了递推方程的基石。
