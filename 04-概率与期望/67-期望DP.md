# 期望DP

> Made by Aki.
> 最后更新于 2026.04.30

## 引言

有一个 $n$ 层的楼梯，你在第 $0$ 层。每次你可以掷一个公平的 $m$ 面骰子，掷出 $k$ 点就走 $k$ 步（向上）。如果向上走 $k$ 步后超过了第 $n$ 层，则停留在第 $n$ 层（到达即结束）。问期望掷多少次骰子才能到达第 $n$ 层？

这种"每步随机前进，问期望步数"的问题，就是**期望 DP** 的典型场景。它的核心思想很直接：对每个状态写出关于期望的方程，利用全期望公式建立递推关系，然后求解。

---

## 1. 期望 DP 的基本范式

### 1.1 全期望公式回顾

在第 64 篇中我们学过全概率公式。对期望也有类似的形式——**全期望公式 (Law of Total Expectation)**：

$$\mathbb{E}[X] = \sum_{y} \mathbb{E}[X \mid Y = y] \cdot P(Y = y)$$

更常用的是**条件期望版本**：将样本空间划分为若干互斥事件 $A_1, A_2, \dots, A_k$：

$$\mathbb{E}[X] = \sum_{i=1}^k \mathbb{E}[X \mid A_i] \cdot P(A_i)$$

### 1.2 DP 设计范式

期望 DP 的一般步骤：

1. **定义状态**：用 $\mathbb{E}[i]$ 表示从状态 $i$ 出发到达目标状态所需的期望步数（或期望得分等）。
2. **列出转移**：从状态 $i$ 出发，所有可能的下一步状态和对应的概率。
3. **建立方程**：利用全期望公式，对每个状态写出方程。
4. **求解方程**：根据依赖关系，可能是顺序递推、高斯消元或自环消去。

## 2. 从简单到复杂

### 2.1 无环情况：DAG 上的期望 DP

如果状态转移构成一个有向无环图（DAG），即不存在环，那么期望 DP 就是简单的递推——从最终状态反向计算到初始状态。

**手算小例子**

**题目**：从 $1$ 到 $n$ 的一条路径，在位置 $i$ 以概率 $\frac{1}{2}$ 前往 $i+1$，以概率 $\frac{1}{2}$ 前往 $i+2$（不能超过 $n$，超出则停在 $n$）。求从 $1$ 到 $n$ 的期望步数。令 $n=5$。

**分析**：

定义 $\mathbb{E}[i]$ 为从位置 $i$ 到 $n$ 的期望步数。边界：$\mathbb{E}[5] = 0$。

$$
\begin{aligned}
\mathbb{E}[4] &= 1 + \frac{1}{2} \cdot \mathbb{E}[5] + \frac{1}{2} \cdot \mathbb{E}[5] = 1 + 0 = 1 \\
\mathbb{E}[3] &= 1 + \frac{1}{2} \cdot \mathbb{E}[4] + \frac{1}{2} \cdot \mathbb{E}[5] = 1 + \frac{1}{2} \cdot 1 + \frac{1}{2} \cdot 0 = 1.5 \\
\mathbb{E}[2] &= 1 + \frac{1}{2} \cdot \mathbb{E}[3] + \frac{1}{2} \cdot \mathbb{E}[4] = 1 + \frac{1}{2} \cdot 1.5 + \frac{1}{2} \cdot 1 = 2.25 \\
\mathbb{E}[1] &= 1 + \frac{1}{2} \cdot \mathbb{E}[2] + \frac{1}{2} \cdot \mathbb{E}[3] = 1 + \frac{1}{2} \cdot 2.25 + \frac{1}{2} \cdot 1.5 = 2.875
\end{aligned}
$$

所以从第 1 层到第 5 层期望约 2.875 步。

这里的关键是每个转移中加上的那个 "1"——代表**当前这一步**本身的计数。如果你在位置 $i$ 做了一次操作（掷骰子），无论结果如何，这一步已经消耗了 1 次操作。

### 2.2 有环情况：自环消去

更常见的情况是——状态转移图中存在环。比如你掷骰子可能原地不动，或者可能后退。

考虑一个最简单的例子：

**问题**：每次掷一枚硬币，如果正面（概率 $p$），游戏结束；如果反面（概率 $q = 1-p$），重新开始（回到同一起点）。求期望掷硬币次数。

**分析**：

令 $\mathbb{E}$ 为从起点到结束的期望次数。

$$
\mathbb{E} = 1 + p \cdot 0 + q \cdot \mathbb{E}
$$

解释：先掷一次（"1"代表这一步），有 $p$ 概率结束（不再需要额外步数），有 $q$ 概率回到起点（还需要 $\mathbb{E}$ 步）。

解这个方程：

$$\mathbb{E} = 1 + q \cdot \mathbb{E}$$

$$\mathbb{E} - q \cdot \mathbb{E} = 1$$

$$\mathbb{E} \cdot (1 - q) = 1$$

$$\mathbb{E} = \frac{1}{1 - q} = \frac{1}{p}$$

这完全符合几何分布的期望——每次以概率 $p$ 成功，期望次数为 $1/p$。所以即使不套用几何分布公式，从期望 DP 的角度也能得到相同结果。

❓ **方程中为什么有个"1"？**

> 这个"1"对应的就是"当前这一步"。无论这一步的结果如何，这一步已经做了，所以它一定会贡献 1 到总步数中。这是期望 DP 里最容易遗漏的地方——每一步操作本身要计 1。

### 2.3 自环消去的一般公式

假设从状态 $i$ 出发：
- 以概率 $p_{i,j}$ 转移到状态 $j$（$j$ 可以是 $i$ 本身）
- 每次转移消耗 1 步
- $\mathbb{E}[i]$ 为从 $i$ 到目标的期望步数

方程：

$$\mathbb{E}[i] = 1 + \sum_{j} p_{i,j} \cdot \mathbb{E}[j]$$

如果 $p_{i,i} > 0$（有自环），把自环项单独写出：

$$\mathbb{E}[i] = 1 + p_{i,i} \cdot \mathbb{E}[i] + \sum_{j \neq i} p_{i,j} \cdot \mathbb{E}[j]$$

整理：

$$\mathbb{E}[i] \cdot (1 - p_{i,i}) = 1 + \sum_{j \neq i} p_{i,j} \cdot \mathbb{E}[j]$$

$$\mathbb{E}[i] = \frac{1 + \sum_{j \neq i} p_{i,j} \cdot \mathbb{E}[j]}{1 - p_{i,i}}$$

这就是自环消去——自环部分通过代数移项转化成了分母 $1 - p_{i,i}$。

## 3. 台阶问题的完整解答

现在回到开篇的问题：$n$ 层楼梯，每次掷 $m$ 面骰子，超出则停在 $n$。

**严格叙述**：从第 0 层出发，每次掷一个 $m$ 面公平骰子，掷出点数 $k \in \{1,\dots,m\}$ 就走 $k$ 步。如果 $i + k \ge n$，则到达第 $n$ 层。求从 0 到 $n$ 的期望步数。

**状态定义**：$\mathbb{E}[i]$ 为从第 $i$ 层出发到第 $n$ 层的期望步数。

**边界**：$\mathbb{E}[n] = 0$。

**转移方程**：

$$\mathbb{E}[i] = 1 + \sum_{k=1}^m \frac{1}{m} \cdot \min(\mathbb{E}[i+k], \mathbb{E}[n]), \quad \text{其中 } i+k \text{ 超过 } n \text{ 时取 } \mathbb{E}[n]=0$$

但这样写有问题——当 $i+k > n$ 时，$i+k$ 不是状态，因为直接到达终点了。更精确地：

$$
\mathbb{E}[i] = 1 + \frac{1}{m} \sum_{k=1}^m \begin{cases}
\mathbb{E}[i+k], & i+k \le n-1 \\
0, & i+k \ge n
\end{cases}
$$

**计算顺序**：由于骰子只向前走，依赖关系是从 $n-1$ 向 $0$ 递推的，没有环。这是 DAG 上的 DP。

```cpp
#include <bits/stdc++.h>
using ll = long long;

// n: 总层数, m: 骰子面数
double solve_stairs(int n, int m) {
    std::vector<double> E(n + 1, 0.0);
    // E[n] = 0 (边界)
    
    for (int i = n - 1; i >= 0; --i) {
        double sum = 0.0;
        for (int k = 1; k <= m; ++k) {
            int nxt = i + k;
            if (nxt >= n) {
                sum += 0;  // 到达终点
            } else {
                sum += E[nxt];
            }
        }
        E[i] = 1.0 + sum / m;
    }
    
    return E[0];
}

int main() {
    int n = 10, m = 6;
    double result = solve_stairs(n, m);
    
    std::cout << "楼梯层数: " << n << std::endl;
    std::cout << "骰子面数: " << m << std::endl;
    std::cout << "期望步数: " << result << std::endl;
    
    return 0;
}
```

## 4. 无穷级数处理

当状态空间无限时（比如问题没有明确的上界），我们需要处理无穷级数。

### 4.1 问题示例

**题目**：从 0 开始，每次以概率 $\frac{1}{2}$ 加 1，以概率 $\frac{1}{2}$ 加 2。问总和首次超过 0（即第一次操作后必然结束了？）...让我们改一个更有趣的：

从 0 开始，每次以概率 $p$ 加 1，以概率 $1-p$ 回到 0。求总和首次达到 $N$ 的期望步数。

### 4.2 思路分析

定义 $\mathbb{E}[i]$ 为从当前总和 $i$ 出发到 $N$ 的期望步数。

$$\mathbb{E}[i] = 1 + p \cdot \mathbb{E}[i+1] + (1-p) \cdot \mathbb{E}[0]$$

边界：$\mathbb{E}[N] = 0$。

这里的关键问题是——$\mathbb{E}[0]$ 出现在了 $\mathbb{E}[1]$ 的方程中，$\mathbb{E}[1]$ 的方程依赖 $\mathbb{E}[0]$，形成了循环依赖。

解决方法是**标准的有环期望 DP 处理**——把所有方程写成一个线性方程组，用高斯消元求解。对于这种特殊形式，其实可以递推求出 $\mathbb{E}[i]$ 关于 $\mathbb{E}[0]$ 的表达式，然后代入边界条件解出 $\mathbb{E}[0]$。

稍后我们会看到更一般的处理方法——在 #71 中，我们会用高斯消元系统地解决这类带环问题。

## 5. 竞赛经典例题：吃糖果问题

**题目**：有 $n$ 种不同类型的糖果，每次等概率随机得到一种。求每种糖果至少吃到一次的期望次数。（优惠券收集问题，我们在 #65 中用指示器技巧处理过。）

现在用期望 DP 来解：

令 $\mathbb{E}[i]$ 为已经集齐 $i$ 种不同糖果后，到集齐全部 $n$ 种的期望步数。

从状态 $i$ 出发：
- 以概率 $\frac{i}{n}$ 得到已有种类——留在状态 $i$
- 以概率 $\frac{n-i}{n}$ 得到新种类——前往状态 $i+1$

$$\mathbb{E}[i] = 1 + \frac{i}{n} \cdot \mathbb{E}[i] + \frac{n-i}{n} \cdot \mathbb{E}[i+1]$$

这种形式有自环（状态 $i$ 转移到自身）。用自环消去：

$$\mathbb{E}[i] \cdot \left(1 - \frac{i}{n}\right) = 1 + \frac{n-i}{n} \cdot \mathbb{E}[i+1]$$

$$\mathbb{E}[i] \cdot \frac{n-i}{n} = 1 + \frac{n-i}{n} \cdot \mathbb{E}[i+1]$$

$$\mathbb{E}[i] = \frac{n}{n-i} + \mathbb{E}[i+1]$$

边界 $\mathbb{E}[n] = 0$，所以：

$$\mathbb{E}[i] = \sum_{k=i}^{n-1} \frac{n}{n-k} = n \sum_{k=1}^{n-i} \frac{1}{k}$$

特别地，$\mathbb{E}[0] = n \sum_{k=1}^n \frac{1}{k}$，和 #65 的结果完全一致。

❓ **优惠券收集问题用期望 DP 和用指示器技巧得到的答案一样，哪种更好？**

> 在这个问题上两种方法等价。指示器技巧更简洁，期望 DP 更"通用"——当问题变得更复杂时，期望 DP 框架更容易扩展（比如不等概率、有费用限制等）。

## 6. 更复杂的期望 DP：用高斯消元

下面是一个简单的带环期望 DP 示例。我们会在 #71 中详细展开。

**题目**：一个 $1 \times n$ 的棋盘，在位置 $i$ 时：
- 以概率 $p$ 跳到 $i+1$（若 $i=n$ 则结束）
- 以概率 $q$ 跳到 $i-1$（若 $i=1$ 则保持在 1）
- 以概率 $r$ 保持在 $i$

其中 $p+q+r=1$。求从位置 $1$ 到位置 $n$ 的期望步数。

这是一个一维随机游走型问题，状态转移有环。我们需要列出所有 $\mathbb{E}[i]$ 的方程，然后统一求解。

我们会在 #71 中详细讲解如何处理这种带环的期望方程。

## 7. 期望 DP 的技巧总结

### 7.1 加 1 法则

每次转移方程中要加 1（这一步本身的计数）。这是最常见的错误来源。

### 7.2 从后往前推

期望 DP 通常从目标状态逆向计算到初始状态，因为边界条件在目标状态。

### 7.3 自环消去

当方程某侧出现 $\mathbb{E}[i]$ 本身时，移项合并、除以系数。这是处理自环的标准方法。

### 7.4 全期望公式的正确使用

$$\mathbb{E}[\text{从当前到目标}] = 1 + \sum_{\text{所有可能转移}} P(\text{转移}) \cdot \mathbb{E}[\text{从新状态到目标}]$$

这个公式是期望 DP 的核心范式。

## 8. 代码实现：通用爬楼梯

```cpp
#include <bits/stdc++.h>
using ll = long long;

// 通用爬楼梯问题
// n: 总层数, 每次可以走 1..m 步
// prob[k]: 走 k 步的概率 (k=1..m)
double solve_stairs_general(int n, int m, const std::vector<double>& prob) {
    std::vector<double> E(n + 1, 0.0);
    
    for (int i = n - 1; i >= 0; --i) {
        double sum = 0.0;
        for (int k = 1; k <= m; ++k) {
            int nxt = std::min(i + k, n);
            if (nxt == n) {
                sum += 0.0;  // 到达终点
            } else {
                sum += prob[k] * E[nxt];
            }
        }
        E[i] = 1.0 + sum;
    }
    
    return E[0];
}

// 带自环的简单模型：每次以 p 前进, 1-p 留在原地
double solve_with_self_loop(int n) {
    double p = 0.6;
    std::vector<double> E(n + 1, 0.0);
    
    for (int i = n - 1; i >= 0; --i) {
        // E[i] = 1 + p*E[i+1] + (1-p)*E[i]
        // 化简：E[i] * p = 1 + p*E[i+1]
        // E[i] = 1/p + E[i+1]
        E[i] = 1.0 / p + E[i + 1];
    }
    
    return E[0];
}

int main() {
    int n = 10;
    
    // 标准骰子
    int m = 6;
    std::vector<double> prob(m + 1, 1.0 / m);
    double result1 = solve_stairs_general(n, m, prob);
    std::cout << "标准骰子爬" << n << "层楼: " << result1 << std::endl;
    
    // 带自环
    double result2 = solve_with_self_loop(n);
    std::cout << "带自环(p=0.6)爬" << n << "层楼: " << result2 << std::endl;
    
    return 0;
}
```

## 10. 小结

期望 DP 的核心思想其实很简单——用全期望公式把"从当前到目标"的期望表示为"1 + 转移概率加权后的下一步期望"。

关键要点：

- **加 1**：每一步操作本身要计入
- **逆向递推**：从目标状态往回推
- **自环消去**：出现自环时通过代数移项解决
- **方程求解**：复杂情况需要高斯消元（#71 详述）

期望 DP 和普通 DP 的最大区别在于：普通 DP 求的是确定的最优值或方案数，而期望 DP 求的是统计平均值。形式上，期望 DP 的转移方程比普通 DP 多了一个"概率加权平均"的步骤。

掌握了期望 DP，你已经可以解决绝大多数竞赛期望题了。下一篇我们将讨论它的"表亲"——概率 DP。
