# 扫描线算法

> Made by Aki.
> 最后更新于 2026.04.30

## 引言

给你 $n$ 个矩形（边与坐标轴平行），求它们覆盖的总面积。$n \le 10^5$，坐标范围 $10^9$。

暴力做法是用二维数组标记每个格子——不可行，坐标太大了。更优雅的暴力是对所有矩形的 $x$ 坐标离散化，然后在每个 $x$ 区间上计算 $y$ 方向的覆盖长度。这是 $O(n^2)$ 的，对 $n=10^5$ 也扛不住。

**扫描线（Sweep Line）** 算法把这个过程优化到 $O(n\log n)$。它的核心思想是：用一条竖直的线从左到右扫过平面，用线段树维护当前 $y$ 方向的覆盖情况。扫描线算法是计算几何和数据结构结合的经典案例。

## 前置知识

- 线段树的**区间覆盖**（不需要 pushdown 的懒标记写法）
- 离散化思想

## 1. 扫描线思想

想象一条竖直线从最左边开始，缓缓向右移动。当它扫过矩形时，矩形覆盖的区域在 $y$ 方向上的长度会发生变化——变化的时机只发生在矩形的**左边界**和**右边界**。

我们把矩形的左右边界看作"事件"：
- 遇到左边界：在 $y$ 区间 $[y_1, y_2]$ 上"增加"一个覆盖
- 遇到右边界：在 $y$ 区间 $[y_1, y_2]$ 上"减少"一个覆盖

每两个相邻事件之间的 $x$ 区间内，$y$ 方向上的覆盖长度 $L$ 是常数。这一段的面积增量就是 $L \times \Delta x$。

```
矩形的左右边界（事件点）：
   |  |     |     |  |
   x1 x2    x3    x4 x5
   
相邻事件间面积 = 当前y覆盖长度 × (x_{i+1} - x_i)
总面积 = 所有相邻事件间的面积之和
```

### 关键问题

我们需要一种数据结构来维护 $y$ 方向上的覆盖长度，支持：
1. 区间加 1（遇到左边界）
2. 区间减 1（遇到右边界）
3. 查询当前被覆盖的总长度

这就是线段树的经典应用——区间加/减 + 查询全局覆盖长度。

## 2. 线段树优化：$cnt$ + $len$，无需 pushdown

对于区间覆盖问题，有一个非常巧妙的线段树写法：每个节点维护两个值：
- `cnt`：该区间被完整覆盖的次数（不是懒标记，是精确值）
- `len`：该区间内被覆盖的总长度

当 `cnt > 0` 时，整个区间被完全覆盖，`len = 区间长度`。
当 `cnt == 0` 时，`len = 左右子节点的 len 之和`。

**关键**：这种写法不需要 pushdown！因为区间加减是对称的，加了多少次就减多少次，`cnt` 永远不会"往下传"——它只在当前节点精确记录覆盖次数。

> ❓ 为什么不需要 pushdown？
>
> 对于矩形面积并的"区间加1"和"区间减1"操作，每次加和减总是成对出现（矩形左边界加1，右边界减1）。当一个区间的 cnt 从 1 减到 0，说明这个区间恰好不再被覆盖——不需要把下方子节点的 cnt 清空，因为只要该区间 cnt > 0，查询 len 时直接返回整段长度，不需要看子节点。

### 数据结构

```cpp
struct SegmentTree {
    struct Node {
        int cnt;    // 被完整覆盖的次数
        double len; // 覆盖长度
    };
    vector<Node> tr;
    vector<double> ys;  // y 坐标离散化
    
    SegmentTree(int n) {
        tr.resize(n * 4);
    }
    
    void build(int u, int l, int r) {
        tr[u].cnt = 0;
        tr[u].len = 0;
        if (l == r) return;
        int mid = (l + r) / 2;
        build(u * 2, l, mid);
        build(u * 2 + 1, mid + 1, r);
    }
    
    void pushUp(int u, int l, int r) {
        if (tr[u].cnt > 0) {
            tr[u].len = ys[r + 1] - ys[l];  // 完全覆盖
        } else if (l == r) {
            tr[u].len = 0;
        } else {
            tr[u].len = tr[u * 2].len + tr[u * 2 + 1].len;
        }
    }
    
    void update(int u, int l, int r, int ql, int qr, int val) {
        if (ql <= l && r <= qr) {
            tr[u].cnt += val;
            pushUp(u, l, r);
            return;
        }
        int mid = (l + r) / 2;
        if (ql <= mid) update(u * 2, l, mid, ql, qr, val);
        if (qr > mid) update(u * 2 + 1, mid + 1, r, ql, qr, val);
        pushUp(u, l, r);
    }
};
```

注意 `pushUp` 中 `ys[r+1] - ys[l]` 的写法——这里 `ys` 是离散化后的 $y$ 坐标数组，区间的长度是离散化后相邻 $y$ 坐标的差。线段树节点 `[l, r]` 对应的是 `ys[l]` 到 `ys[r+1]` 这个区间。

## 3. 矩形面积并的完整实现

```cpp
#include <bits/stdc++.h>
using namespace std;
const double eps = 1e-9;

int sgn(double x) {
    if (fabs(x) < eps) return 0;
    if (x > 0) return 1;
    return -1;
}

struct Event {
    double x, y1, y2;
    int type;  // 1 = 入边（左边界），-1 = 出边（右边界）
    bool operator<(const Event &other) const {
        if (sgn(x - other.x) != 0) return x < other.x;
        return type > other.type;  // 先处理入边
    }
};

// 线段树（同上）
struct SegTree {
    struct Node { int cnt; double len; };
    vector<Node> tr;
    vector<double> ys;
    
    SegTree(const vector<double> &y) : ys(y) {
        tr.resize(y.size() * 4);
        build(1, 0, y.size() - 2);
    }
    
    void build(int u, int l, int r) {
        tr[u].cnt = tr[u].len = 0;
        if (l == r) return;
        int mid = (l + r) / 2;
        build(u * 2, l, mid);
        build(u * 2 + 1, mid + 1, r);
    }
    
    void pushUp(int u, int l, int r) {
        if (tr[u].cnt > 0) {
            tr[u].len = ys[r + 1] - ys[l];
        } else if (l == r) {
            tr[u].len = 0;
        } else {
            tr[u].len = tr[u * 2].len + tr[u * 2 + 1].len;
        }
    }
    
    void update(int u, int l, int r, int ql, int qr, int val) {
        if (ql <= l && r <= qr) {
            tr[u].cnt += val;
            pushUp(u, l, r);
            return;
        }
        int mid = (l + r) / 2;
        if (ql <= mid) update(u * 2, l, mid, ql, qr, val);
        if (qr > mid) update(u * 2 + 1, mid + 1, r, ql, qr, val);
        pushUp(u, l, r);
    }
};

double unionArea(vector<pair<Point, Point>> rects) {
    vector<Event> events;
    vector<double> ys;
    
    for (const auto &rect : rects) {
        double x1 = rect.first.x, y1 = rect.first.y;
        double x2 = rect.second.x, y2 = rect.second.y;
        events.push_back({x1, y1, y2, 1});
        events.push_back({x2, y1, y2, -1});
        ys.push_back(y1);
        ys.push_back(y2);
    }
    
    // 离散化 y
    sort(ys.begin(), ys.end());
    ys.erase(unique(ys.begin(), ys.end(), [](double a, double b) {
        return sgn(a - b) == 0;
    }), ys.end());
    
    // 排序事件
    sort(events.begin(), events.end());
    
    SegTree seg(ys);
    double ans = 0;
    
    for (size_t i = 0; i < events.size(); i++) {
        if (i > 0) {
            ans += seg.tr[1].len * (events[i].x - events[i - 1].x);
        }
        int l = lower_bound(ys.begin(), ys.end(), events[i].y1) - ys.begin();
        int r = lower_bound(ys.begin(), ys.end(), events[i].y2) - ys.begin() - 1;
        seg.update(1, 0, ys.size() - 2, l, r, events[i].type);
    }
    
    return ans;
}
```

### 手算例子

两个矩形：$R_1(0,0)-(2,2)$，$R_2(1,1)-(3,3)$。

事件列表（按 $x$ 排序）：
1. $x=0$，$y\in[0,2]$，type=1（入）
2. $x=1$，$y\in[1,3]$，type=1（入）
3. $x=2$，$y\in[0,2]$，type=-1（出）
4. $x=3$，$y\in[1,3]$，type=-1（出）

$y$ 离散化：$[0, 1, 2, 3]$，对应区间索引 $[0,2]$。

处理过程：
1. $x=0$：更新 $y\in[0,2]$，cnt+1。当前覆盖长度 = 2。ans += 0（第一个事件不计算）
2. $x=1$：$\Delta x=1$，ans += $2 \times 1 = 2$。更新 $y\in[1,3]$，cnt+1。此时 $[0,1]$ 区间 cnt=1，$[1,2]$ 区间 cnt=2，覆盖长度 = 3。
3. $x=2$：$\Delta x=1$，ans += $3 \times 1 = 5$。更新 $y\in[0,2]$，cnt-1。此时 $[0,1]$ 区间 cnt=0，$[1,2]$ 区间 cnt=1，覆盖长度 = 2（$[1,3]$ 部分）。
4. $x=3$：$\Delta x=1$，ans += $2 \times 1 = 7$。更新 $y\in[1,3]$，cnt-1。覆盖长度 = 0。

最后 ans = 7。验证：两个 $2\times2$ 正方形重叠 $1\times1$，总面积 $=4+4-1=7$。正确！

## 4. 矩形周长并

求并集图形的周长。周长由两部分组成：

- **横边**：扫描线在 $x$ 方向移动时，$y$ 覆盖长度的变化量
- **竖边**：在事件点处，$y$ 方向覆盖的总长度

### 竖边的计算

在每个事件位置，竖边长度就是当前 $y$ 覆盖长度的变化量。更准确地说，竖边长度的增量等于 $|当前覆盖长度 - 上次覆盖长度|$。

### 横边的计算

横边需要额外维护区间被覆盖的"段数"。比如 $y$ 方向覆盖了 $[0,1]$ 和 $[2,3]$（两段），那横边就有 4 条（每段上下各一条）。

```cpp
struct Node {
    int cnt;      // 完整覆盖次数
    double len;   // 覆盖总长度
    int segs;     // 被覆盖的段数（不连续区间的个数）
    bool lc, rc;  // 左端点是否被覆盖，右端点是否被覆盖
};

void pushUp(int u, int l, int r) {
    if (tr[u].cnt > 0) {
        tr[u].len = ys[r + 1] - ys[l];
        tr[u].lc = tr[u].rc = true;
        tr[u].segs = 1;
    } else if (l == r) {
        tr[u].len = 0;
        tr[u].lc = tr[u].rc = false;
        tr[u].segs = 0;
    } else {
        tr[u].len = tr[u * 2].len + tr[u * 2 + 1].len;
        tr[u].lc = tr[u * 2].lc;
        tr[u].rc = tr[u * 2 + 1].rc;
        tr[u].segs = tr[u * 2].segs + tr[u * 2 + 1].segs;
        if (tr[u * 2].rc && tr[u * 2 + 1].lc)
            tr[u].segs--;  // 中间连续，段数合并
    }
}
```

横边的长度就是 $segs \times 2 \times \Delta x$（每个覆盖段贡献上下两条横边）。

> ❓ 为什么横边是 $segs \times 2 \times \Delta x$？
>
> 在相邻事件之间，$y$ 方向的覆盖情况不变。每个被覆盖的 $y$ 区间顶部和底部各有一条水平边，穿过这段 $\Delta x$。如果 $y$ 方向被分成 3 段，那就有 6 条水平边。

完整的周长：

```cpp
double perimeter(vector<pair<Point, Point>> rects) {
    // 同面积并的扫描线 + 事件处理
    // ... 事件排序、y离散化等同上 ...
    
    double ans = 0;
    double lastLen = 0;
    int lastSegs = 0;
    
    for (size_t i = 0; i < events.size(); i++) {
        if (i > 0) {
            double dx = events[i].x - events[i - 1].x;
            // 横边：前一个状态的段数 × 2 × dx
            ans += lastSegs * 2 * dx;
            // 竖边：|当前覆盖长度 - 上次覆盖长度|
            ans += fabs(seg.tr[1].len - lastLen);
        }
        
        // 更新覆盖
        int l = lower_bound(ys.begin(), ys.end(), events[i].y1) - ys.begin();
        int r = lower_bound(ys.begin(), ys.end(), events[i].y2) - ys.begin() - 1;
        seg.update(1, 0, ys.size() - 2, l, r, events[i].type);
        
        lastLen = seg.tr[1].len;
        lastSegs = seg.tr[1].segs;
    }
    // 最后还要加上最右侧的竖边
    ans += seg.tr[1].len;
    
    return ans;
}
```

> ❓ 最后为什么还要加一次 seg.tr[1].len？
>
> 每个矩形最右侧也有竖边，它们在事件处理的 $\Delta x$ 计算中没被覆盖到（$\Delta x$ 是到上一个事件的距离）。实际上应该在循环结束后加上最终状态的 $y$ 覆盖长度（即最右侧的竖边）。但更稳妥的做法是在事件列表末尾加一个虚拟事件来触发最后一次竖边计算。

## 5. 扫描线算法的更多应用

扫描线算法可以解决许多其他几何问题：
- **矩形并集的轮廓**：结合面积和周长
- **平面点集的最近点对**：另一种解法（但分治法更常用）
- **二维区间查询**：离线处理
- **Voronoi 图的构建**：扫描线算法（Fortune 算法）

## 小结

扫描线算法把二维问题转化为一维数据结构问题：

- **事件**：矩形的左右边界，标记入/出类型
- **排序**：事件按 $x$ 排序
- **线段树**：维护 $y$ 方向的覆盖情况
  - 面积并：维护覆盖长度 $len$
  - 周长并：额外维护段数 $segs$ 和边界标记 $lc, rc$
- **关键技巧**：$cnt$ + $len$ 写法不需要 pushdown
