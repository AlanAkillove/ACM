## 引言

二维计算几何学完了，如果题目升级到三维呢？三维空间中的几何问题比二维复杂一个维度——不仅有更多的边界情况，还有更复杂的数学工具。

三维计算几何在竞赛中出现的频率比二维低，但一旦出现，就是"区分题"。三维凸包、三维最近点对、三维旋转卡壳——这些都是 ACM 中较难的题目。

这一篇我们从三维几何的基础开始，逐步到三维凸包的增量法实现。

## 1. 三维点与向量

```cpp
struct Point3 {
    double x, y, z;
    Point3(double x = 0, double y = 0, double z = 0) : x(x), y(y), z(z) {}
};
using Vector3 = Point3;

Point3 operator+(const Point3 &a, const Point3 &b) { return Point3(a.x + b.x, a.y + b.y, a.z + b.z); }
Point3 operator-(const Point3 &a, const Point3 &b) { return Point3(a.x - b.x, a.y - b.y, a.z - b.z); }
Point3 operator*(const Point3 &a, double t) { return Point3(a.x * t, a.y * t, a.z * t); }

double dot(const Vector3 &a, const Vector3 &b) {
    return a.x * b.x + a.y * b.y + a.z * b.z;
}

double length(const Vector3 &a) {
    return sqrt(dot(a, a));
}
```

## 2. 三维叉积与混合积

三维叉积的结果是一个**向量**，垂直于两个输入向量：

$$\vec{a} \times \vec{b} = (a_y b_z - a_z b_y,\; a_z b_x - a_x b_z,\; a_x b_y - a_y b_x)$$

叉积的长度 $|\vec{a} \times \vec{b}| = |\vec{a}||\vec{b}|\sin\theta$，等于以 $\vec{a}, \vec{b}$ 为边的平行四边形面积。

三维叉积的方向由右手定则确定。

```cpp
Vector3 cross(const Vector3 &a, const Vector3 &b) {
    return Vector3(
        a.y * b.z - a.z * b.y,
        a.z * b.x - a.x * b.z,
        a.x * b.y - a.y * b.x
    );
}
```

**混合积（Triple Product）**：$(\vec{a} \times \vec{b}) \cdot \vec{c}$。它的绝对值等于以 $\vec{a}, \vec{b}, \vec{c}$ 为棱的**平行六面体的体积**。

```cpp
double tripleProduct(const Vector3 &a, const Vector3 &b, const Vector3 &c) {
    return dot(cross(a, b), c);
}
```

> ❓ 混合积的正负号代表什么？
>
> 正号表示 $a, b, c$ 构成右手系（从 $a$ 转到 $b$ 的方向，$c$ 在拇指方向），负号表示左手系。在三维凸包中，我们用它来判断一个点在一个面的哪一侧。

## 3. 平面的表示

在三维空间中，平面可以用**点法式**表示：平面上一点 $P_0$ 和法向量 $\vec{n}$。平面上任意点 $P$ 满足：

$$\vec{n} \cdot (P - P_0) = 0$$

或者用**一般式**：$ax + by + cz + d = 0$，其中 $(a,b,c)$ 是法向量。

```cpp
struct Plane {
    Vector3 n;  // 法向量
    double d;   // 满足 n·P + d = 0
    
    Plane() {}
    Plane(const Point3 &p1, const Point3 &p2, const Point3 &p3) {
        n = cross(p2 - p1, p3 - p1);
        n = n * (1.0 / length(n));  // 归一化
        d = -dot(n, p1);
    }
    
    // 判断点 p 在平面的哪一侧
    // >0: 法向量方向侧, =0: 平面上, <0: 法向量反方向侧
    double side(const Point3 &p) const {
        return dot(n, p) + d;
    }
};
```

点到平面的距离：

```cpp
double distToPlane(const Point3 &p, const Plane &pl) {
    return fabs(pl.side(p)) / length(pl.n);
}
```

## 4. 多面体体积

四面体（四个点）的体积：$\frac{1}{6} |(\vec{AB} \times \vec{AC}) \cdot \vec{AD}|$

```cpp
double tetrahedronVolume(const Point3 &a, const Point3 &b, 
                          const Point3 &c, const Point3 &d) {
    return fabs(tripleProduct(b - a, c - a, d - a)) / 6.0;
}
```

对于任意多面体，可以选一个参考点（通常选原点），把每个三角面与原点连成四面体，求和（需要注意符号，凸多面体直接用 fabs）：

```cpp
double polyhedronVolume(const vector<Point3> &verts, 
                         const vector<tuple<int, int, int>> &faces) {
    double vol = 0;
    Point3 origin(0, 0, 0);
    for (const auto &f : faces) {
        int i = get<0>(f), j = get<1>(f), k = get<2>(f);
        vol += tripleProduct(verts[i] - origin, 
                             verts[j] - origin, 
                             verts[k] - origin);
    }
    return fabs(vol) / 6.0;
}
```

注意：这里假设所有面的法向量指向外部（逆时针顺序从外部看）。如果没有这个保证，应该对每个面用指向外部的法向量方向取正。

## 5. 三维凸包：增量法

三维凸包问题：给定 $n$ 个三维空间中的点，求包含所有点的最小凸多面体。

### 算法思路

**增量法（Incremental Algorithm）**：每次加入一个点，如果点在当前凸包内部，直接忽略；否则，删除从该点能"看到"的面，加入新的面连接该点和凸包上未被删除的边。

这个算法的复杂度是 $O(n^2)$——每次加入一个新点，需要遍历当前凸包的所有面来判断是否可见。

### 算法步骤

1. 先选 4 个不共面的点构成初始四面体
2. 对于每个新点 $p$：
   a. 遍历当前凸包的所有面，标记哪些面对 $p$ 可见（点在面的外侧）
   b. 所有可见的面将被删除
   c. 从这些可见面的边界边（即被一个可见面和一个不可见面共享的边）出发，与 $p$ 组成新的面
   d. 将新面加入凸包

### 判断可见性

对点 $p$ 和面 $f$（由顶点 $a, b, c$ 逆时针构成），如果 $\text{tripleProduct}(b-a, c-a, p-a) > 0$，则 $p$ 在面的法向量方向侧（即可见面——对于从凸包外部看向凸包的法向量方向）。

```cpp
struct Face {
    int v[3];  // 三个顶点的索引（逆时针，法向量朝外）
    Face(int a = 0, int b = 0, int c = 0) { v[0] = a; v[1] = b; v[2] = c; }
    
    Vector3 normal(const vector<Point3> &pts) const {
        return cross(pts[v[1]] - pts[v[0]], pts[v[2]] - pts[v[0]]);
    }
    
    bool canSee(const vector<Point3> &pts, const Point3 &p) const {
        return sgn(dot(normal(pts), p - pts[v[0]])) > 0;
    }
};
```

### 完整实现

```cpp
#include <bits/stdc++.h>
using namespace std;
const double eps = 1e-9;

int sgn(double x) {
    if (fabs(x) < eps) return 0;
    if (x > 0) return 1;
    return -1;
}

struct Point3 {
    double x, y, z;
    Point3(double x = 0, double y = 0, double z = 0) : x(x), y(y), z(z) {}
};
using Vector3 = Point3;

Point3 operator-(const Point3 &a, const Point3 &b) { return Point3(a.x - b.x, a.y - b.y, a.z - b.z); }
double dot(const Vector3 &a, const Vector3 &b) { return a.x * b.x + a.y * b.y + a.z * b.z; }
Vector3 cross(const Vector3 &a, const Vector3 &b) {
    return Vector3(a.y * b.z - a.z * b.y, a.z * b.x - a.x * b.z, a.x * b.y - a.y * b.x);
}
double length(const Vector3 &a) { return sqrt(dot(a, a)); }

struct Face {
    int v[3];
    Face(int a = 0, int b = 0, int c = 0) { v[0] = a; v[1] = b; v[2] = c; }
    Vector3 normal(const vector<Point3> &pts) const {
        return cross(pts[v[1]] - pts[v[0]], pts[v[2]] - pts[v[0]]);
    }
    bool canSee(const vector<Point3> &pts, const Point3 &p) const {
        return sgn(dot(normal(pts), p - pts[v[0]])) > 0;
    }
};

// 三维凸包（增量法）
vector<Face> convexHull3D(vector<Point3> pts) {
    int n = pts.size();
    vector<Face> faces;
    
    // 初始建一个四面体
    faces.push_back(Face(0, 1, 2));
    faces.push_back(Face(0, 2, 3));
    faces.push_back(Face(0, 3, 1));
    faces.push_back(Face(1, 3, 2));
    
    for (int i = 4; i < n; i++) {
        const Point3 &p = pts[i];
        vector<Face> newFaces;
        vector<vector<bool>> edge(n, vector<bool>(n, false));
        
        for (const auto &f : faces) {
            if (f.canSee(pts, p)) {
                // 标记该面的三条边
                for (int j = 0; j < 3; j++) {
                    int a = f.v[j], b = f.v[(j + 1) % 3];
                    // 如果这条边被标记两次（两个相邻面都可见），则删除
                    // 如果只标记一次（一个面可见，一个不可见），则为边界边
                    if (edge[a][b]) edge[a][b] = edge[b][a] = false;
                    else edge[a][b] = edge[b][a] = true;
                }
            } else {
                newFaces.push_back(f);  // 保留不可见的面
            }
        }
        
        // 从边界边构建新面
        for (const auto &f : faces) {
            if (f.canSee(pts, p)) {
                for (int j = 0; j < 3; j++) {
                    int a = f.v[j], b = f.v[(j + 1) % 3];
                    if (edge[a][b]) {
                        newFaces.push_back(Face(a, b, i));
                    }
                }
            }
        }
        
        faces = newFaces;
    }
    
    return faces;
}
```

### 边界情况的处理

1. **四点共面**：初始建四面体时，需要找到 4 个不共面的点。如果所有点共面，退化为二维凸包问题。
2. **点在面上**：如果点在某个面上，`canSee` 返回 false（因为用 `> 0` 判断），点不会被添加到面中，但也不影响后续点的处理。
3. **精度问题**：混合积判断符号时，eps 的设置很关键。

### 复杂度

增量法最坏情况下每步都需要 $O(m)$ 时间（$m$ 为当前面数），面数最坏 $O(n)$，所以总复杂度 $O(n^2)$。但随机化输入后期望复杂度也是 $O(n^2)$（三维凸包的面数本身是 $O(n)$ 的）。

## 小结

三维几何是二维几何的自然推广：

- **三维点/向量**：多了一个 $z$ 坐标
- **三维叉积**：结果是向量（不是标量），垂直于两个输入向量
- **混合积**：判断点在面的哪一侧，计算体积
- **三维凸包**：增量法 $O(n^2)$，每次加入一个点，维护凸包的面集合

三维几何在竞赛中出现的频率不高，但三维凸包是一个很好的"攻坚"练习。掌握它之后，很多三维问题就不那么可怕了。
