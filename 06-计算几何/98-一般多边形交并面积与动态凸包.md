这两个问题在实际工程中都很重要，竞赛中也时有出现。

## 1. 凸多边形交的交替推进算法

两个凸多边形的交仍然是凸多边形（或空集）。对于凸多边形，有 $O(n+m)$ 的高效算法——**交替推进法**。

### 算法思想

两个凸多边形 $P$ 和 $Q$ 都是逆时针的。用两个指针 $i$ 和 $j$ 分别指向 $P$ 和 $Q$ 的当前顶点。每次根据当前边的"推进方向"决定 $i$ 还是 $j$ 前进：

- 对于 $P$ 的边 $p_i p_{i+1}$ 和 $Q$ 的边 $q_j q_{j+1}$
- 判断两条边的方向关系：如果 $P$ 的边在 $Q$ 的边的"右侧"，推进 $i$；否则推进 $j$

算法需要特别注意：
1. 两条边相交时，把交点加入结果
2. 一个顶点在另一个多边形内部时，把该顶点加入结果

### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;
const double eps = 1e-9;

int sgn(double x) { /* ... */ }
struct Point { double x, y; /* 运算 */ };
double cross(const Vector &a, const Vector &b) { /* ... */ }
double dot(const Vector &a, const Vector &b) { /* ... */ }

// 点 p 是否在线段 ab 上（含端点）
bool onSegment(const Point &p, const Point &a, const Point &b) {
    return sgn(cross(p - a, b - a)) == 0 &&
           sgn(dot(p - a, p - b)) <= 0;
}

// 线段相交，返回交点
// 0: 不相交, 1: 交于一点, 2: 重合
int segIntersect(const Point &a, const Point &b, const Point &c, const Point &d, Point &p) {
    // ... 标准线段相交，解参数方程求交点 ...
}

// 判断点是否在多边形内部（含边界）
bool pointInPolygon(const Point &p, const vector<Point> &poly) {
    int n = poly.size();
    int wn = 0;
    for (int i = 0; i < n; i++) {
        int j = (i + 1) % n;
        if (onSegment(p, poly[i], poly[j])) return true;
        int k = sgn(cross(poly[j] - poly[i], p - poly[i]));
        int d1 = sgn(poly[i].y - p.y);
        int d2 = sgn(poly[j].y - p.y);
        if (k > 0 && d1 <= 0 && d2 > 0) wn++;
        if (k < 0 && d2 <= 0 && d1 > 0) wn--;
    }
    return wn != 0;
}

// 凸多边形交（交替推进法）
vector<Point> convexPolygonIntersect(vector<Point> P, vector<Point> Q) {
    int n = P.size(), m = Q.size();
    if (n < 3 || m < 3) return {};
    
    vector<Point> res;
    int i = 0, j = 0;
    
    // 最多跑 (n+m) 步
    for (int step = 0; step < 2 * (n + m) && res.size() < n + m; step++) {
        Point a = P[i], b = P[(i + 1) % n];
        Point c = Q[j], d = Q[(j + 1) % m];
        
        Vector va = b - a, vb = d - c;
        double cva = cross(va, vb);  // 两条边的方向关系
        
        Point p;
        bool hasInter = segIntersect(a, b, c, d, p);
        
        // 如果两条边相交，加入交点
        if (hasInter) {
            if (res.empty() || sgn(length(p - res.back()) > eps))
                res.push_back(p);
        }
        
        // 交替推进
        if (sgn(cross(vb, a - c)) >= 0 && sgn(cross(vb, b - c)) >= 0) {
            // P 的这条边完全在 Q 内部，加入终点
            if (pointInPolygon(b, Q))
                res.push_back(b);
            i = (i + 1) % n;
        } else if (sgn(cross(va, c - a)) >= 0 && sgn(cross(va, d - a)) >= 0) {
            // Q 的这条边完全在 P 内部，加入终点
            if (pointInPolygon(d, P))
                res.push_back(d);
            j = (j + 1) % m;
        } else {
            // 否则按叉积符号决定推进哪个
            if (sgn(cva) >= 0) {
                i = (i + 1) % n;
            } else {
                j = (j + 1) % m;
            }
        }
    }
    
    // 特殊情况：一个多边形完全在另一个内部
    if (res.empty()) {
        if (pointInPolygon(P[0], Q)) return P;
        if (pointInPolygon(Q[0], P)) return Q;
    }
    
    // 去重和去除共线点
    if (res.size() >= 3) {
        vector<Point> hull;
        for (const auto &p : res) {
            while (hull.size() >= 2) {
                Point a = hull[hull.size() - 2];
                Point b = hull.back();
                if (sgn(cross(b - a, p - b)) <= 0)
                    hull.pop_back();
                else break;
            }
            hull.push_back(p);
        }
        return hull;
    }
    
    return res;
}
```

> ❓ 这个算法为什么是 $O(n+m)$ 的？
>
> 每次迭代至少推进 $i$ 或 $j$ 中的一个指针。两个指针都最多绕多边形一圈，所以迭代次数不超过 $2(n+m)$。

## 2. 一般多边形的布尔运算

对于一个凹多边形，求交、并、差就不能用交替推进法了。需要更通用的方法——**平面扫描法**。

### 凹多边形求交的流程

1. 找到两个多边形所有边的交点
2. 把多边形在交点处"分割"成子段
3. 根据布尔运算类型（交/并/差），判断每个子段是否在结果中
   - 交：取在两个多边形**内部**或边界上的子段
   - 并：取至少在一个多边形**内部**的子段
   - 差（P - Q）：取在 $P$ 中但不在 $Q$ 中的子段
4. 把选出的子段连接成多边形

这个方法复杂度 $O((n+m)\log(n+m))$，实现比较复杂。竞赛中遇到凹多边形布尔运算时，建议优先考虑是否有特殊的简化方法。

## 3. 动态凸包基础

**动态凸包**：支持在点集中插入（和删除）点，同时维护凸包。

动态凸包有两种常见实现方式：

### 方法一：set 维护上下凸壳（Andrew 动态化）

对于 $x$ 坐标不重复的点，可以用两个 `std::set` 分别维护上凸壳和下凸壳。

每次插入新点 $(x, y)$：
1. 检查新点是否在当前凸包内（是则跳过）
2. 找到 $x$ 坐标相邻的左右两个凸包顶点
3. 向左检查：新点是否导致左侧顶点被"包裹"（非凸性），不断删除
4. 向右检查：同理

```cpp
struct DynamicConvexHull {
    set<Point> hull;  // 按 x 排序的凸包顶点
    
    bool isInside(const Point &p) {
        // 找到 x 相邻的点
        auto it = hull.lower_bound(p);
        if (it == hull.end() || it == hull.begin())
            return false;  // 边界上的点一定在凸包上
        auto right = it;
        auto left = prev(it);
        // 检查 p 是否在 left-right 线的"下方"（对于下凸壳）
        return sgn(cross(*right - *left, p - *left)) <= 0;
    }
    
    void insert(const Point &p) {
        if (isInside(p)) return;
        
        auto it = hull.insert(p).first;
        
        // 向左检查并删除
        while (it != hull.begin() && prev(it) != hull.begin()) {
            auto a = prev(prev(it));
            auto b = prev(it);
            if (sgn(cross(*b - *a, p - *b)) <= 0)
                hull.erase(b);
            else break;
        }
        
        // 向右检查并删除
        while (next(it) != hull.end() && next(next(it)) != hull.end()) {
            auto a = next(it);
            auto b = next(a);
            if (sgn(cross(*a - p, *b - p)) <= 0)
                hull.erase(a);
            else break;
        }
    }
};
```

注意：上面的代码只维护了**下凸壳**。上凸壳需要对称维护（相当于把 $y$ 取反）。

### 方法二：CDQ 分治（离线）

如果问题是**离线**的（知道所有插入/删除操作的时间点），可以用 CDQ 分治：

1. 把每个点的存活时间区间算出来（从插入到删除）
2. 把时间区间按中点划分到 CDQ 的左右两半
3. 对每个 CDQ 节点，把该节点的所有点做一次静态凸包
4. 回答每个时间点的查询时，合并所有祖先节点的凸包结果

```cpp
struct Event {
    int l, r;  // 存活时间区间
    Point p;
};

vector<Point> ans;  // 每个时间点的凸包

void cdq(int l, int r, vector<Event> &events) {
    if (l == r) {
        // 收集当前时间点的贡献
        return;
    }
    int mid = (l + r) / 2;
    
    // 把完全覆盖 [l, r] 的点加入当前节点
    vector<Point> cur;
    for (const auto &e : events) {
        if (e.l <= l && e.r >= r) {
            cur.push_back(e.p);
        }
    }
    
    // 对 cur 做静态凸包
    auto hull = convexHull(cur);
    // 将凸包信息存储在对应时间点的答案中
    
    // 递归处理左右区间
    cdq(l, mid, events);
    cdq(mid + 1, r, events);
}
```

### 方法三：二进制分组

对于**只插入不删除**的动态凸包，可以用二进制分组：
1. 把当前点集按 2 的幂分成 $O(\log n)$ 组
2. 每个组维护一个静态凸包
3. 插入点时，新点单独成一组（大小为 1），不断合并大小相同的组

查询最远点对等操作时，在各组之间做旋转卡壳。

## 4. 动态凸包的复杂度比较

| 方法 | 插入 | 删除 | 查询 | 在线/离线 | 实现难度 |
|------|------|------|------|-----------|---------|
| set 上下凸壳 | $O(\log n)$ | - | $O(\log n)$ | 在线 | 中 |
| CDQ 分治 | - | - | $O(\log^2 n)$ | 离线 | 高 |
| 二进制分组 | $O(\log^2 n)$ | - | $O(\log^2 n)$ | 在线 | 中 |

> ❓ 什么情况下用哪种方法？
>
> 如果只有插入（加点的动态凸包），用 set 维护上下凸壳最方便，可以直接用 `std::set` 的迭代器操作。如果有删除，但题目接受离线，用 CDQ 分治。如果必须在线的增删查，那需要考虑更高级的数据结构（如 Li Chao 线段树的变种）。

## 小结

这一篇讨论的是几何计算中更复杂的问题：

- **凸多边形交**：交替推进法 $O(n+m)$，两条边的关系决定指针移动
- **一般多边形布尔运算**：平面扫描法，找交点 -> 分割 -> 筛选 -> 拼接
- **动态凸包**：
  - set 维护上下凸壳（只插入，在线）
  - CDQ 分治（离线，支持删除）
  - 二进制分组（只插入，在线）

熟练掌握这些工具后，大部分计算几何竞赛题都能找到合适的解法。
