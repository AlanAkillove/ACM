# 三角剖分与平面图

> Made by Aki.
> 最后更新于 2026.04.30

## 引言

一个简单的凸多边形，你可以随便选一个顶点连接其他顶点，轻松把它分成若干个三角形。但是——如果多边形是**凹**的呢？如果给的是散乱的平面点集呢？

把多边形或点集分解成三角形，叫做**三角剖分（Triangulation）**。它是计算机图形学的基础（三维模型的渲染就是先剖分成三角形），也是很多几何算法的预处理步骤。

另一种视角是**平面图（Planar Graph）**——边可以在平面上不相交地画出来的图。平面图有一个非常漂亮的公式 $V - E + F = 2$（欧拉公式），它把顶点数、边数和面数联系起来。平面图对偶图的性质、点定位等问题，在计算几何中都有重要应用。

## 1. 简单多边形的三角剖分

**简单多边形**：边不相交的多边形（可以是凹的）。

三角剖分问题：在简单多边形内添加不相交的对角线，把多边形分成三角形的集合。

**性质**：任何简单 $n$ 边形经过三角剖分，恰好得到 $n-2$ 个三角形。

### 耳切法（Ear Clipping）

耳切法是最直观的三角剖分方法。它的思想是：**任何简单多边形至少有两个"耳朵"（ear）**。

什么是耳朵？多边形连续三个顶点 $A, B, C$，如果 $AC$ 是多边形内部的一条对角线（$B$ 是凸点），那么三角形 $ABC$ 就是一个耳朵。把这个三角形"切掉"，剩下的仍然是简单多边形。

> ❓ 怎么判断一个顶点是凸点还是凹点？
>
> 对于逆时针多边形，用叉积判断：$\text{cross}(B-A, C-B) > 0$ 为凸点（左转），$< 0$ 为凹点（右转）。

> ❓ 为什么任何简单多边形至少有两个耳朵？
>
> 这个定理的证明有点深度，但直观上可以这样理解：简单多边形的内角和为 $(n-2)\pi$。"耳朵"对应内角 $<\pi$ 且中间没有其他顶点穿过的凸顶点。如果只有一个耳朵，其他顶点都在它的"对面"，那多边形就退化成了三角形。实际上，任何简单多边形至少有 2 个不相邻的耳朵（Meister's Two Ears Theorem）。

### 耳切法实现

```cpp
#include <bits/stdc++.h>
using namespace std;
const double eps = 1e-9;

int sgn(double x) { /* ... */ }
struct Point { double x, y; /* 运算重载 */ };
double cross(const Vector &a, const Vector &b) { return a.x * b.y - a.y * b.x; }

// 判断点 p 是否在线段 ab 上
bool onSegment(const Point &p, const Point &a, const Point &b) {
    return sgn(cross(p - a, b - a)) == 0 &&
           sgn(min(a.x, b.x) - p.x) <= 0 && sgn(p.x - max(a.x, b.x)) <= 0 &&
           sgn(min(a.y, b.y) - p.y) <= 0 && sgn(p.y - max(a.y, b.y)) <= 0;
}

// 判断线段 ab 和 cd 是否相交（含端点）
bool segIntersect(const Point &a, const Point &b, const Point &c, const Point &d) {
    // ... 标准线段相交判断 ...
}

// 判断三角形 ABC 是否是耳朵
bool isEar(const vector<Point> &poly, int i, int j, int k) {
    // i, j, k 是连续的三个顶点索引
    // 检查三角形内是否有其他顶点
    int n = poly.size();
    for (int t = 0; t < n; t++) {
        if (t == i || t == j || t == k) continue;
        
        // 用叉积判断 poly[t] 是否在三角形内部
        // 或者用面积法
        double a1 = cross(poly[j] - poly[i], poly[t] - poly[i]);
        double a2 = cross(poly[k] - poly[j], poly[t] - poly[j]);
        double a3 = cross(poly[i] - poly[k], poly[t] - poly[k]);
        
        // 如果同号且在三角形内部（对逆时针顶点）
        if (sgn(a1) > 0 && sgn(a2) > 0 && sgn(a3) > 0) return false;
        if (sgn(a1) < 0 && sgn(a2) < 0 && sgn(a3) < 0) return false;
        // 如果在边上（不是顶点）也视为不合法
        if ((sgn(a1) == 0 && onSegment(poly[t], poly[i], poly[j])) ||
            (sgn(a2) == 0 && onSegment(poly[t], poly[j], poly[k])) ||
            (sgn(a3) == 0 && onSegment(poly[t], poly[i], poly[k])))
            return false;
    }
    return true;
}

// 耳切法三角剖分，返回三角形列表
vector<tuple<int, int, int>> triangulate(vector<Point> poly) {
    int n = poly.size();
    if (n <= 3) {
        if (n == 3) return {{0, 1, 2}};
        return {};
    }
    
    // 确保逆时针
    double area = 0;
    for (int i = 0; i < n; i++)
        area += cross(poly[i], poly[(i + 1) % n]);
    if (sgn(area) < 0) reverse(poly.begin(), poly.end());
    
    vector<tuple<int, int, int>> triangles;
    vector<int> idx(n);
    iota(idx.begin(), idx.end(), 0);
    
    while (idx.size() > 3) {
        int m = idx.size();
        bool found = false;
        for (int t = 0; t < m; t++) {
            int i = idx[t];
            int j = idx[(t + 1) % m];
            int k = idx[(t + 2) % m];
            
            // 先检查 j 是否是凸点（对逆时针多边形，叉积 > 0）
            if (sgn(cross(poly[j] - poly[i], poly[k] - poly[j])) <= 0)
                continue;  // 凹点不可能是耳朵
            
            // 检查是否是合法的耳朵
            if (isEar(poly, i, j, k)) {
                triangles.push_back({i, j, k});
                idx.erase(idx.begin() + (t + 1) % m);
                found = true;
                break;
            }
        }
        if (!found) break;  // 失败（通常不会发生）
    }
    
    // 最后三个点构成一个三角形
    triangles.push_back({idx[0], idx[1], idx[2]});
    return triangles;
}
```

### 手算例子

多边形（逆时针）：$A(0,0), B(2,0), C(3,1), D(2,2), E(0,2), F(1,1)$

```
    E(0,2) ---- D(2,2)
     |    \     /
     |     F(1,1)
     |      |
    A(0,0) ---- B(2,0) --- C(3,1)
```

这是一个凹多边形（$F$ 是凹进去的顶点）。

检查耳朵：
- $A(0,0), B(2,0), C(3,1)$：$B$ 是凸点，三角形 $ABC$ 内有没有其他点？$F$ 不在内（检查叉积），$D$ 不在内——$ABC$ 是耳朵！
- 切掉 $ABC$，剩下 $A, C, D, E, F$（注意顶点重新连接）
- 继续切耳朵…

## 2. 平面图与欧拉公式

**平面图**：可以在平面上画出，使得任何两条边不相交（只在顶点处相交）的图。

对于连通平面图，**欧拉公式**成立：
$$V - E + F = 2$$

其中 $V$ 是顶点数，$E$ 是边数，$F$ 是面数（包含外部无限面）。

> ❓ 这个公式在计算几何中有什么用？
>
> 很多几何问题的复杂度分析依赖于欧拉公式。比如平面图的边数 $E \le 3V - 6$（当 $V \ge 3$ 时）。这意味着平面图是"稀疏"的——边数最多是顶点数的线性倍，而不是 $O(V^2)$。这个性质在设计平面图算法时至关重要。

### 从欧拉公式推导的常用结论

1. **任何简单平面图** $E \le 3V - 6$（$V\ge 3$）
2. **二分平面图** $E \le 2V - 4$
3. **任何简单平面图至少有一个顶点的度数 $\le 5$**

证明思路：每个面至少 3 条边，每条边最多被两个面共享，所以 $2E \ge 3F$。代入欧拉公式 $V - E + F = 2$ 得到 $E \le 3V - 6$。

## 3. 平面图的点定位

**点定位（Point Location）**：给定一个平面图（把平面分成多个面区域），问一个查询点落在哪个面内。

经典解法：**梯形图（Trapezoidal Map）** + **持久化线段树**。

### 梯形图思路

1. 对平面图的所有顶点按 $x$ 排序，做垂直扫描线
2. 在扫描线经过顶点时，动态维护扫描线与平面图边的交点（$y$ 坐标）
3. 在扫描线事件之间，这些交点是按 $y$ 有序的，把带状区域分成一个个梯形
4. 每个梯形对应一个节点，查询时根据 $x, y$ 坐标沿梯形图走下去

梯形图的结构可以在 $O(n\log n)$ 时间内构建，查询 $O(\log n)$。

### 简化方案：扫描线 + 二分

对于竞赛中的常用场景（查询点很少时），可以使用更简单的方法：

对于每个查询点，从该点向右作射线（如水平射线），统计与多边形边的交点数。如果交点为奇数，点在多边形内；偶数则在多边形外。

但对于平面图（多个面），需要在每个面的边界上进行射线法判断——如果面很多，效率较低。

## 4. 三角剖分的更多视角

### Delaunay 三角剖分

Delaunay 三角剖分是点集三角剖分中"最均衡"的一种：对所有三角形，其外接圆内不包含其他点。这种剖分最大化最小角，避免出现"瘦长"的三角形。

Delaunay 三角剖分广泛应用于：
- 有限元网格生成
- 地理信息系统（TIN 数字高程模型）
- 三维重建

## 小结

三角剖分和平面图是计算几何中偏理论的部分：

- **耳切法**：简单多边形三角剖分的经典算法，$O(n^2)$，实现直观
- **欧拉公式** $V - E + F = 2$：分析平面图复杂度的核心工具
- **平面图边数** $E \le 3V - 6$：可用于证明许多几何问题的复杂度下限
- **点定位**：梯形图 + 持久化线段树，查询 $O(\log n)$
