# SG 函数与 Sprague-Grundy 定理

> Made by Aki.
> 最后更新于 2026.04.30

## 引言

到目前为止，我们学过的每一种博弈（巴什博弈、威佐夫博弈、Nim 游戏、阶梯 Nim）都有其独特的解法。但你会不会觉得，每次遇到一个新游戏都要从头分析规律，很累？

有没有一种**通用的方法**，可以系统性地解决任何 impartial combinatorial game？

答案是肯定的。这就是 **Sprague-Grundy 定理**（以下简称 SG 定理），它被认为是组合博弈论中最重要的定理。SG 定理告诉我们：

> 任何一个 impartial combinatorial game，都等价于一个 Nim 游戏。每个游戏局面可以映射为一个非负整数（SG 值），而多个游戏的组合就等价于将这些 SG 值异或。

这意味着：**只要学会了 SG 函数和 SG 定理，你就能解决所有公平组合博弈问题。** 这是一个非常强大的工具。

## 从 N/P 态到 SG 函数

### N/P 态的局限性

在之前的文章中，我们只区分了两种状态：必胜（N）和必败（P）。但有些时候，光知道「胜或败」是不够的。

举个例子，考虑两个独立的游戏 $G_1$ 和 $G_2$，你可以选择在其中一个游戏中操作一步。整个游戏的胜负规则是什么？

如果 $G_1$ 是 N 态，$G_2$ 是 P 态，那整体的胜负如何判断？答案是：**P 态是「0」，N 态是「非 0」，整体胜负就是 Nim-sum**。

但这太粗糙了。两个不同的 N 态可能并不等价——比如，一步就能赢的 N 态和需要很多步才能赢的 N 态，在组合中会有不同的作用。

SG 函数就是对每个状态赋予一个**更精细的数值**，这个数值反映了该状态在组合中的「权重」。

### mex 运算

SG 函数的核心是一个简单的运算——**mex**（Minimum EXcluded value），即「不在集合中的最小非负整数」。

$$mex(S) = \min\{k \ge 0 \mid k \notin S\}$$

例如：
- $mex(\{0,1,2\}) = 3$
- $mex(\{0,2,3\}) = 1$
- $mex(\{1,2,3\}) = 0$
- $mex(\emptyset) = 0$

mex 运算在 SG 函数中扮演的角色是「计算当前状态的 SG 值」。

### SG 函数的定义

对于一个游戏状态 $x$，记它能直接到达的所有状态的集合为 $F(x)$（即 $x$ 的「后继状态集」）。定义：

$$SG(x) = mex(\{SG(y) \mid y \in F(x)\})$$

即：当前状态的 SG 值 = 所有后继状态的 SG 值组成的集合的 mex。

**边界条件**：终止状态（没有后继的状态）的 SG 值为 $mex(\emptyset) = 0$。

> ❓ 这跟 N/P 态有什么关系？
> 
> 联系非常紧密：
> - $SG(x) = 0$ 对应 P 态（必败态）
> - $SG(x) \ne 0$ 对应 N 态（必胜态）
> 
> 因为如果所有后继的 SG 值都不为 0（集合中没有 0），则 mex 结果为 0，即该状态为 P。这正是 P 态的定义！
> 
> 所以说，SG 函数是 N/P 态的推广——从「只分 0 和非 0」推广到了「区分不同的非 0 值」。

### 手算例子：取石子游戏的 SG 函数

还是最基础的取石子游戏：一堆 $n$ 颗石子，每次可取 $1$ 或 $2$ 颗，取走最后一颗胜。

我们来手算每个状态的 SG 值：

- $SG(0) = mex(\emptyset) = 0$
- $SG(1) = mex(\{SG(0)\}) = mex(\{0\}) = 1$
- $SG(2) = mex(\{SG(1), SG(0)\}) = mex(\{1,0\}) = 2$
- $SG(3) = mex(\{SG(2), SG(1)\}) = mex(\{2,1\}) = 0$
- $SG(4) = mex(\{SG(3), SG(2)\}) = mex(\{0,2\}) = 1$
- $SG(5) = mex(\{SG(4), SG(3)\}) = mex(\{1,0\}) = 2$
- $SG(6) = mex(\{SG(5), SG(4)\}) = mex(\{2,1\}) = 0$

模式：$SG(n) = n \bmod 3$。

验证：$n = 0,3,6$ 时 SG = 0（P 态），$n = 1,4$ 时 SG = 1，$n = 2,5$ 时 SG = 2。确实符合我们之前的结论——$n$ 是 3 的倍数时必败。

## 有向图游戏

任何一个 impartial combinatorial game 都可以用**有向图**来表示：
- 每个节点是一个游戏状态
- 每条边是一个合法操作
- 出度为 0 的节点是终止状态

**SG 函数就是在有向图上做的一种递归标记**——从终止状态出发，逆着边计算每个节点的 SG 值。

```cpp
// 有向图游戏的 SG 值计算（记忆化搜索）
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

const int MAXN = 1e5 + 5;
vector<int> g[MAXN];  // 有向图
int sg[MAXN];          // SG 值，-1 表示未计算

int dfs(int u) {
    if (sg[u] != -1) return sg[u];
    set<int> s;
    for (int v : g[u]) {
        s.insert(dfs(v));
    }
    // 计算 mex
    int mex = 0;
    while (s.count(mex)) mex++;
    return sg[u] = mex;
}

int main() {
    memset(sg, -1, sizeof(sg));
    // 建图后调用 dfs(起始状态)
    return 0;
}
```

## Sprague-Grundy 定理

现在是最核心的部分。SG 定理告诉我们**如何处理多个游戏的组合**。

### 游戏的和

假设有两个独立的游戏 $G_1$ 和 $G_2$。定义它们的「和」$G = G_1 + G_2$ 为：**每回合，玩家选择其中一个游戏，并在其中走一步**。游戏在 $G_1$ 和 $G_2$ 都无法操作时结束（即两个游戏都到达终止状态）。

更一般地，$n$ 个游戏的和 $G = G_1 + G_2 + \cdots + G_n$ 就是每次选一个游戏走一步。

### SG 定理的内容

**定理（Sprague-Grundy, 1935）**：对于 $n$ 个独立游戏的和 $G = G_1 + G_2 + \cdots + G_n$，有：
$$SG(G) = SG(G_1) \oplus SG(G_2) \oplus \cdots \oplus SG(G_n)$$

其中 $\oplus$ 表示异或（XOR）。

换句话说：**整个组合游戏的 SG 值，等于各子游戏 SG 值的异或**。而整体先手必胜当且仅当这个异或值不为 0。

> ❓ 这看起来跟 Nim 游戏一模一样？
> 
> 没错！因为 Nim 游戏中，每个大小为 $k$ 的石子堆可以视为一个子游戏，其 SG 值恰好为 $k$。多个 Nim 堆的和的 SG 值就是 $k_1 \oplus k_2 \oplus \cdots \oplus k_n$。这与 Bouton 定理完全一致。
>
> 事实上，SG 定理就是 Bouton 定理的推广——**任何 impartial 组合游戏都可以等价地转化为 Nim 游戏**。

### SG 定理的证明思路

我们需要证明：
1. $SG(G_1) \oplus SG(G_2) \oplus \cdots \oplus SG(G_n) = 0$ 时，游戏 $G$ 的所有后继的 SG 异或都不为 0。
2. 异或值不为 0 时，存在一个后继的 SG 异或为 0。

这与 Bouton 定理的证明如出一辙。实际上，证明过程就是「把 SG 值看作 Nim 堆的大小，把移动看作改变某个 SG 值」。

**证明概要**：

设 $S = SG(G_1) \oplus SG(G_2) \oplus \cdots \oplus SG(G_n)$。

**如果 $S = 0$**：选手在某个子游戏 $G_i$ 中走一步，将其状态变为 $G_i'$。新的异或和为：
$$S' = S \oplus SG(G_i) \oplus SG(G_i') = SG(G_i) \oplus SG(G_i')$$

由于 $SG(G_i') \ne SG(G_i)$（因为 $SG(G_i')$ 是 $SG(G_i)$ 的一个后继，两者不可能相等），所以 $S' \ne 0$。

**如果 $S \ne 0$**：设 $d$ 为 $S$ 的最高位。存在某个 $i$ 使得 $SG(G_i)$ 在第 $d$ 位上为 1。令 $SG(G_i') = SG(G_i) \oplus S$。由于 $SG(G_i)$ 和 $S$ 在第 $d$ 位上都是 1，异或后 $SG(G_i')$ 在第 $d$ 位上为 0，所以 $SG(G_i') < SG(G_i)$。

根据 SG 函数的定义，$SG(G_i) = mex(\{SG(y) \mid y \in F(G_i)\})$，所有小于 $SG(G_i)$ 的非负整数都在后继 SG 值集合中。所以 $SG(G_i')$ 一定对应某个后继状态 $G_i'$。

操作后：$S' = S \oplus SG(G_i) \oplus SG(G_i') = S \oplus SG(G_i) \oplus (SG(G_i) \oplus S) = 0$。

证毕。

### 直观理解

SG 定理的美妙之处在于它将复杂的博弈问题拆解为三步：

1. **分解**：将整个游戏拆分成若干个独立的子游戏
2. **计算**：计算每个子游戏的 SG 值（通常通过 DP/记忆化搜索/找规律）
3. **组合**：将所有 SG 值异或，非 0 则必胜

每个子游戏的计算和最核心的部分——是正确建模状态和转移。一旦做好这一步，组合部分就是一行异或代码。

## SG 函数的深层理解

### 为什么是 mex？

mex 运算确保了 SG 值具有「后继的 SG 值集合恰好是 $0,1,\dots,SG(x)-1$ 的某个超集」的性质。具体来说：

如果 $SG(x) = k$，那么 $x$ 的后继状态的 SG 值包含了 $0,1,\dots,k-1$ 中的至少一个（实际上，SG 值等于 $k$ 意味着 $0,1,\dots,k-1$ 中每个值都在后继中出现），且不包含 $k$。

这个性质保证了：**从 SG 值为 $k$ 的状态，可以走到 SG 值为任意 $0 \le j < k$ 的状态**。这正是 Nim 游戏中的核心性质——一堆 $k$ 颗石子可以取到任意少于 $k$ 的数量。

### 一个更复杂的例子

**问题**：一堆石子，每次可以取 1、3 或 4 颗，取走最后一颗胜。计算 $SG(n)$ 并找出规律。

计算：
- $SG(0) = 0$
- $SG(1) = mex(\{SG(0)\}) = mex(\{0\}) = 1$
- $SG(2) = mex(\{SG(1)\}) = mex(\{1\}) = 0$
- $SG(3) = mex(\{SG(2), SG(0)\}) = mex(\{0,0\}) = mex(\{0\}) = 1$
- $SG(4) = mex(\{SG(3), SG(1), SG(0)\}) = mex(\{1,1,0\}) = mex(\{0,1\}) = 2$
- $SG(5) = mex(\{SG(4), SG(2), SG(1)\}) = mex(\{2,0,1\}) = mex(\{0,1,2\}) = 3$
- $SG(6) = mex(\{SG(5), SG(3), SG(2)\}) = mex(\{3,1,0\}) = 2$
- $SG(7) = mex(\{SG(6), SG(4), SG(3)\}) = mex(\{2,2,1\}) = mex(\{1,2\}) = 0$
- $SG(8) = mex(\{SG(7), SG(5), SG(4)\}) = mex(\{0,3,2\}) = 1$
- $SG(9) = mex(\{SG(8), SG(6), SG(5)\}) = mex(\{1,2,3\}) = 0$

SG 序列：$0,1,0,1,2,3,2,0,1,0,\dots$

看起来并没有一个简单的周期性公式（虽然最终会周期化，但周期可能比较长）。这就是为什么实际竞赛中，对于这种**取子集合给定**的博弈，我们会用打表找规律或者直接 DP。

## 将游戏分解为子游戏

SG 定理最强大的应用在于：**当原游戏可以分解为多个独立的子游戏时**。

例如，一个棋盘上有多个棋子，每个棋子可以独立移动——那么每个棋子就是一个子游戏。或者一个图上有多个互不连通的组件，每个组件独立——那么每个组件就是一个子游戏。

关键在于识别哪些部分是「独立的」。独立意味着：**操作一个子游戏不会影响其他子游戏的状态**。

```cpp
// 计算取子集合为 {1, 3, 4} 的 SG 值
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

const int MAXN = 1e6 + 5;
int sg[MAXN];

int main() {
    vector<int> moves = {1, 3, 4};
    int n;
    cin >> n;

    sg[0] = 0;
    for (int i = 1; i <= n; i++) {
        set<int> s;
        for (int x : moves) {
            if (x <= i) s.insert(sg[i - x]);
        }
        int mex = 0;
        while (s.count(mex)) mex++;
        sg[i] = mex;
    }

    for (int i = 0; i <= n; i++) {
        cout << "SG(" << i << ") = " << sg[i] << endl;
    }
    return 0;
}
```

### 更一般的形式：多个游戏组合的判定

假设有 $k$ 个独立游戏，每个游戏的当前状态对应的 SG 值分别为 $s_1, s_2, \dots, s_k$。则整体胜负判定：

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
    int k;
    cin >> k;
    ll xor_sum = 0;
    for (int i = 0; i < k; i++) {
        ll sg_val;
        cin >> sg_val;
        xor_sum ^= sg_val;
    }
    if (xor_sum == 0) {
        cout << "Second wins!" << endl;
    } else {
        cout << "First wins!" << endl;
    }
    return 0;
}
```

## 小结

- **SG 函数**：$SG(x) = mex(\{SG(y) \mid y \in F(x)\})$，将每个状态映射为一个非负整数
- **SG 值**：0 对应必败态，非 0 对应必胜态
- **SG 定理**：多个独立游戏组合的 SG 值 = 各子游戏 SG 值的异或
- **求解步骤**：分解 → 计算各子游戏 SG 值 → 异或判定
- **核心意义**：任何 impartial 组合游戏都可以等价为 Nim 游戏

SG 定理是博弈论中最强大的工具之一。下一讲我们将通过大量具体例子，学习如何实际应用 SG 函数解决各种博弈问题。
