# 阶梯 Nim

> Made by Aki.
> 最后更新于 2026.04.30

## 引言

想象这样一个游戏：有 $n$ 级台阶（编号 $1$ 到 $n$），每一级台阶上放着若干颗石子。你每次可以选择一级台阶（至少第 2 级），从该级上取任意正整数颗石子（至少 1 颗），**移到下一级台阶**上。石子移到第 0 级（地面）后就不再参与游戏。取走最后一颗石子（即把所有石子都移到地面）的人获胜。

你可能会想：这不就是把 Nim 游戏的「取走」变成了「下移」吗？那跟 Nim 有什么区别？

我们来简单试一下。假设只有 2 级台阶，第 1 级有 $a_1$ 颗，第 2 级有 $a_2$ 颗。

- 如果操作第 1 级，石子会被移到地面——相当于这堆石子被「取走」了。
- 如果操作第 2 级，石子会被移到第 1 级——相当于石子从「不能直接取」变成了「能被下一手取走」。

这里的关键是：**并非所有台阶上的石子都等价**。奇数台阶和偶数台阶的石子的「命运」截然不同。

这听起来和 Nim 不一样——但它其实可以转化为 Nim。而且转化方式极其巧妙。

## 从简单例子开始

让我们先手算一个小规模的阶梯 Nim。

### 只有 1 级台阶

石子全部在第 1 级，每次可以取若干移到地面。这跟普通的 Nim 游戏中的一堆石子完全一样！先手可以一次性全部移到地面，必胜。

Nim-sum 表示：$a_1 \ne 0$ 时先手必胜。

### 只有 2 级台阶

第 1 级有 $a_1$ 颗，第 2 级有 $a_2$ 颗。

我们可以手动分析几种情况：

- $(0,0)$：P（终止态）
- $(0,1)$：只能操作第 2 级，移 1 颗到第 1 级 → $(1,0)$。轮到对手，对手可以直接移走第 1 级的所有石子获胜。所以 $(0,1)$ 是 N。
- $(1,0)$：可以直接移走第 1 级的石子获胜。N。
- $(1,1)$：可以：
  - 操作第 1 级：移 1 颗到地面 → $(0,1)$（N）
  - 操作第 2 级：移 1 颗到第 1 级 → $(2,0)$（N）
  所有后继都是 N，所以 $(1,1)$ 是 P。

这里我们看到 $(1,1)$ 是 P 态，$a_1=1$。让我们试试更多：
- $(2,0)$：N
- $(2,1)$：
  - 操作第 1 级：移 1 → $(1,1)$（P）或移 2 → $(0,1)$（N）
  存在操作到 P 态，所以 $(2,1)$ 是 N。
- $(0,2)$：操作第 2 级移 2 → $(2,0)$（N），移 1 → $(1,0)$（N）。所有后继是 N？等等，移 1 到第 1 级得到 $(1,0)$，第 1 级有 1 颗——这是 N，对手可以取走获胜。让我们再试试 $(0,2)$ 的其他选择：
  - 移 2 到第 1 级 → $(2,0)$（N）
  - 移 1 到第 1 级 → $(1,0)$（N）
  所有后继是 N，所以 $(0,2)$ 是 P！

$(0,2)$ 是 P，$(1,1)$ 是 P……好像 $a_1$ 为偶数时是 P？

验证：$(0,0)$ — $a_1=0$ 偶数 P ✓，$(2,0)$ — $a_1=2$ 偶数，但我们之前说 $(2,0)$ 是 N——因为先手可以直接取走 2 颗获胜。不对！

我犯错了。$(2,0)$ 的意思是第 1 级有 2 颗，第 2 级有 0 颗。先手可以一次全部移走，直接获胜。所以 $(2,0)$ 不是 P。

这说明我的猜想错了。重新思考。

> ❓ 我们是否忽略了什么？为什么 $(0,2)$ 是 P 但 $(2,0)$ 不是 P？
> 
> $(0,2)$ 时只有一颗在第 2 级，先手必须把它移到第 1 级。然后就轮到对手操作第 1 级获胜。所以 $(0,2)$ 先手必败。
>
> 而 $(2,0)$ 时先手可以直接操作第 1 级获胜。
>
> 区别在于：**第 1 级的石子可以直接取走（一步获胜）**，而第 2 级的石子必须先经过第 1 级才能被取走。换句话说，第 1 级的石子是「可用的」，第 2 级的石子是「备用」的。

### 核心洞察

让我们一步步总结：

- 当一名玩家把第 2 级的石子移到第 1 级时，下一名玩家可以通过移走第 1 级的石子来获得（或消耗）这个优势。
- 第 1 级的石子可以直接决定胜负——这是「奇数级」。
- 第 2 级的石子必须经过一次转移才能影响胜负——这是「偶数级」。

事实上，阶梯 Nim 的核心结论是：

> **阶梯 Nim 等价于对奇数级台阶上的石子数做 Nim 游戏。**

换句话说，**只异或所有奇数台阶上的石子数**，结果为 0 则先手必败，否则先手必胜。

## 阶梯 Nim 定理

**定理**：在阶梯 Nim 中，先手必胜当且仅当所有奇数编号台阶上石子数的异或和不为 0。

即：$$S = a_1 \oplus a_3 \oplus a_5 \oplus \cdots \ne 0 \iff \text{First wins}$$

### 证明思路

我们从公平博弈的 N/P 态定义出发，证明两件事：

**1. 如果奇数台阶的异或和为 0，则无论怎么操作，都会使奇数台阶的异或和变为非 0。**

操作分为两种：
- 操作奇数台阶（如第 $2k+1$ 级）：从该级取 $x$ 颗移到下一级（偶数级）。这会改变 $a_{2k+1}$，从而改变奇数异或和。由于 $a_{2k+1} > a_{2k+1} - x \ge 0$，异或和必发生变化且变非 0（与 Nim 证明相同）。
- 操作偶数台阶（如第 $2k$ 级）：从该级取 $x$ 颗移到下一级（奇数级）。这不会改变奇数台阶的异或和（因为没动奇数台阶的当前数量）？等等——它增加了奇数台阶的数量！

这里有一个微妙之处：操作偶数台阶会把石子加到奇数台阶上，从而改变奇数异或和。

让我们重新审视线索。

实际上，正确的证明需要把偶数台阶的操作也纳入考虑：

**情况 1**：操作奇数台阶 $i$（$i$ 为奇数）
- 取 $x$ 颗从第 $i$ 级移到第 $i-1$ 级（偶数级）
- $a_i$ 减少 $x$，其他奇数台阶不变
- 奇数异或和从 0 变为 $0 \oplus a_i \oplus (a_i - x) \ne 0$（因为 $a_i \ne a_i - x$）

**情况 2**：操作偶数台阶 $i$（$i$ 为偶数）
- 取 $x$ 颗从第 $i$ 级移到第 $i-1$ 级（奇数级）
- $a_{i-1}$ 增加 $x$（$i-1$ 是奇数）
- 奇数异或和从 0 变为 $0 \oplus a_{i-1} \oplus (a_{i-1} + x)$
- 由于 $a_{i-1} \oplus (a_{i-1} + x)$ 一般不为 0，奇数异或和变为非 0

所以无论操作哪级台阶，奇数异或和都会从 0 变为非 0。

**2. 如果奇数台阶的异或和 $S \ne 0$，则存在操作使奇数异或和变为 0。**

这类似于 Bouton 定理的构造方法：
- 找到 $S$ 的最高位，选择某个奇数台阶 $i$，使其在该位上为 1
- 令 $a_i' = a_i \oplus S$，从第 $i$ 级取 $a_i - a_i'$ 颗移到下一级
- 如果 $a_i' > a_i$ 怎么办？这表示我们需要增加石子！但在原始操作中，我们只能减少石子，不能增加。

这里有一个问题：如果 $a_i \oplus S > a_i$，我们不能通过取石子来实现。但在 Nim 中，由于 $S$ 的最高位与 $a_i$ 相同，$a_i \oplus S$ 的最高位被消去，因此必然小于 $a_i$。这个论证在阶梯 Nim 的奇数台阶上也成立（因为只看奇数台阶时，这就是一个普通的 Nim 游戏）。

**3. 还要考虑「操作偶数台阶」来改变奇数异或和的可能性。**

实际上，对于 $S \ne 0$ 的情况，最简单的构造就是直接操作某个奇数台阶，像普通 Nim 一样把它的石子数变为 $a_i \oplus S$。

那如果 $a_i \oplus S = 0$ 呢？那意味着我们把这级的所有石子都移到下一级——这时下一级（偶数级）的石子数增加了，这可能会导致对手反过来操作这个偶数级来破坏局面。

这并没有违反必胜性，因为在 Nim 的对称策略中，我们只需要保证**每一步**都把奇数异或和归零即可。无论对手怎么走，他都会破坏归零状态，我们再修复。

> ❓ 听起来有点绕。能不能更直观地解释？
> 
> 可以这样想：奇数台阶上的石子就像 Nim 游戏中的石子堆，偶数台阶只是「缓冲区」。把石子从奇数级移到偶数级相当于「取走」——石子离开了奇数级，不再影响奇数异或和。把石子从偶数级移到奇数级相当于「给对手送石子」——表面上让奇数异或和变了，但对手可以立刻把这些多出来的石子再移到偶数级，从而恢复局面。
>
> 实际上，有一种更漂亮的证法：定义一个「配对策略」。如果奇数异或和为 0，对手可以采用策略：当你操作奇数级时，他按 Nim 策略回应；当你操作偶数级时，他把你刚移下来的石子原封不动地再移到下一级——这相当于「抵消」了你的操作。

## 代码实现

阶梯 Nim 的代码非常简单，本质上就是对奇数台阶做异或：

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
    int n;
    cin >> n;
    ll xor_sum = 0;
    for (int i = 1; i <= n; i++) {
        ll x;
        cin >> x;
        if (i % 2 == 1) {  // 奇数台阶
            xor_sum ^= x;
        }
    }
    if (xor_sum == 0) {
        cout << "Second wins!" << endl;
    } else {
        cout << "First wins!" << endl;
    }
    return 0;
}
```

### 手算例子

**例 1**：有 4 级台阶，石子数分别为 $(3, 1, 4, 2)$。判断先手是否必胜。

奇数台阶：第 1 级 = 3，第 3 级 = 4。
异或和：$3 \oplus 4 = 7 \ne 0$。先手必胜。

**例 2**：有 5 级台阶，石子数分别为 $(2, 5, 2, 3, 1)$。

奇数台阶：第 1 级 = 2，第 3 级 = 2，第 5 级 = 1。
异或和：$2 \oplus 2 \oplus 1 = 1 \ne 0$。先手必胜。

先手应该怎么走？找到奇数异或和 $S = 1$ 的最高位——第 0 位（$2^0 = 1$）。
哪些奇数台阶在第 0 位上是 1？第 5 级（$1=1_2$）和第 1 级（$2=10_2$ 的第 0 位是 0）、第 3 级（$2=10_2$ 的第 0 位是 0），所以只有第 5 级。

取 $a_5' = 1 \oplus 1 = 0$，即把第 5 级的 1 颗石子全部移到第 4 级。

操作后：$(2, 5, 2, 3, 0) \rightarrow (2, 5, 2, 4, 0)$——等等，第 5 级的 1 颗移到第 4 级，所以第 4 级从 3 变成 4。奇数异或和：$2 \oplus 2 \oplus 0 = 0$ ✓

## 树上扩展：Tree Nim / 阶梯 Nim 的树上版本

阶梯 Nim 有一个很自然的推广——**树上 Nim**（又称 Tree Nim 或 Staircase Nim on Tree）。

问题描述：给定一棵有根树，每个节点上有若干石子。每次操作可以选择一个非根节点，将该节点上的任意正整数颗石子移到其父节点。将石子移到根节点即视为被取走。问先手是否必胜？

结论：**将所有深度为奇数的节点上的石子数异或起来**。结果为 0 则先手必败，否则先手必胜。

为什么？因为树上的每条从根到叶子的路径都可以看作一个阶梯，而不同路径之间的操作互不干扰。深度从根（深度 0）开始计数，根节点的孩子深度为 1（奇数）。

> ❓ 跟普通阶梯 Nim 一样吗？
> 
> 完全一样的思想。奇数深度节点等价于「奇数台阶」，偶数深度节点等价于「偶数台阶」。关键区别在于：树上的分支节点可能有多条路径汇聚，但 SG 定理保证了独立子游戏的异或和性。

### 树上 Nim 的代码

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

const int MAXN = 1e5 + 5;
vector<int> g[MAXN];
ll val[MAXN];
int depth[MAXN];

void dfs(int u, int p) {
    depth[u] = (p == -1 ? 0 : depth[p] + 1);
    for (int v : g[u]) {
        if (v != p) dfs(v, u);
    }
}

int main() {
    int n;
    cin >> n;
    for (int i = 1; i <= n; i++) {
        cin >> val[i];
    }
    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
        g[v].push_back(u);
    }

    dfs(1, -1);  // 假设根为 1

    ll xor_sum = 0;
    for (int i = 1; i <= n; i++) {
        if (depth[i] % 2 == 1) {  // 奇数深度
            xor_sum ^= val[i];
        }
    }

    cout << (xor_sum ? "First wins!" : "Second wins!") << endl;
    return 0;
}
```

## 经典题目分析

### 题目：POJ 1704 - Georgia and Bob

**题意**：在一条直线上有 $n$ 个棋子，位置分别为 $p_1 < p_2 < \dots < p_n$。每次可以选择一个棋子向左移动任意格（不能越过其他棋子），不能移动者输。问先手胜负。

**分析**：
这个题看着跟阶梯 Nim 八竿子打不着，但其实可以通过巧妙的配对转化为阶梯 Nim。

将棋子两两配对（$(p_1,p_2)$，$(p_3,p_4)$，……）。将每对之间的间距视为石子数。如果 $n$ 为奇数，可以在 $p_0 = 0$ 处放一个虚拟棋子。

移动一个棋子时：
- 如果移动的是每对的左边棋子，相当于减少了相邻棋子之间的间距（取走石子）
- 如果移动的是每对的右边棋子，相当于增加了间距（给对手送石子）

这其实就是阶梯 Nim！位置编号的奇偶性决定了是奇数级还是偶数级。

具体到代码：

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
    int T;
    cin >> T;
    while (T--) {
        int n;
        cin >> n;
        vector<int> p(n);
        for (int i = 0; i < n; i++) cin >> p[i];
        sort(p.begin(), p.end());

        int xor_sum = 0;
        for (int i = n - 1; i >= 0; i -= 2) {
            int gap;
            if (i == 0) {
                gap = p[0] - 1;  // 到左边界
            } else {
                gap = p[i] - p[i-1] - 1;
            }
            xor_sum ^= gap;
        }

        cout << (xor_sum ? "Georgia will win" : "Bob will win") << endl;
    }
    return 0;
}
```

## 小结

- **阶梯 Nim**：石子只能向下移动，不能直接取走
- **核心结论**：只异或奇数编号台阶上的石子数即可判定胜负
- **直觉理解**：奇数级石子是「有效堆」，偶数级是「缓冲区」
- **树上推广**：深度为奇数的节点上的石子异或和
- **经典转化**：Georgia and Bob 等看似无关的问题通过配对转化为阶梯 Nim

阶梯 Nim 的巧妙之处在于「等价变换」的思想——把一个看起来更复杂的游戏通过配对的技巧转化成我们已经熟悉的 Nim 游戏。这种转化思想在博弈论中非常常见，也是我们下一篇文章中 SG 定理的核心思路。
