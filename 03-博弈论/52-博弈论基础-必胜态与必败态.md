# 博弈论基础：必胜态与必败态

> Made by Aki.
> 最后更新于 2026.04.30

## 一、引言

想象这样一个场景：你和朋友面对面坐着，桌上有一堆石子，共 7 颗。规则很简单——两人轮流取石子，每次可以取 1 颗或 2 颗，取走最后一颗的人获胜。你先手。

你会怎么玩？直觉上，你可能觉得先手有优势。但真的是这样吗？我们可以把这个游戏的所有状态画出来，看看是否存在一种「必胜策略」。

这就是博弈论的起点——**用状态和转移来分析游戏**。竞赛中绝大多数博弈问题，最终都归结为判断「先手必胜还是必败」，而这一切都建立在本篇文章的核心概念之上：**必胜态与必败态**。

## 三、核心推导

### 3.1 游戏的形式化描述

在讨论博弈之前，我们需要把「游戏」这件事说清楚。竞赛中遇到的博弈问题，通常满足以下条件：

- **双人回合制**：两名玩家轮流行动
- **零和**：一方的收益等于另一方的损失
- **确定信息**：双方都知道游戏的全部状态
- **无随机因素**：不涉及概率
- **有限步内结束**：游戏不会无限进行下去

这类游戏被称为 **Impartial Combinatorial Game**（ impartial 意为「公平」，指双方可走的操作完全相同）。我们所有的讨论都将建立在这一框架之上。

### 3.2 必胜态（N-position）与必败态（P-position）

### 从一个简单例子出发

回到开头的取石子游戏：共 7 颗石子，每人每次取 1 或 2 颗，取走最后一颗胜。

我们来思考一个问题：**假如轮到某人的时候，桌上只剩 1 颗石子，他会是什么处境？**

显然，他可以取 1 颗，拿走最后一颗，直接获胜。所以「还剩 1 颗石子」是**对当前操作者有利的状态**。

那 2 颗呢？他取 2 颗，同样直接获胜。

那 3 颗呢？他如果取 1 颗，留给对手 2 颗——对手就赢了。他如果取 2 颗，留给对手 1 颗——对手还是赢了。无论他怎么走，对手都能在下一轮获胜。所以「还剩 3 颗石子」是**对当前操作者不利的状态**。

我们把前一种状态称为 **N-position**（Next player wins，即「当前操作者必胜」），后一种称为 **P-position**（Previous player wins，即「上一位操作者必胜」，换句话说，「当前操作者必败」）。

> ❓ N 和 P 容易记混怎么办？
> 
> 我的记忆方式：N 代表 Next（轮到谁谁赢），P 代表 Previous（上一手的人赢）。轮到 P 状态时，当前玩家必败，上一手的人必胜——所以上一手的人「赢了就走」，留下 P 态给你。

### 递归定义

更严格地，我们可以递归定义：

$$
\begin{aligned}
&\text{如果一个状态没有任何后继状态（即终止状态），那么它是 P-position。} \\
&\text{如果一个状态存在某个后继状态是 P-position，那么它是 N-position。} \\
&\text{如果一个状态的所有后继状态都是 N-position，那么它是 P-position。}
\end{aligned}
$$

> ❓ 终止状态为什么是 P-position？
> 
> 终止状态意味着游戏已经结束，轮到当前玩家时无棋可走，他直接输了。所以当前玩家必败，当然是 P-position。

### 手算例子：取石子游戏

我们用 $f(n)$ 表示 $n$ 颗石子时当前操作者的胜负状态（N 或 P）。

- $f(0) = P$：0 颗石子，无棋可走，必败。
- $f(1) = N$：可取 1 颗，走到 $f(0)=P$。
- $f(2) = N$：可取 2 颗，走到 $f(0)=P$。
- $f(3) = P$：可取 1→$f(2)=N$，可取 2→$f(1)=N$，所有后继都是 N，所以是 P。
- $f(4) = N$：可取 1→$f(3)=P$，存在后继为 P，所以是 N。
- $f(5) = N$：可取 2→$f(3)=P$，存在后继为 P。
- $f(6) = P$：可取 1→$f(5)=N$，可取 2→$f(4)=N$，全为 N 所以是 P。
- $f(7) = N$：可取 1→$f(6)=P$，所以是 N。

所以先手在 7 颗石子时是必胜的。具体策略：取 1 颗，留给对手 $f(6)=P$ 的状态，即 6 颗石子时轮到对手走——对手无论取 1 还是 2，都会留下 5 或 4（都是 N），你再应机应对。

> ❓ 我看出规律了：$n$ 是 3 的倍数时是 P，否则是 N。
> 
> 没错！这正是巴什博弈的核心结论。我们会在下一篇文章中详细讨论。

### 关键理解

从上例我们可以提炼出博弈分析的核心思想——**不要分析具体怎么走，而是分析状态的本质**。必胜态和必败态的递归定义告诉我们：

- **找到 P-position 就找到了突破口**。因为从 N-position 到 P-position 的一步，就是「必胜策略的一步」。
- **分析博弈树**：如果把所有状态看作节点，操作看作有向边，那么整个游戏就是一个有向图。我们要做的就是在有向图上标记每个节点的 N/P 属性。

### 3.3 状态图与博弈树

对于更复杂的游戏，手算每一步就不现实了。更系统的方式是画出**状态转移图**（博弈树）。

假设我们有 $n$ 颗石子，每次可以取 $1,2,3$ 颗。状态图如下（画出 $n=0$ 到 $n=6$ 的部分）：

```
0(P) ← 1(N) ← 2(N) ← 3(N) ← 4(P) ← 5(N) ← 6(N) ← 7(N) ← 8(P) ...
```

其中箭头表示「可以走到」的关系（比如从 3 可以走到 2、1、0）。

为什么 $f(0)=P$？因为对手已经取走了最后一颗，轮到我们时没石子了，我们输了。

我们可以看到，$n=0,4,8,\dots$ 是 P，其余是 N。这有一个模式：P 态恰好是 $n$ 是 4 的倍数的情况，因为每次最多取 3 颗。

> ❓ 等等，刚才取 1 或 2 时 P 态是 $n$ 是 3 的倍数；现在取 1~3 时 P 态是 $n$ 是 4 的倍数。这之间有什么关系？
> 
> 问得好！我们注意到：P 态之间的间隔等于「最大可取数 + 1」。每次可以取 $1\sim m$ 颗时，P 态就是 $n \bmod (m+1) = 0$。这正是下一篇文章要讲的巴什博弈。

### 3.4 对称策略与模仿策略

在取石子游戏中，我们实际上在用一种更直观的策略思想——**对称策略**。

假设游戏规则改为：两堆石子，每堆 $n$ 颗，每次可以从任意一堆中取任意数量（至少 1 颗，可以全部取走）。后手有什么策略？

后手可以这样玩：**无论先手从哪一堆取多少颗，后手就从另一堆取相同数量**。这样，两堆始终相等。最终后手取走最后一颗获胜。

这不是偶然——不对称的游戏也可以通过分析 N/P 态找到必胜策略。对称策略只是 N/P 态分析的一种直观体现。

### 3.5 博弈分析的通用框架

总结一下，解决一个博弈问题的通用步骤：

1. **定义状态**：把游戏局面用一个或一组变量表示
2. **确定转移**：当前状态可以走到哪些后续状态
3. **标记胜败**：从终止状态出发，逆向递推标记每个状态的 N/P
4. **构造策略**：如果当前是 N，找一步到 P 的操作

在实际竞赛中，第 3 步通常有三种实现方式：
- **手算小数据找规律**（适合简单博弈）
- **DP/记忆化搜索**（适合状态空间有限的博弈）
- **SG 函数**（适合可以分解为子游戏的博弈，后文会讲）

## 四、代码实现

下面是一个用 DP 分析取石子游戏的通用模板。给定 $n$ 颗石子，每次可取集合 $S$ 中的任意颗数，判断每个状态的胜败。

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

const int MAXN = 1e6 + 5;
bool win[MAXN];  // win[i] = true 表示 i 颗石子时先手必胜

int main() {
    int n;          // 石子总数
    int m;          // 可取方案数
    cin >> n >> m;
    vector<int> moves(m);
    for (int i = 0; i < m; i++) cin >> moves[i];

    // 边界：0 颗石子，必败
    win[0] = false;

    // 递推计算 1..n
    for (int i = 1; i <= n; i++) {
        win[i] = false;  // 先假设必败
        for (int x : moves) {
            if (x > i) continue;
            if (win[i - x] == false) {
                // 存在一个后继是必败态 → 当前是必胜态
                win[i] = true;
                break;
            }
        }
    }

    cout << (win[n] ? "First" : "Second") << " wins!" << endl;
    return 0;
}
```

这个模板虽然简单，但体现了博弈 DP 的核心逻辑：**一个状态是必胜态，当且仅当存在一个后继状态是必败态**。

### 一个更通用的记忆化搜索版本

当状态空间不是线性的（比如二维棋盘），用记忆化搜索更方便：

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

// 假设游戏状态用一个整数 state 表示
const int MAX_STATE = 1e5 + 5;
int memo[MAX_STATE];  // -1: 未计算, 0: 必败, 1: 必胜

vector<int> get_next_states(int state) {
    // 根据游戏规则生成所有后继状态
    // 根据具体问题实现
}

bool dfs(int state) {
    if (memo[state] != -1) return memo[state];
    for (int nxt : get_next_states(state)) {
        if (!dfs(nxt)) {
            return memo[state] = 1;  // 存在必败后继 → 必胜
        }
    }
    return memo[state] = 0;  // 所有后继都是必胜 → 必败
}
```

## 六、小结

本篇我们建立了博弈论的基础框架：

- **必胜态（N-position）**：存在一步操作能将对手送入必败态
- **必败态（P-position）**：所有操作都将对手送入必胜态
- 博弈分析的核心在于**状态的 N/P 判定**和**转移**
- 竞赛中最常用的分析工具是：DP 递推 / 找规律 / SG 函数

理解这些概念后，下一讲我们将进入最经典的两种具体博弈模型——巴什博弈和威佐夫博弈。
