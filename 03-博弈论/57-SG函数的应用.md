# SG 函数的应用

> Made by Aki.

## 引言

在上一篇文章中，我们学习了 SG 函数的定义和 Sprague-Grundy 定理。但理论是一回事，实际解题又是另一回事。

SG 函数在竞赛中最常见的应用方式是**打表找规律**——对于多数博弈问题，直接计算 SG 值到很大的范围并不现实（状态空间太大），但 SG 值序列往往具有周期性，找到周期后就可以 $O(1)$ 判断任意状态的 SG 值。

在本篇中，我们将通过一系列经典案例，展示 SG 函数的具体应用技巧。

## 技巧一：直接 DP 计算 SG

### 案例 1：取石子游戏（取子集合给定）

这是 SG 函数最直接的应用。给定可以取的石子数集合 $S$，计算每个石子数 $n$ 的 SG 值。

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

const int MAXN = 10005;
int sg[MAXN];

void compute_sg(int n, const vector<int>& moves) {
    sg[0] = 0;
    for (int i = 1; i <= n; i++) {
        set<int> s;
        for (int x : moves) {
            if (x <= i) s.insert(sg[i - x]);
        }
        int mex = 0;
        while (s.count(mex)) mex++;
        sg[i] = mex;
    }
}
```

**时间复杂度**：$O(n \times |S|)$，其中 $|S|$ 是取子集合的大小。

### 案例 2：多堆取石子游戏（S-Nim）

**问题**（POJ 2960 / S-Nim）：有 $k$ 堆石子，每堆 $a_i$ 颗。每次可以从一堆中取 $x$ 颗，其中 $x$ 属于集合 $S$。问先手胜负。

这就是 S-Nim——每堆石子是一个独立的子游戏，整体 SG 值为各堆 SG 值的异或。

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

const int MAXN = 10005;
int sg[MAXN];

int main() {
    int m;  // 取子集合大小
    while (cin >> m, m) {
        vector<int> moves(m);
        for (int i = 0; i < m; i++) cin >> moves[i];

        // 计算 SG 值，范围到 10000 就够了
        sg[0] = 0;
        for (int i = 1; i < MAXN; i++) {
            set<int> s;
            for (int x : moves) {
                if (x <= i) s.insert(sg[i - x]);
            }
            int mex = 0;
            while (s.count(mex)) mex++;
            sg[i] = mex;
        }

        int t;  // 测试组数
        cin >> t;
        while (t--) {
            int k;  // 堆数
            cin >> k;
            int xor_sum = 0;
            for (int i = 0; i < k; i++) {
                int x;
                cin >> x;
                xor_sum ^= sg[x];
            }
            cout << (xor_sum ? 'W' : 'L');
        }
        cout << endl;
    }
    return 0;
}
```

### 手算例子

取子集合 $S = \{1, 2, 3\}$。我们之前在巴什博弈中知道，这就是每堆可以取 $1 \sim 3$ 颗，$n \bmod 4 = 0$ 时必败。

验证 SG 值：
- $SG(0) = 0$
- $SG(1) = mex(\{SG(0)\}) = mex(\{0\}) = 1$
- $SG(2) = mex(\{SG(1), SG(0)\}) = mex(\{1,0\}) = 2$
- $SG(3) = mex(\{SG(2), SG(1), SG(0)\}) = mex(\{2,1,0\}) = 3$
- $SG(4) = mex(\{SG(3), SG(2), SG(1)\}) = mex(\{3,2,1\}) = 0$
- $SG(5) = mex(\{SG(4), SG(3), SG(2)\}) = mex(\{0,3,2\}) = 1$

模式：$SG(n) = n \bmod 4$。与巴什博弈的结论一致——$n \bmod 4 = 0$ 时 SG 为 0（必败）。

## 技巧二：打表找规律

许多博弈问题的 SG 值序列会呈现周期性。找到周期后，就可以直接 $O(1)$ 计算任意 $n$ 的 SG 值。

### 案例 3：每次取 2 的幂次颗石子

**问题**：一堆石子，每次可以取 $1, 2, 4, 8, 16, \dots$ 颗（2 的幂次）。取走最后一颗胜。

手动计算小数据：

- $SG(0) = 0$
- $SG(1) = mex(\{SG(0)\}) = mex(\{0\}) = 1$
- $SG(2) = mex(\{SG(1), SG(0)\}) = mex(\{1,0\}) = 2$
- $SG(3) = mex(\{SG(2), SG(1)\}) = mex(\{2,1\}) = 0$
- $SG(4) = mex(\{SG(3), SG(2), SG(0)\}) = mex(\{0,2,0\}) = 1$
- $SG(5) = mex(\{SG(4), SG(3), SG(1)\}) = mex(\{1,0,1\}) = 2$
- $SG(6) = mex(\{SG(5), SG(4), SG(2)\}) = mex(\{2,1,2\}) = 0$

模式：$0, 1, 2, 0, 1, 2, 0, \dots$ 周期为 3！

所以规律是：$SG(n) = n \bmod 3$。

> ❓ 这跟每次取 1 或 2 的结果一样！怎么解释？
> 
> 因为 $4$ 及更大的 $2$ 的幂次其实不影响 SG 值的计算——从 $SG(3)$ 开始，所有能取的大数（4, 8, ...）都不产生新的 mex 值，因为 $0,1,2$ 已经被更小的取子覆盖了。最终 mex 只依赖前 $3$ 个值，所以周期为 3。

### 案例 4：每次可以取 1 或 2 或 3 颗——但 SG 值可能不同

等一下，我们上面已经算过取 $\{1,2,3\}$ 的情况，其 SG 序列是 $n \bmod 4$。这与取 2 的幂次的结果不同——尽管两者都包含 $1,2$。

这是因为取子集合的差异导致可到达的 SG 值集合不同，从而 mex 结果不同。

当取子集合为 $\{1,2,4,8,\dots\}$ 时，$n=3$ 只能走到 $SG(2)=2$ 和 $SG(1)=1$，mex 得到 0；而取 $\{1,2,3\}$ 时，$n=3$ 还能走到 $SG(0)=0$，mex 得到 3。

**核心原则**：**SG 值的规律完全由取子集合决定，不同的集合会导出完全不同的周期**。

### 打表代码模板

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

void find_pattern(int limit, const vector<int>& moves) {
    vector<int> sg(limit + 1, 0);
    for (int i = 1; i <= limit; i++) {
        set<int> s;
        for (int x : moves) {
            if (x <= i) s.insert(sg[i - x]);
        }
        int mex = 0;
        while (s.count(mex)) mex++;
        sg[i] = mex;
    }

    // 输出前若干个 SG 值找规律
    for (int i = 0; i <= min(50, limit); i++) {
        cout << i << ": " << sg[i] << endl;
    }
}
```

## 技巧三：经典游戏的 SG 推导

### 案例 5：两堆石子——取相同数量

**问题**：有两堆石子 $(x,y)$，每次可以从两堆中取相同正整数颗（至少 1 颗）。取走最后一颗胜。

这是一个子游戏（两堆一起考虑，不能分别操作）。SG 函数定义为 $SG(x,y)$：

- $SG(0,0) = 0$
- $SG(1,1)$：可以走到 $(0,0)$，SG=0 → $mex(\{0\}) = 1$
- $SG(2,2)$：可以走到 $(0,0), (1,1)$，SG 值集合 $\{0,1\}$ → $mex = 2$
- $SG(k,k)$：可以走到 $(0,0), (1,1), \dots, (k-1,k-1)$，SG 值集合 $\{0,1,\dots,k-1\}$ → $mex = k$

所以 $SG(k,k) = k$。那么 $SG(x,y)$ 其中 $x \ne y$ 呢？

- $SG(0,1)$：只能走到 $(0,0)$，SG=0 → $mex = 1$
- $SG(0,2)$：只能走到 $(0,0), (0,1)$，SG 值 $\{0,1\}$ → $mex = 2$
- $SG(0,k)$：$mex(\{0,1,\dots,k-1\}) = k$

有趣的是，这个游戏的 SG 值规律很简单：$SG(x,y) = \max(x,y)$。

> ❓ 等等，如果 $\max(x,y) = k$，为什么 SG 值就是 $k$？
> 
> 我们来想：假设 $x \le y$，那么 $(x,y)$ 能走到所有 $(x-t, y-t)$ 其中 $1 \le t \le x$。这些后继的 max 值覆盖了 $0,1,\dots,y-1$ 中的一部分。经过完整的数学归纳法可以证明 $SG(x,y) = y$（假设 $y \ge x$）。

### 案例 6：Nim 堆的 SG 值

我们之前说过，Nim 游戏中大小为 $k$ 的一堆石子，其 SG 值为 $k$。现在我们可以用 SG 函数的定义来验证：

对于大小为 $k$ 的 Nim 堆，后继状态的大小可以是 $0,1,\dots,k-1$。根据归纳假设，这些后继的 SG 值分别为 $0,1,\dots,k-1$。所以当前 SG 值为 $mex(\{0,1,\dots,k-1\}) = k$。

这正是 Nim 游戏与 SG 定理完美契合的原因。

## 技巧四：博弈问题的 SG 转化

### 案例 7：棋盘上的移动

**问题**：一个 $n \times m$ 的棋盘，左上角 $(1,1)$ 有一颗棋子。每次可以将棋子向右或向下移动任意格（至少 1 格，但不能超出棋盘）。不能移动者输。

这个游戏可以看作一个二维的 DAG，每个格子是一个状态。我们可以直接计算 SG 值：

- 终点 $(n,m)$：SG = 0（无法移动）
- $(n,m-1)$：只能向右走到 $(n,m)$，后继 SG=0 → mex=1
- $(n-1,m)$：只能向下走到 $(n,m)$，后继 SG=0 → mex=1
- $(n-1,m-1)$：可以走到 $(n,m-1)$ 和 $(n-1,m)$，后继 SG 值 $\{1,1\}$ → mex = 0

继续算下去，会发现一个模式：**SG 值取决于到右下角的曼哈顿距离的奇偶性**？不对，这其实是二维的 Nim，SG 值会更复杂。

但我们可以注意到：向右和向下的移动完全独立！所以这个游戏可以分解为**两个独立子游戏的和**：
- 子游戏 1：在水平方向上的移动（每次向右任意格）
- 子游戏 2：在垂直方向上的移动（每次向下任意格）

水平方向：从位置 $x$ 移动到 $(n,m)$，其实就是一堆大小为 $n-x$ 的石子——可以直接取任意颗。SG 值为 $n-x$。
垂直方向：SG 值为 $m-y$。

整体 SG 值：$(n-x) \oplus (m-y)$。

这就是 SG 定理的威力——把一个二维问题分解为一维子问题。

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
    int n, m, x, y;
    cin >> n >> m >> x >> y;
    // 从 (x,y) 出发走到 (n,m)
    ll sg_val = (n - x) ^ (m - y);
    if (sg_val == 0) {
        cout << "Second wins!" << endl;
    } else {
        cout << "First wins!" << endl;
    }
    return 0;
}
```

### 案例 8：分裂游戏（Multi-SG 的思想）

**问题**：一堆石子，每次操作可以取走若干颗，然后将剩下的石子分成两堆（非空）。取走最后一颗者胜。

这涉及「分裂」操作——一个状态可以分裂为两个独立子游戏。

计算 SG 值：
- $SG(0) = 0$
- $SG(1)$：只能取走 1 颗，无法分裂 → $mex(\{SG(0)\}) = 1$
- $SG(2)$：可以：
  - 取 1 颗，剩余 1 颗不分 → $SG(1)=1$
  - 取 2 颗，剩余 0 → $SG(0)=0$
  - 取 1 颗，剩余 1 颗分成两堆？分成两堆至少每堆 1 颗，但剩余只有 1 颗分不成两堆非空
  后继 SG 值集合 $\{1,0\}$ → $mex = 2$
- $SG(3)$：
  - 取 1 颗，剩 2（不分）→ $SG(2)=2$
  - 取 2 颗，剩 1 → $SG(1)=1$
  - 取 3 颗，剩 0 → $SG(0)=0$
  - 取 1 颗，剩 2 分成 $(1,1)$ → 两个子游戏 SG 值为 $SG(1) \oplus SG(1) = 1 \oplus 1 = 0$
    等等，这里要小心：分裂后的游戏是两个独立子游戏的「和」，它的 SG 值是各子 SG 的异或。
    所以后继 SG 值为 $SG(1)=1$（取走 1 后剩余 1 不分）和 $SG(1) \oplus SG(1) = 0$（取 1 后剩 2 分成 1+1）
  - 取 2 颗，剩 1 → 不分（只有 1 颗分不了），SG=1
  - 取 1 颗，剩 2 也能不分 → SG=2
  所有后继 SG 值：$\{2, 1, 0, 0\text{(来自分裂)}\} = \{0,1,2\}$ → $mex = 3$

继续算下去，会发现 $SG(n) = n$。这个游戏的 SG 值跟普通 Nim 一样！

> ❓ 分裂不影响结果？
> 
> 对，因为「分成两堆」实际上给了对手更多的操作空间，但 SG 值的计算已经自动考虑了这种影响。在这个具体例子中，分裂并没有改变 SG 值，但并不是所有分裂游戏都如此——这就是下一讲要讨论的 Multi-SG。

## 实战：综合运用

### 问题：棋盘格游戏

**题意**：一个 $1 \times n$ 的棋盘，每个格子要么空着要么有一个棋子。每次可以选择一个棋子向左移动任意格，不能越过其他棋子，不能移动者输。求先手胜负。

**分析**：这是我们在阶梯 Nim 中遇到过的 Georgia and Bob 问题。现在用 SG 函数重新理解。

将棋子按位置排序 $p_1 < p_2 < \dots < p_k$。每两个相邻棋子之间的间隔可以看作一个 Nim 堆？不完全是——因为移动一个棋子会同时改变两个间隔。

正确的分解方式是：将棋子两两配对。在第 $i$ 对中（两个相邻的棋子），它们之间的空格数决定了这对棋子的 SG 值。经过计算，每个这样的间隔就等价于一个 Nim 堆，大小等于间隔数。所以整体 SG 值就是所有间隔的异或。

## 小结

SG 函数的应用技巧可以总结为：

1. **直接 DP**：状态空间不大时，直接计算 SG 值并异或
2. **打表找规律**：计算小范围 SG 值，观察周期或数学公式
3. **分解为子游戏**：用 SG 定理将复杂问题拆解
4. **转化思想**：看似不同的问题往往可以转化为已知模型（Nim、巴什等）

SG 函数并不是一个固定的代码模板，而是一种**分析框架**。掌握了这个框架，你可以系统地解决任何 impartial 组合博弈问题。
