# Every-SG、Multi-SG 与 Moore's Nim

> Made by Aki.
> 最后更新于 2026.04.30

## 引言

前面我们学到的 SG 定理处理的是「选择其中一个子游戏进行一步操作」的回合制博弈。但在竞赛中，你可能会遇到三种进阶变体：

1. **Every-SG**：每个回合，**所有**还在进行中的子游戏**都必须**各走一步
2. **Multi-SG**：一步操作可以**将一个子游戏分裂为多个子游戏**
3. **Moore's Nim / NimK**：每次可以**同时在多堆中取石子**

这三种变体在经典博弈的基础上增加了新的维度，但各自的解决方法又有着精妙的统一性。

## Every-SG

### 问题定义

**Every-SG**（有时也称 **All-SG** 或 **Total Nim**）：游戏由多个子游戏组成。在每一回合中，当前玩家必须**在所有尚未结束的子游戏中各走一步**。如果一个子游戏已经结束（没有合法的操作），则在后续回合中跳过它。

### 胜负判定

在 Every-SG 中，仅靠 SG 值的异或是不够的，因为每个回合玩家必须操作**所有**子游戏。这意味着游戏持续的时间取决于**最长**的那个子游戏——先手希望延长自己必胜的子游戏、缩短自己必败的子游戏，后手则相反。

**Every-SG 的胜负判定**：
- 如果存在一个子游戏对于先手是必败的（SG=0），那么先手是否必败？不一定。
- 关键变量是每个子游戏的 **step 值**——即从当前状态到游戏结束的**最长步数**（假设双方都采用最优策略）。

更精确地说：

1. 先计算每个子游戏的 SG 值
2. 再计算每个子游戏的 **step 值**：
   - 终止状态：$step = 0$
   - 必胜态（SG ≠ 0）：$step = \max(\text{所有必败后继的 step}) + 1$
   - 必败态（SG = 0）：$step = \min(\text{所有后继的 step}) + 1$
3. 如果存在一个子游戏的 SG = 0（先手必败），且该子游戏的 step 值是**全局最大**的，则先手可以在「必败」的子游戏中拖延时间，在「必胜」的子游戏中加速获胜——综合结果是先手可能还是能赢。

**Every-SG 的最终定理**：
- 先手获胜当且仅当**所有 SG 值最大的子游戏（即 SG 值最大的那些子游戏）的 step 值的最大值是奇数**？

实际上，Every-SG 的结论更简单：**先手胜当且仅当存在子游戏 SG ≠ 0？** 不，不对。

正确的 Every-SG 判定方法是：

1. 设 $maxStep$ 为所有子游戏中 step 值的最大值
2. 如果 $maxStep$ 为奇数，则先手胜；为偶数则后手胜

注意，这里 step 值的计算依赖于双方都采用最优策略——在必胜态下尽量延长游戏，在必败态下尽量缩短游戏。

### step 值的计算

```cpp
// 计算 SG 值和 step 值
int sg[MAXN], step[MAXN];

void compute(int limit) {
    sg[0] = 0; step[0] = 0;
    for (int i = 1; i <= limit; i++) {
        set<int> s;
        int max_step_lose = -1;   // 必败后继的最大 step
        int min_step_win = INT_MAX; // 必胜后继的最小 step

        for (int move : moves) {
            if (move > i) continue;
            int j = i - move;
            s.insert(sg[j]);
            if (sg[j] == 0) {
                // 必败后继
                max_step_lose = max(max_step_lose, step[j]);
            } else {
                // 必胜后继
                min_step_win = min(min_step_win, step[j]);
            }
        }

        int mex = 0;
        while (s.count(mex)) mex++;
        sg[i] = mex;

        if (sg[i] != 0) {
            // 必胜态：走一步到必败态，且走最慢的那个
            step[i] = max_step_lose + 1;
        } else {
            // 必败态：所有后继都是必胜态，走最快的那个
            step[i] = min_step_win + 1;
        }
    }
}
```

### 手算例子

考虑一个 Every-SG 游戏，包含两个独立的 Nim 堆：
- 堆 A：大小为 1
- 堆 B：大小为 2

在 Every-SG 规则下，每回合必须从两个堆中各取石子（只要堆还没空）。

先算每个堆的 SG 和 step 值（作为独立的 Nim 游戏）：
- 堆 A（n=1）：$SG=1, step=?$
  - 可以走到 0（就是取走整堆），0 是必败态，step(0)=0
  - SG=1（必胜），$step(1) = \max(\text{必败后继 step}) + 1 = 0 + 1 = 1$
- 堆 B（n=2）：$SG=2, step=?$
  - 可以走到 0（取完）和 1（取 1 颗）
  - 0 是必败态，step(0)=0
  - 1 是必胜态（SG=1≠0），step(1)=1
  - SG=2（必胜），$step(2) = \max(\text{必败后继 step}) + 1 = 0 + 1 = 1$

两个子游戏的 step 值都是 1，最大值 = 1（奇数）→ 先手胜。

验证一下：每回合必须从两个堆中各取石子。

初始局面：（1, 2）
- 先手从 A 取走 1（A 结束），从 B 取走 1 → 局面变为（0, 1）
- 后手从 B 取走 1（B 结束）→ 所有游戏结束，后手完成了最后一步 → 先手胜。

等等，在 Every-SG 中，谁完成最后一步？当所有子游戏都结束后，无法进行操作的玩家输。所以上一步完成所有子游戏的玩家获胜。

这里后手取走了 B 的最后一颗石子，所有子游戏结束，轮到先手无棋可走——先手输？！

让我们重新分析：初始（A=1, B=2），先手从 A 取 1（只剩 B=2），从 B 取 1（B=1）。局面变为（A=0, B=1）。

A 已经结束，所以下一回合后手只需要（也必须）从 B 操作：从 B 取 1 → 局面变为（0, 0）。所有子游戏结束，接下来轮到先手无棋可走 → 先手输。

所以按照这个走法先手输了！那先手有没有更好的走法？

先手从 A 取 1，从 B 取 2（取完），变为（0, 0）。先手自己结束了所有子游戏，轮到后手无棋可走 → 后手输。

所以先手应该取走 B 的全部石子，而不是只取 1 颗。

> ❓ 等等，所以 Every-SG 其实就是「每个子游戏独立操作，但必须同时在所有活着的子游戏中各走一步」？那先手可以直接从 B 取 2 让 B 结束，这样对面就只剩 A=1 要操作了？
>
> 对的！Every-SG 的关键在于：先手可以决定让哪些子游戏「早点结束」，哪些「晚点结束」。如果先手能**在自己处于必胜的子游戏中拖延时间，在必败的子游戏中迅速结束**，就有可能整体获胜。
>
> 在这个例子中，B（n=2）先手必胜（SG≠0），但先手选择在 B 上快速结束（取 2 颗），因为让 B 结束后，对手只需要在 A（n=1）上操作，而 A 的下一步就是结束——但结束的是对手，所以对手最后一步完成后轮到先手无棋可走？不对——
>
> 重新想：每回合每个活着的子游戏都必须走一步。初始两堆都活着。先手从 A 取 1（A 结束）、从 B 取 2（B 结束）——所有子游戏结束。轮到后手无棋可走 → 后手输！
>
> 所以这个局面的确是先手胜。$maxStep$ 的逻辑是正确的——B 的 step = 1，A 的 step = 1，最大值 1（奇数）→ 先手胜。

### 竞赛应用

Every-SG 在竞赛中出现的频率不如标准 SG 高，但一旦出现，就是区分度的来源。关键在于**正确计算每个子游戏的 step 值**。

**典型题目**：洛谷 P4279 - 小约翰的游戏（原 HDU 3595 / POJ 3480 同类题）

这道题中，每个子游戏是「每次可以从一堆中取另一堆数量的倍数颗石子」——即 Euclid's Game（欧几里得游戏）的 Every-SG 版本。

```cpp
// 洛谷 P4279 (Misère Nim) 的核心代码框架
#include <bits/stdc++.h>
using namespace std;

// 计算 (a,b) 的 SG 和 step
pair<int, int> compute(int a, int b) {
    if (a < b) swap(a, b);
    if (b == 0) return {0, 0};  // SG=0, step=0

    auto [sg_next, step_next] = compute(b, a % b);
    // 后继的 SG 和 step

    int k = a / b;
    if (k >= 2) {
        // 可以一步跳到多个不同的状态
        // 这里利用欧几里得游戏的特殊性质
        // 当 k >= 2 时，当前玩家可以控制胜负
        return {1, step_next + 1};
    } else {
        // k == 1，只能走到 compute(b, a%b)
        int sg = (sg_next == 0) ? 1 : 0;
        int step = (sg == 1) ? (step_next + 1) : (step_next + 1);
        // 如果当前必胜，step = max(所有必败后继 step) + 1
        // 如果当前必败，step = min(所有后继 step) + 1
        // 由于只有唯一后继，step 就是 step_next + 1
        return {sg, step + 1};
    }
}
```

## Multi-SG

### 问题定义

**Multi-SG**（多重 SG）：在普通 SG 的基础上增加一种操作——可以将当前子游戏**分裂**为多个子游戏。分裂后的多个子游戏是独立的，整体 SG 值为各子游戏 SG 值的异或。

### 与普通 SG 的关系

普通 SG 中，一个状态的 SG 值是 $mex(\{SG(所有后继)\})$。

在 Multi-SG 中，后继状态可能是一个**子游戏集合**（因为分裂）。一个状态 $x$ 可以：
1. 走一步到另一个状态 $y$（普通操作）
2. 分裂成两个或多个状态 $(y_1, y_2, \dots, y_k)$（分裂操作），此时整体 SG 值为 $SG(y_1) \oplus SG(y_2) \oplus \dots \oplus SG(y_k)$

所以 Multi-SG 的 SG 值递推公式：

$$
SG(x) = mex(\{SG(y) \mid y \text{ 是普通后继}\} \cup \{SG(y_1) \oplus SG(y_2) \oplus \dots \oplus SG(y_k) \mid \text{分裂后继}\})
$$

### 手算例子：每次取若干颗，剩下的可以分成两堆

我们之前在 #57 中已经遇到过分裂游戏（每次取走石子后，剩余石子必须或不必须分成两堆）。那里我们发现 $SG(n) = n$——分裂不影响 SG 值。

但那是特定规则下的结果。现在我们来看一个真正改变 SG 值的 Multi-SG。

**规则**：一堆石子，每次操作可以：
1. 取走 1 颗石子
2. 将一堆石子分成三堆非空石子（每堆至少 1 颗）

取走最后一颗胜。注意这里「取走 1 颗」和「分裂」是两种不同的操作。

计算 SG 值：

- $SG(0) = 0$
- $SG(1)$：只能取走 1 颗（到 0）→ $mex(\{SG(0)\}) = mex(\{0\}) = 1$
- $SG(2)$：只能取走 1 颗（到 1），不能分成三堆（每堆至少 1，3 颗才够）→ $mex(\{SG(1)\}) = mex(\{1\}) = 0$
- $SG(3)$：
  - 取 1 → 到 2，SG=0
  - 分三堆：只能分成 (1,1,1)，整体 SG = $1 \oplus 1 \oplus 1 = 1$
  - 后继 SG 值集合 $\{0, 1\}$ → $mex(\{0,1\}) = 2$
- $SG(4)$：
  - 取 1 → 到 3，SG=2
  - 分三堆：(1,1,2)，整体 SG = $1 \oplus 1 \oplus 0 = 0$
  - 后继 SG 值集合 $\{2, 0\}$ → $mex(\{0,2\}) = 1$
- $SG(5)$：
  - 取 1 → 到 4，SG=1
  - 分三堆：(1,1,3)，整体 SG = $1 \oplus 1 \oplus 2 = 2$
  - 分三堆：(1,2,2)，整体 SG = $1 \oplus 0 \oplus 0 = 1$
  - 后继 SG 值集合 $\{1, 2, 1\} = \{1, 2\}$ → $mex(\{1,2\}) = 0$

SG 序列：$0, 1, 0, 2, 1, 0, \dots$

这个序列不同于普通 Nim（$SG(n) = n$），也不同于巴什（$n \bmod 4$）。它有自己的规律。打表发现周期为 4：$n \bmod 4 = 0$ 时 SG=0，$n \bmod 4 = 1$ 时 SG=1，$n \bmod 4 = 2$ 时 SG=0，$n \bmod 4 = 3$ 时 SG=2。

所以分裂操作确实改变了 SG 值的结构。

> ❓ 那 Multi-SG 在竞赛中怎么处理？总不能每次都手算打表吧？
>
> 对的，Multi-SG 的典型做法就是**打表找规律**。先计算小范围的 SG 值，观察周期或递推关系，然后写出 $O(1)$ 的判定公式。分裂操作往往会产生美妙的数学规律。

## Moore's Nim / NimK

### 问题定义

**Moore's Nim**（也称为 **NimK** 或 **Moore's Nim**）：有 $n$ 堆石子，每次可以选择**最多 $k$ 堆**（$k \ge 1$），从每堆中取任意正整数颗（可以不同数量）。取走最后一颗石子者胜。

当 $k = 1$ 时，就是标准 Nim。

当 $k = n$ 时，就是 Moore's Nim 的极端情况——每次可以从所有堆中取石子（但最多从 $n$ 堆中取），这意味着每次可以选择任意多堆。

### 判定定理

**Moore's Nim / NimK 定理**：
将每堆石子数写成二进制。从低位到高位，如果**每个二进制位上 1 的个数模 $(k+1)$ 都等于 0**，则先手必败；否则先手必胜。

$$
\text{先手必败} \iff \forall j \ge 0, \left(\sum_{i=1}^{n} bit_j(a_i)\right) \bmod (k+1) = 0
$$

其中 $bit_j(a_i)$ 是 $a_i$ 的第 $j$ 个二进制位（0 或 1）。

### 手算例子

**例 1：标准 Nim（k=1）**

要求每个二进制位上 1 的个数模 2 = 0——这就是所有位的异或和为 0。与 Bouton 定理一致。

**例 2：k=2，两堆石子 [3, 3]**

二进制：
- 3 = 011（二进制位 2^1 和 2^0）
- 3 = 011

每个位上的 1 的个数：
- 位 0：2 个 1，模 3 = 2 ≠ 0 → 先手必胜

验证：[3, 3] 在 k=2 时，先手可以从两堆各取 1 颗 → [2, 2]：二进制 010 和 010
- 位 0：0 个 1，模 3 = 0
- 位 1：2 个 1，模 3 = 2 ≠ 0

仍然非零？那从一堆取 2 颗，一堆取 1 颗呢？[1, 2]：001 和 010
- 位 0：1 个 1，模 3 = 1 ≠ 0

再试：[3, 3] 取成 [0, 0]？可以，规则是「最多 k 堆」——取完两堆所有石子！先手直接获胜。

所以 [3, 3] 确实是先手必胜 ✓

**例 3：k=2，三堆石子 [1, 2, 3]**

二进制：001, 010, 011
- 位 0：1（001）+ 0（010）+ 1（011）= 2 个 1，模 3 = 2 ≠ 0 → 先手必胜

验证：先手可以从三堆中各取一些，目标是使每个二进制位上的 1 个数模 3 = 0。

在位 0 上有 2 个 1，需要变成 0 或 3 个 1（模 3 为 0）。在位 1 上有 2 个 1（010 和 011），同样需要调整。

如果从堆 3（3 颗）取走 2 颗 → [1, 2, 1]：001, 010, 001
- 位 0：2 个 1，模 3 = 2 ≠ 0

如果从堆 3 取 1 颗 → [1, 2, 2]：001, 010, 010
- 位 0：1 个 1，模 3 = 1 ≠ 0

如果从堆 2 取走 2 → [1, 0, 3]：001, 000, 011
- 位 0：2 个 1，模 3 = 2 ≠ 0

看起来不太好找。让我们用定理来找策略：当前局面各位上 1 的个数：
- 位 0：2
- 位 1：2

我们需要在每个位上让 1 的个数变成 0 或 3（模 3 = 0），但总共只有 3 堆，最多调整 3 堆。

可以同时从 2 堆或 3 堆中取石子。从堆 1（001）取走 1 → [0, 2, 3]：000, 010, 011
- 位 0：1，模 3 = 1

从堆 1 取 1，从堆 2 取 2 → [0, 0, 3]：000, 000, 011
- 位 0：1（来自 011）

从堆 1 取 1，从堆 3 取 2 → [0, 2, 1]：000, 010, 001
- 位 0：1

从堆 2 取 2，从堆 3 取 1 → [1, 0, 2]：001, 000, 010
- 位 0：1，位 1：1

从三堆都取：堆 1 取 1，堆 2 取 1，堆 3 取 2 → [0, 1, 1]：000, 001, 001
- 位 0：2

从三堆都取：堆 1 取 1，堆 2 取 2，堆 3 取 3（全取）→ [0, 0, 0]
- 所有位 0，模 3 = 0 ✓

所以直接取走所有石子就赢了！答案是先手必胜 ✓

**例 4：k=2，各二进制位 1 的个数都是 3 的倍数**

[1, 2, 3, 4]：001, 010, 011, 100
- 位 0：001 + 011 = 2 个 1，模 3 = 2 ≠ 0

找一个是真正的 NimK 必败态：
[7, 7, 7]：111, 111, 111
- 位 0：3 个 1，模 3 = 0
- 位 1：3 个 1，模 3 = 0
- 位 2：3 个 1，模 3 = 0

所以 [7, 7, 7] 在 k=2 下是必败态！

验证：先手可以从最多两堆中取石子。无论怎么取，总有一些二进制位上 1 的个数不再是 3 的倍数。

具体走法及对手回应省略，但定理保证了这个结论。

> ❓ 为什么是模 (k+1) 而不是别的数？
>
> 这个定理的证明依赖于一点：当所有位模 (k+1) 为 0 时，一次最多操作 k 堆，每堆可以改变任意多个二进制位，但**最多只能在 k 堆的范围内改变**，所以不可能在所有位上保持模 (k+1) 为 0。反过来，如果存在某个位模 (k+1) 不为 0，总可以找到一种取法使其变为 0。

### NimK 的代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

bool nimk_win(int n, int k, const vector<ll>& a) {
    int max_bit = 0;
    for (ll x : a) {
        int b = 64 - __builtin_clzll(x);  // 最高位
        max_bit = max(max_bit, b);
    }

    for (int j = 0; j < max_bit; j++) {
        int cnt = 0;
        for (ll x : a) {
            if ((x >> j) & 1) cnt++;
        }
        if (cnt % (k + 1) != 0) return true;  // 先手必胜
    }
    return false;  // 先手必败
}

int main() {
    int n, k;
    cin >> n >> k;
    vector<ll> a(n);
    for (int i = 0; i < n; i++) cin >> a[i];
    cout << (nimk_win(n, k, a) ? "First" : "Second") << " wins!" << endl;
    return 0;
}
```

复杂度：$O(n \cdot \log \max a_i)$。

## 三种进阶变体的对比

| 变体 | 核心规则 | 判定方法 |
|:---|:---|:---|
| **Every-SG** | 每回合所有子游戏各走一步 | 计算 step 值，maxStep 为奇数则先手胜 |
| **Multi-SG** | 可以分裂子游戏 | SG 递推（含分裂后的异或和），打表找规律 |
| **Moore's Nim** | 每次最多操作 k 堆 | 每位二进制 1 的个数模 (k+1) 为 0 则必败 |

## 小结

三种进阶博弈变体展示了 SG 理论的延展性：

- **Every-SG** 打破了「每回合只动一个子游戏」的限制，引入了 step 值来量化游戏的「长度」，通过控制游戏节奏（加速/拖延）来获得全局胜利
- **Multi-SG** 引入了分裂操作，仍然可以通过 SG 函数分析，但递推式需要包含分裂后的异或和——打表找规律是常用策略
- **Moore's Nim / NimK** 将 Nim 推广到「每次操作多堆」的情况，判定条件从「异或为 0」推广到「每位 1 的个数模 (k+1) 为 0」

这三种变体虽然各自有不同的规则和结论，但都建立在 SG 函数和 Nim 的基本框架之上——理解基础越扎实，面对变体时的适应能力就越强。
