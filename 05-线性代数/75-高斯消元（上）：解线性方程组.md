## 一、引言

小明去水果店买了 3 个苹果、2 个香蕉和 1 个橙子，花了 14 元。小红买了 1 个苹果、1 个香蕉和 2 个橙子，花了 11 元。小刚买了 2 个苹果、3 个香蕉和 1 个橙子，花了 15 元。问苹果、香蕉、橙子各多少钱？

设价格分别为 $x,y,z$，方程组如下：

$$
\begin{cases}
3x + 2y + z = 14 \\
x + y + 2z = 11 \\
2x + 3y + z = 15
\end{cases}
$$

初中我们学过代入消元法和加减消元法。但如果变量个数扩大到几十上百个呢？手算显然不现实。

这就是高斯消元法要解决的问题——一个系统化的、适用于计算机的消元算法。它可能是线性代数中最实用的算法了：从解方程组、算行列式、求逆矩阵到图论中的概率计算，到处都有它的身影。

## 三、核心推导

### 3.1 增广矩阵

先来看上面的三元方程组。把系数写成矩阵 $A$，未知数 $x$，常数 $b$：

$$
A = \begin{bmatrix}
3 & 2 & 1 \\
1 & 1 & 2 \\
2 & 3 & 1
\end{bmatrix}, \quad
x = \begin{bmatrix} x \\ y \\ z \end{bmatrix}, \quad
b = \begin{bmatrix} 14 \\ 11 \\ 15 \end{bmatrix}
$$

方程组就是 $Ax = b$。把 $A$ 和 $b$ 拼接在一起，得到**增广矩阵** $[A \mid b]$：

$$
[A \mid b] = \left[\begin{array}{ccc|c}
3 & 2 & 1 & 14 \\
1 & 1 & 2 & 11 \\
2 & 3 & 1 & 15
\end{array}\right]
$$

我们要做的事，就是通过对这个矩阵进行**行变换**，把它变成一种可以直接读出解的形式。

### 3.2 三种基本行变换

高斯消元只允许三种行变换操作，这三种变换不会改变方程组的解：

1. **交换两行**：$R_i \leftrightarrow R_j$
2. **某行乘以非零常数**：$R_i \leftarrow k \cdot R_i$（$k \neq 0$）
3. **某行加上另一行的倍数**：$R_i \leftarrow R_i + k \cdot R_j$

这三种操作的组合，足以把任意矩阵化简为**行阶梯形**甚至**行最简形**。

### 3.3 消元过程——手动模拟

我们用上面的例子，一步步消元。

**第 1 步**：找到第 1 列中绝对值最大的元素（叫**主元**）所在的行。

第 1 列是 $3, 1, 2$，绝对值最大的是 $3$，在第 1 行。所以直接以第 1 行为基准行，不需要交换。

用第 1 行消去其他行的第 1 列。第 2 行第 1 列是 $1$，要让 $1 \to 0$：$R_2 \leftarrow R_2 - \frac{1}{3}R_1$。第 3 行第 1 列是 $2$：$R_3 \leftarrow R_3 - \frac{2}{3}R_1$。

计算过程：

$$
R_2 - \frac{1}{3}R_1: \quad [1-\frac{3}{3},\; 1-\frac{2}{3},\; 2-\frac{1}{3},\; 11-\frac{14}{3}] = [0,\; \frac{1}{3},\; \frac{5}{3},\; \frac{19}{3}]
$$
$$
R_3 - \frac{2}{3}R_1: \quad [2-\frac{6}{3},\; 3-\frac{4}{3},\; 1-\frac{2}{3},\; 15-\frac{28}{3}] = [0,\; \frac{5}{3},\; \frac{1}{3},\; \frac{17}{3}]
$$

矩阵变成：

$$
\left[\begin{array}{ccc|c}
3 & 2 & 1 & 14 \\
0 & \frac{1}{3} & \frac{5}{3} & \frac{19}{3} \\
0 & \frac{5}{3} & \frac{1}{3} & \frac{17}{3}
\end{array}\right]
$$

**第 2 步**：看第 2 行第 2 列开始的子矩阵，找第 2 列（从第 2 行开始往下看）绝对值最大的元素做主元。

第 2 列剩余部分：$\frac{1}{3}, \frac{5}{3}$，最大是 $\frac{5}{3}$ 在第 3 行，交换 $R_2 \leftrightarrow R_3$：

$$
\left[\begin{array}{ccc|c}
3 & 2 & 1 & 14 \\
0 & \frac{5}{3} & \frac{1}{3} & \frac{17}{3} \\
0 & \frac{1}{3} & \frac{5}{3} & \frac{19}{3}
\end{array}\right]
$$

用第 2 行消去第 3 行的第 2 列：$R_3 \leftarrow R_3 - \frac{1/3}{5/3}R_2 = R_3 - \frac{1}{5}R_2$。

$$
R_3 - \frac{1}{5}R_2: \quad [0,\; \frac{1}{3}-\frac{1}{5}\cdot\frac{5}{3},\; \frac{5}{3}-\frac{1}{5}\cdot\frac{1}{3},\; \frac{19}{3}-\frac{1}{5}\cdot\frac{17}{3}] = [0,\; 0,\; \frac{24}{15},\; \frac{78}{15}]
$$

最终得到：

$$
\left[\begin{array}{ccc|c}
3 & 2 & 1 & 14 \\
0 & \frac{5}{3} & \frac{1}{3} & \frac{17}{3} \\
0 & 0 & \frac{24}{15} & \frac{78}{15}
\end{array}\right]
$$

这就是**行阶梯形**（Row Echelon Form, REF）——从左上到右下呈阶梯状，每行的第一个非零元素（主元）在下一行主元的左边。

### 3.4 回代求解

从行阶梯形可以直接回代求解。第 3 行对应：

$$
\frac{24}{15}z = \frac{78}{15} \implies z = \frac{78}{24} = 3.25
$$

代入第 2 行：

$$
\frac{5}{3}y + \frac{1}{3} \cdot 3.25 = \frac{17}{3} \implies y = \frac{17 - 3.25}{5} = 2.75
$$

代入第 1 行：

$$
3x + 2\cdot 2.75 + 3.25 = 14 \implies x = \frac{14 - 5.5 - 3.25}{3} = 1.75
$$

所以答案：苹果 1.75 元，香蕉 2.75 元，橙子 3.25 元。

❓ **消元过程中出现了分数，能不能避免？**

在实数（浮点数）计算时，分数不可避免，我们用 double 存储即可。但是如果是在模 $p$ 意义下，我们也可以用乘法逆元避免分数。后面第 76 篇会讲。

另一种思路是使用**整数高斯消元**（辗转相除法消元），只用整数运算而不产生分数，但代价是数值可能变得非常大。

### 3.5 列主元选取——为什么要选大的主元？

你可能注意到了，我们在每一步都选了「当前列中绝对值最大的元素」做主元。为什么不直接按顺序消元？

来看一个例子。方程组：

$$
\begin{cases}
10^{-8}x + y = 1 \\
x + y = 2
\end{cases}
$$

如果不交换行，直接用第 1 行的 $10^{-8}$ 做主元消去第 2 行的 $x$，消元因子是 $\frac{1}{10^{-8}} = 10^8$。由于浮点数精度有限，$10^8$ 乘以第 1 行后再减到第 2 行会导致灾难性的精度损失。

这就是**列主元选取**（Partial Pivoting）的必要性——通过选绝对值最大的主元来最小化浮点误差。

### 3.6 无解与无穷多解的判定

不是所有线性方程组都有唯一解。在高斯消元的过程中，我们可以通过阶梯形矩阵来判断：

**无解**：消元过程中出现某一行形如 $[0\;0\;\cdots\;0 \mid c]$，其中 $c \neq 0$。这对应方程 $0 = c$，显然无解。

**无穷多解**：消元完成后，有效方程数（非零行数）少于变量数。这种情况下，自由变量的个数 = 变量总数 - 矩阵的秩。

例如方程组：

$$
\begin{cases}
x + y + z = 1 \\
2x + 2y + 2z = 2
\end{cases}
$$

消元后第 2 行会变成 $[0\;0\;0 \mid 0]$，实际只有 1 个有效方程。3 个变量，1 个有效方程，有 2 个自由变量，无穷多解。

## 四、代码实现

```cpp
#include <bits/stdc++.h>
using ll = long long;
using db = double;
const db EPS = 1e-9;

// 高斯消元解线性方程组
// a 是增广矩阵 [A | b]，n 个方程，m 个变量 (m = n 通常)
// 返回: 0-唯一解, 1-无穷多解, -1-无解
int gauss(vector<vector<db>> &a, vector<db> &sol) {
    int n = (int)a.size();      // 方程数
    int m = (int)a[0].size() - 1; // 变量数
    vector<int> where(m, -1);   // 第 i 个变量的主元在第几行

    int row = 0;
    for (int col = 0; col < m && row < n; col++) {
        // 选主元：找第 col 列中绝对值最大的行
        int sel = row;
        for (int i = row; i < n; i++)
            if (fabs(a[i][col]) > fabs(a[sel][col]))
                sel = i;
        if (fabs(a[sel][col]) < EPS) continue;  // 该列全为 0，跳过
        // 交换到当前行
        swap(a[row], a[sel]);
        where[col] = row;

        // 主元归一（可选：消元时可以把主元所在行整行归一）
        // 但不必须，我们可以在回代时处理

        // 消去其他行的第 col 列
        for (int i = 0; i < n; i++) {
            if (i != row) {
                db c = a[i][col] / a[row][col];
                for (int j = col; j <= m; j++)
                    a[i][j] -= a[row][j] * c;
            }
        }
        row++;
    }

    // 检查无解
    for (int i = row; i < n; i++)
        if (fabs(a[i][m]) > EPS) return -1;

    // 检查无穷解
    if (row < m) return 1;

    // 唯一解 —— 回代求出解
    sol.assign(m, 0);
    for (int i = 0; i < m; i++)
        sol[i] = a[where[i]][m] / a[where[i]][i];
    return 0;
}
```

**使用示例**：

```cpp
int main() {
    int n = 3;
    vector<vector<db>> a = {
        {3, 2, 1, 14},
        {1, 1, 2, 11},
        {2, 3, 1, 15}
    };
    vector<db> sol;
    int res = gauss(a, sol);
    if (res == 0) {
        for (int i = 0; i < n; i++)
            cout << "x" << i << " = " << sol[i] << "\n";
    }
}
```

代码的逻辑等价于我们手动模拟的过程：逐列扫描 -> 选主元 -> 交换 -> 消元。注意这里我们用了「全消元」的方式（消元时同时消去上方和下方的行），这样消元完成后对角线上就是主元，回代非常方便。

## 五、复杂度与正确性分析

**时间复杂度**：$O(n^3)$。对每一列，我们扫描 $O(n)$ 行选主元，然后对剩余的 $O(n)$ 行各做一次 $O(n)$ 列的消元操作。总操作次数约 $\frac{2}{3}n^3$。

**空间复杂度**：$O(n^2)$，存储增广矩阵。

**数值稳定性**：列主元选取保证了主元的绝对值是当前列中的最大值，从而最小化了消元因子 $c = a_{ik}/a_{kk}$ 的绝对值，减少了浮点数误差。

**正确性**：三种行变换保持解集不变，这是线代中基本定理。经过这些变换得到的阶梯形矩阵与原矩阵同解。

## 六、典型应用场景

1. **解线性方程组**：最常见的用途，电路分析、化学方程配平等
2. **图上期望/概率的计算**：对于有环的期望 DP，通常需要列线性方程组求解（参见第 71 篇）
3. **求解线性规划**：单纯形法中也用到消元思想

## 七、小结

高斯消元的核心思想可以用三个词概括：**主元、消元、回代**。它把解线性方程组这个看起来复杂的问题，变成了一个可以机械化执行的算法，时间复杂度 $O(n^3)$。

今天我们学习了实数域下的高斯消元——选列主元、消元到行阶梯形、回代求解，以及无解和无穷多解的判定。但高斯消元的用处远不止于此。
