## 一、引言

在打 Codeforces 的时候，你遇到了一道计数题，数据范围是 $n \le 10^{18}$。你尝试暴力计算了小规模的结果：$n = 0, 1, 2, \dots, 10$ 时的答案依次是 $1, 2, 4, 7, 13, 24, 44, 81, 149, 274, 504$。

这东西看起来可能有递推关系。手动观察一下：$4 = 1 + 2 + 1?$ 不对；$7 = 4 + 2 + 1?$ 好像猜对了。$13 = 7 + 4 + 2?$ 也对。$24 = 13 + 7 + 4?$ 也对。所以递推式似乎是 $a_n = a_{n-1} + a_{n-2} + a_{n-3}$，三阶递推。

但如果你碰到一个更复杂的数列，肉眼看不出来递推式呢？或者递推阶数很高（比如 20 阶）呢？

这时就需要 **Berlekamp-Massey 算法**（简称 BM 算法）。它能根据数列的前若干项，自动求出最小的线性递推式。然后配合第 74 篇的矩阵快速幂或第 79 篇的 Kitamasa 方法，就能 $O(k^2 \log n)$ 求出第 $n$ 项。

竞赛中 BM 算法最常见的套路是**打表 + BM + 快速求第 n 项**：先写暴力在小范围算好前几十项，喂给 BM 求出递推式，然后利用递推式求 $n$ 很大时的结果。

## 三、核心推导

### 3.1 问题定义

给定一个数列 $\{a_0, a_1, a_2, \dots, a_{N-1}\}$。我们希望找到一个最短的递推式：

$$
a_i = c_1 a_{i-1} + c_2 a_{i-2} + \cdots + c_k a_{i-k}, \quad \forall i \ge k
$$

使得前 $N$ 项都满足这个递推关系（或者说，递推式「拟合」已知数据）。这里的 $k$ 就是递推的阶数，$c_1, c_2, \dots, c_k$ 是递推系数。

BM 算法在已知项数 $N$ 是递推阶数 $k$ 的大约 2 倍时能正确工作（$N \ge 2k$）。

### 3.2 从递推失败开始

BM 算法的核心思想很有趣：我们不是一开始就猜一个递推式，而是**不断修正**它。

我们从空递推式开始，依次处理数列的每一项。如果当前递推式对某一项预测错了，就修正递推式，使它对这一项也正确。

用一个典型例子来说明。考虑数列：$a = \{1, 1, 2, 3, 5, 8\}$（Fibonacci）。

**初始化**：
- 当前递推式 $C = \{1\}$（空的，对应 $a_i = 0$ 对所有 $i$）
- 我们还记录「上次失败」的信息：上次在哪一项错了、当时的递推式是什么

**处理 $i = 0$**（$a_0 = 1$）：

当前递推式 $C$ 预测 $a_0 = 0$，实际是 $1$，所以有误差 $\Delta = 1 - 0 = 1$。

我们需要修正。因为我们之前从未失败过，我们构造一个新的递推式来拟合第 0 项：$C = \{1, 0\}$，对应 $a_i = 1 \cdot a_{i-1}$。但实际上这个递推式只是「刚刚修好第 0 项」，对其他项不一定成立。

记录失败位置：$pos = 0$，$failC = \{1\}$（旧的 $C$），$failDelta = 1$。

**处理 $i = 1$**（$a_1 = 1$）：

用 $C = \{1, 0\}$ 预测：预测值 = $1 \times a_0 = 1$，实际是 $1$，$\Delta = 0$。正确。

**处理 $i = 2$**（$a_2 = 2$）：

预测值 = $1 \times a_1 = 1$，实际是 $2$，$\Delta = 1$。错了！

上次失败在 $pos = 0$，当时 $failDelta = 1$，$failC = \{1\}$。我们知道「上次的那套递推式」在步子差等于 $failDelta$ 时没能应对。现在构造修正项：

$$
R = \{1, 0, \dots, 0, -1\}
$$

其中的 1 来自于 $\Delta / failDelta = 1/1 = 1$，$-1$ 是上个递推式 $failC = \{1\}$ 取反再偏移。具体来说，修正项 $R$ 放在位置 $i - pos - 1 = 2 - 0 - 1 = 1$ 之后：

$$
R = \{1, 0, 0, \dots, 0, -1\}
$$

这里 $-1$ 出现在偏移后的位置。我们把当前递推式「拉长」加上修正项：

新递推长度 $= \max(|C|, |R|)$，对应系数相加。

$C$ 原来长度是 2（系数是 $\{1,0\}$，但 $a_i = 1 \cdot a_{i-1}$ 实际上是 1 阶递推）。

哎，说了这么多术语，不如直接举例来得清楚。好在 BM 的代码实现并不长，我们先给出算法流程，再一步步手算。

### 3.3 BM 算法伪码

```
C = {1}          // 当前递推式（多项式形式，C[0]=1 是常数项）
B = {1}          // 上次修正的递推式
L = 0            // 当前递推阶数
m = 1            // 最近一次修正的位置偏移
b = 1            // 最近一次修正的 Delta

for i from 0 to N-1:
    // 用当前 C 预测 a_i
    d = a_i + Σ_{j=1}^{L} C[j] * a_{i-j}
    if |d| < eps: continue   // 预测正确
    
    // 预测失败，需要修正
    T = C
    // 新的 C = 旧的 C - (d/b) * x^m * B
    // 具体：把 B 的系数乘上 d/b，向右偏移 m 位，然后在 C 中减去
    C = C - (d/b) * shift(B, m)
    if 2*L <= i:
        L = i + 1 - L
        B = T
        b = d
        m = 1
    else:
        m = m + 1
```

这里多项式 $C$ 表示递推式：$C[0] = 1$（常数项），$a_i = -\sum_{j=1}^{L} C[j] \cdot a_{i-j}$。

### 3.4 手动模拟 Fibonacci

数列：$a = [1, 1, 2, 3, 5, 8]$，在模 1e9+7 下计算（这里全部在整数域，系数可能为负数，我们取模保证正数）。

初始：$C = [1], B = [1], L = 0, m = 1, b = 1$

**i=0, a[0]=1**：
预测值 = $-\sum C[j]a_{-j} = 0$（因为 $L=0$）
$d = 1 - 0 = 1$

失败。$T = C = [1]$

$C = C - (d/b) \cdot \text{shift}(B, m) = [1] - 1 \cdot \text{shift}([1], 1) = [1] - [0, 1] = [1, -1]$
即 $C = [1, -1]$，意味着 $a_i = -(-1)a_{i-1} = 1 \cdot a_{i-1}$。

检查 $2L \le i$：$2 \times 0 \le 0$ 成立。
更新：$L = i+1-L = 1$，$B = T = [1]$，$b = d = 1$，$m = 1$。

**i=1, a[1]=1**：
$d = a_1 + C[1] \cdot a_0 = 1 + (-1) \times 1 = 0$。正确，无事发生。

**i=2, a[2]=2**：
$d = a_2 + C[1]a_1 = 2 + (-1) \times 1 = 1$。失败！

$T = C = [1, -1]$
$C = C - (1/1) \cdot \text{shift}(B, m) = [1, -1] - \text{shift}([1], 1) = [1, -1] - [0, 1] = [1, -2]$

检查 $2L \le i$：$2 \times 1 \le 2$ 成立。
更新：$L = i+1-L = 2+1-1=2$，$B = T = [1, -1]$，$b = d = 1$，$m = 1$。

$C = [1, -2]$ 对应递推：$a_i = -(-2)a_{i-1} = 2a_{i-1}$？这不对，Fibonacci 是 $a_i = a_{i-1} + a_{i-2}$。

因为 $C$ 长度不够。继续往下跑。

**i=3, a[3]=3**：
$d = a_3 + C[1]a_2 + C[2]a_1 = 3 + (-2) \times 2 + 0 \times 1 = 3 - 4 = -1$。失败！

$T = C = [1, -2]$
$C = C - (d/b) \cdot \text{shift}(B, m) = [1, -2] - (-1/1) \cdot \text{shift}([1, -1], 1)$
$\text{shift}([1, -1], 1) = [0, 1, -1]$
$C = [1, -2] - (-1)[0, 1, -1] = [1, -2] + [0, 1, -1] = [1, -1, -1]$

检查 $2L \le i$：$2 \times 2 \le 3$ 不成立。
$m = m + 1 = 2$。
$C = [1, -1, -1]$ 对应 $a_i = -(-1)a_{i-1} - (-1)a_{i-2} = a_{i-1} + a_{i-2}$。正确！

**i=4, a[4]=5**：
$d = a_4 + C[1]a_3 + C[2]a_2 = 5 + (-1)\cdot3 + (-1)\cdot2 = 5 - 3 - 2 = 0$。正确。

**i=5, a[5]=8**：
$d = a_5 + C[1]a_4 + C[2]a_3 = 8 + (-1)\cdot5 + (-1)\cdot3 = 8 - 5 - 3 = 0$。正确。

最终 $C = [1, -1, -1]$，即 $a_i = a_{i-1} + a_{i-2}$。

❓ **为什么 $C$ 的多项式系数的符号是反的？**

BM 算法的多项式用的是「特征多项式」的形式：$C(x) = 1 - c_1 x - c_2 x^2 - \cdots - c_k x^k$。数列满足 $\sum_{j=0}^k C[j] \cdot a_{i-j} = 0$，其中 $C[0] = 1$。

所以 $C = [1, -1, -1]$ 表示 $1\cdot a_i + (-1)a_{i-1} + (-1)a_{i-2} = 0$，即 $a_i = a_{i-1} + a_{i-2}$。转换回我们习惯的递推系数时取负数即可。

### 3.5 竞赛套路：打表 + BM

BM 在竞赛中最常见的用法是：

1. 先写暴力 DP 或搜索，计算出小规模（通常 $n = 0 \sim 50$）的结果
2. 把数列输出到控制台
3. 用 BM 算法求出递推式
4. 用矩阵快速幂或 Kitamasa 方法（第 79 篇）求出第 $n$ 项

这个套路最关键的地方在于：**怎么确信递推式是整数系数的线性递推？** 很多计数问题天然满足这个性质（因为 DP 转移是线性的），但也有一些问题不满足（包含非线性 DP 或涉及组合数求和等）。需要直觉判断一下。

## 四、代码实现

```cpp
#include <bits/stdc++.h>
using ll = long long;
using poly = vector<ll>;

const ll MOD = 1'000'000'007;

// Berlekamp-Massey 算法，输入数列 a[0..N-1]，输出最短递推多项式 C
// 数列满足: a[i] = -Σ C[j] * a[i-j] for j=1..k, 其中 C[0]=1
poly berlekamp_massey(const poly &a) {
    poly C(1, 1);  // 当前递推多项式
    poly B(1, 1);  // 上次修正的递推式
    ll b = 1;      // 上次修正的 Delta
    int L = 0, m = 1;

    for (int i = 0; i < (int)a.size(); i++) {
        // 计算当前预测误差 d
        ll d = a[i];
        for (int j = 1; j <= L; j++)
            d = (d + C[j] * a[i - j]) % MOD;
        if (d == 0) { m++; continue; }

        // 预测失败，修正
        poly T = C;
        // 修正 C = C - (d/b) * shift(B, m)
        ll coef = d * mod_pow(b, MOD - 2) % MOD; // d/b
        if (C.size() < B.size() + m)
            C.resize(B.size() + m, 0);
        for (int j = 0; j < (int)B.size(); j++)
            C[j + m] = (C[j + m] - coef * B[j] % MOD + MOD) % MOD;

        // 是否需要更新 B
        if (2 * L <= i) {
            L = i + 1 - L;
            B = T;
            b = d;
            m = 1;
        } else {
            m++;
        }
    }
    // C 为递推多项式，方便后续使用，可以转成递推系数
    // 递推式: a[n] = -C[1]*a[n-1] - C[2]*a[n-2] - ... - C[k]*a[n-k]
    return C;
}

// 用 BM 找到的递推式来求第 n 项（配合线性递推模块）
ll linear_recurrence(const poly &a, const poly &C, ll n) {

    // 先用矩阵快速幂实现一个简单的版本
    int k = (int)C.size() - 1;
    if (n < (int)a.size()) return a[n] % MOD;
```

### 5.1 使用示例

假设我们对某个问题暴力计算得到前 10 项：`a = [1, 2, 4, 7, 13, 24, 44, 81, 149, 274]`

```cpp
int main() {
    poly a = {1, 2, 4, 7, 13, 24, 44, 81, 149, 274};
    poly C = berlekamp_massey(a);
    // C = [1, -1, -1, -1]  意味着递推式 a[n] = a[n-1] + a[n-2] + a[n-3]
    cout << "递推阶数: " << C.size() - 1 << "\n";
    for (int i = 1; i < (int)C.size(); i++)
        cout << "c[" << i << "] = " << (-C[i] + MOD) % MOD << "\n";
}
```

输出：
```
递推阶数: 3
c[1] = 1
c[2] = 1
c[3] = 1
```

和我们引言中的观察一致！

❓ **BM 算法的正确性前提是什么？**

BM 算法需要数列本身确实满足某个线性递推关系——也就是说，数列是由某个 $k$ 阶线性齐次递推生成的。如果数列是随机的，BM 也会强行「拟合」出一个递推式，但那个递推式没有预测能力。

实用经验：提供至少 2 倍于递推阶数的项数，通常就能得到正确结果。如果有噪声，BM 可能给出（递推式的）阶数很高的解，但实际有用的还是最短的那个。

## 五、复杂度与正确性分析

**时间复杂度**：$O(N^2)$，其中 $N$ 是提供的项数。每次修正时，$C$ 和 $B$ 的长度最多为 $N$。

对于竞赛中常见的 $N = 50 \sim 2000$，这个复杂度是完全可以接受的。

**空间复杂度**：$O(N)$，存储递推式系数。

**正确性**：

BM 算法的正确性基于线性递推空间上的如下性质：如果当前递推式 $C$ 在位置 $i$ 出错了（即 $d \neq 0$），则可以通过上次出错的递推式 $B$ 来构造一个修正项，使得修正后的递推式对位置 $i$ 也正确。这个修正过程保持了递推式是最短的性质，这由 2L 条件和 B 的更新机制保证。

正式证明需要用到生成函数和线性递推的代数理论，但在竞赛中只需知道它是正确的即可。

## 六、典型应用场景

1. **猜递推**：暴力打表小范围数据，用 BM 自动求出递推式，然后用矩阵快速幂求第 $n$ 项
2. **DP 优化**：某些 DP 的转移矩阵虽大，但实际递推阶数很小，BM 可以验证递推性质
3. **生成函数推导**：配合多项式工具，BM 可以用于有理生成函数的恢复

## 七、小结

Berlekamp-Massey 算法是竞赛中「打表骗递推」的神器。核心流程三步走：

1. **暴力打表**：计算数列前 $2k \sim 3k$ 项（$k$ 是猜测的递推阶数）
2. **BM 求递推**：自动找出最短线性递推式
3. **快速求第 n 项**：用矩阵快速幂或 Kitamasa 方法（下一篇要讲的内容）

本质上，BM 算法让我们在不了解问题深层结构的情况下，也能利用「线性」这一普遍性质来加速计算。下一篇我们将深入讨论 **Kitamasa 方法** 和 **Bostan-Mori 算法**——BM 找到递推式之后，如何更快地求出第 $n$ 项。
