# 线性基（XOR 基）

> Made by Aki.
> 最后更新于 2026.04.30

---

## 一、引言

新年联欢晚会上，主持人拿出一个袋子，里面装有 $n$ 张卡片，每张上写着一个正整数。观众可以从袋子中任意选取一些卡片（可以选任意张，每张最多选一次），将选中卡片上的数字进行异或（XOR）运算，得到的结果就是你的奖金。你希望最大化奖金数额。

如果 $n=1000$ 甚至 $n=10^5$，暴力枚举所有子集显然不可能。但是这个问题有一个非常优雅的解法——**线性基**（Linear Basis, XOR Basis）。

线性基的核心思想是：给定一组数，我们可以找到一组「基向量」，使得原数的所有 XOR 子集都能由基向量的 XOR 子集表示，且基向量的数量不超过数字的二进制位数（通常是 60 或 30，取决于值域）。对于最大异或和问题，我们只需要在基向量上应用贪心策略即可。

---

## 二、前置知识

- 异或运算（XOR，写作 $\oplus$）：$0\oplus0=0, 0\oplus1=1, 1\oplus0=1, 1\oplus1=0$
- 异或的性质：满足交换律和结合律，$a \oplus a = 0$，$a \oplus 0 = a$
- 二进制表示与位操作
- 向量空间的基本概念（线性无关、基、张成空间），但本篇会从直觉出发讲解，不需要严谨的线性代数背景

## 三、核心推导

### 3.1 问题转化

我们考虑 XOR 子集问题的本质。

给定一个集合 $S = \{a_1, a_2, \dots, a_n\}$，定义它的张成空间 $\langle S \rangle$ 为所有形如 $a_{i_1} \oplus a_{i_2} \oplus \cdots \oplus a_{i_k}$ 的数的集合（$k \ge 0$，空子集的结果为 0）。

我们需要在这个空间中求某个极值（最大、最小、第 k 小等）。

这里的关键是：如果我们能找到一组**极简的生成元**，它们数量少、形式规整，那么在这个生成元上做各种操作就很方便了。

### 3.2 从插入一个数开始

线性基本质上是一组用贪心方法维护的二进制向量。我们用一个数组 `basis[d]` 表示：最高位是第 $d$ 位的基向量（如果存在）。

维护线性基的核心操作是**插入一个数**。

**插入算法**（直觉版）：对于要插入的数 $x$，从高到低扫描它的二进制位。如果第 $d$ 位是 1：
- 如果 `basis[d]` 还不存在（= 0），就把 $x$ 存到 `basis[d]`，插入成功
- 如果 `basis[d]` 已经存在，就把 $x$ 异或上 `basis[d]`（消去第 $d$ 位的 1），继续往低位扫描

**手算小例子**：依次插入 $5(101_2), 3(011_2), 6(110_2)$。

初始：`basis = [0, 0, 0]`（假设最大二进制位是第 2 位，即数值范围 0-7）。

**插入 5 (101)**：
- 第 2 位是 1，`basis[2]` 为 0，存入 → `basis[2] = 101`

  现在：`basis = [-, -, 101]`（用 - 表示 0）

**插入 3 (011)**：
- 第 2 位（二进制 100）是 0，跳过
- 第 1 位（二进制 010）是 1，`basis[1]` 为 0，存入 → `basis[1] = 011`

  现在：`basis = [-, 011, 101]`

**插入 6 (110)**：
- 第 2 位（100）是 1，`basis[2] = 101` 存在，异或：$110 \oplus 101 = 011$
- 第 1 位（010）是 1，`basis[1] = 011` 存在，异或：$011 \oplus 011 = 000$
- 结果为 0，说明 $6$ 可以被已有的基表示（实际上 $6 = 5 \oplus 3$），插入失败（但这是有用的信息）

最终线性基：`basis = [-, 011, 101]`，共 2 个基向量。

### 3.3 为什么插入算法有效？

这个算法的本质是**高斯消元**的异或版本。

我们来理解一下：当我们把 $x$ 与 `basis[d]` 异或时，实际上是在做「在第 $d$ 位消元」，就像普通高斯消元中把 $a_{ik}/a_{kk}$ 倍的第 $k$ 行减到第 $i$ 行一样。

最终插入的结果是：线性基中每个 `basis[d]` 的最高位都是 $d$，且互不相同。这意味着基向量之间是「线性无关」的（没有任何一个可以由其他几个异或得到）。

❓ **为什么线性基的大小最多是 60？**

因为 `basis[d]` 的下标 $d$ 是二进制位数。如果数字最大是 $2^{60}$，那最多有 60 个二进制位，所以最多 60 个基向量。对于 int 范围（$2^{31}$）内的数，最多 30 个基向量。

这个性质保证了线性基无论插入多少个数，空间占用都是 $O(\log \max a_i)$ 的。

### 3.4 最大异或和

利用已构建的线性基，求最大异或和变得非常简单。

贪心策略：从高到低枚举基向量，如果当前答案 $ans$ 异或上 `basis[d]` 会变大，就异或上它。

为什么从高位开始？因为高位二进制位对数值大小的影响是决定性的：即使低位全部是 1，也比不上把高位从 0 变成 1 带来的提升。

**手算小例子**：用上面的线性基求最大异或和。

线性基：`basis[2] = 101`, `basis[1] = 011`

$ans$ 初始为 0。

- 第 2 位：$ans \oplus 101 = 101 > 0$，取 → $ans = 101$（即 5）
- 第 1 位：$ans \oplus 011 = 110 > 101$，取 → $ans = 110$（即 6）

所以最大异或和是 6。

验证一下：原集合 $\{5,3,6\}$ 的所有非空 XOR 子集为：$\{5,3,6,5\oplus3=6,5\oplus6=3,3\oplus6=5,5\oplus3\oplus6=0\}$，最大值确实是 6。

### 3.5 第 k 小异或和

求第 k 小异或和稍微复杂一些。思路是**重构线性基**，使之变成「简化阶梯形」。

重构步骤：对于每个 `basis[d]`，用更高位的基向量去消去 `basis[d]` 中更高位的 1。这样每个基向量就只在自己的最高位是 1，其余高位都是 0。

重构后的基向量按最高位从小到大排好，就形成了类似二进制位权重的结构。然后我们把 $k$ 按二进制拆分，对应位为 1 就异或上对应的基向量。

**手算小例子**：

上面线性基已经是简化形式了——`basis[2] = 101`, `basis[1] = 011`。按最高位排序后，$b_1 = 011$（最高位第 1 位），$b_2 = 101$（最高位第 2 位）。

求第 k 小（$k=1,2,3,4,\dots$）：
- $k=1$（二进制 $001$）：只有第 0 位（最低位）为 1，对应 $b_1 = 3$，所以第 1 小是 $3$（不包含空子集的 0）
- $k=2$（二进制 $010$）：对应 $b_2 = 5$，第 2 小是 $5$
- $k=3$（二进制 $011$）：$b_1 \oplus b_2 = 6$，第 3 小是 $6$

验证一下：所有非空异或值从小到大排序：$3,5,6$。正确！

❓ **$k=0$ 呢？**

$k=0$ 对应空子集，结果是 0。但要注意：只有当线性基的大小小于原集合大小时（说明存在线性相关），0 才是可达的。如果原集合中的数线性无关，0 不可达（空子集是否算入视题目要求而定）。

## 四、代码实现

```cpp
#include <bits/stdc++.h>
using ll = long long;

struct LinearBasis {
    static const int LOG = 60; // 视值域调整，long long 范围用 60
    vector<ll> basis;
    int sz;

    LinearBasis() {
        basis.assign(LOG + 1, 0);
        sz = 0;
    }

    // 插入一个数，返回是否成功（true = 加入了新的基向量）
    bool insert(ll x) {
        for (int d = LOG; d >= 0; d--) {
            if (!(x >> d & 1)) continue;
            if (basis[d]) {
                x ^= basis[d];
            } else {
                basis[d] = x;
                sz++;
                return true;
            }
        }
        return false; // x 变成了 0，说明可被已有基表示
    }

    // 查询最大异或和
    ll max_xor() {
        ll ans = 0;
        for (int d = LOG; d >= 0; d--) {
            if ((ans ^ basis[d]) > ans)
                ans ^= basis[d];
        }
        return ans;
    }

    // 查询最小异或和（不考虑空子集）
    ll min_xor() {
        for (int d = 0; d <= LOG; d++)
            if (basis[d]) return basis[d];
        return 0;
    }

    // 重构为简化阶梯形，为第 k 小做准备
    void rebuild() {
        for (int d = LOG; d >= 0; d--) {
            if (!basis[d]) continue;
            for (int j = d - 1; j >= 0; j--) {
                if (basis[d] >> j & 1)
                    basis[d] ^= basis[j];
            }
        }
    }

    // 查询第 k 小异或和（rebuild 后调用）
    ll kth(ll k) {
        // 重建
        rebuild();
        // 收集非零基向量
        vector<ll> v;
        for (int d = 0; d <= LOG; d++)
            if (basis[d]) v.push_back(basis[d]);
        // 是否可表示 0
        bool zero = (sz < /* 插入的总数，需外部传入 */); // 需要知道原集合大小
        // 如果 zero=false，k++ 跳过空集（0不算）
        if (!zero) k++;
        if (k > (1LL << v.size())) return -1; // 超出范围
        ll ans = 0;
        for (int i = 0; i < (int)v.size(); i++)
            if (k >> i & 1) ans ^= v[i];
        return ans;
    }
};
```

❓ **为什么 `rebuild` 中要从高位往低位做消元？**

如果我们从高位往低位做消元：对于第 $d$ 位，用 $basis[d]$ 去消去更低位的基向量中第 $d$ 位的 1。这样能保证最终每个基向量的最高位唯一，且低位基向量中不会出现高位基向量的最高位。

必须注意方向——如果从低位向高位做，消元后基向量的形式就不是我们想要的二进制位权重形式了。

## 五、复杂度与正确性分析

**插入复杂度**：$O(\log M)$ 每次，其中 $M = \max a_i$。每次插入最多扫描 $\log M$ 个二进制位。

**最大异或和**：$O(\log M)$，从高位到低位过一遍。

**第 k 小**：重构 $O(\log^2 M)$，查询 $O(\log M)$。重构时每个基向量要消去低位中它出现的位，最多 $O(\log M)$ 次操作。

**空间复杂度**：$O(\log M)$。

**正确性证明概要**：

1. 插入过程保证基向量之间线性无关（$basis[d]$ 的最高位 $d$ 互不相同）
2. 任意原数的异或和，可以通过多次异或基向量得到（反之亦然），所以线性基张成的空间等于原集合张成的空间
3. 最大异或和的贪心策略正确：因为高位 1 的价值大于所有低位 1 之和（$2^d > 2^{d-1} + 2^{d-2} + \cdots + 1$）
4. 第 k 小的正确性：重构后的基向量组成了一个「二进制表示」的基，第 i 个基向量是否选择对应 $k$ 的第 i 位

## 六、典型应用场景

1. **最大异或和**：给定一个数的集合，选取子集使异或和最大
2. **第 k 小异或和**：查询子集异或的所有可能值中第 k 小的
3. **贪心选择判定**：判断某个数是否能被已有集合中的元素异或得到
4. **线性基的合并**：两个线性基的合并就是把一个的所有元素插入另一个，$O(\log^2 M)$ 复杂度

**推荐练习题**：

## 七、小结

线性基的本质就是**二进制位视角下的高斯消元**。它把一组数的异或信息压缩到 $O(\log M)$ 的极小空间中，让我们可以在压缩后的基上高效求解最大、最小、第 k 小等异或相关问题。

线性代数的思想在这里给出了令人惊叹的体现：「无关信息」被消去，「核心结构」被保留。值得一提的是，这里的「基」正是线性代数中「向量空间的基」在异或空间下的特例——向量空间 $(\mathbb{Z}_2)^k$ 上的基。
