## 三、场景一：邻接矩阵幂与路径计数

### 3.1 基本概念

图的**邻接矩阵** $A$ 定义为：

$$A_{ij} = \begin{cases}
1 & \text{如果 } i \to j \text{ 有边} \\
0 & \text{否则}
\end{cases}$$

加权的版本：$A_{ij} = w(i, j)$。

**基本定理**：$(A^k)_{ij}$ 等于从 $i$ 到 $j$ 恰好走 $k$ 步的路径数量。

这个定理的证明很简单：矩阵乘法 $A^2$ 的 $(i, j)$ 条目是 $\sum_k A_{ik} A_{kj}$，即从 $i$ 经中间点 $k$ 到 $j$ 的路径数求和。归纳即得。

**应用 1：计数所有长度不超过 $K$ 的路径**

求 $\sum_{k=0}^K (A^k)_{ij}$。这可以用矩阵幂的几何级数公式：

$$(I - A^{K+1})(I - A)^{-1}$$

前提是 $I - A$ 可逆。或者用分治方法：令 $S(n) = I + A + A^2 + \cdots + A^n$，有递归：

$$S(2m) = S(m) \cdot (I + A^{m+1})$$

这个递归可以在 $O(n^3 \log K)$ 内计算（每次递归需要一次矩阵乘法和一次矩阵加法）。

**应用 2：有向图的可达性**

如果只关心是否可达（不关心路径数），可以把邻接矩阵的乘法改成布尔运算（AND/OR）。或者直接用 Floyd-Warshall。

但如果 n 很大（比如 $n = 10^5$），矩阵幂就不好使了。这时应该用 BFS/DFS。

### 3.2 手算例子

考虑一个有向图，顶点 0, 1, 2，边：$0 \to 1$, $1 \to 2$, $2 \to 0$, $0 \to 2$。

邻接矩阵：

$$A = \begin{pmatrix}
0 & 1 & 1 \\
0 & 0 & 1 \\
1 & 0 & 0
\end{pmatrix}$$

求从 0 到 2 走恰好 3 步的路径数。

$$A^2 = \begin{pmatrix}
0 & 1 & 1 \\
0 & 0 & 1 \\
1 & 0 & 0
\end{pmatrix}
\begin{pmatrix}
0 & 1 & 1 \\
0 & 0 & 1 \\
1 & 0 & 0
\end{pmatrix}
= \begin{pmatrix}
1 \cdot 1 + 1 \cdot 1 & 0 + 0 + 0 & 1 \cdot 0 + 1 \cdot 1 \\
\dots & \dots & \dots \\
\dots & \dots & \dots
\end{pmatrix}$$

详细计算：
$(A^2)_{00} = A_{01}A_{10} + A_{02}A_{20} = 1 \times 0 + 1 \times 1 = 1$
$(A^2)_{01} = A_{01}A_{11} + A_{02}A_{21} = 1 \times 0 + 1 \times 0 = 0$
$(A^2)_{02} = A_{01}A_{12} + A_{02}A_{22} = 1 \times 1 + 1 \times 0 = 1$

$(A^2)_{10} = A_{12}A_{20} = 1 \times 1 = 1$
$(A^2)_{11} = 0$
$(A^2)_{12} = 0$

$(A^2)_{20} = A_{20}A_{00} + A_{21}A_{10} = 1 \times 0 + 0 \times 0 = 0$
$(A^2)_{21} = A_{20}A_{01} + A_{21}A_{11} = 1 \times 1 + 0 \times 0 = 1$
$(A^2)_{22} = A_{20}A_{02} + A_{21}A_{12} = 1 \times 1 + 0 \times 1 = 1$

所以：

$$A^2 = \begin{pmatrix}
1 & 0 & 1 \\
1 & 0 & 0 \\
0 & 1 & 1
\end{pmatrix}$$

$$A^3 = A^2 \cdot A = \begin{pmatrix}
1 & 0 & 1 \\
1 & 0 & 0 \\
0 & 1 & 1
\end{pmatrix}
\begin{pmatrix}
0 & 1 & 1 \\
0 & 0 & 1 \\
1 & 0 & 0
\end{pmatrix}
= \begin{pmatrix}
1 & 1 & 1 \\
0 & 1 & 1 \\
1 & 0 & 1
\end{pmatrix}$$

$(A^3)_{02} = 1$，所以从 0 到 2 恰好 3 步有 1 条路径。

验证：$0 \to 1 \to 2 \to 0$（不对，这是到 0）。$0 \to 2 \to 0 \to 1$（这是到 1）。$0 \to 1 \to 2 \to 0 \to 1 \to 2$（步数不对）。手动枚举 3 步从 0 到 2：
- $0 \to 1 \to 2$ 只有 2 步
- $0 \to 1 \to 2 \to 0 \to 2$ 要 4 步
- $0 \to 2 \to 0 \to 2$ 只要 3 步！

确实有 1 条路径：$0 \to 2 \to 0 \to 2$。

❓ **如果图有重边（加权）呢？**

邻接矩阵 $A_{ij}$ 存放边数（或权重），$(A^k)_{ij}$ 就是所有 $k$ 步路径的权重乘积之和。这在**计数 DP** 中很常用。

### 3.3 竞赛技巧：快速幂 + DP 思想

路径计数问题本质上就是矩阵乘法。但某些题目不会直接给你邻接矩阵，需要你**构造**一个合适的矩阵。

例如，考虑从 $i$ 走到 $j$，每次可以向前走 1 步或 2 步，求 $k$ 步后从 0 到 $n$ 的路径数。

这实际上是一个 Fibonacci 类型的递推，可以用矩阵快速幂解决。矩阵形式：

$$\begin{pmatrix}
a_{k} \\ a_{k-1}
\end{pmatrix}
= \begin{pmatrix}
1 & 1 \\
1 & 0
\end{pmatrix}
\begin{pmatrix}
a_{k-1} \\ a_{k-2}
\end{pmatrix}$$

其中 $a_k$ 表示走 $k$ 步到达目标的方法数。

**关键是识别出转移可以用矩阵表示**，然后选择合适的方法（矩阵快速幂 / Kitamasa / 特征根法）求解。

## 四、场景二：图上随机游走的矩阵形式

### 4.1 转移矩阵

在无向图上做随机游走，从当前节点 $i$，等概率走到相邻节点。设 $d_i$ 是节点 $i$ 的度数。转移矩阵 $P$ 定义为：

$$P_{ij} = \begin{cases}
1/d_i & \text{如果 } (i, j) \in E \\
0 & \text{否则}
\end{cases}$$

那么 $k$ 步后的概率分布是 $\pi_k = \pi_0 \cdot P^{k}$（行向量形式）。

对于**有向图**（比如搜索引擎的 PageRank），转移矩阵类似但需要考虑死胡同（sink node）的问题，于是引入阻尼因子 $\alpha$：

$$P = \alpha \cdot M + (1 - \alpha) \cdot \frac{1}{n} J$$

其中 $M$ 是标准转移矩阵，$J$ 是全 1 矩阵。

### 4.2 平稳分布

对于不可约非周期马尔可夫链，$\pi_{\infty} = \lim_{k\to\infty} \pi_0 P^k$ 存在，且满足 $\pi = \pi P$。

求解 $\pi P = \pi$ 等价于解线性方程组 $\pi (I - P) = 0$，加约束 $\sum \pi_i = 1$。

这就是一个**高斯消元**问题（#75）。

**无向图特例**：对于连通无向图，平稳分布非常简洁：

$$\pi_i = \frac{d_i}{2|E|}$$

完全不需要解线性方程组！验证：$\pi P = ?$

$(\pi P)_j = \sum_i \pi_i \cdot P_{ij} = \sum_{(i,j)\in E} \frac{d_i}{2|E|} \cdot \frac{1}{d_i} = \frac{d_j}{2|E|} = \pi_j$

成立。所以无向图上的平稳分布只依赖于度数，非常方便。

### 4.3 首达时间与期望步数

设 $H_{ij}$ 是从 $i$ 第一次到达 $j$ 的期望步数（hit time）。

对于无向图，有一个经典公式：

$$H_{ij} + H_{ji} = 2|E| \cdot R_{ij}$$

其中 $R_{ij}$ 是 $i$ 和 $j$ 之间的**有效电阻**（把图的每条边看作 1 欧姆电阻）。

这个公式把随机游走的期望时间与电路理论联系起来了！计算 $R_{ij}$ 可以用**Kirchhoff 矩阵树定理**配合高斯消元。

### 4.4 手算例子

考虑一个三角形图（3 个点，每对节点之间都有边）。

每个点的度数 $d_i = 2$，所以平稳分布 $\pi_i = 2/6 = 1/3$。

转移矩阵：

$$P = \begin{pmatrix}
0 & 1/2 & 1/2 \\
1/2 & 0 & 1/2 \\
1/2 & 1/2 & 0
\end{pmatrix}$$

计算 $P^2$：

$$P^2 = \begin{pmatrix}
1/2 & 1/4 & 1/4 \\
1/4 & 1/2 & 1/4 \\
1/4 & 1/4 & 1/2
\end{pmatrix}$$

可以验证 $P^k$ 趋于 $\begin{pmatrix} 1/3 & 1/3 & 1/3 \\ 1/3 & 1/3 & 1/3 \\ 1/3 & 1/3 & 1/3 \end{pmatrix}$。

这验证了平稳分布就是 $\pi = (1/3, 1/3, 1/3)$。

求从 0 到 1 的首达时间 $H_{01}$：

解线性方程组。设 $h_i = \mathbb{E}[\text{从 i 到 1 的步数}]$。

$h_1 = 0$（已经在目标点）。
$h_0 = 1 + \frac12 h_1 + \frac12 h_2 = 1 + \frac12 h_2$
$h_2 = 1 + \frac12 h_0 + \frac12 h_1 = 1 + \frac12 h_0$

解：$h_2 = 1 + \frac12 h_0$，代入 $h_0 = 1 + \frac12 (1 + \frac12 h_0) = 1 + \frac12 + \frac14 h_0$
$h_0 - \frac14 h_0 = \frac32$
$\frac34 h_0 = \frac32$
$h_0 = 2$

所以 $H_{01} = 2$（从 0 出发平均需要 2 步到达 1）。

验证有效电阻公式：$|E| = 3$，三角形中任意两点的有效电阻 $R_{01} = \frac{2}{3}$（将三条 1 欧姆电阻接成三角形，$0$ 和 $1$ 之间的等效电阻等于 $1 \parallel (1+1) = \frac{2}{3}$）。

$2|E| \cdot R_{01} = 2 \times 3 \times \frac{2}{3} = 4$

但 $H_{01}+H_{10} = 2 + 2 = 4$。成立！

❓ **这个有效电阻公式有什么用？**

它把随机游走问题转化成了电路问题。如果图是二维网格或其他规则结构，有效电阻已知，就可以直接得到期望步数，不需要解线性方程组。

### 4.5 竞赛中的应用

随机游走在竞赛中最常见的题型是：

1. **给定图和参数，求 $k$ 步后的概率分布**：用矩阵快速幂
2. **求期望步数**：解线性方程组（高斯消元）
3. **求首达时间**：高斯消元，或者利用对称性简化
4. **结合概率 DP**：对大规模问题需要用数据结构优化

**矩阵形式的优势**：把随机游走问题统一到线性代数框架下，可以用矩阵快速幂、高斯消元等标准工具处理。

## 五、场景三：线性规划简介

### 5.1 什么是线性规划

**线性规划**（Linear Programming, LP）是在一组线性不等式约束下，优化一个线性目标函数的问题：

$$\begin{aligned}
\text{maximize } & c^T x \\
\text{subject to } & Ax \le b \\
& x \ge 0
\end{aligned}$$

其中 $x \in \mathbb{R}^n$ 是决策变量，$A \in \mathbb{R}^{m \times n}$ 是约束矩阵，$b \in \mathbb{R}^m$，$c \in \mathbb{R}^n$。

**背景**：线性规划不在传统 ICPC 算法竞赛的考点范围内，但是——**ICPC World Finals 偶尔会考**。例如，WF 2016 有一道题可以用线性规划解释（实际是识别出整数规划形式，然后利用特殊性质简化为最大流）。另外，离线算法竞赛中偶尔出现需要解线性规划的思路题。

### 5.2 单纯形法思想

**单纯形法**（Simplex Algorithm）是最常用的线性规划求解算法。

核心思想：
1. 从可行域的一个顶点开始
2. 沿着约束边走到相邻的更好顶点
3. 重复直到最优

虽然单纯形法在最坏情况下是指数时间的，但在实践中非常快。竞赛中通常实现的是**简化单纯形法**（针对标准形式）。

对算法竞赛而言，关键不是实现完整的单纯形法（大多数情况下用不到），而是**识别出线性规划模型**，然后用现成的库或简化方法处理。

### 5.3 竞赛中能用 LP 解决的问题特征

1. **资源分配**：多种资源上限，多种产出，最大化收益
2. **网络流**：最大流和最小费用流是线性规划的特例
3. **博弈**：二人零和博弈 = 线性规划对偶
4. **近似算法**：某些 NP-hard 问题的 LP 松弛

### 5.4 ICPC WF 2016 案例

ICPC World Finals 2016 的某道题（具体是「Ordering Pizza」或类似分配问题）其实可以用整数规划建模，但因为特殊的结构（约束矩阵是全幺模的，totally unimodular），整数规划解和线性规划松弛解一致，最终可以用最大流算法解决。

这里的要点是：**理解线性规划可以帮助你识别出问题的结构**，即使最终用的算法不是标准 LP。

### 5.5 对偶与最大流

线性规划的**对偶定理**：原问题的最优值等于对偶问题的最优值（强对偶定理）。

对一个最大流问题：
- 原问题：求从源点到汇点的最大流量（线性约束：容量限制、流量守恒）
- 对偶问题：求最小割（边权和最小的割集）

这正是最大流-最小割定理！对偶性不仅提供了理论证明，还给出了算法思路。

## 六、代码示例

### 6.1 随机游走首达时间（高斯消元）

```cpp
#include <bits/stdc++.h>
using namespace std;

const double EPS = 1e-9;

// 高斯消元求解线性方程组
vector<double> gauss(vector<vector<double>> a, vector<double> b) {
    int n = a.size();
    for (int i = 0; i < n; i++) a[i].push_back(b[i]);

    for (int col = 0; col < n; col++) {
        int sel = col;
        for (int i = col; i < n; i++)
            if (abs(a[i][col]) > abs(a[sel][col])) sel = i;
        if (abs(a[sel][col]) < EPS) continue;
        swap(a[col], a[sel]);

        for (int i = col + 1; i <= n; i++)
            a[col][i] /= a[col][col];
        a[col][col] = 1;

        for (int i = 0; i < n; i++) {
            if (i == col) continue;
            double c = a[i][col];
            for (int j = col; j <= n; j++)
                a[i][j] -= c * a[col][j];
        }
    }

    vector<double> x(n);
    for (int i = 0; i < n; i++) x[i] = a[i][n];
    return x;
}

// 计算随机游走的首达时间
// 图用邻接表表示，目标节点是 target
vector<double> hitting_time(const vector<vector<int>>& adj, int target) {
    int n = adj.size();
    // 方程组: h_i = 1 + sum_{j in N(i)} (1/deg_i) * h_j, for i != target
    // h_target = 0
    vector<vector<double>> A(n, vector<double>(n, 0));
    vector<double> b(n, 0);

    for (int i = 0; i < n; i++) {
        if (i == target) {
            A[i][i] = 1;
            b[i] = 0;
            continue;
        }
        int deg = adj[i].size();
        A[i][i] = 1;
        for (int j : adj[i]) {
            A[i][j] = -1.0 / deg;
        }
        b[i] = 1;
    }

    return gauss(A, b);
}
```

### 6.2 邻接矩阵快速幂（路径计数）

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
using matrix = vector<vector<ll>>;

const ll MOD = 1'000'000'007;

matrix mat_mul(const matrix& A, const matrix& B) {
    int n = A.size();
    matrix C(n, vector<ll>(n, 0));
    for (int i = 0; i < n; i++)
        for (int k = 0; k < n; k++)
            if (A[i][k])
                for (int j = 0; j < n; j++)
                    C[i][j] = (C[i][j] + A[i][k] * B[k][j]) % MOD;
    return C;
}

matrix mat_pow(matrix A, ll k) {
    int n = A.size();
    matrix res(n, vector<ll>(n, 0));
    for (int i = 0; i < n; i++) res[i][i] = 1;
    while (k) {
        if (k & 1) res = mat_mul(res, A);
        A = mat_mul(A, A);
        k >>= 1;
    }
    return res;
}

// 计算从 s 到 t 恰好 k 步的路径数
ll count_paths(const matrix& adj, int s, int t, ll k) {
    auto Ak = mat_pow(adj, k);
    return Ak[s][t];
}

// 计算从 s 到 t 不超过 K 步的路径数
ll count_paths_up_to(const matrix& adj, int s, int t, ll K) {
    int n = adj.size();
    // 使用等比数列分治法
    // 返回 I + A + A^2 + ... + A^n
    // 这里简化实现，直接累加（当 K 不大时）
    matrix sum(n, vector<ll>(n, 0));
    for (int i = 0; i < n; i++) sum[i][i] = 1;
    matrix cur = {{0}}; // 占位，实际应快速求幂加
    // 用二分法
    function<matrix(matrix, ll)> geo_sum = [&](matrix A, ll n) -> matrix {
        if (n == 0) {
            matrix I(n, vector<ll>(n, 0));
            for (int i = 0; i < n; i++) I[i][i] = 1;
            return I;
        }
        if (n % 2 == 0)
            return geo_sum(A, n-1) + mat_pow(A, n);
        matrix half = geo_sum(A, n/2);
        return half * (I + mat_pow(A, n/2 + 1));
    };
    // 实际使用需完善
    return 0;
}
```

### 7.2 综合题型示例

**题型特征**：拿到题后如果发现需要处理以下元素，可以考虑线性代数方法：

1. **大范围递推**（$n \le 10^9$ 或 $10^{18}$）→ 矩阵快速幂 / Kitamasa
2. **图上路径计数** → 邻接矩阵幂
3. **随机游走概率/期望** → 转移矩阵 + 高斯消元
4. **稀疏矩阵**（$n$ 很大但非零元素不多）→ Wiedemann 算法
5. **环形卷积** → 循环矩阵 + FFT
6. **存在唯一的多项式**（自然数幂和、DP 结果）→ 拉格朗日插值

---

## 八、小结

线性代数是算法竞赛中不可或缺的工具。从最基础的矩阵乘法到复杂的黑盒算法，核心思路是一脉相承的：**用矩阵表示变换，用线性结构简化问题**。

本系列 10 篇文章覆盖了竞赛中最常用的线性代数技术：

| 篇号 | 内容 | 核心方法 |
|------|------|---------|
| #74 | 矩阵基本运算与快速幂 | 模板，$O(n^3 \log k)$ |
| #75-#76 | 高斯消元 | 解方程组，$O(n^3)$ |
| #77 | 线性基 | 异或空间，$O(n \log V)$ |
| #78 | Berlekamp-Massey | 由数列求递推，$O(n^2)$ |
| #79 | Kitamasa / Bostan-Mori | 递推求第 $n$ 项，$O(k^2 \log n)$ |
| #80 | 范德蒙德 / 拉格朗日插值 | 多项式插值，$O(k^2)$ 或 $O(k)$ |
| #81 | 循环矩阵 / Toeplitz | 结构化矩阵，$O(n \log n)$ |
| #82 | 黑盒线性代数 | 稀疏矩阵迭代，$O(md + d^2 \log k)$ |

**关键原则**：
- 先判断问题是否有线性结构
- 选择最匹配的工具（不是所有问题都需要 $O(n^3)$ 的矩阵运算）
- 注意模运算下的限制（特征值、行列式等在模域中可能退化）
