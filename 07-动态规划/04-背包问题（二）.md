上一篇我们学习了三种基础背包问题。这一篇我们将探讨更复杂的背包变种：分组背包、混合背包和背包求方案数。

---

## 一、引言

来看一个更复杂的问题：

> 有 $n$ 件物品和一个容量为 $W$ 的背包。这些物品被分成 $k$ 组，每组内的物品互斥（每组最多选一件）。第 $i$ 件物品的重量为 $w_i$，价值为 $v_i$，属于第 $g_i$ 组。求背包能装下的最大总价值。

这就是**分组背包问题**——在 01 背包的基础上增加了分组约束。

如果物品的选择还有其他限制呢？比如有些物品只能选一次，有些可以选无限次，有些有数量限制——这就是**混合背包问题**。

如果问题不是求最大价值，而是求有多少种方式达到某个价值呢？这就是**背包求方案数问题**。

这些变种展示了背包问题的灵活性和广泛应用。

## 三、核心推导

### 3.1 分组背包

#### 问题定义

有 $n$ 件物品和一个容量为 $W$ 的背包。这些物品被分成 $k$ 组，每组内的物品互斥（每组最多选一件）。第 $i$ 件物品的重量为 $w_i$，价值为 $v_i$。求背包能装下的最大总价值。

#### 状态定义

我们定义 $dp[i][j]$ 为：**前 $i$ 组物品，背包容量为 $j$ 时，能获得的最大价值**。

#### 状态转移方程

对于第 $i$ 组，我们有两种选择：

1. **不选第 $i$ 组的任何物品**：$dp[i][j] = dp[i-1][j]$
2. **选第 $i$ 组的第 $k$ 件物品**（前提是 $j \ge w_k$）：$dp[i][j] = dp[i-1][j-w_k] + v_k$

取所有可能选择的最大值：

$$
dp[i][j] = \max \begin{cases}
dp[i-1][j] \\
\max_{k \in \text{group}_i, j \ge w_k} \{dp[i-1][j-w_k] + v_k\}
\end{cases}
$$

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

struct Item {
    int w, v;
};

int knapsackGroup(int W, vector<vector<Item>>& groups) {
    int k = groups.size();
    vector<int> dp(W + 1, 0);
    
    for (int i = 0; i < k; i++) {
        // 注意：逆序遍历，保证每组只选一件
        for (int j = W; j >= 0; j--) {
            for (auto& item : groups[i]) {
                if (j >= item.w) {
                    dp[j] = max(dp[j], dp[j - item.w] + item.v);
                }
            }
        }
    }
    
    return dp[W];
}
```

**❓ 为什么分组背包要逆序遍历容量？**

因为每组最多选一件。如果正序遍历，$dp[j-w_k]$ 可能已经被更新过（即当前组被选了多件），这就不符合分组背包的定义了。

**❓ 遍历顺序为什么是「组 → 容量 → 物品」？**

如果改成「组 → 物品 → 容量」，那么在处理同一组的不同物品时，可能会出现「选了两件」的情况（因为 $dp[j-w_k]$ 可能已经被同组的另一件物品更新过）。

### 3.2 混合背包

#### 问题定义

有 $n$ 件物品和一个容量为 $W$ 的背包。第 $i$ 件物品的重量为 $w_i$，价值为 $v_i$，类型为 $t_i$：
- $t_i = 0$：01 背包（只能选一次）
- $t_i = 1$：完全背包（可以选无限次）
- $t_i = 2$：多重背包（最多选 $s_i$ 次）

求背包能装下的最大总价值。

#### 解决思路

混合背包的核心思想是：**根据物品类型选择不同的遍历顺序**。

- 01 背包：逆序遍历容量
- 完全背包：正序遍历容量
- 多重背包：二进制拆分后逆序遍历

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int knapsackMixed(int W, vector<int>& w, vector<int>& v, vector<int>& t, vector<int>& s) {
    int n = w.size();
    vector<int> dp(W + 1, 0);
    
    for (int i = 0; i < n; i++) {
        if (t[i] == 0) {
            // 01 背包：逆序遍历
            for (int j = W; j >= w[i]; j--) {
                dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
            }
        } else if (t[i] == 1) {
            // 完全背包：正序遍历
            for (int j = w[i]; j <= W; j++) {
                dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
            }
        } else {
            // 多重背包：二进制拆分后逆序遍历
            int k = 1;
            int remaining = s[i];
            while (k <= remaining) {
                int weight = k * w[i];
                int value = k * v[i];
                for (int j = W; j >= weight; j--) {
                    dp[j] = max(dp[j], dp[j - weight] + value);
                }
                remaining -= k;
                k *= 2;
            }
            if (remaining > 0) {
                int weight = remaining * w[i];
                int value = remaining * v[i];
                for (int j = W; j >= weight; j--) {
                    dp[j] = max(dp[j], dp[j - weight] + value);
                }
            }
        }
    }
    
    return dp[W];
}
```

### 3.3 背包求方案数

#### 问题定义

有 $n$ 件物品和一个容量为 $W$ 的背包。第 $i$ 件物品的重量为 $w_i$。每件物品只能选一次，求有多少种方式恰好装满背包。

#### 状态定义

我们定义 $dp[j]$ 为：**容量为 $j$ 时，恰好装满背包的方案数**。

#### 状态转移方程

对于每件物品 $i$，如果选择它，方案数增加 $dp[j-w_i]$：

$$
dp[j] += dp[j-w_i]
$$

#### 边界条件

$dp[0] = 1$（容量为 0 时，有一种方案：什么都不选）

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int countWays(int W, vector<int>& w) {
    int n = w.size();
    vector<int> dp(W + 1, 0);
    dp[0] = 1;  // 边界条件
    
    for (int i = 0; i < n; i++) {
        for (int j = W; j >= w[i]; j--) {
            dp[j] += dp[j - w[i]];
        }
    }
    
    return dp[W];
}
```

**❓ 为什么边界条件是 $dp[0] = 1$？**

因为「什么都不选」也是一种方案。如果 $dp[0] = 0$，那么所有方案数都会是 0。

**❓ 为什么要求「恰好装满」？**

如果问题改为「不超过容量 $W$」，那么最终答案应该是 $\sum_{j=0}^{W} dp[j]$。

---

### 3.4 背包求最优方案

除了求最大价值，我们有时还需要知道具体选了哪些物品。

#### 解决思路

在计算 $dp$ 数组的同时，记录每个状态是从哪个状态转移过来的。最后从最终状态回溯，即可得到最优方案。

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

vector<int> knapsack01WithPath(int W, vector<int>& w, vector<int>& v) {
    int n = w.size();
    vector<int> dp(W + 1, 0);
    vector<vector<bool>> choice(n, vector<bool>(W + 1, false));
    
    for (int i = 0; i < n; i++) {
        for (int j = W; j >= w[i]; j--) {
            if (dp[j - w[i]] + v[i] > dp[j]) {
                dp[j] = dp[j - w[i]] + v[i];
                choice[i][j] = true;
            }
        }
    }
    
    // 回溯得到最优方案
    vector<int> path;
    int j = W;
    for (int i = n - 1; i >= 0; i--) {
        if (choice[i][j]) {
            path.push_back(i);
            j -= w[i];
        }
    }
    reverse(path.begin(), path.end());
    return path;
}
```

## 四、代码实现

以下是本篇所有核心问题的代码模板。

### 分组背包

```cpp
struct Item { int w, v; };

int knapsackGroup(int W, vector<vector<Item>>& groups) {
    int k = groups.size();
    vector<int> dp(W + 1, 0);
    for (int i = 0; i < k; i++) {
        for (int j = W; j >= 0; j--) {
            for (const auto& item : groups[i]) {
                if (j >= item.w) {
                    dp[j] = max(dp[j], dp[j - item.w] + item.v);
                }
            }
        }
    }
    return dp[W];
}
```

### 混合背包

```cpp
int knapsackMixed(int W, vector<int>& w, vector<int>& v, vector<int>& t, vector<int>& s) {
    int n = w.size();
    vector<int> dp(W + 1, 0);
    for (int i = 0; i < n; i++) {
        if (t[i] == 0) {
            for (int j = W; j >= w[i]; j--) {
                dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
            }
        } else if (t[i] == 1) {
            for (int j = w[i]; j <= W; j++) {
                dp[j] = max(dp[j], dp[j - w[i]] + v[i]);

            int k = 1;

                k *= 2;
```

## 五、复杂度与正确性分析

| 问题 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 分组背包 | $O(k \cdot W \cdot \max(|group_i|))$ | $O(W)$ |
| 混合背包 | $O(W \sum \log s_i)$ | $O(W)$ |
| 背包求方案数 | $O(nW)$ | $O(W)$ |

**正确性保证**：

1. **分组背包**：遍历顺序「组 → 容量 → 物品」保证每组最多选一件。

2. **混合背包**：根据物品类型选择正确的遍历顺序（01 逆序、完全正序、多重二进制拆分后逆序）。

3. **背包求方案数**：$dp[0] = 1$ 的边界条件保证了「什么都不选」也是一种方案。

## 六、典型应用场景

背包问题的变种在竞赛中的应用非常广泛，以下是几个典型场景：

1. **分组选择**：在多个互斥选项中选择最优组合。

2. **混合限制**：物品有不同的选择限制（只能选一次、可以选多次等）。

3. **计数问题**：计算有多少种方式达到某个状态。

## 七、小结

本文介绍了三种背包问题的变种：**分组背包**、**混合背包** 和 **背包求方案数**。

四个核心要点：

1. 分组背包的关键是遍历顺序：「组 → 容量 → 物品」，逆序遍历容量。

2. 混合背包的核心思想是根据物品类型选择不同的遍历顺序。

3. 背包求方案数时，边界条件 $dp[0] = 1$ 至关重要。

4. 背包问题的变种展示了动态规划的灵活性，掌握基础模型后可以轻松应对各种变种。
