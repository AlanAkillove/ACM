> Made by Aki.

在前面的学习中，我们遇到了很多 $O(n^2)$ 或 $O(n^3)$ 的 DP 问题。当 $n$ 很大时，这些复杂度可能无法满足要求。本篇将介绍几种常见的 DP 优化技巧，帮助我们将时间复杂度降低一个数量级。

---

## 一、引言

来看一个经典问题：

> 有一个长度为 $n$ 的数组 $a$，求长度不超过 $m$ 的连续子数组的最大和。

暴力解法是枚举所有长度不超过 $m$ 的子数组，时间复杂度为 $O(nm)$。当 $n = 10^5, m = 10^5$ 时，会超时。

有没有更高效的方法？答案是**单调队列优化**——一种将 $O(nm)$ 优化到 $O(n)$ 的技巧。

## 三、核心推导

### 3.1 单调队列优化

#### 问题定义

有一个长度为 $n$ 的数组 $a$，求长度不超过 $m$ 的连续子数组的最大和。

#### 朴素解法

暴力枚举所有长度不超过 $m$ 的子数组，时间复杂度为 $O(nm)$。

#### 前缀和 + 单调队列优化

我们定义前缀和 $prefix[i] = \sum_{k=1}^{i} a[k]$，那么区间 $[j+1, i]$ 的和为 $prefix[i] - prefix[j]$。

对于每个 $i$，我们需要在滑动窗口 $[i-m, i-1]$ 中找到最小的 $prefix[j]$：

$$
ans = \max_{1 \le i \le n} \{prefix[i] - \min_{i-m \le j < i} \{prefix[j]\}\}
$$

这是一个**滑动窗口最小值**问题，可以用单调队列在 $O(1)$ 时间内解决。

**单调队列的性质**：
- 队列中的元素从队首到队尾单调递增
- 队首元素始终是当前窗口的最小值

**操作流程**：
1. 将 $prefix[0] = 0$ 入队
2. 对于每个 $i$，移除队首超出窗口的元素
3. 队首元素就是当前窗口的最小值
4. 移除队尾大于等于 $prefix[i]$ 的元素，然后将 $i$ 入队

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int maxSubarraySum(vector<int>& a, int m) {
    int n = a.size();
    vector<int> prefix(n + 1, 0);
    for (int i = 1; i <= n; i++) {
        prefix[i] = prefix[i - 1] + a[i - 1];
    }
    
    deque<int> q;  // 存储下标，单调递增
    q.push_back(0);  // prefix[0] = 0
    
    int ans = INT_MIN;
    for (int i = 1; i <= n; i++) {
        // 移除队首超出窗口的元素
        while (!q.empty() && q.front() < i - m) {
            q.pop_front();
        }
        
        // 当前窗口的最小值就是队首
        ans = max(ans, prefix[i] - prefix[q.front()]);
        
        // 移除队尾大于等于 prefix[i] 的元素
        while (!q.empty() && prefix[q.back()] >= prefix[i]) {
            q.pop_back();
        }
        
        q.push_back(i);
    }
    
    return ans;
}
```

**时间复杂度**：$O(n)$
**空间复杂度**：$O(n)$

**❓ 为什么单调队列的时间复杂度是 $O(n)$？**

因为每个元素最多入队一次、出队一次，总操作次数是 $O(n)$。

### 3.2 斜率优化

#### 问题定义

给定一个长度为 $n$ 的数组 $a$，将其分成若干段，每段 $[j+1, i]$ 的代价为 $(\sum_{k=j+1}^{i} a_k + C)^2$，其中 $C$ 是常数。求最小总代价。

这个问题在洛谷上以「玩具装箱」（P3195）闻名，是斜率优化的经典例题。

#### 朴素 DP

定义 $dp[i]$ 为前 $i$ 个元素的最小总代价，$prefix[i] = \sum_{k=1}^{i} a_k$：

$$
dp[i] = \min_{0 \le j < i} dp[j] + (prefix[i] - prefix[j] + C)^2
$$

朴素实现的复杂度是 $O(n^2)$。

#### 转化为斜率优化形式

展开平方项，令 $s[i] = prefix[i]$：

$$
dp[i] = \min_{0 \le j < i} dp[j] + s[i]^2 + (s[j] + C)^2 - 2 \cdot s[i] \cdot (s[j] + C)
$$

整理后：

$$
dp[i] - s[i]^2 = \min_{0 \le j < i} \{dp[j] + (s[j] + C)^2 - 2 \cdot s[i] \cdot (s[j] + C)\}
$$

令：
- $y_j = dp[j] + (s[j] + C)^2$
- $k_i = 2 \cdot s[i]$（随 $i$ 递增而增大）
- $x_j = s[j] + C$

则方程变为：

$$
dp[i] - s[i]^2 = \min_{0 \le j < i} \{y_j - k_i \cdot x_j\}
$$

这相当于在每个候选点 $(x_j, y_j)$ 中，找一条斜率为 $k_i$ 的直线，使得截距最小。

**关键洞察**：由于 $k_i$ 单调递增，最优转移点也单调递增。我们可以用单调队列维护一个**下凸包**，每次从队首取最优解，从队尾插入新点。

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

// 计算 y_j = dp[j] + (s[j] + C)^2
ll get_y(vector<ll>& dp, vector<ll>& s, int j, ll C) {
    return dp[j] + (s[j] + C) * (s[j] + C);
}

// 计算 x_j = s[j] + C
ll get_x(vector<ll>& s, int j, ll C) {
    return s[j] + C;
}

// 比较三点 (j1, j2, j3) 的斜率，判断 j2 是否在凸包内（无用）
// cross = (y3-y1)*(x2-x1) - (y2-y1)*(x3-x1)
bool is_bad(vector<ll>& dp, vector<ll>& s, int j1, int j2, int j3, ll C) {
    ll y1 = get_y(dp, s, j1, C);
    ll y2 = get_y(dp, s, j2, C);
    ll y3 = get_y(dp, s, j3, C);
    ll x1 = get_x(s, j1, C);
    ll x2 = get_x(s, j2, C);
    ll x3 = get_x(s, j3, C);
    return (y3 - y1) * (x2 - x1) <= (y2 - y1) * (x3 - x1);
}

int main() {
    int n;
    ll C;
    cin >> n >> C;
    
    vector<ll> a(n), s(n + 1, 0);
    for (int i = 0; i < n; i++) {
        cin >> a[i];
        s[i + 1] = s[i] + a[i];
    }
    
    vector<ll> dp(n + 1, 0);
    deque<int> q;
    q.push_back(0);  // j = 0
    
    for (int i = 1; i <= n; i++) {
        ll k = 2 * s[i];
        
        // 移除队首中不如 j+1 优的转移点
        while (q.size() >= 2) {
            ll y1 = get_y(dp, s, q[0], C);
            ll y2 = get_y(dp, s, q[1], C);
            ll x1 = get_x(s, q[0], C);
            ll x2 = get_x(s, q[1], C);
            if (y2 - k * x2 <= y1 - k * x1) {
                q.pop_front();
            } else {
                break;
            }
        }
        
        // 计算 dp[i]
        int j = q.front();
        dp[i] = dp[j] + (s[i] - s[j] + C) * (s[i] - s[j] + C);
        
        // 将 i 插入队尾，维护凸包
        while (q.size() >= 2) {
            int j1 = q[q.size() - 2];
            int j2 = q.back();
            if (is_bad(dp, s, j1, j2, i, C)) {
                q.pop_back();
            } else {
                break;
            }
        }
        q.push_back(i);
    }
    
    cout << dp[n] << endl;
    return 0;
}
```

**时间复杂度**：$O(n)$
**空间复杂度**：$O(n)$

**❓ 斜率优化的核心思想是什么？**

斜率优化的核心思想是：将 DP 方程转化为 $y = kx + b$ 的形式，然后利用 $k$ 的单调性维护一个凸包，从而在 $O(1)$ 时间内找到最优转移点。

**❓ 如何判断是否能用斜率优化？**

1. DP 方程可以写成 $dp[i] = \min_{j < i} \{a[i] \cdot b[j] + c[j]\} + d[i]$ 的形式
2. $a[i]$ 和 $b[j]$ 其中之一单调（通常是 $a[i]$ 单调）
3. 满足这两个条件时，可以用斜率优化将 $O(n^2)$ 降到 $O(n)$

### 3.3 四边形不等式优化

#### 问题定义

有 $n$ 个物品排成一排，第 $i$ 个物品的重量为 $w_i$。将这些物品分成 $k$ 组，每组的代价为组内物品重量之和的平方。求最小总代价。

#### 朴素 DP

定义 $dp[i][j]$ 为将前 $i$ 个物品分成 $j$ 组的最小代价：

$$
dp[i][j] = \min_{j-1 \le k < i} \{dp[k][j-1] + (sum[i] - sum[k])^2\}
$$

其中 $sum[i] = \sum_{t=1}^{i} w_t$。

朴素实现的时间复杂度是 $O(kn^2)$。

#### 四边形不等式优化

**四边形不等式**：如果对于任意 $a \le b \le c \le d$，都有 $w(a, c) + w(b, d) \le w(a, d) + w(b, c)$，则称 $w$ 满足四边形不等式。

**关键定理**：如果 $w(i, j)$ 满足四边形不等式，那么 $dp[i][j]$ 的最优决策点 $K[i][j]$ 满足 $K[i][j-1] \le K[i][j] \le K[i+1][j]$。

利用这个单调性，我们可以将时间复杂度优化到 $O(kn)$。

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

long long minCostPartition(vector<int>& w, int k) {
    int n = w.size();
    vector<long long> sum(n + 1, 0);
    for (int i = 1; i <= n; i++) {
        sum[i] = sum[i - 1] + w[i - 1];
    }
    
    // dp[i][j] 表示前 i 个物品分成 j 组的最小代价
    vector<vector<long long>> dp(n + 1, vector<long long>(k + 1, 1e18));
    dp[0][0] = 0;
    
    // K[i][j] 表示 dp[i][j] 的最优决策点
    // 注意：为了安全访问 K[n+1][j]，需分配 n+2 大小的数组
    vector<vector<int>> K(n + 2, vector<int>(k + 1, 0));
    
    for (int j = 1; j <= k; j++) {
        K[n + 1][j] = n;  // 哨兵
        for (int i = n; i >= j; i--) {
            // 枚举决策点，范围是 [K[i][j-1], K[i+1][j]]
            for (int t = K[i][j - 1]; t <= K[i + 1][j]; t++) {
                long long cost = dp[t][j - 1] + (sum[i] - sum[t]) * (sum[i] - sum[t]);
                if (cost < dp[i][j]) {
                    dp[i][j] = cost;
                    K[i][j] = t;
                }
            }
        }
    }
    
    return dp[n][k];
}
```

**时间复杂度**：$O(kn)$（每个 $j$ 的枚举总次数为 $O(n)$）
**空间复杂度**：$O(kn)$

**❓ 如何判断一个问题是否满足四边形不等式？**

通常需要数学证明。常见的满足四边形不等式的函数包括：
- $w(i, j) = (sum[j] - sum[i])^2$
- $w(i, j) = (j - i)^2$
- $w(i, j) = \sum_{t=i}^{j} c_t$（当 $c_t \ge 0$ 时）

## 五、复杂度与正确性分析

| 优化方法 | 原始复杂度 | 优化后复杂度 | 适用条件 |
|---------|-----------|-------------|---------|
| 单调队列 | $O(nm)$ | $O(n)$ | 滑动窗口最值 |
| 斜率优化 | $O(n^2)$ | $O(n)$ | DP 方程可转化为 $y = kx + b$ |
| 四边形不等式 | $O(kn^2)$ | $O(kn)$ | 代价函数满足四边形不等式 |

**正确性保证**：

1. **单调队列优化**：利用单调性维护滑动窗口最值，保证每次转移都是最优的。

2. **斜率优化**：利用凸包性质，保证找到的转移点是最优的。

3. **四边形不等式优化**：利用决策单调性，缩小枚举范围，保证不遗漏最优解。

## 六、典型应用场景

DP 优化在竞赛中的应用非常广泛，以下是几个典型场景：

1. **滑动窗口问题**：滑动窗口最大值、滑动窗口平均值等。

2. **序列划分问题**：将序列分成若干段，使得总代价最小。

3. **资源分配问题**：在有限的资源下，如何分配使得收益最大。

## 七、小结

本文介绍了三种常见的 DP 优化技巧：**单调队列优化**、**斜率优化** 和 **四边形不等式优化**。

四个核心要点：

1. 单调队列优化适用于滑动窗口最值问题，可以将 $O(nm)$ 优化到 $O(n)$。

2. 斜率优化适用于 DP 方程可转化为 $y = kx + b$ 形式的问题，可以将 $O(n^2)$ 优化到 $O(n)$。

3. 四边形不等式优化适用于代价函数满足四边形不等式的问题，可以将 $O(kn^2)$ 优化到 $O(kn \log n)$。

4. 优化的前提是理解原始 DP 方程，找到可以利用的单调性或凸性。
