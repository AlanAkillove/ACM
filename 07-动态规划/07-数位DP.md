# 数位 DP

> Made by Aki.
> 最后更新于 2026.06.03

数位 DP 是一类处理与数字位数相关的问题的动态规划方法。它的核心思想是按位处理数字，利用记忆化搜索避免重复计算。

---

## 一、引言

来看一个经典问题：

> 给定两个整数 $L$ 和 $R$，求区间 $[L, R]$ 中有多少个数满足：数位上不包含数字 4。
> 例如：$L = 1, R = 10$，答案是 9（只有 4 不满足）。

如果 $R$ 很大（比如 $10^{18}$），暴力枚举每个数显然会超时。我们需要一种更高效的方法。

这就是**数位 DP** 的典型应用——通过按位处理数字，将问题转化为状态转移问题。

## 三、核心推导

### 3.1 基础数位 DP

#### 问题定义

给定两个整数 $L$ 和 $R$，求区间 $[L, R]$ 中有多少个数满足：数位上不包含数字 4。

#### 核心思想

我们不直接计算 $[L, R]$ 的答案，而是先计算 $[0, R]$ 的答案，再减去 $[0, L-1]$ 的答案。

如何计算 $[0, R]$ 的答案？我们从高位到低位逐位处理：

- 如果当前位填的数字小于 $R$ 对应位的数字，那么后续位可以随便填（不受限制）
- 如果当前位填的数字等于 $R$ 对应位的数字，那么后续位仍然受限

#### 状态定义

我们定义 $dp[pos][tight]$ 为：**处理到第 $pos$ 位，是否受上限限制（$tight$）时，满足条件的数字个数**。

- $pos$：当前处理到第几位（从高位到低位）
- $tight$：是否受上限限制（0 或 1）

#### 状态转移方程

$$
dp[pos][tight] = \sum_{d=0}^{maxd} dp[pos+1][tight \& (d == maxd)]
$$

其中 $maxd$ 是当前位能填的最大数字：
- 如果 $tight = 1$，$maxd = R[pos]$
- 如果 $tight = 0$，$maxd = 9$

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int dp[20][2];  // dp[pos][tight]
vector<int> digits;  // R 的各位数字

int dfs(int pos, int tight) {
    if (pos == digits.size()) {
        return 1;  // 一个有效数字
    }
    if (dp[pos][tight] != -1) {
        return dp[pos][tight];
    }
    
    int maxd = tight ? digits[pos] : 9;
    int res = 0;
    for (int d = 0; d <= maxd; d++) {
        if (d == 4) continue;  // 跳过数字 4
        res += dfs(pos + 1, tight && (d == maxd));
    }
    
    return dp[pos][tight] = res;
}

int solve(int R) {
    digits.clear();
    while (R > 0) {
        digits.push_back(R % 10);
        R /= 10;
    }
    reverse(digits.begin(), digits.end());
    memset(dp, -1, sizeof(dp));
    return dfs(0, 1);
}

int main() {
    int L, R;
    cin >> L >> R;
    cout << solve(R) - solve(L - 1) << endl;
    return 0;
}
```

**❓ 为什么需要 $tight$ 状态？**

因为当 $tight = 1$ 时，当前位不能超过 $R$ 对应位的数字；当 $tight = 0$ 时，当前位可以填 0-9 的任意数字。



### 3.2 计数问题

#### 问题定义

给定一个整数 $R$，求 $[1, R]$ 中所有数的数位之和。
例如：$R = 13$，数位之和 = $1+2+3+4+5+6+7+8+9+1+0+1+1+1+2+1+3 = 55$。

> 验证：$1$ 到 $9$ 的数位和为 $45$，$10\sim 13$ 的数位和为 $(1+0)+(1+1)+(1+2)+(1+3)=1+2+3+4=10$，总计 $45+10=55$。

#### 状态定义

我们定义 $dp[pos][tight]$ 为：**处理到第 $pos$ 位，是否受上限限制时，所有数的数位之和**。

#### 状态转移方程

$$
dp[pos][tight] = \sum_{d=0}^{maxd} (d + dp[pos+1][tight \& (d == maxd)])
$$

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

pair<long long, long long> dp[20][2];  // {count, sum}
vector<int> digits;

pair<long long, long long> dfs(int pos, int tight) {
    if (pos == digits.size()) {
        return {1, 0};  // 1 个数字，数位和为 0
    }
    if (dp[pos][tight].first != -1) {
        return dp[pos][tight];
    }
    
    int maxd = tight ? digits[pos] : 9;
    long long cnt = 0, sum = 0;
    for (int d = 0; d <= maxd; d++) {
        auto [child_cnt, child_sum] = dfs(pos + 1, tight && (d == maxd));
        cnt += child_cnt;
        sum += child_sum + d * child_cnt;  // d 要乘以子树的数字个数
    }
    
    return dp[pos][tight] = {cnt, sum};
}

long long solve(int R) {
    if (R < 0) return 0;
    digits.clear();
    while (R > 0) {
        digits.push_back(R % 10);
        R /= 10;
    }
    reverse(digits.begin(), digits.end());
    fill(&dp[0][0], &dp[0][0] + 20 * 2, make_pair(-1LL, 0LL));
    return dfs(0, 1).second;
}

int main() {
    int L, R;
    cin >> L >> R;
    cout << solve(R) - solve(L - 1) << endl;
    return 0;
}
```

### 3.3 数字游戏

#### 问题定义

给定两个整数 $L$ 和 $R$，求区间 $[L, R]$ 中有多少个数满足：相邻数位不相同。
例如：$L = 1, R = 100$，答案是 90（11, 22, 33, ..., 99 不满足）。

#### 状态定义

我们定义 $dp[pos][tight][prev][started]$ 为：**处理到第 $pos$ 位，是否受上限限制，前一位数字为 $prev$，是否已经开始（即前导零阶段已结束）时，满足条件的数字个数**。

其中 $prev$ 的范围是 $0\sim 9$（实际数字），另外用特殊值 $10$ 表示「前导零状态」（还未出现非零数字）。

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int dp[20][2][11][2];  // dp[pos][tight][prev][started]
vector<int> digits;

int dfs(int pos, int tight, int prev, int started) {
    if (pos == digits.size()) {
        return 1;  // 一个有效数字
    }
    if (dp[pos][tight][prev][started] != -1) {
        return dp[pos][tight][prev][started];
    }
    
    int maxd = tight ? digits[pos] : 9;
    int res = 0;
    for (int d = 0; d <= maxd; d++) {
        int nxt_started = started || (d != 0);
        if (started && d == prev) continue;  // 相邻数位不能相同（仅在已开始后检查）
        int nxt_prev = nxt_started ? d : 10;  // 10 表示"前导零状态"（特殊值不与 0-9 冲突）
        res += dfs(pos + 1, tight && (d == maxd), nxt_prev, nxt_started);
    }
    
    dp[pos][tight][prev][started] = res;
    return res;
}

int solve(int R) {
    digits.clear();
    while (R > 0) {
        digits.push_back(R % 10);
        R /= 10;
    }
    reverse(digits.begin(), digits.end());
    memset(dp, -1, sizeof(dp));
    return dfs(0, 1, 10, 0);  // prev=10 表示前导零状态，started=0 表示还没开始
}

int main() {
    int L, R;
    cin >> L >> R;
    cout << solve(R) - solve(L - 1) << endl;
    return 0;
}
```

**❓ 为什么需要 $started$ 状态？**

前导零（leading zeros）是数位 DP 中的常见陷阱。例如数字 $5$ 在处理 3 位数时会被表示为 `005`，但中间的 0 不应该参与「相邻数位是否相同」的判断。$started$ 标志位区分了前导零阶段（未开始）和实际数字阶段（已开始），避免前导零之间的 0 被错误地视为"相邻相同"。

## 四、代码实现

以下是本篇所有核心问题的代码模板。

### 基础数位 DP（不含数字 4）

```cpp
int dp[20][2];
vector<int> digits;

int dfs(int pos, int tight) {
    if (pos == digits.size()) return 1;
    if (dp[pos][tight] != -1) return dp[pos][tight];
    int maxd = tight ? digits[pos] : 9;
    int res = 0;
    for (int d = 0; d <= maxd; d++) {
        if (d == 4) continue;
        res += dfs(pos + 1, tight && (d == maxd));
    }
    return dp[pos][tight] = res;
}
```

### 计数问题（数位之和）

```cpp
pair<long long, long long> dp[20][2];  // {count, sum}
vector<int> digits;

pair<long long, long long> dfs(int pos, int tight) {
    if (pos == digits.size()) return {1, 0};
    if (dp[pos][tight].first != -1) return dp[pos][tight];
    int maxd = tight ? digits[pos] : 9;
    long long cnt = 0, sum = 0;
    for (int d = 0; d <= maxd; d++) {
        auto [child_cnt, child_sum] = dfs(pos + 1, tight && (d == maxd));
        cnt += child_cnt;
        sum += child_sum + d * child_cnt;
    }
    return dp[pos][tight] = {cnt, sum};
}
```

### 数字游戏（相邻数位不相同）

```cpp
int dp[20][2][11][2];
vector<int> digits;

int dfs(int pos, int tight, int prev, int started) {
    if (pos == digits.size()) return 1;
    if (dp[pos][tight][prev][started] != -1) return dp[pos][tight][prev][started];
    int maxd = tight ? digits[pos] : 9;
    int res = 0;
    for (int d = 0; d <= maxd; d++) {
        int nxt_started = started || (d != 0);
        if (started && d == prev) continue;
        int nxt_prev = nxt_started ? d : 10;
        res += dfs(pos + 1, tight && (d == maxd), nxt_prev, nxt_started);
    }
    return dp[pos][tight][prev][started] = res;
}
```

## 五、复杂度与正确性分析

| 问题 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 基础数位 DP | $O(20 \times 2 \times 10)$ | $O(20 \times 2)$ |
| 计数问题 | $O(20 \times 2 \times 10)$ | $O(20 \times 2)$ |
| 数字游戏 | $O(20 \times 2 \times 11 \times 2 \times 10)$ | $O(20 \times 2 \times 11 \times 2)$ |

**正确性保证**：

1. **状态定义正确**：$dp[pos][tight]$ 明确定义为「处理到第 $pos$ 位，是否受上限限制时的答案」。

2. **状态转移正确**：枚举当前位所有可能的数字，递归计算后续位的答案。

3. **记忆化正确**：相同状态的答案相同，避免重复计算。

4. **边界条件正确**：$pos == digits.size()$ 时返回 1（一个有效数字）。

## 六、典型应用场景

数位 DP 在竞赛中的应用非常广泛，以下是几个典型场景：

1. **计数问题**：计算区间内满足某些条件的数字个数。

2. **数字性质**：判断数字是否满足某些性质（不含某个数字、相邻数位不同等）。

3. **数位统计**：统计区间内所有数的数位之和、数位之积等。

**推荐练习题**：

- [洛谷 P2657 windy 数](https://www.luogu.com.cn/problem/P2657) — 相邻数位之差 ≥ 2（需修改条件判断）
- [LeetCode 233. 数字 1 的个数](https://leetcode.cn/problems/number-of-digit-one/) — 数位统计
- [LeetCode 902. 最大为 N 的数字组合](https://leetcode.cn/problems/numbers-at-most-n-given-digit-set/) — 数位 DP 变种
- [Codeforces 55D - Beautiful numbers](https://codeforces.com/problemset/problem/55/D) — 数位 DP + LCM

## 七、小结

本文介绍了数位 DP 的基本思想和三个经典问题：**基础数位 DP**、**计数问题** 和 **数字游戏**。

四个核心要点：

1. 数位 DP 的核心思想是按位处理数字，利用记忆化搜索避免重复计算。

2. $tight$ 状态表示是否受上限限制，是数位 DP 的关键状态。

3. 区间问题可以通过 $solve(R) - solve(L-1)$ 的方式转化为前缀问题。

4. 根据问题的需要，可以增加其他状态（如前一位数字、是否包含某个数字等）。
