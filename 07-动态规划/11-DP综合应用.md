## 三、核心推导

### 3.1 字符串 DP

#### 问题定义

给定一个字符串 $s$ 和一个字典 $wordDict$，判断 $s$ 是否可以被拆分为字典中的一个或多个单词。

#### 状态定义

我们定义 $dp[i]$ 为：**字符串 $s$ 的前 $i$ 个字符是否可以被拆分**。

#### 状态转移方程

对于 $dp[i]$，我们枚举最后一个单词的起始位置 $j$：

$$
dp[i] = \bigvee_{j=0}^{i-1} (dp[j] \land (s[j:i] \in wordDict))
$$

其中 $s[j:i]$ 表示 $s$ 的第 $j$ 到第 $i-1$ 个字符。

#### 边界条件

$dp[0] = true$（空字符串可以被拆分）

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

bool wordBreak(string s, vector<string>& wordDict) {
    int n = s.size();
    unordered_set<string> dict(wordDict.begin(), wordDict.end());
    
    vector<bool> dp(n + 1, false);
    dp[0] = true;
    
    for (int i = 1; i <= n; i++) {
        for (int j = 0; j < i; j++) {
            if (dp[j] && dict.count(s.substr(j, i - j))) {
                dp[i] = true;
                break;
            }
        }
    }
    
    return dp[n];
}
```

**时间复杂度**：$O(n^3)$（最坏情况，$s.substr$ 每次创建子串需 $O(i-j)$ 时间）
**空间复杂度**：$O(n)$

**❓ 为什么 $dp[0] = true$？**

因为空字符串可以被拆分（什么都不选）。如果 $dp[0] = false$，那么所有结果都会是 false。

### 3.2 计数 DP

#### 问题定义

给定一个字符串 $s$ 和一个字符串 $t$，求 $s$ 中有多少个子序列等于 $t$。
例如：$s = "rabbbit"$，$t = "rabbit"$，答案是 3。

#### 状态定义

我们定义 $dp[i][j]$ 为：**$s$ 的前 $i$ 个字符中，有多少个子序列等于 $t$ 的前 $j$ 个字符**。

#### 状态转移方程

对于 $s[i-1]$ 和 $t[j-1]$：

- 如果 $s[i-1] = t[j-1]$，我们可以选择匹配或不匹配：
  - 匹配：$dp[i][j] += dp[i-1][j-1]$
  - 不匹配：$dp[i][j] += dp[i-1][j]$
- 如果 $s[i-1] \ne t[j-1]$，只能不匹配：
  - $dp[i][j] = dp[i-1][j]$

所以状态转移方程为：

$$
dp[i][j] = \begin{cases}
dp[i-1][j-1] + dp[i-1][j], & \text{if } s[i-1] = t[j-1] \\
dp[i-1][j], & \text{otherwise}
\end{cases}
$$

#### 边界条件

$dp[i][0] = 1$（空字符串是任何字符串的子序列）
$dp[0][j] = 0$（$j > 0$，空字符串没有非空子序列）

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int numDistinct(string s, string t) {
    int m = s.size(), n = t.size();
    // 注意：答案可能超出 int 范围，建议根据题目要求使用 long long
    vector<vector<long long>> dp(m + 1, vector<long long>(n + 1, 0));
    
    for (int i = 0; i <= m; i++) {
        dp[i][0] = 1;
    }
    
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (s[i - 1] == t[j - 1]) {
                dp[i][j] = dp[i - 1][j - 1] + dp[i - 1][j];
            } else {
                dp[i][j] = dp[i - 1][j];
            }
        }
    }
    
    return dp[m][n];
}
```

**时间复杂度**：$O(mn)$
**空间复杂度**：$O(mn)$

### 3.3 DP 与贪心结合

#### 问题定义

有 $n$ 个活动，第 $i$ 个活动的开始时间为 $s_i$，结束时间为 $e_i$。选择一些活动，使得它们互不重叠，且数量最多。

这是一个经典的**活动选择问题**，通常用贪心算法解决。但我们可以用 DP 的思路来理解它。

#### 贪心解法

按照结束时间排序，每次选择结束时间最早且不与已选活动重叠的活动。

```cpp
#include <bits/stdc++.h>
using namespace std;

int maxActivities(vector<pair<int, int>>& activities) {
    sort(activities.begin(), activities.end(), [](const auto& a, const auto& b) {
        return a.second < b.second;
    });
    
    int count = 0;
    int last_end = -1;
    for (auto& [s, e] : activities) {
        if (s >= last_end) {
            count++;
            last_end = e;
        }
    }
    
    return count;
}
```

**时间复杂度**：$O(n \log n)$
**空间复杂度**：$O(1)$

#### DP 解法

定义 $dp[i]$ 为考虑前 $i$ 个活动时，最多能选择多少个活动：

$$
dp[i] = \max(dp[i-1], dp[j] + 1)
$$

其中 $j$ 是最大的满足 $e_j \le s_i$ 的活动。

```cpp
#include <bits/stdc++.h>
using namespace std;

int maxActivitiesDP(vector<pair<int, int>>& activities) {
    int n = activities.size();
    sort(activities.begin(), activities.end(), [](const auto& a, const auto& b) {
        return a.second < b.second;
    });
    
    vector<int> dp(n + 1, 0);
    for (int i = 1; i <= n; i++) {
        auto& [s, e] = activities[i - 1];
        // 二分查找最大的 j 满足 e_j <= s
        int j = upper_bound(activities.begin(), activities.begin() + i, s,
            [](int val, const auto& act) { return val < act.second; }) - activities.begin();
        dp[i] = max(dp[i - 1], dp[j] + 1);
    }
    
    return dp[n];
}
```

**时间复杂度**：$O(n \log n)$
**空间复杂度**：$O(n)$

**❓ 为什么贪心和 DP 都能解决这个问题？**

因为这个问题具有**贪心选择性质**：选择结束时间最早的活动一定是最优的。这是贪心算法能工作的前提。DP 解法更通用，可以处理更复杂的情况（比如每个活动有权重）。

## 五、复杂度与正确性分析

| 问题 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 字符串 DP | $O(n^3)$ | $O(n)$ |
| 计数 DP | $O(mn)$ | $O(mn)$ |
| 活动选择（贪心） | $O(n \log n)$ | $O(1)$ |
| 活动选择（DP） | $O(n \log n)$ | $O(n)$ |

**正确性保证**：

1. **字符串 DP**：枚举所有可能的分割点，取最优值。

2. **计数 DP**：利用子序列的性质，匹配或不匹配当前字符。

3. **活动选择**：贪心选择结束时间最早的活动，保证最优性。

## 六、典型应用场景

DP 综合应用在竞赛中的应用非常广泛，以下是几个典型场景：

1. **字符串问题**：单词拆分、正则表达式匹配等。

## 七、小结

本文介绍了三种综合性较强的 DP 问题：**字符串 DP**、**计数 DP** 和 **DP 与贪心结合**。

四个核心要点：

1. 字符串 DP 通常需要枚举分割点或匹配位置，时间复杂度较高。

2. 计数 DP 利用子序列的性质，通过匹配或不匹配当前字符来转移状态。

3. DP 与贪心结合时，需要判断问题是否具有贪心选择性质。

4. 综合问题往往需要灵活运用多种 DP 思想，没有固定的套路。
