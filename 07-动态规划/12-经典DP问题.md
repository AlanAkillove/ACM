## 一、引言

来看两个问题：

> **问题一**：你是一个专业的小偷，计划偷窃沿街的房屋。每间房内藏有一定的现金，相邻的房屋装有连通的防盗系统——如果两间相邻的房屋在同一晚被偷，会触发报警。给定一个非负整数数组，求能偷到的最大金额。
> **问题二**：给定一个数组 `prices[i]` 表示股票在第 $i$ 天的价格。你可以进行最多 $k$ 笔交易（一笔交易包含一次买入和一次卖出），且卖出后才能再次买入。求能获得的最大利润。

这两个问题有什么共同点？每个位置都有多个「状态」——偷或不偷、持有股票或空仓——并且状态之间按照固定规则转移。这就是 **状态机 DP** 的典型特征：$dp[i][state]$ 表示第 $i$ 个位置处于状态 $state$ 时的最优值，状态按照拓扑序转移。

本文选取了六组经典 DP 系列问题，每组问题都遵循「入门 → 进阶 → 变种」的递进路径。目标是让你看到：**看似不同的题目，背后往往是同一个 DP 模型的变形**。

## 三、核心推导

### 3.1 打家劫舍系列——状态机 DP 入门

打家劫舍系列是状态机 DP 最简单的入门模型。核心约束只有一条：**相邻不能同时选**。

#### 3.1.1 打家劫舍 I：线性相邻约束

**问题定义**：给定数组 `nums`，不能选相邻元素，最大化所选元素之和。

**状态定义**：$dp[i][0/1]$ 表示考虑前 $i$ 间房屋，第 $i$ 间**不偷 / 偷**时能获得的最大金额。

**状态转移方程**：

$$
dp[i][0] = \max(dp[i-1][0], dp[i-1][1]) \\
dp[i][1] = dp[i-1][0] + nums[i]
$$

**边界条件**：$dp[0][0] = 0,\ dp[0][1] = nums[0]$

**空间优化**：由于 $dp[i]$ 只依赖 $dp[i-1]$，可以压缩到两个变量：

```cpp
int rob(vector<int>& nums) {
    int no = 0, take = nums[0];
    for (int i = 1; i < nums.size(); i++) {
        int new_no = max(no, take);
        int new_take = no + nums[i];
        no = new_no;
        take = new_take;
    }
    return max(no, take);
}
```

**复杂度**：$O(n)$ 时间，$O(1)$ 空间。

**❓ 为什么不能只定义 $dp[i]$ 为前 $i$ 间的最大金额？**

也是可以的！$dp[i] = \max(dp[i-1], dp[i-2] + nums[i])$ ——这就是状态机压缩后的形式。但状态机视角更容易推广到更复杂的约束（如树形、带权）。

#### 3.1.2 打家劫舍 II：环形

**问题定义**：房屋首尾相连成环，其他条件不变。

核心思路：**分类讨论**——把环拆成两条线。第一间偷则不偷最后一间，第一间不偷则最后一间可选。

```cpp
int rob(vector<int>& nums) {
    int n = nums.size();
    if (n == 1) return nums[0];
    // case 1: 不偷第一间 → 可以在 [1, n-1] 任意偷
    // case 2: 不偷最后一间 → 可以在 [0, n-2] 任意偷
    return max(rob_range(nums, 0, n - 2),
               rob_range(nums, 1, n - 1));
}
```

其中 `rob_range` 是 3.1.1 的线性版本。

**复杂度**：$O(n)$。

#### 3.1.3 打家劫舍 III：树形

**问题定义**：房屋排列从数组变成了二叉树。相邻约束变成了「父子不能同时选」。

这就是树形 DP 中的**最大权独立集**问题。用后序遍历（DFS）逐节点计算：

```cpp
struct Node {
    int val;
    Node *left, *right;
};

pair<int, int> dfs(Node* u) {
    if (!u) return {0, 0};  // {不偷u, 偷u}
    auto [l_no, l_take] = dfs(u->left);
    auto [r_no, r_take] = dfs(u->right);
    int no = max(l_no, l_take) + max(r_no, r_take);
    int take = u->val + l_no + r_no;
    return {no, take};
}

int rob(Node* root) {
    auto [no, take] = dfs(root);
    return max(no, take);
}
```

**复杂度**：$O(n)$。

**对比**：从线性到环形到树形——三种变种覆盖了 DP 中最常见的三种结构（数组、环、树），但核心的「相邻约束」一以贯之。

### 3.2 股票买卖系列——状态机 DP 进阶

股票系列的每一道题都能用统一的状态机框架解决。我们先给出通用模型，再逐一实例化。

**通用状态机**：在第 $i$ 天，我们关心两个维度：
- 当前持有几笔**已完成**的交易（$k$）
- 当前是否**持有股票**（$0$ 空仓，$1$ 持仓）

定义 $dp[i][k][s]$ 为第 $i$ 天结束时，已完成 $k$ 笔交易、持股状态为 $s$ 时的最大利润。

#### 3.2.1 买卖一次（$k = 1$）

最简单的情况。可以在第 $i$ 天卖出（完成交易）或不操作：

$$
dp[i][1][0] = \max(dp[i-1][1][0], dp[i-1][0][1] + prices[i]) \\
dp[i][0][1] = \max(dp[i-1][0][1], -prices[i])
$$

初始：$dp[0][0][1] = -prices[0]$，其余为 $0$。

#### 3.2.2 无限次交易（$k = \infty$）

$k$ 维度可以去掉。只有两个状态：空仓/持仓。

$$
dp[i][0] = \max(dp[i-1][0], dp[i-1][1] + prices[i]) \\
dp[i][1] = \max(dp[i-1][1], dp[i-1][0] - prices[i])
$$

```cpp
int maxProfit(vector<int>& prices) {
    int no = 0, hold = -prices[0];
    for (int i = 1; i < prices.size(); i++) {
        int new_no = max(no, hold + prices[i]);
        int new_hold = max(hold, no - prices[i]);
        no = new_no;
        hold = new_hold;
    }
    return no;
}
```

**复杂度**：$O(n)$ 时间，$O(1)$ 空间。

#### 3.2.3 最多 $k$ 次交易

定义 $dp[t][0/1]$ 为已经完成 $t$ 笔交易时空仓/持仓的最大利润：

```cpp
int maxProfit(int k, vector<int>& prices) {
    int n = prices.size();
    if (k >= n / 2) {  // k 很大时退化为无限次交易
        int ans = 0;
        for (int i = 1; i < n; i++)
            ans += max(0, prices[i] - prices[i-1]);
        return ans;
    }
    vector<vector<int>> dp(k + 1, vector<int>(2, -1e9));
    dp[0][0] = 0;
    dp[0][1] = -prices[0];
    for (int i = 1; i < n; i++) {
        for (int t = 1; t <= k; t++) {
            dp[t][0] = max(dp[t][0], dp[t][1] + prices[i]);
            dp[t][1] = max(dp[t][1], dp[t-1][0] - prices[i]);
        }
        dp[0][1] = max(dp[0][1], -prices[i]);
    }
    int ans = 0;
    for (int t = 0; t <= k; t++) ans = max(ans, dp[t][0]);
    return ans;
}
```

**关键技巧**：当 $k \ge n/2$ 时，问题退化为无限次交易，直接累加所有上涨价差。

#### 3.2.4 冷冻期与手续费

**冷冻期**（卖出后第二天不能买入）：需要在状态中额外记录冷冻状态。

$$
dp[i][0] = \max(dp[i-1][0], dp[i-1][1] + prices[i]) \\
dp[i][1] = \max(dp[i-1][1], dp[i-2][0] - prices[i]) \quad \text{（冷冻期：只能从两天前的不持股状态买入）}
$$

**手续费**：每次交易扣 $fee$，只需在卖出时减去即可。

$$
dp[i][0] = \max(dp[i-1][0], dp[i-1][1] + prices[i] - fee) \\
dp[i][1] = \max(dp[i-1][1], dp[i-1][0] - prices[i])
$$

#### 3.2.5 状态机 DP 通用框架

```
dp[i][state] = 第 i 天处于 state 时的最优值
转移 = 按状态图逐边更新（枚举每条出边）
答案 = 最后一天所有终态中的最优值
```

**关键步骤**：
1. 画出状态转移图（每个状态有哪些入边/出边）
2. 初始状态赋值（第 0 天各状态的值）
3. 按拓扑序逐天转移
4. 空间优化（如只需保留前 1 或 2 天）

### 3.3 路径与网格 DP

网格 DP 是最直观的二维 DP：状态就是格子坐标，转移看移动方向。

#### 3.3.1 不同路径

**问题定义**：$m \times n$ 网格，从左上角走到右下角，每次只能向右或向下，求不同路径数。

$$
dp[i][j] = dp[i-1][j] + dp[i][j-1]
$$

边界：$dp[0][*] = dp[*][0] = 1$。

**含障碍**（LeetCode 63）：遇到障碍物时 $dp[i][j] = 0$。

```cpp
int uniquePathsWithObstacles(vector<vector<int>>& grid) {
    int m = grid.size(), n = grid[0].size();
    vector<long long> dp(n, 0);
    dp[0] = 1;
    for (int i = 0; i < m; i++) {
        if (grid[i][0]) dp[0] = 0;  // 起点/列首有障碍
        for (int j = 1; j < n; j++) {
            if (grid[i][j]) dp[j] = 0;
            else dp[j] += dp[j-1];
        }
    }
    return dp[n-1];
}
```

#### 3.3.2 最小路径和

$$
dp[i][j] = grid[i][j] + \min(dp[i-1][j], dp[i][j-1])
$$

#### 3.3.3 三角形最小路径和

从三角形顶部到底部的最小路径和。可以**从底向上**递推，避免边界判断：

```cpp
int minimumTotal(vector<vector<int>>& triangle) {
    vector<int> dp = triangle.back();
    for (int i = triangle.size() - 2; i >= 0; i--)
        for (int j = 0; j <= i; j++)
            dp[j] = triangle[i][j] + min(dp[j], dp[j+1]);
    return dp[0];
}
```

**复杂度**：$O(n^2)$ 时间，$O(n)$ 空间。

### 3.4 零钱兑换与背包变种

零钱兑换问题本质上是背包问题的变形。掌握它们的关键在于区分**组合数**与**排列数**——遍历顺序决定了答案是组合还是排列。

#### 3.4.1 零钱兑换（完全背包·最小个数）

**问题定义**：给定不同面额的硬币 `coins` 和一个总金额 `amount`，求凑成总金额所需的最少硬币个数（每种硬币可以使用无限次）。

$$
dp[i] = \min_{c \in coins} dp[i - c] + 1
$$

```cpp
int coinChange(vector<int>& coins, int amount) {
    vector<int> dp(amount + 1, amount + 1);
    dp[0] = 0;
    for (int i = 1; i <= amount; i++)
        for (int c : coins)
            if (i >= c) dp[i] = min(dp[i], dp[i - c] + 1);
    return dp[amount] > amount ? -1 : dp[amount];
}
```

**复杂度**：$O(amount \times |coins|)$。

#### 3.4.2 零钱兑换 II（完全背包·方案数·组合）

**问题定义**：求凑成总金额的方案数（不考虑顺序，即组合数）。

**外层循环硬币，内层循环金额**——保证每种组合只被统计一次：

```cpp
int change(int amount, vector<int>& coins) {
    vector<unsigned long long> dp(amount + 1, 0);
    dp[0] = 1;
    for (int c : coins)
        for (int i = c; i <= amount; i++)
            dp[i] += dp[i - c];
    return dp[amount];
}
```

#### 3.4.3 组合总和 IV（完全背包·方案数·排列）

**问题定义**：求凑成目标数的方案数（考虑顺序，即排列数）。

**外层循环金额，内层循环硬币**——保证所有排列都被统计：

```cpp
int combinationSum4(vector<int>& nums, int target) {
    vector<unsigned long long> dp(target + 1, 0);
    dp[0] = 1;
    for (int i = 1; i <= target; i++)
        for (int n : nums)
            if (i >= n) dp[i] += dp[i - n];
    return dp[target];
}
```

**❓ 为什么遍历顺序决定组合还是排列？**

- 先硬币再金额：每个硬币被「固定」为选取顺序中的一分子，不会重复考虑同组硬币的不同排列。
- 先金额再硬币：对每个金额考虑了所有可能的「最后一个硬币」，自然得到了所有排列。

#### 3.4.4 目标和（01 背包变形）

**问题定义**：给定数组 `nums` 和目标值 `target`，给每个数前添加 `+` 或 `-`，使总和等于 `target`，求方案数。

设所有添加 `+` 的数之和为 $P$，添加 `-` 的绝对值之和为 $N$：
- $P + N = sum$
- $P - N = target$
- 解得 $P = (sum + target) / 2$

问题转化为：从数组中选若干数，使其和为 $P$ 的方案数——**01 背包求方案数**。

```cpp
int findTargetSumWays(vector<int>& nums, int target) {
    int sum = accumulate(nums.begin(), nums.end(), 0);
    if (abs(target) > sum || (sum + target) % 2) return 0;
    int W = (sum + target) / 2;
    vector<int> dp(W + 1, 0);
    dp[0] = 1;
    for (int x : nums)
        for (int j = W; j >= x; j--)
            dp[j] += dp[j - x];
    return dp[W];
}
```

### 3.5 回文 DP

回文问题的 DP 解法通常有两种视角：区间 DP（状态是子区间）和中心扩展（非 DP 但更高效）。

#### 3.5.1 最长回文子序列（区间 DP）

**问题定义**：给定字符串 `s`，求最长回文子序列的长度。

$$
dp[i][j] = \begin{cases}
dp[i+1][j-1] + 2, & \text{if } s[i] = s[j] \\
\max(dp[i+1][j], dp[i][j-1]), & \text{otherwise}
\end{cases}
$$

**遍历顺序**：按区间长度从小到大。

```cpp
int longestPalindromeSubseq(string s) {
    int n = s.size();
    vector<vector<int>> dp(n, vector<int>(n, 0));
    for (int i = n - 1; i >= 0; i--) {
        dp[i][i] = 1;
        for (int j = i + 1; j < n; j++) {
            if (s[i] == s[j])
                dp[i][j] = dp[i+1][j-1] + 2;
            else
                dp[i][j] = max(dp[i+1][j], dp[i][j-1]);
        }
    }
    return dp[0][n-1];
}
```

**复杂度**：$O(n^2)$ 时间，$O(n^2)$ 空间（可优化到 $O(n)$）。

#### 3.5.2 最长回文子串（区间 DP / 中心扩展）

区间 DP 做法与 3.5.1 类似，记录最大值对应的子串。更高效的是中心扩展法：

```cpp
string longestPalindrome(string s) {
    int n = s.size(), start = 0, len = 0;
    auto expand = [&](int l, int r) {
        while (l >= 0 && r < n && s[l] == s[r]) l--, r++;
        if (r - l - 1 > len) start = l + 1, len = r - l - 1;
    };
    for (int i = 0; i < n; i++) {
        expand(i, i);     // 奇数长度
        expand(i, i + 1); // 偶数长度
    }
    return s.substr(start, len);
}
```

**复杂度**：$O(n^2)$ 时间，$O(1)$ 空间。

### 3.6 竞赛综合题

前五节都是经典面试题。竞赛中的 DP 题目往往更强调**综合运用**和**优化技巧**。这里精选几道代表性题目。

#### 3.6.1 CF 1155D — Beautiful Array（三状态 DP）

**问题描述**：给定数组 $a$ 和一个整数 $x$，你可以选择至多一个连续子数组，将其每个元素乘以 $x$。求能得到的最大子数组和。

**核心思路**：维护三个状态——还没乘过 $x$、正在乘 $x$ 的区间中、已经乘完 $x$。

$$
dp[0] = \max(0, dp[0] + a_i)             \quad \text{（从未乘 } x \text{）}\\
dp[1] = \max(dp[0], dp[1]) + x \cdot a_i \quad \text{（正在乘 } x \text{）}\\
dp[2] = \max(dp[1], dp[2]) + a_i         \quad \text{（已经乘完 } x \text{）}
$$

```cpp
long long maxSubarrayWithMultiply(vector<int>& a, int x) {
    long long ans = 0, d0 = 0, d1 = 0, d2 = 0;
    for (int v : a) {
        d2 = max(d1, d2) + v;     // 乘完后恢复原值
        d1 = max(d0, d1) + 1LL * x * v;  // 正在乘
        d0 = max(0LL, d0 + v);    // 从未乘过
        ans = max({ans, d0, d1, d2});
    }
    return ans;
}
```

**为什么好题**：三状态并不是题目直接给出的，而是从「最多一段乘 $x$」这个约束中自己设计出来的——这正是竞赛 DP 的核心能力。

#### 3.6.2 CF 1092F — Tree with Maximum Cost（换根 DP）

**问题描述**：给一棵树，每个节点有权值。定义一个节点的「代价」为所有节点到它的距离乘以权值之和，求最大代价。

**换根 DP**（二次扫描）的标准流程：

1. **第一次 DFS**：以 1 为根，计算 `sum[u]`（子树权值和）和 `dp[u]`（子树内的代价和）。
2. **第二次 DFS**：将根从父节点 $p$ 换到子节点 $u$：
   - 子节点 $u$ 子树中的节点距离减 1（代价减少 $sum[u]$）
   - 其余节点距离加 1（代价增加 $totalSum - sum[u]$）
   - $dp[u] = dp[p] - sum[u] + (totalSum - sum[u])$

```cpp
vector<vector<int>> g;
vector<long long> val, dp, sum;
long long total;

void dfs1(int u, int p, int depth) {
    sum[u] = val[u];
    dp[0] += depth * val[u];  // 以 0 为根的初始代价
    for (int v : g[u]) if (v != p) {
        dfs1(v, u, depth + 1);
        sum[u] += sum[v];
    }
}

void dfs2(int u, int p) {
    for (int v : g[u]) if (v != p) {
        dp[v] = dp[u] - sum[v] + (total - sum[v]);
        dfs2(v, u);
    }
}
```

**复杂度**：$O(n)$。

#### 3.6.3 AtCoder DP K — Stones（博弈 DP）

**问题描述**：有 $n$ 堆石子，每堆 $a_i$ 个。两人轮流取石子，每次可以取 $a_i$ 个（来自给定的集合）。无法取的人输。判断先手是否必胜。

$dp[i] = true$ 表示剩余 $i$ 个石子时先手必胜：

$$
dp[i] = \bigvee_{a \in A, a \le i} \lnot dp[i - a]
$$

即：只要存在一种取法使得对手面临必败态，当前就是必胜态。

```cpp
vector<bool> dp(K + 1, false);
for (int i = 1; i <= K; i++)
    for (int a : A)
        if (i >= a && !dp[i - a])
            dp[i] = true;
```

**复杂度**：$O(K \cdot n)$。

#### 3.6.4 AtCoder DP M — Candies（前缀和优化 DP）

**问题描述**：$n$ 个小朋友分 $k$ 颗糖，第 $i$ 个小朋友最多拿 $a_i$ 颗，求分配方案数。

$$
dp[i][j] = \sum_{t = \max(0, j - a_i)}^{j} dp[i-1][t]
$$

直接枚举 $t$ 是 $O(nk^2)$。但求和区间是连续的，可以用**前缀和**优化：

```cpp
vector<long long> dp(K + 1, 0), pre(K + 1, 0);
dp[0] = 1;
for (int i = 0; i < n; i++) {
    pre[0] = dp[0];
    for (int j = 1; j <= K; j++)
        pre[j] = (pre[j-1] + dp[j]) % MOD;
    for (int j = 0; j <= K; j++) {
        int l = max(0, j - a[i]);
        dp[j] = (pre[j] - (l ? pre[l-1] : 0) + MOD) % MOD;
    }
}
```

**复杂度**：$O(nk)$（前缀和将内层循环从 $O(k)$ 降为 $O(1)$）。

## 四、代码实现

以下代码模板可在 [codes/](codes/) 目录下找到完整实现。

### 打家劫舍 I（线性）

```cpp
int rob(vector<int>& nums) {
    int no = 0, take = nums[0];
    for (int i = 1; i < nums.size(); i++) {
        int new_no = max(no, take);
        int new_take = no + nums[i];
        no = new_no, take = new_take;
    }
    return max(no, take);
}
```

### 股票买卖（无限次交易，状态机版）

```cpp
int maxProfit(vector<int>& prices) {
    int no = 0, hold = -prices[0];
    for (int i = 1; i < prices.size(); i++) {
        int new_no = max(no, hold + prices[i]);
        int new_hold = max(hold, no - prices[i]);
        no = new_no, hold = new_hold;
    }
    return no;
}
```

### 换根 DP（树的代价）

```cpp
void dfs1(int u, int p, int d);  // 计算初始 dp 和子树和
void dfs2(int u, int p);         // 换根转移
long long solve() {

    dfs2(0, -1);
    return *max_element(dp.begin(), dp.end());
}
```

### 前缀和优化 DP（分糖果）

```cpp
long long distribute(vector<int>& a, int K) {
    vector<long long> dp(K + 1, 0), pre(K + 1);
    dp[0] = 1;
    for (int x : a) {
        partial_sum(dp.begin(), dp.end(), pre.begin());
        for (int j = 0; j <= K; j++) {
            int l = max(0, j - x);
            dp[j] = (pre[j] - (l ? pre[l-1] : 0) + MOD) % MOD;
        }
    }
    return dp[K];
}
```

## 六、典型应用场景

1. **序列相邻约束问题**：任何具有「相邻元素冲突」性质的问题都可以用打家劫舍的状态机模型解决。例如：选课排座位、活动安排等。

2. **状态随时间变化的问题**：当问题可以分解为时间轴上的一组离散状态（持有/空仓、空闲/忙碌等）时，状态机 DP 是首选框架。股票买卖是它的标准范例。

3. **网格路径问题**：走格子类问题（机器人寻路、三角形路径等）直接用网格 DP。

4. **背包变形问题**：「选或不选 + 计数」类问题，关键是判断遍历顺序——先物品还是先容量。

5. **竞赛综合问题**：
   - 需要自己设计状态的题（如 CF 1155D 的三状态）
   - 需要换根扫描的树上问题（如 CF 1092F）
   - 需要数学优化降维的计数 DP（如 AtCoder DP M 的前缀和优化）

## 七、小结

本文通过六组经典问题展示了 DP 的几个高阶视角：

1. **状态机 DP**：打家劫舍和股票买卖本质上共享同一套建模框架——在时间轴上定义有限个离散状态，按规则转移。区别只在于状态数量和转移图的复杂度。

2. **网格 DP 与背包变种**：基础 DP 模型的经典应用。遍历方向的细微差别（先物品还是先容量）会导致完全不同的语义（组合 vs 排列）。

3. **区间 DP**：回文子序列问题展示了如何在区间上定义状态、按长度转移。

4. **竞赛综合 DP**：换根 DP、三状态 DP、博弈 DP、前缀和优化——这些是竞赛中高于模板题的综合能力要求。掌握了它们，意味着你已经能够：
   - 从题目描述中**设计**合适的 DP 状态
   - 选择正确的**优化技巧**把复杂度降到可接受范围
   - 灵活运用**换根**等进阶手法
