# 概率与期望 DP

> Made by Aki.
> 最后更新于 2026.06.03

概率与期望 DP 是一类处理概率和期望问题的动态规划方法。它的核心思想是利用期望的线性性质，将复杂问题分解为子问题。

## 一、引言

来看一个经典问题：

> 有一个 $n$ 面的骰子，每面的概率相等。求掷骰子直到出现大于等于 $k$ 的数的期望次数。

这个问题涉及到概率和期望，用普通的 DP 思路不太容易解决。我们需要引入**期望 DP** 的概念。

## 三、核心推导

### 3.1 期望 DP 基础

#### 问题定义

有一个 $n$ 面的骰子，每面的概率相等。求掷骰子直到出现大于等于 $k$ 的数的期望次数。

#### 状态定义

我们定义 $dp[i]$ 为：**当前数为 $i$ 时，掷骰子直到出现大于等于 $k$ 的数的期望次数**。

#### 状态转移方程

对于 $i < k$，我们掷一次骰子，有 $n$ 种可能的结果：

$$
dp[i] = 1 + \frac{1}{n} \sum_{j=1}^{n} dp[j]
$$

对于 $i \ge k$，已经满足条件，$dp[i] = 0$。

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, k;
    cin >> n >> k;
    
    // dp[i] 表示当前数为 i 时，还需要掷的期望次数
    vector<double> dp(n + 1, 0);
    
    // 计算所有 dp[j] 的和
    double sum = 0;
    for (int j = 1; j < k; j++) {
        sum += dp[j];
    }
    
    // 所有 dp[i] (i < k) 都相等，设其值为 x
    // x = 1 + (1/n) * [(k-1) * x + (n-k+1) * 0]
    // x = 1 + (k-1)/n * x
    // x * (n-k+1)/n = 1
    // x = n / (n-k+1)
    
    double ans = (double)n / (n - k + 1);
    cout << fixed << setprecision(10) << ans << endl;
    
    return 0;
}
```

**❓ 为什么所有 $dp[i]$ ($i < k$) 都相等？**

因为骰子每面的概率相等，且转移方程对所有 $i < k$ 都是一样的。所以它们的期望值相同。

### 3.2 博弈期望

#### 问题定义

A 和 B 玩一个游戏，轮流掷一个 $n$ 面的骰子。谁先掷出大于等于 $k$ 的数谁赢。A 先手，求 A 赢的概率。

#### 状态定义

我们定义 $dp[i]$ 为：**当前轮到玩家 $i$ 掷骰子时，$i$ 赢的概率**。

#### 状态转移方程

对于玩家 $i$，掷一次骰子，有 $\frac{k-1}{n}$ 的概率继续游戏，有 $\frac{n-k+1}{n}$ 的概率直接赢：

$$
dp[i] = \frac{n-k+1}{n} + \frac{k-1}{n} \cdot dp[1-i]
$$

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, k;
    cin >> n >> k;
    
    // dp[0] 表示 A 赢的概率，dp[1] 表示 B 赢的概率
    vector<double> dp(2);
    
    // 解方程：
    // dp[0] = (n-k+1)/n + (k-1)/n * dp[1]
    // dp[1] = (n-k+1)/n + (k-1)/n * dp[0]
    // 由于 dp[0] + dp[1] = 1，解得：
    // 设 p = dp[0]，由对称性 dp[1] = 1-p
    // p = (n-k+1)/n + (k-1)/n * (1-p)
    // 解得：p = n / (n+k-1)
    dp[0] = (double)n / (n + k - 1);
    
    cout << fixed << setprecision(10) << dp[0] << endl;
    
    return 0;
}
```

**❓ 为什么 $dp[0] + dp[1] = 1$？**

因为游戏一定会结束（概率为 1），所以 A 赢的概率 + B 赢的概率 = 1。

### 3.3 绿豆蛙的归宿

#### 问题定义

有一个 $n$ 个节点的有向图，从节点 1 出发，每条边有一个概率。求到达节点 $n$ 的期望步数。

#### 状态定义

我们定义 $dp[i]$ 为：**从节点 $i$ 出发，到达节点 $n$ 的期望步数**。

#### 状态转移方程

对于节点 $i$，它有 $m$ 条出边，每条边 $(i, j)$ 的概率为 $p_{ij}$：

$$
dp[i] = \sum_{(i, j)} p_{ij} \cdot (dp[j] + 1)
$$

#### 边界条件

$dp[n] = 0$（已经到达终点）

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 100005;
vector<pair<int, double>> adj[MAXN];  // 邻接表，存 (子节点, 概率)
double dp[MAXN];
bool vis[MAXN];

double dfs(int u, int n) {
    if (u == n) return 0;
    if (vis[u]) return dp[u];
    vis[u] = true;
    
    dp[u] = 0;
    for (auto& [v, p] : adj[u]) {
        dp[u] += p * (dfs(v, n) + 1);
    }
    
    return dp[u];
}

int main() {
    int n, m;
    cin >> n >> m;
    
    for (int i = 0; i < m; i++) {
        int u, v;
        double p;
        cin >> u >> v >> p;
        adj[u].push_back({v, p});
    }
    
    cout << fixed << setprecision(10) << dfs(1, n) << endl;
    
    return 0;
}
```

**时间复杂度**：$O(n + m)$
**空间复杂度**：$O(n + m)$

**❓ 为什么用记忆化搜索而不是递推？**

本题的图是 **DAG（有向无环图）**，因此可以使用记忆化搜索（或拓扑排序递推）。对于有环图，期望 DP 需要建立线性方程组并用高斯消元法求解，远超本文范围。

### 3.4 博弈 DP（确定性博弈）

第 3.2 节的「博弈期望」是从概率角度分析随机游戏——掷骰子、抽牌等。但竞赛中另一类常见问题是**确定性博弈**：两名玩家轮流操作，信息完全公开，没有随机因素。这类问题同样可以用 DP 求解，核心是**必胜态与必败态的递归定义**。

#### 问题定义

> 有 $n$ 堆石子，每堆数量为 $a_1, a_2, \dots, a_n$。两名玩家轮流取石子，每次可以从任意一堆中取走任意正整数个石子。无法取石子的人输。判断先手是否必胜。

这就是经典的 **Nim 游戏**：先手必胜当且仅当 $a_1 \oplus a_2 \oplus \cdots \oplus a_n \neq 0$（异或和不为零）。但更一般的博弈问题无法用这样简单的公式，需要 DP。

#### 博弈 DP 的核心框架

**状态定义**：$dp[i]$ 表示剩余 $i$ 个石子时，当前操作的一方是否能必胜（$true$ = 必胜态）。

**转移**：当前玩家可以取 $a_1, a_2, \dots, a_k$ 个石子（给定集合中的值）。如果存在一种取法使得对手进入**必败态**，则当前是必胜态：

$$
dp[i] = \bigvee_{a \in A,\ a \le i} \lnot dp[i - a]
$$

**边界**：$dp[0] = false$（没有石子可取，当前玩家输）。

```cpp
// AtCoder DP K - Stones
// 给定 n 种取法 A[0..n-1]，总石子数 K
vector<bool> dp(K + 1, false);
for (int i = 1; i <= K; i++) {
    for (int a : A) {
        if (i >= a && !dp[i - a]) {
            dp[i] = true;
            break;
        }
    }
}
cout << (dp[K] ? "First" : "Second") << endl;
```

**复杂度**：$O(K \cdot n)$。

#### 区间博弈

另一种常见形式是**区间博弈**：在序列两端轮流取数，双方都想最大化自己的总和。

> 给定一个序列 $a[1..n]$，两人轮流从两端取走一个数，求先手能获得的最大总和。

定义 $dp[l][r]$ 为子数组 $[l, r]$ 上先手能获得的最大分数差（先手总分 - 后手总分）：

$$
dp[l][r] = \max(a[l] - dp[l+1][r], a[r] - dp[l][r-1])
$$

```cpp
vector<vector<int>> dp(n, vector<int>(n, 0));
for (int i = 0; i < n; i++) dp[i][i] = a[i];
for (int len = 2; len <= n; len++) {
    for (int l = 0, r = l + len - 1; r < n; l++, r++) {
        dp[l][r] = max(a[l] - dp[l+1][r], a[r] - dp[l][r-1]);
    }
}
// 先手最大总和 = (总和 + dp[0][n-1]) / 2
```

#### SG 函数

对于更复杂的博弈（多个独立子游戏组合），仅靠必胜/必败态不够。**SG 函数**（Sprague-Grundy）是更强大的工具。对于每个状态，其 SG 值定义为所有后继状态的 SG 值集合的 mex（最小非负整数）。多个独立游戏的 SG 值等于各子游戏 SG 值的异或。

但 SG 函数的系统讲解需要专门的篇幅，本文仅做概念介绍。竞赛中大多数博弈 DP 直接用必胜态/必败态转移即可求解。

### 绿豆蛙的归宿

```cpp
const int MAXN = 100005;
vector<pair<int, double>> adj[MAXN];
double dp[MAXN];
bool vis[MAXN];

double dfs(int u, int n) {
    if (u == n) return 0;
    if (vis[u]) return dp[u];
    vis[u] = true;
    dp[u] = 0;

```

### 博弈 DP（取石子）

```cpp
bool canWin(int K, vector<int>& A) {
    vector<bool> dp(K + 1, false);
    for (int i = 1; i <= K; i++)
        for (int a : A)
            if (i >= a && !dp[i - a])
                { dp[i] = true; break; }
    return dp[K];
}
```

### 博弈 DP（区间取数）

```cpp
int maxDiff(vector<int>& a) {
    int n = a.size();
    vector<vector<int>> dp(n, vector<int>(n, 0));
    for (int i = 0; i < n; i++) dp[i][i] = a[i];
    for (int len = 2; len <= n; len++)
        for (int l = 0, r = l + len - 1; r < n; l++, r++)
            dp[l][r] = max(a[l] - dp[l+1][r], a[r] - dp[l][r-1]);
    int total = accumulate(a.begin(), a.end(), 0);
    return (total + dp[0][n-1]) / 2;  // 先手最大总和
}
```

## 五、复杂度与正确性分析

| 问题 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 期望 DP 基础 | $O(1)$ | $O(1)$ |
| 博弈期望 | $O(1)$ | $O(1)$ |
| 绿豆蛙的归宿 | $O(n + m)$ | $O(n + m)$ |
| 博弈 DP（取石子） | $O(K \cdot n)$ | $O(K)$ |
| 博弈 DP（区间取数） | $O(n^2)$ | $O(n^2)$ |

**正确性保证**：

1. **期望的线性性质**：$E[X + Y] = E[X] + E[Y]$，这是期望 DP 的数学基础。

2. **全期望公式**：$E[X] = \sum_{i} P(A_i) \cdot E[X | A_i]$，这是状态转移的理论依据。

3. **边界条件正确**：$dp[n] = 0$（已经到达终点）。

## 六、典型应用场景

概率与期望 DP 在竞赛中的应用非常广泛，以下是几个典型场景：

1. **随机过程**：掷骰子、抽卡等随机过程的期望。

2. **博弈问题**：轮流操作的游戏，求某一方获胜的概率。

3. **图论问题**：在有向图上随机游走的期望步数。

## 七、小结

本文介绍了概率与期望 DP 的基本思想和四个经典问题：**期望 DP 基础**、**博弈期望**、**绿豆蛙的归宿** 和 **博弈 DP**。

五个核心要点：

1. 期望 DP 利用期望的线性性质，将复杂问题分解为子问题。

2. 状态转移方程通常为 $dp[i] = \sum_{j} p_{ij} \cdot (dp[j] + cost)$。

3. 记忆化搜索是处理图上期望 DP 的常用方法，可以自动处理环的情况。

4. 概率和期望问题需要扎实的概率论基础，建议先复习相关知识。

5. 确定性博弈 DP 的核心是「必胜态存在一条到必败态的转移」这一递归定义，与概率博弈的期望值计算有本质区别。
