## 一、引言

来看两个经典问题：

> **问题一**：给定一个整数数组 $nums$，找到其中最长严格递增子序列的长度。
> 例如：$nums = [10, 9, 2, 5, 3, 7, 101, 18]$，最长递增子序列是 $[2, 3, 7, 101]$，长度为 4。

> **问题二**：给定两个字符串 $text1$ 和 $text2$，返回它们的最长公共子序列的长度。
> 例如：$text1 = "abcde"$，$text2 = "ace"$，最长公共子序列是 $"ace"$，长度为 3。

这两个问题有什么共同点？它们都需要在一个序列上进行决策，且当前决策会影响后续选择。这就是**线性 DP** 的典型特征。

本文将详细推导这两个经典问题的解法，并介绍它们的一个重要应用——编辑距离。

## 三、核心推导

### 3.1 最长递增子序列（LIS）

#### 问题定义

给定一个长度为 $n$ 的数组 $nums$，找到最长严格递增子序列的长度。

**子序列**：从原数组中删除一些（可以是零个）元素，不改变剩余元素的相对顺序。

#### 状态定义

我们定义 $dp[i]$ 为：**以 $nums[i]$ 结尾的最长递增子序列的长度**。

为什么这样定义？因为当我们考虑 $nums[i]$ 时，我们需要知道：如果把 $nums[i]$ 放在子序列的最后，最长能有多长？

#### 状态转移方程

对于每个 $nums[i]$，我们需要检查它前面的所有元素 $nums[j]$（$j < i$）：

- 如果 $nums[j] < nums[i]$，说明 $nums[i]$ 可以接在以 $nums[j]$ 结尾的子序列后面
- 此时以 $nums[i]$ 结尾的 LIS 长度 = 以 $nums[j]$ 结尾的 LIS 长度 + 1

所以状态转移方程为：

$$
dp[i] = \max_{0 \le j < i, nums[j] < nums[i]} \{dp[i],dp[j]+1\} 
$$

如果不存在这样的 $j$（即 $nums[i]$ 比前面所有元素都小），则 $dp[i] = 1$。

#### 边界条件

$dp[i] = 1$（每个元素自身就是一个长度为 1 的子序列）

#### 最终答案

$\max_{0 \le i < n} \{dp[i]\}$（所有位置的 LIS 长度的最大值）

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int lengthOfLIS(vector<int>& nums) {
    int n = nums.size();
    vector<int> dp(n, 1);  // 初始化为 1
    
    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (nums[j] < nums[i]) {
                dp[i] = max(dp[i], dp[j] + 1);
            }
        }
    }
    
    return *max_element(dp.begin(), dp.end());
}
```

**时间复杂度**：$O(n^2)$
**空间复杂度**：$O(n)$

#### 优化：$O(n \log n)$ 解法

> 贪心+二分

观察状态转移方程，我们发现：对于每个 $nums[i]$，我们需要找到所有 $nums[j] < nums[i]$ 中 $dp[j]$ 的最大值。

我们可以维护一个数组 $tails$，其中 $tails[k]$ 表示长度为 $k+1$ 的递增子序列的最小末尾元素。

**关键洞察**：$tails$ 数组一定是递增的。因为如果长度为 $k+1$ 的子序列末尾元素比长度为 $k$ 的还小，那长度为 $k$ 的子序列末尾元素可以替换为更小的值。

对于每个 $nums[i]$，我们在 $tails$ 中二分查找第一个 $\ge nums[i]$ 的位置 $pos$：
- 如果 $pos = tails.size()$，说明 $nums[i]$ 比所有元素都大，可以扩展子序列长度
- 否则，更新 $tails[pos] = nums[i]$
示例：
$nums=[10,9,2,5,3,7,101,18]$ 

| num   | 10  | 9   | 2   | 5   | 3   | 7     | 101       | 18       |
| ----- | --- | --- | --- | --- | --- | ----- | --------- | -------- |
| tails | 10  | 9   | 2   | 2,5 | 2,3 | 2,3,7 | 2,3,7,101 | 2,3,7,18 |
故 $tais.size()=LIS.size()=4$ 。


```cpp
#include <bits/stdc++.h>
using namespace std;

int lengthOfLIS(vector<int>& nums) {
    vector<int> tails;
    for (int num : nums) {
        auto it = lower_bound(tails.begin(), tails.end(), num);
        if (it == tails.end()) {
            tails.push_back(num);
        } else {
            *it = num;
        }
    }
    return tails.size();
}
```

**时间复杂度**：$O(n \log n)$
**空间复杂度**：$O(n)$

**❓ 为什么 $tails$ 数组的长度就是 LIS 的长度？**

因为 $tails[k]$ 存储的是长度为 $k+1$ 的递增子序列的最小末尾元素。如果 $tails$ 有 $m$ 个元素，说明存在长度为 $m$ 的递增子序列。而且由于我们总是更新为更小的值，这个长度是最优的。

### 3.2 最长公共子序列（LCS）

#### 问题定义

给定两个字符串 $text1$ 和 $text2$，返回它们的最长公共子序列的长度。

#### 状态定义

我们定义 $dp[i][j]$ 为：**$text1$ 的前 $i$ 个字符和 $text2$ 的前 $j$ 个字符的最长公共子序列长度**。

#### 状态转移方程

对于 $text1[i-1]$ 和 $text2[j-1]$（注意下标从 0 开始，但 $dp$ 从 1 开始）：

- 如果 $text1[i-1] = text2[j-1]$，说明这两个字符可以匹配，$dp[i][j] = dp[i-1][j-1] + 1$
- 如果 $text1[i-1] \ne text2[j-1]$，说明这两个字符不匹配，我们有两种选择：
  - 跳过 $text1[i-1]$，看 $text1$ 的前 $i-1$ 个字符和 $text2$ 的前 $j$ 个字符
  - 跳过 $text2[j-1]$，看 $text1$ 的前 $i$ 个字符和 $text2$ 的前 $j-1$ 个字符
  - 取两者的最大值

所以状态转移方程为：

$$
dp[i][j] = \begin{cases}
dp[i-1][j-1] + 1, & \text{if } text1[i-1] = text2[j-1] \\
\max(dp[i-1][j], dp[i][j-1]), & \text{otherwise}
\end{cases}
$$

#### 边界条件

$dp[0][j] = 0$（$text1$ 为空串）
$dp[i][0] = 0$（$text2$ 为空串）

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int longestCommonSubsequence(string text1, string text2) {
    int m = text1.size(), n = text2.size();
    vector<vector<int>> dp(m + 1, vector<int>(n + 1, 0));
    
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (text1[i - 1] == text2[j - 1]) {
                dp[i][j] = dp[i - 1][j - 1] + 1;
            } else {
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);
            }
        }
    }
    return dp[m][n];
}
```

**时间复杂度**：$O(mn)$
**空间复杂度**：$O(mn)$

**空间优化**：由于 $dp[i][j]$ 只依赖于 $dp[i-1][j-1]$、$dp[i-1][j]$ 和 $dp[i][j-1]$，我们可以用滚动数组优化到 $O(n)$。

### 3.3 编辑距离

#### 问题定义

给定两个字符串 $word1$ 和 $word2$，计算将 $word1$ 转换成 $word2$ 所需的最少操作数。允许的操作有：
- 插入一个字符
- 删除一个字符
- 替换一个字符

#### 状态定义

我们定义 $dp[i][j]$ 为：**$word1$ 的前 $i$ 个字符转换成 $word2$ 的前 $j$ 个字符所需的最少操作数**。

#### 状态转移方程

对于 $word1[i-1]$ 和 $word2[j-1]$：

- 如果 $word1[i-1] = word2[j-1]$，不需要操作，$dp[i][j] = dp[i-1][j-1]$
- 如果 $word1[i-1] \ne word2[j-1]$，我们有三种选择：
  - 替换：将 $word1[i-1]$ 替换为 $word2[j-1]$，$dp[i][j] = dp[i-1][j-1] + 1$
  - 删除：删除 $word1[i-1]$，$dp[i][j] = dp[i-1][j] + 1$
  - 插入：在 $word1$ 末尾插入 $word2[j-1]$，$dp[i][j] = dp[i][j-1] + 1$
  - 取三者的最小值

所以状态转移方程为：

$$
dp[i][j] = \begin{cases}
dp[i-1][j-1], & \text{if } word1[i-1] = word2[j-1] \\
\min(dp[i-1][j-1], dp[i-1][j], dp[i][j-1]) + 1, & \text{otherwise}
\end{cases}
$$

#### 边界条件

$dp[0][j] = j$（$word1$ 为空串，需要插入 $j$ 个字符）
$dp[i][0] = i$（$word2$ 为空串，需要删除 $i$ 个字符）

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int minDistance(string word1, string word2) {
    int m = word1.size(), n = word2.size();
    vector<vector<int>> dp(m + 1, vector<int>(n + 1));
    
    // 初始化边界条件
    for (int i = 0; i <= m; i++) dp[i][0] = i;
    for (int j = 0; j <= n; j++) dp[0][j] = j;
    
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (word1[i - 1] == word2[j - 1]) {
                dp[i][j] = dp[i - 1][j - 1];
            } else {
                dp[i][j] = min({dp[i - 1][j - 1], dp[i - 1][j], dp[i][j - 1]}) + 1;
            }
        }
    }
    
    return dp[m][n];
}
```

**时间复杂度**：$O(mn)$
**空间复杂度**：$O(mn)$

**❓ 编辑距离和 LCS 有什么关系？**

编辑距离和 LCS 都是经典的字符串 DP 问题。它们的状态定义和转移方程非常相似，但目标不同：
- LCS 求的是「最长公共子序列」
- 编辑距离求的是「最少操作次数」

实际上，编辑距离可以看作 LCS 的推广：当只允许插入和删除操作时，编辑距离 = $m + n - 2 \times LCS$。

## 五、复杂度与正确性分析

| 问题 | 时间复杂度 | 空间复杂度 | 空间优化后 |
|------|-----------|-----------|-----------|
| LIS（朴素） | $O(n^2)$ | $O(n)$ | - |
| LIS（优化） | $O(n \log n)$ | $O(n)$ | - |
| LCS | $O(mn)$ | $O(mn)$ | $O(n)$ |
| 编辑距离 | $O(mn)$ | $O(mn)$ | $O(n)$ |

**正确性保证**：

1. **状态定义正确**：每个 $dp$ 值都有明确的含义，且能覆盖所有可能的情况。

2. **状态转移正确**：每个状态都考虑了所有可能的转移，且取最优值。

3. **边界条件正确**：初始化时考虑了最简单的情况（空串、单个字符）。

4. **计算顺序正确**：从左到右、从上到下计算，保证依赖的状态已经计算过。

## 六、典型应用场景

线性 DP 在竞赛中的应用非常广泛，以下是几个典型场景：

1. **序列问题**：在数组或字符串上进行决策，如 LIS、LCS。

2. **字符串匹配**：编辑距离、正则表达式匹配等。

3. **路径问题**：网格路径、数字三角形等。

4. **股票问题**：买卖股票的最佳时机等。

## 七、小结

本文介绍了三个经典的线性 DP 问题：**最长递增子序列（LIS）**、**最长公共子序列（LCS）** 和 **编辑距离**。

四个核心要点：

1. 线性 DP 的特征是在序列上进行决策，当前决策会影响后续选择。

2. LIS 有 $O(n^2)$ 和 $O(n \log n)$ 两种解法，后者利用了 $tails$ 数组的单调性。

3. LCS 和编辑距离都是二维 DP，状态转移方程非常相似，可以相互转化。

4. 空间优化是常见的技巧，当状态转移只依赖前一行时可以用滚动数组。
