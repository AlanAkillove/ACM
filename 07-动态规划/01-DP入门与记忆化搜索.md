## 一、引言

来看一个经典问题：

> 有一只青蛙，每次可以跳 1 级台阶或 2 级台阶。问：跳上 $n$ 级台阶有多少种不同的方法？

比如 $n = 3$ 时，有 3 种跳法：
- 1 → 1 → 1
- 1 → 2
- 2 → 1

你可能会想：这不就是斐波那契数列吗？$f(n) = f(n-1) + f(n-2)$。

但等等——为什么是斐波那契数列？这个递推式是怎么来的？如果我们用递归实现，为什么 $n = 50$ 时程序会卡住不动？

这些问题引出了动态规划的核心思想：**如何高效地计算具有重叠子问题的递推关系**。

本文将从最基础的概念建起——递归、记忆化搜索、递推——为后续所有 DP 专题打下地基。即使你之前对动态规划只有「递推」的模糊印象，读完这篇也应该能回答三个问题：

1. 什么问题适合用动态规划？
2. 记忆化搜索和递推有什么区别？
3. 为什么动态规划能避免重复计算？

## 三、核心推导

### 3.1 从递归说起——自然的思维方式

对于青蛙跳台阶问题，我们自然会这样思考：

> 要跳到第 $n$ 级台阶，最后一步有两种选择：
> - 从第 $n-1$ 级跳 1 级上来
> - 从第 $n-2$ 级跳 2 级上来

所以，跳到第 $n$ 级的方法数 = 跳到第 $n-1$ 级的方法数 + 跳到第 $n-2$ 级的方法数。

用数学语言描述：

$$
f(n) = f(n-1) + f(n-2), \quad f(1) = 1, \quad f(2) = 2
$$

直接翻译成递归代码：

```cpp
int climbStairs(int n) {
    if (n <= 2) return n;
    return climbStairs(n - 1) + climbStairs(n - 2);
}
```

这段代码简洁优美，但有个致命问题：**太慢了**。

**❓ 为什么慢？** 让我们画出 $n = 5$ 时的递归调用树：

```
                    f(5)
                   /    \
               f(4)      f(3)
              /    \    /    \
          f(3)   f(2) f(2)  f(1)
         /    \
     f(2)   f(1)
```

注意到了吗？$f(3)$ 被计算了 2 次，$f(2)$ 被计算了 3 次。当 $n$ 增大时，重复计算的数量呈指数级增长。

**时间复杂度**：$O(2^n)$——这就是为什么 $n = 50$ 时程序会卡住。

---

### 3.2 记忆化搜索——避免重复计算

既然问题是「重复计算」，解决方案自然就是「算过的不再算」。

我们可以用一个数组把已经计算过的结果存起来：

```cpp
vector<int> memo;  // 动态数组，根据 n 调整大小

int climbStairs(int n) {
    memo.resize(n + 1, -1);
    return dfs(n);
}

int dfs(int n) {
    if (n <= 2) return n;
    if (memo[n] != -1) return memo[n];  // 已经算过，直接返回
    memo[n] = dfs(n - 1) + dfs(n - 2);  // 记录结果
    return memo[n];
}
```

这就是**记忆化搜索**（Memoization）——在递归的基础上，用数组记录已计算的结果。

**时间复杂度**：$O(n)$——每个状态只计算一次。

**空间复杂度**：$O(n)$——递归栈深度 + 记忆化数组。

**直觉理解**：想象你在爬一棵树，每到一个节点就做标记。下次再路过时，看到标记就直接跳过，不再重复探索。

---

### 3.3 递推——自底向上的动态规划

记忆化搜索是「自顶向下」的——从大问题出发，分解成小问题。

我们也可以「自底向上」——先解决小问题，再逐步构建大问题的解：

```cpp
int climbStairs(int n) {
    if (n <= 2) return n;
    vector<int> dp(n + 1);
    dp[1] = 1;
    dp[2] = 2;
    for (int i = 3; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    return dp[n];
}
```

这就是**递推**——也是动态规划最经典的形式。

**❓ 记忆化搜索和递推有什么区别？**

| 特点 | 记忆化搜索 | 递推 |
|------|-----------|------|
| 方向 | 自顶向下 | 自底向上 |
| 实现方式 | 递归 + 数组 | 循环 + 数组 |
| 代码风格 | 更自然，接近思维过程 | 更高效，无递归开销 |
| 适用场景 | 状态转移复杂、不规则 | 状态转移简单、规律 |

**关键洞察**：两种方法本质相同——都是用数组记录已计算的结果，避免重复计算。区别只在于计算的顺序。

---

### 3.4 空间优化——只保留必要的信息

观察递推式 $f(n) = f(n-1) + f(n-2)$，我们发现：计算 $f(n)$ 时只需要 $f(n-1)$ 和 $f(n-2)$，不需要更早的结果。

所以我们可以只用两个变量：

```cpp
int climbStairs(int n) {
    if (n <= 2) return n;
    int prev2 = 1, prev1 = 2;  // f(1) 和 f(2)
    for (int i = 3; i <= n; i++) {
        int curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

**空间复杂度**：从 $O(n)$ 降到 $O(1)$。

这是动态规划中常见的优化技巧：**滚动数组**或**状态压缩**。当状态转移只依赖前几个状态时，我们可以只保留必要的信息。

---

### 3.5 动态规划的本质——状态与转移

让我们总结一下动态规划的核心思想：

1. **状态**：用一个数组（或变量）表示问题在某个阶段的解。比如 $dp[i]$ 表示跳到第 $i$ 级台阶的方法数。

2. **状态转移方程**：描述状态之间的关系。比如 $dp[i] = dp[i-1] + dp[i-2]$。

3. **边界条件**：最简单情况的解。比如 $dp[1] = 1, dp[2] = 2$。

4. **最优子结构**：大问题的解可以由小问题的解组合而成。

5. **无后效性**：一旦某个状态确定，它之后的变化不再受之前状态的影响。

**❓ 什么问题适合用动态规划？**

满足以下两个条件的问题可以考虑动态规划：

1. **最优子结构**：问题可以分解为子问题，且子问题的最优解能组合成原问题的最优解。

2. **重叠子问题**：在求解过程中，同一个子问题会被多次计算。

如果不满足条件 1，可能需要用贪心或其他方法。
如果不满足条件 2，直接递归就够了，不需要记忆化。

## 四、代码实现

以下是本篇所有核心方法的代码模板。

### 方法一：记忆化搜索

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    vector<int> memo;
    
    int climbStairs(int n) {
        memo.assign(n + 1, -1);
        return dfs(n);
    }
    
    int dfs(int n) {
        if (n <= 2) return n;
        if (memo[n] != -1) return memo[n];
        memo[n] = dfs(n - 1) + dfs(n - 2);
        return memo[n];
    }
};
```

## 五、复杂度与正确性分析

| 方法 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 朴素递归 | $O(2^n)$ | $O(n)$（递归栈） |
| 记忆化搜索 | $O(n)$ | $O(n)$ |
| 递推 | $O(n)$ | $O(n)$ |
| 空间优化 | $O(n)$ | $O(1)$ |

**正确性保证**：

1. **状态定义正确**：$dp[i]$ 明确定义为「跳到第 $i$ 级台阶的方法数」。

2. **状态转移正确**：最后一步只能从 $i-1$ 或 $i-2$ 跳上来，所以 $dp[i] = dp[i-1] + dp[i-2]$。

3. **边界条件正确**：$dp[1] = 1$（只有一种跳法），$dp[2] = 2$（两种跳法）。

4. **计算顺序正确**：递推时从 $i = 3$ 开始，保证 $dp[i-1]$ 和 $dp[i-2]$ 已经计算过。

## 六、典型应用场景

动态规划在竞赛中的应用非常广泛，以下是几个典型场景：

1. **计数问题**：计算有多少种方法达到某个状态。例如：爬楼梯、走网格路径。

2. **最优化问题**：求最大值或最小值。例如：最长递增子序列、背包问题。

3. **可行性问题**：判断是否能达到某个状态。例如：能否用给定硬币凑出某个金额。

**推荐练习题**（仅涉及已学知识：递推、状态转移）：

- [LeetCode 746. 使用最小花费爬楼梯](https://leetcode.cn/problems/min-cost-climbing-stairs/) — 变种爬楼梯，练习状态转移

## 七、小结

本文建立了动态规划最底层的基础设施：**递归**是自然的思维方式但效率低下，**记忆化搜索**通过记录结果避免重复计算，**递推**用循环替代递归更高效，**空间优化**只保留必要的信息。

四个核心要点：

1. 动态规划的本质是「用空间换时间」——记录已计算的结果，避免重复计算。
2. 记忆化搜索和递推本质相同，区别在于计算方向（自顶向下 vs 自底向上）。
3. 适合 DP 的问题必须满足「最优子结构」和「重叠子问题」两个条件。
4. 空间优化是常见的技巧，当状态转移只依赖前几个状态时可以大幅减少空间。
