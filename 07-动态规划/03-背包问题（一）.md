# 背包问题（一）

> Made by Aki.
> 最后更新于 2026.06.03

背包问题是动态规划中最经典、最重要的模型之一。掌握背包问题的核心思想，对于学习其他类型的 DP 问题有重要帮助。

---

## 一、引言

来看一个经典问题：

> 你有一个容量为 $W$ 的背包，有 $n$ 件物品，第 $i$ 件物品的重量为 $w_i$，价值为 $v_i$。每件物品只能选一次，求背包能装下的最大总价值。

这就是**01 背包问题**——每件物品只有「选」和「不选」两种状态。

如果每件物品可以选无限次呢？那就是**完全背包问题**。

如果每件物品有数量限制呢？那就是**多重背包问题**。

这三种背包问题是背包问题的基础，掌握它们是学习更复杂 DP 问题的关键。

## 三、核心推导

### 3.1 01 背包

#### 问题定义

有 $n$ 件物品和一个容量为 $W$ 的背包。第 $i$ 件物品的重量为 $w_i$，价值为 $v_i$。每件物品只能选一次，求背包能装下的最大总价值。

#### 状态定义

我们定义 $dp[i][j]$ 为：**前 $i$ 件物品，背包容量为 $j$ 时，能获得的最大价值**。

#### 状态转移方程

对于第 $i$ 件物品，我们有两种选择：

1. **不选第 $i$ 件物品**：$dp[i][j] = dp[i-1][j]$
2. **选第 $i$ 件物品**（前提是 $j \ge w_i$）：$dp[i][j] = dp[i-1][j-w_i] + v_i$

取两者的最大值：

$$
dp[i][j] = \begin{cases}
dp[i-1][j], & \text{if } j < w_i \\
\max(dp[i-1][j], dp[i-1][j-w_i] + v_i), & \text{otherwise}
\end{cases}
$$

#### 边界条件

$dp[0][j] = 0$（没有物品可选）
$dp[i][0] = 0$（背包容量为 0）

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int knapsack01(int W, vector<int>& w, vector<int>& v) {
    int n = w.size();
    vector<vector<int>> dp(n + 1, vector<int>(W + 1, 0));
    
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= W; j++) {
            if (j < w[i - 1]) {
                dp[i][j] = dp[i - 1][j];
            } else {
                dp[i][j] = max(dp[i - 1][j], dp[i - 1][j - w[i - 1]] + v[i - 1]);
            }
        }
    }
    
    return dp[n][W];
}
```

**时间复杂度**：$O(nW)$
**空间复杂度**：$O(nW)$

#### 空间优化

观察状态转移方程，$dp[i][j]$ 只依赖于 $dp[i-1][j]$ 和 $dp[i-1][j-w_i]$。我们可以用一维数组优化：

```cpp
int knapsack01(int W, vector<int>& w, vector<int>& v) {
    int n = w.size();
    vector<int> dp(W + 1, 0);
    
    for (int i = 0; i < n; i++) {
        for (int j = W; j >= w[i]; j--) {  // 注意：逆序遍历
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
        }
    }
    
    return dp[W];
}
```

**❓ 为什么 01 背包要逆序遍历？**

因为每个物品只能选一次。如果正序遍历，$dp[j-w_i]$ 可能已经被更新过（即当前物品被选了多次），这就不符合 01 背包的定义了。

逆序遍历保证 $dp[j-w_i]$ 还是上一行的值，即当前物品只被考虑了一次。

### 3.2 完全背包

#### 问题定义

有 $n$ 件物品和一个容量为 $W$ 的背包。第 $i$ 件物品的重量为 $w_i$，价值为 $v_i$。每件物品可以选无限次，求背包能装下的最大总价值。

#### 状态定义

与 01 背包相同：$dp[i][j]$ 表示前 $i$ 件物品，背包容量为 $j$ 时，能获得的最大价值。

#### 状态转移方程

对于第 $i$ 件物品，我们有多种选择：
- 不选：$dp[i][j] = dp[i-1][j]$
- 选 1 次：$dp[i][j] = dp[i-1][j-w_i] + v_i$
- 选 2 次：$dp[i][j] = dp[i-1][j-2w_i] + 2v_i$
- ...

但这样太复杂了。我们换个思路：

**关键洞察**：$dp[i][j]$ 可以从 $dp[i][j-w_i]$ 转移过来（即「选第 $i$ 件物品」这个决策可以重复多次）。

所以状态转移方程为：

$$
dp[i][j] = \begin{cases}
dp[i-1][j], & \text{if } j < w_i \\
\max(dp[i-1][j], dp[i][j-w_i] + v_i), & \text{otherwise}
\end{cases}
$$

注意与 01 背包的区别：这里是 $dp[i][j-w_i]$，而不是 $dp[i-1][j-w_i]$。

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int knapsackComplete(int W, vector<int>& w, vector<int>& v) {
    int n = w.size();
    vector<vector<int>> dp(n + 1, vector<int>(W + 1, 0));
    
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= W; j++) {
            if (j < w[i - 1]) {
                dp[i][j] = dp[i - 1][j];
            } else {
                dp[i][j] = max(dp[i - 1][j], dp[i][j - w[i - 1]] + v[i - 1]);
            }
        }
    }
    
    return dp[n][W];
}
```

#### 空间优化

```cpp
int knapsackComplete(int W, vector<int>& w, vector<int>& v) {
    int n = w.size();
    vector<int> dp(W + 1, 0);
    
    for (int i = 0; i < n; i++) {
        for (int j = w[i]; j <= W; j++) {  // 注意：正序遍历
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
        }
    }
    
    return dp[W];
}
```

**❓ 为什么完全背包要正序遍历？**

因为每个物品可以选无限次。正序遍历保证 $dp[j-w_i]$ 可能已经被更新过（即当前物品被选了多次），这正是完全背包所需要的。

**对比 01 背包和完全背包的空间优化代码，唯一的区别就是遍历顺序：01 背包逆序，完全背包正序。**

### 3.3 多重背包

#### 问题定义

有 $n$ 件物品和一个容量为 $W$ 的背包。第 $i$ 件物品的重量为 $w_i$，价值为 $v_i$，数量为 $s_i$。求背包能装下的最大总价值。

#### 方法一：直接拆分

最直观的方法是把每件物品拆分成 $s_i$ 个独立的物品，然后用 01 背包解决。

```cpp
int knapsackMultiple(int W, vector<int>& w, vector<int>& v, vector<int>& s) {
    vector<int> ws, vs;
    for (int i = 0; i < w.size(); i++) {
        for (int j = 0; j < s[i]; j++) {
            ws.push_back(w[i]);
            vs.push_back(v[i]);
        }
    }
    return knapsack01(W, ws, vs);
}
```

**时间复杂度**：$O(W \sum s_i)$——当 $s_i$ 很大时会超时。

#### 方法二：二进制优化

**关键洞察**：任何正整数 $s$ 都可以表示为若干个 2 的幂次之和（二进制分解）。

例如：$s = 13 = 1 + 2 + 4 + 6$（注意最后一个是 6，不是 8，因为 $1+2+4+8=15 > 13$）。

我们可以把第 $i$ 件物品拆分成 $O(\log s_i)$ 个物品，然后用 01 背包解决。

```cpp
int knapsackMultiple(int W, vector<int>& w, vector<int>& v, vector<int>& s) {
    vector<int> ws, vs;
    for (int i = 0; i < w.size(); i++) {
        int k = 1;
        int remain = s[i];  // 用局部变量避免修改输入参数
        while (k <= remain) {
            ws.push_back(k * w[i]);
            vs.push_back(k * v[i]);
            remain -= k;
            k *= 2;
        }
        if (remain > 0) {
            ws.push_back(remain * w[i]);
            vs.push_back(remain * v[i]);
        }
    }
    return knapsack01(W, ws, vs);
}
```

**时间复杂度**：$O(W \sum \log s_i)$——比直接拆分快很多。

#### 方法三：单调队列优化

更高级的优化方法是使用单调队列，可以在 $O(nW)$ 时间内解决多重背包问题。这将在后续的 DP 优化篇中讲解。

```cpp
int knapsackMultiple(int W, vector<int>& w, vector<int>& v, vector<int>& s) {
    vector<int> ws, vs;
    for (int i = 0; i < w.size(); i++) {
        int k = 1;
        while (k <= s[i]) {
            ws.push_back(k * w[i]);
            vs.push_back(k * v[i]);
            s[i] -= k;
            k *= 2;
        }
        if (s[i] > 0) {
            ws.push_back(s[i] * w[i]);
            vs.push_back(s[i] * v[i]);
        }
    }
    return knapsack01(W, ws, vs);
}
```

## 五、复杂度与正确性分析

| 问题 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 01 背包 | $O(nW)$ | $O(nW)$ → $O(W)$ |
| 完全背包 | $O(nW)$ | $O(nW)$ → $O(W)$ |
| 多重背包（直接拆分） | $O(W \sum s_i)$ | $O(W)$ |
| 多重背包（二进制优化） | $O(W \sum \log s_i)$ | $O(W)$ |

**正确性保证**：

1. **状态定义正确**：$dp[i][j]$ 明确定义为「前 $i$ 件物品，容量为 $j$ 时的最大价值」。

2. **状态转移正确**：每个状态都考虑了「选」和「不选」两种情况，且取最优值。

3. **遍历顺序正确**：01 背包逆序保证每件物品只选一次，完全背包正序允许每件物品选多次。

4. **边界条件正确**：$dp[0][j] = 0$（没有物品）和 $dp[i][0] = 0$（容量为 0）。

## 七、小结

本文介绍了三种基础背包问题：**01 背包**、**完全背包** 和 **多重背包**。

四个核心要点：

1. 01 背包和完全背包的唯一区别是遍历顺序：01 背包逆序，完全背包正序。

2. 背包问题的空间优化是常见的技巧，可以将空间复杂度从 $O(nW)$ 降到 $O(W)$。

3. 多重背包可以通过二进制优化将时间复杂度从 $O(W \sum s_i)$ 降到 $O(W \sum \log s_i)$。

4. 背包问题是许多复杂 DP 问题的基础模型，掌握它对后续学习非常重要。
