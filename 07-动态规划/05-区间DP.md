# 区间 DP

> Made by Aki.
> 最后更新于 2026.06.03

区间 DP 是动态规划中一类经典问题，它的核心思想是在区间上进行决策，逐步合并区间直到得到最终答案。

---

## 一、引言

来看一个经典问题：

> 有 $n$ 堆石子排成一排，每堆石子有一定的数量。每次可以合并相邻的两堆石子，合并的代价为这两堆石子的数量之和。求将所有石子合并成一堆的最小总代价。

例如：$n = 4$，石子数量为 $[1, 3, 5, 2]$。
- 合并方案一：先合并 1 和 3（代价 4），再合并 4 和 5（代价 9），最后合并 9 和 2（代价 11），总代价 = 4 + 9 + 11 = 24
- 合并方案二：先合并 3 和 5（代价 8），再合并 1 和 8（代价 9），最后合并 9 和 2（代价 11），总代价 = 8 + 9 + 11 = 28

哪种合并顺序能让总代价最小？

这就是**石子合并问题**——经典的区间 DP 问题。它的特点是：决策的顺序会影响最终结果，我们需要找到最优的合并顺序。

## 三、核心推导

### 3.1 石子合并（直线版）

#### 问题定义

有 $n$ 堆石子排成一排，第 $i$ 堆有 $a_i$ 个石子。每次可以合并相邻的两堆石子，合并的代价为这两堆石子的数量之和。求将所有石子合并成一堆的最小总代价。

#### 状态定义

我们定义 $dp[i][j]$ 为：**将第 $i$ 堆到第 $j$ 堆石子合并成一堆的最小代价**。

#### 状态转移方程

对于区间 $[i, j]$，我们枚举最后一次合并的位置 $k$（$i \le k < j$）：

- 先合并 $[i, k]$ 成一堆，代价为 $dp[i][k]$
- 再合并 $[k+1, j]$ 成一堆，代价为 $dp[k+1][j]$
- 最后合并这两堆，代价为 $\sum_{t=i}^{j} a_t$

所以状态转移方程为：

$$
dp[i][j] = \min_{i \le k < j} \{dp[i][k] + dp[k+1][j]\} + \sum_{t=i}^{j} a_t
$$

#### 边界条件

$dp[i][i] = 0$（只有一堆石子，不需要合并）

#### 前缀和优化

为了快速计算 $\sum_{t=i}^{j} a_t$，我们可以预处理前缀和：

$$
sum[i] = \sum_{t=1}^{i} a_t
$$

那么 $\sum_{t=i}^{j} a_t = sum[j] - sum[i-1]$

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int stoneMerge(vector<int>& a) {
    int n = a.size();
    vector<int> sum(n + 1, 0);
    for (int i = 1; i <= n; i++) {
        sum[i] = sum[i - 1] + a[i - 1];
    }
    
    vector<vector<int>> dp(n + 1, vector<int>(n + 1, 0));
    
    // 枚举区间长度
    for (int len = 2; len <= n; len++) {
        // 枚举左端点
        for (int i = 1; i + len - 1 <= n; i++) {
            int j = i + len - 1;  // 右端点
            dp[i][j] = INT_MAX;
            // 枚举分割点
            for (int k = i; k < j; k++) {
                dp[i][j] = min(dp[i][j], dp[i][k] + dp[k + 1][j] + sum[j] - sum[i - 1]);
            }
        }
    }
    
    return dp[1][n];
}
```

**时间复杂度**：$O(n^3)$
**空间复杂度**：$O(n^2)$

**❓ 为什么要按区间长度枚举？**

因为计算 $dp[i][j]$ 时，我们需要用到更小区间的值 $dp[i][k]$ 和 $dp[k+1][j]$。按长度枚举保证计算大区间时，所有小区间都已经计算过。

### 3.2 环形石子合并

#### 问题定义

与石子合并相同，但石子排成一个环形。即第 $n$ 堆和第 $1$ 堆相邻。

#### 解决思路

**关键洞察**：环形问题可以通过「断环为链」转化为直线问题。

具体做法：将数组复制一份接在后面，变成长度为 $2n$ 的数组。然后在这个数组上做区间 DP，最后取所有长度为 $n$ 的区间的最小值。

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int stoneMergeCircle(vector<int>& a) {
    int n = a.size();
    // 断环为链
    vector<int> b(2 * n);
    for (int i = 0; i < n; i++) {
        b[i] = a[i];
        b[i + n] = a[i];
    }
    
    // 在长度为 2n 的数组上做区间 DP
    vector<int> sum(2 * n + 1, 0);
    for (int i = 1; i <= 2 * n; i++) {
        sum[i] = sum[i - 1] + b[i - 1];
    }
    
    vector<vector<int>> dp(2 * n + 1, vector<int>(2 * n + 1, 0));
    
    for (int len = 2; len <= n; len++) {
        for (int i = 1; i + len - 1 <= 2 * n; i++) {
            int j = i + len - 1;
            dp[i][j] = INT_MAX;
            for (int k = i; k < j; k++) {
                dp[i][j] = min(dp[i][j], dp[i][k] + dp[k + 1][j] + sum[j] - sum[i - 1]);
            }
        }
    }
    
    // 取所有长度为 n 的区间的最小值
    int ans = INT_MAX;
    for (int i = 1; i + n - 1 <= 2 * n; i++) {
        ans = min(ans, dp[i][i + n - 1]);
    }
    return ans;
}
```

**时间复杂度**：$O(n^3)$（虽然数组长度翻倍，但区间长度最多为 $n$）
**空间复杂度**：$O(n^2)$

### 3.3 戳气球

#### 问题定义

有 $n$ 个气球，每个气球上有一个数字。戳破第 $i$ 个气球可以获得 $nums[i-1] \times nums[i] \times nums[i+1]$ 个金币（假设边界外的气球数字为 1）。求戳破所有气球能获得的最大金币数。

#### 关键难点

这个问题的难点在于：戳破一个气球后，左右相邻的气球会变成新的相邻关系。如果按照「戳破」的顺序思考，状态会非常复杂。

**逆向思维**：我们不考虑「戳破」的顺序，而是考虑「添加」的顺序。假设最后戳破的是第 $k$ 个气球，那么戳破它时，左右边界是固定的（因为其他气球都已经戳破了）。

#### 状态定义

我们定义 $dp[i][j]$ 为：**戳破区间 $(i, j)$ 内所有气球（不包括 $i$ 和 $j$）能获得的最大金币数**。

注意：这里的区间是开区间，$i$ 和 $j$ 是边界，不被戳破。

#### 状态转移方程

假设最后戳破的是第 $k$ 个气球（$i < k < j$），那么：

- 戳破 $(i, k)$ 内所有气球的收益：$dp[i][k]$
- 戳破 $(k, j)$ 内所有气球的收益：$dp[k][j]$
- 戳破第 $k$ 个气球的收益：$nums[i] \times nums[k] \times nums[j]$

所以状态转移方程为：

$$
dp[i][j] = \max_{i < k < j} \{dp[i][k] + dp[k][j] + nums[i] \times nums[k] \times nums[j]\}
$$

#### 边界条件

$dp[i][j] = 0$（当 $j = i + 1$ 时，区间内没有气球）

#### 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int maxCoins(vector<int>& nums) {
    int n = nums.size();
    // 在两端添加虚拟气球（数字为 1）
    vector<int> a(n + 2);
    a[0] = 1;
    a[n + 1] = 1;
    for (int i = 1; i <= n; i++) {
        a[i] = nums[i - 1];
    }
    
    vector<vector<int>> dp(n + 2, vector<int>(n + 2, 0));
    
    // 枚举区间长度
    for (int len = 3; len <= n + 2; len++) {
        // 枚举左端点
        for (int i = 0; i + len - 1 <= n + 1; i++) {
            int j = i + len - 1;
            // 枚举最后戳破的气球
            for (int k = i + 1; k < j; k++) {
                dp[i][j] = max(dp[i][j], dp[i][k] + dp[k][j] + a[i] * a[k] * a[j]);
            }
        }
    }
    
    return dp[0][n + 1];
}
```

**时间复杂度**：$O(n^3)$
**空间复杂度**：$O(n^2)$

**❓ 为什么要在两端添加虚拟气球？**

因为状态转移方程中需要 $nums[i]$ 和 $nums[j]$ 作为边界。添加虚拟气球后，原来的第一个和最后一个气球就有了固定的左右边界。

## 五、复杂度与正确性分析

| 问题 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 石子合并 | $O(n^3)$ | $O(n^2)$ |
| 环形石子合并 | $O(n^3)$ | $O(n^2)$ |
| 戳气球 | $O(n^3)$ | $O(n^2)$ |

**正确性保证**：

1. **状态定义正确**：$dp[i][j]$ 明确定义为「处理区间 $[i, j]$ 的最优解」。

2. **状态转移正确**：枚举所有可能的分割点，取最优值。

3. **计算顺序正确**：按区间长度从小到大计算，保证依赖的状态已经计算过。

4. **边界条件正确**：石子合并中 $dp[i][i] = 0$（单堆不需要合并）；戳气球中当 $j = i + 1$ 时 $dp[i][j] = 0$（区间内无气球）。

## 七、小结

本文介绍了三个经典的区间 DP 问题：**石子合并**、**环形石子合并** 和 **戳气球**。

四个核心要点：

1. 区间 DP 的特征是在区间上进行决策，状态 $dp[i][j]$ 通常表示处理区间 $[i, j]$ 的最优解。

2. 状态转移方程通常是枚举分割点 $k$，取最优值。

3. 计算顺序按区间长度从小到大，保证依赖的状态已经计算过。

4. 环形问题可以通过「断环为链」转化为直线问题。
