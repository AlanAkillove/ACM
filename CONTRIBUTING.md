## 行为准则

本项目致力于为所有参与者提供友善、包容的环境。

- **尊重他人**：对不同水平的读者保持耐心，避免居高临下的语言
- **建设性反馈**：指出问题时，尽量附带改进建议或理由
- **聚焦内容**：讨论应围绕内容本身，而非个人

## 如何贡献

### 报告问题

发现以下问题时，欢迎提交 [Issue](https://github.com/AlanAkillove/ACM/issues)：

- **内容错误**：推导错误、笔误、公式渲染问题
- **链接失效**：文档内部引用或外部题目链接失效
- **表达不清**：某段内容难以理解，需要补充解释或重写
- **例题过时**：推荐题目已失效、更换或不再适合

提交 Issue 时请注明涉及的文档编号（如 #9）和具体位置。

### 提交内容

你可以通过 Pull Request 提交以下类型的内容：

- **修正**：修复推导错误、笔误、格式问题
- **完善**：扩展现有章节、增加手算例子、补充易错点
- **新增**：增加新例题、新小节、新练习题
- **代码**：补充参考代码（C++17 或 Python 3）

### 其他方式

- 在 Discussions 中讨论内容编排或选题方向
- 在相关 Issue 下参与讨论和投票

## 内容规范

### 文档结构

每篇文档遵循统一结构：

```
# 标题
> 作者信息与最后更新日期

## 一、引言
## 二、前置知识（引用已有篇章）
## 三、核心推导
## 四、代码实现（可选）
## 五、复杂度与正确性分析
## 六、典型应用场景（可选）
## 七、小结
```

### 风格指南

- **数学公式**：使用 LaTeX，行内用 `$...$`，独立公式用 `$$...$$`
- **代码**：优先 C++17，可选配 Python 3；关键算法附复杂度注释
- **引用**：引用已有篇章时使用相对链接，如 `[第 7 篇](../01-数论/07-快速幂与慢速乘.md)`
- **难度标识**：🟢 入门 🟡 进阶 🔴 高阶 🟣 专题/综合
- **语言**：中文编写，术语首次出现时附英文（如「筛法（Sieve）」）

### 文件命名

- 编号 + 中文标题，如 `09-乘法逆元.md`
- 文件夹按板块编号命名：`01-数论`、`02-组合数学` ...

## PR 流程

1. Fork 本仓库
2. 从 `main` 创建新分支：`git checkout -b docs/fix-xxx` 或 `feat/add-xxx`
3. 做出修改，确保内容准确、格式一致
4. 提交 PR，标题使用 Conventional Commits 格式：
   - `docs:` — 文档内容变更（修正、完善、新增章节）
   - `feat:` — 新功能（如新增代码实现）
   - `fix:` — 错误修复
   - `refactor:` — 结构重组（不改变内容含义）
5. 在 PR 描述中说明改动内容和原因
6. 等待审核与反馈

### PR 检查清单

提交前请确认：

- [ ] 数学推导和公式正确无误
- [ ] 所有链接（内部引用和外部题目链接）有效
- [ ] 新增内容未与已有内容重复
- [ ] 格式与现有风格一致（标题层级、公式排版等）
- [ ] 若新增练习题，已检查题目与文档主题匹配

## 代码规范

### C++17

- 使用 `long long` 代替 `int` 进行模运算
- 模运算变量以 `p` 或 `MOD` 命名
- 关键步骤添加简短注释

### Python 3

- 使用 `pow(a, p-2, p)` 进行模逆元计算
- 避免 `import *`
- 函数命名使用 `snake_case`
