# 扩展欧几里得算法

> Made by Aki.
> 最后更新于 2026.04.29

第 3 篇裴蜀定理告诉我们：$ax + by = \gcd(a, b)$ 一定有整数解，但没告诉我们解在哪。本篇回答这个问题——用扩展欧几里得算法（exgcd）在 $O(\log n)$ 时间内求出那组 $(x, y)$。

## 一、引言

第 3 篇结尾我们留了一个悬念：$3x + 5y = 1$ 的解在哪？今天来解决它。

扩欧不仅是求一组解的问题。它的输出 $(x, y)$ 是后续系列中**乘法逆元、线性同余方程、中国剩余定理**的基础组件。如果你只能记住一个算法的推导过程，exgcd 应该排在候选名单的前列。

## 三、核心推导

### 3.1 从辗转相除法到扩展——思路的起源

回顾辗转相除法求 $\gcd(a, b)$ 的递归过程：

```
gcd(a, b):
    if b == 0: return a
    return gcd(b, a mod b)
```

每层递归执行 $a \gets b$，$b \gets a \bmod b$。当 $b = 0$ 时，此时 $a$ 就是 $\gcd$。

扩展欧几里得的思路是：**在递归返回的过程中，一边回溯一边构造出 $x, y$**。

具体来说，如果递归调用的下层已经求出了满足 $b \cdot x' + (a \bmod b) \cdot y' = \gcd(a, b)$ 的 $(x', y')$，我们能否从这组解推导出当前层需要的 $a \cdot x + b \cdot y = \gcd(a, b)$ 的解 $(x, y)$？

答案是可以——而且只需要一次代数变换。

---

### 3.2 核心递推关系——回溯时如何更新 $x, y$

假设递归调用 `exgcd(b, a % b)` 已经返回了满足下式的 $(x', y')$：

$$
b \cdot x' + (a \bmod b) \cdot y' = g
$$

其中 $g = \gcd(a, b)$。

由带余除法：$a \bmod b = a - \left\lfloor \frac{a}{b} \right\rfloor \cdot b$。代入：

$$
b \cdot x' + \left(a - \left\lfloor \frac{a}{b} \right\rfloor \cdot b\right) \cdot y' = g
$$

整理 $a$ 和 $b$ 的系数：

$$
a \cdot \textcolor{red}{y'} + b \cdot \left(\textcolor{red}{x' - \left\lfloor \frac{a}{b} \right\rfloor \cdot y'}\right) = g
$$

由此读出当前层的解：

$$
\begin{cases}
x = y' \\[4pt]
y = x' - \left\lfloor \frac{a}{b} \right\rfloor \cdot y'
\end{cases}
$$

这就是 exgcd 的核心递推公式。**注意 $x$ 取的是下层递归的 $y'$，这个交换是关键。**

**递归基**：当 $b = 0$ 时，$\gcd(a, 0) = a$。此时方程是 $a \cdot x + 0 \cdot y = a$，最简解为 $x = 1, y = 0$。

❓ **为什么 $b = 0$ 时 $y$ 可以取任意值？** 因为 $b = 0$，$y$ 的系数为零，它对结果没有影响。我们取 $y = 0$ 是为了回溯时方便递推。

---

### 3.3 完整手动模拟

求 $3x + 5y = 1$ 的一组解。即 $\gcd(3, 5) = 1$。

**递归深入（与普通欧几里得相同）**：

```
exgcd(3, 5):
    3 ÷ 5 = 0 余 3  →  调用 exgcd(5, 3)

exgcd(5, 3):
    5 ÷ 3 = 1 余 2  →  调用 exgcd(3, 2)

exgcd(3, 2):
    3 ÷ 2 = 1 余 1  →  调用 exgcd(2, 1)

exgcd(2, 1):
    2 ÷ 1 = 2 余 0  →  调用 exgcd(1, 0)

exgcd(1, 0):
    b == 0, return (g=1, x=1, y=0)   ← 递归基
```

**递归回溯（递推更新 $x, y$）**：

| 层 | $a$ | $b$ | 下层 $(x', y')$ | $q = \lfloor a/b \rfloor$ | 本层 $(x, y) = (y', x' - q \cdot y')$ |
|:--:|:---:|:---:|:---:|:---:|------|
| 5 | 1 | 0 | — | — | $(1, 0)$ ← 递归基 |
| 4 | 2 | 1 | $(1, 0)$ | $2$ | $(0, \; 1 - 2 \cdot 0) = (0, 1)$ |
| 3 | 3 | 2 | $(0, 1)$ | $1$ | $(1, \; 0 - 1 \cdot 1) = (1, -1)$ |
| 2 | 5 | 3 | $(1, -1)$ | $1$ | $(-1, \; 1 - 1 \cdot (-1)) = (-1, 2)$ |
| 1 | 3 | 5 | $(-1, 2)$ | $0$ | $(2, \; -1 - 0 \cdot 2) = (2, -1)$ |

最终：$x = 2$，$y = -1$。

验证：$3 \times 2 + 5 \times (-1) = 6 - 5 = 1$。✓

❓ **每一步的 $q = \lfloor a/b \rfloor$ 对应什么？** 就是辗转相除法中那一步的商。在 exgcd 的代码中，这通常写为 `a / b`（整数除法自动向下取整）。

---

### 3.4 从特解到通解——与第 3 篇的知识衔接

exgcd 求出的是一个**特解** $(x_0, y_0)$。回顾第 3 篇 3.4 节的通解公式，$ax + by = c$ 的通解为：

$$
\begin{cases}
x = x_0 + \dfrac{b}{g} \cdot t \\[8pt]
y = y_0 - \dfrac{a}{g} \cdot t
\end{cases}
\quad (t \in \mathbb{Z})
$$

其中 $g = \gcd(a, b)$。

**应用**：如果题目要求的是最小正整数解（极其常见），只需在特解上调整 $t$ 使 $x$ 落在 $[1, b/g]$ 范围内：

```cpp
ll dx = b / g;
x = (x % dx + dx) % dx;        // 调整到 [0, dx)
if (x == 0) x = dx;            // 如果要正整数解而非非负解，需要这一步
```

---

### 3.5 求解 $ax + by = c$（$c$ 不一定是 $\gcd$）

exgcd 解出的是 $ax_0 + by_0 = g$。对于一般的 $ax + by = c$：

1. 若 $c \bmod g \neq 0$ → 无解（裴蜀定理）。
2. 否则，将特解放大：$x_1 = x_0 \cdot \frac{c}{g}$，$y_1 = y_0 \cdot \frac{c}{g}$。
3. 通解在此基础上加上周期的整倍数。

这个普适流程能够处理所有二元一次不定方程的整数解问题——从简单的同余方程 $ax \equiv 1 \pmod{b}$ 到带约束的正整数解计数问题。

## 四、代码实现

### 4.1 扩展欧几里得算法（递归版）

```cpp
using ll = long long;

// 返回 gcd(a, b)，同时通过引用返回 x, y，满足 ax + by = gcd(a, b)
ll exgcd(ll a, ll b, ll &x, ll &y) {
    if (b == 0) {
        x = 1;
        y = 0;
        return a;
    }
    ll g = exgcd(b, a % b, y, x);
    y -= a / b * x;
    return g;
}
```

注意参数传递时的 `(y, x)` 交换——这正是 3.2 节推导中「本层的 $x$ 是下层的 $y'$」在代码中的体现。回溯时的 `y -= a / b * x` 对应 $y = x' - \lfloor a/b \rfloor \cdot y'$（此时传进来的 $x$ 已经存的是下层的 $y'$，$y$ 存的是下层的 $x'$，刚好对上）。

### 4.2 求解 $ax + by = c$ 的完整模板

```cpp
// 求解 ax + by = c
// 返回 true 如果有解，通过引用返回通解参数
bool solve_diophantine(ll a, ll b, ll c, ll &x, ll &y, ll &dx, ll &dy) {
    ll g = exgcd(abs(a), abs(b), x, y);
    if (c % g != 0) return false;
    x *= c / g;
    y *= c / g;
    if (a < 0) x = -x;
    if (b < 0) y = -y;
    dx = abs(b) / g;   // x 的周期
    dy = abs(a) / g;   // y 的周期
    return true;
}
```

### 4.3 适配到最小正整数解

```cpp
// 返回 ax + by = c 的最小正整数 x 解（无解返回 -1）
ll min_positive_x(ll a, ll b, ll c) {
    ll x, y, dx, dy;
    if (!solve_diophantine(a, b, c, x, y, dx, dy)) return -1;
    x = (x % dx + dx) % dx;
    if (x == 0) x = dx;   // 正整数
    return x;
}
```

使用示例：

```cpp
ll x, y;
ll g = exgcd(3, 5, x, y);
// g = 1, x = 2, y = -1
// 验证：3*2 + 5*(-1) = 1 ✓

// 洛谷 P1082：求 ax ≡ 1 (mod b) 的最小正整数解
ll ans = min_positive_x(3, 7, 1);
// ans = 5，因为 3*5 = 15 ≡ 1 (mod 7)
```

## 五、复杂度与正确性分析

**时间复杂度**：exgcd 的递归深度与辗转相除法完全一致——每步将较大的数至少减半，深度为 $O(\log \min(a, b))$。每个节点的操作（一次整数乘法和一次减法）是 $O(1)$。总复杂度 $O(\log \min(a, b))$。

**空间复杂度**：递归深度 $O(\log n)$，对应调用栈空间。也可以写成迭代版（见下文），空间 $O(1)$。

**正确性**：3.2 节的递推公式是代数恒等变换，逻辑上等价于辗转相除法的正确性。只要递归基正确（$b=0$ 时 $\gcd(a,0)=a$，$x=1$, $y=0$ 满足 $a \cdot 1 + 0 \cdot 0 = a$），整条回溯链保证正确。

**迭代版（可选）**：

```cpp
// exgcd 迭代版，避免递归栈开销
ll exgcd_iter(ll a, ll b, ll &x, ll &y) {
    x = 1, y = 0;
    ll x1 = 0, y1 = 1;
    while (b) {
        ll q = a / b;
        tie(x, x1) = make_pair(x1, x - q * x1);
        tie(y, y1) = make_pair(y1, y - q * y1);
        tie(a, b) = make_pair(b, a - q * b);
    }
    return a;
}
```

## 六、典型应用场景

exgcd 在竞赛中有三个核心用途：

1. **求模逆元**（第 9 篇）：$\gcd(a, m) = 1$ 时，解 $ax + my = 1$ 得到的 $x$ 就是 $a^{-1} \pmod{m}$。

2. **解同余方程**（第 12 篇）：$ax \equiv b \pmod{m}$ 等价于不定方程 $ax + my = b$，exgcd 直接求解。

3. **中国剩余定理**（第 13 篇）：CRT 中需要计算 $M_i \cdot M_i^{-1} \pmod{m_i}$，这个逆元用 exgcd 求解。

**推荐练习题**（仅涉及已学知识：模运算 + GCD + 裴蜀定理 + exgcd）：

- [洛谷 P1082](https://www.luogu.com.cn/problem/P1082) — 同余方程。$ax \equiv 1 \pmod{b}$ 求最小正整数解。exgcd 一步出结果，是 exgcd 的"Hello World"
- [洛谷 P5656](https://www.luogu.com.cn/problem/P5656) — 【模板】二元一次不定方程。$ax + by = c$ 的完整分类讨论——无解/有整数解无正整数解/有正整数解，通解公式的直接演练

## 七、小结

exgcd 是基础篇的收官之作。它和裴蜀定理的关系可以这样理解：

- **裴蜀定理**（第 3 篇）说"$\gcd(a, b)$ 可以写成 $ax + by$ 的形式"——存在性。
- **扩展欧几里得**（本篇）说"我来把具体的 $x, y$ 算出来"——构造性。

这两篇合在一起，完整地解决了二元一次不定方程的理论问题和计算问题。
