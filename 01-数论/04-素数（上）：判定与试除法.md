# 素数（上）：判定与试除法

> Made by Aki.
> 最后更新于 2026.04.29

素数是数论的「原子」——每个大于 $1$ 的整数都可以唯一地分解为素数的乘积。本篇解决最基础的问题：给你一个数，判断它是不是素数。

---

## 一、引言

> 给定一个正整数 $n$（$1 \le n \le 10^{12}$），判断它是否为素数。如果是，输出 `Yes`，否则输出 `No`。

如果没有任何优化，你可能这样写：

```cpp
bool isPrime(int n) {
    if (n <= 1) return false;
    for (int i = 2; i < n; i++)
        if (n % i == 0) return false;
    return true;
}
```

对 $n = 10^9$，这个循环要跑 $10^9$ 次，超时是必然的。但有一个简单到不可思议的优化：**我们只需要检查到 $\lfloor\sqrt{n}\rfloor$**。为什么？这正是本篇要讲清楚的第一个问题。

当 $n$ 大到 $10^{18}$ 时，即使 $O(\sqrt{n})$ 也不够快。这时需要 Miller-Rabin 素性测试——一个能以 $O(\log n)$ 概率判定素数的精妙算法。本篇也会给出它的完整模板。

## 三、核心推导

### 3.1 素数的定义

一个大于 $1$ 的正整数 $n$，如果只能被 $1$ 和它本身整除，就称为**素数**（prime）。不是素数且大于 $1$ 的称为**合数**（composite）。

$1$ 既不是素数也不是合数。最小的素数是 $2$，也是唯一的偶素数。

❓ **为什么素数这么重要？** 算术基本定理（第 6 篇）告诉我们，每个大于 $1$ 的整数都可以唯一地写成素数的乘积。这使得素数成为所有整数的「基本构建块」——就像化学中的元素一样。

---

### 3.2 试除法——为什么只需要试到 $\sqrt{n}$？

**定理**：如果 $n$ 是合数，它一定有一个素因子 $p \le \sqrt{n}$。

**证明**：设 $n$ 是合数，则存在 $a, b$ 满足 $n = a \times b$，其中 $1 < a \le b < n$。

若 $a > \sqrt{n}$ 且 $b > \sqrt{n}$，则 $a \times b > \sqrt{n} \times \sqrt{n} = n$，矛盾。所以 $\min(a, b) \le \sqrt{n}$。

$\min(a, b)$ 要么自身是素数，要么可以继续分解为更小的素因子（这些素因子也 $\le \min(a, b) \le \sqrt{n}$）。因此 $n$ 必有一个 $\le \sqrt{n}$ 的素因子。

**这个简单的定理将试除的范围从 $O(n)$ 降到了 $O(\sqrt{n})$**，是竞赛中最基础的优化之一。

---

### 3.3 试除法的进一步优化

**优化一：只试除 $2$ 和奇数。**

除了 $2$ 以外，所有素数都是奇数。所以我们可以先判断是否被 $2$ 整除，然后从 $3$ 开始只检查奇数：

```cpp
if (n % 2 == 0) return n == 2;
for (ll i = 3; i * i <= n; i += 2)
    if (n % i == 0) return false;
```

这样循环次数从 $\sqrt{n}$ 降到 $\sqrt{n}/2$。

**优化二：只试除 $6k \pm 1$ 形式的数。**

所有 $\ge 5$ 的素数都可表示为 $6k \pm 1$（因为 $6k, 6k\pm2, 6k+3$ 都有小因子）。因此可以进一步减少试除量。

```cpp
if (n % 2 == 0 || n % 3 == 0) return n == 2 || n == 3;
for (ll i = 5; i * i <= n; i += 6)
    if (n % i == 0 || n % (i + 2) == 0) return false;
```

循环次数降到 $\sqrt{n}/3$。

**试除法总结**：时间 $O(\sqrt{n})$，空间 $O(1)$。适用于 $n \le 10^{12}$ 且查询次数较少的场景。如果需要批量判断（如筛出 $1$ 到 $10^7$ 的所有素数），试除法太慢，应该改用筛法——那是第 5 篇的内容。

---

### 3.4 Miller-Rabin 素性测试——处理大整数

当 $n$ 大到 $10^{18}$ 时，$\sqrt{n} = 10^9$，试除法一次需要 $10^9$ 步，不可接受。而 Miller-Rabin 算法可以在 $O(k \log n)$ 的时间内以极高的正确率判断素性。

Miller-Rabin 基于两个定理：**费马小定理**和**二次探测定理**。这两个定理的完整证明会在第 11 篇给出。本篇聚焦于它们在素数判定中的具体运用。

---

#### 3.4.1 费马小定理（Fermat's Little Theorem）

> 如果 $p$ 是素数，$a$ 不是 $p$ 的倍数，那么
>
> $$
> a^{p-1} \equiv 1 \pmod{p}
> $$

这给出了一个素数判定的**必要条件**：对于要判定的数 $n$，选一个底数 $a$（$1 < a < n$），如果 $a^{n-1} \not\equiv 1 \pmod{n}$，那 $n$ 一定不是素数。

但费马小定理的**逆命题不成立**：存在合数（卡迈克尔数，如 $561$）满足 $\forall a, \gcd(a,n)=1$ 有 $a^{n-1} \equiv 1 \pmod{n}$。所以仅有费马测试是不够的。

---

#### 3.4.2 二次探测定理

> 如果 $p$ 是奇素数，且 $x^2 \equiv 1 \pmod{p}$，那么 $x \equiv \pm 1 \pmod{p}$。

这个定理补上了费马测试的漏洞。Miller-Rabin 的完整流程如下：

**算法——Miller-Rabin 单轮测试（底数为 $a$）**：

1. 将 $n-1$ 写成 $d \cdot 2^r$，其中 $d$ 是奇数。
2. 计算 $x = a^d \bmod n$。
3. 如果 $x = 1$ 或 $x = n-1$，本轮通过。
4. 重复 $r-1$ 次：令 $x = x^2 \bmod n$。如果 $x = n-1$，本轮通过。如果 $x = 1$，本轮不通过（因为这说明前一个 $x$ 的平方是 $1$ 但自身不是 $\pm 1$）。
5. 如果循环结束仍未通过，本轮不通过。

**直觉理解**：第四步在「追溯」$a^{n-1}$ 的计算过程。如果 $n$ 真的是素数，那么 $a^{n-1} \equiv 1$，而且在这条平方链上，$1$ 只能从 $\pm 1$ 平方得来。如果有一步突然跳出 $1$ 而前值不是 $\pm 1$，$n$ 就一定不是素数。

---

#### 3.4.3 确定性底数集合

对于单轮测试，一个合数被误判为素数的概率不超过 $1/4$。重复 $k$ 轮，误判率不超过 $(1/4)^k$。

但对于 `long long` 范围内的数（$n < 2^{64}$），已经有人证明了只需测试固定的 7 个底数就能做到 100% 正确判定：

$$
a \in \{2, 325, 9375, 28178, 450775, 9780504, 1795265022\}
$$

对于 $n < 2^{32}$（即 `int` 范围），只需前三个底数 $\{2, 7, 61\}$ 即可。

❓ **这意味着什么？** 对于竞赛中出现的所有整数（通常不超过 $10^{18}$），Miller-Rabin 实际上是**确定性的**算法，而不是概率性的。你可以完全信赖它的结果。

---

#### 3.4.4 手算一个例子

判断 $n = 221$ 是否为素数（底数 $a = 2$）。

$n - 1 = 220 = 55 \times 2^2$，所以 $d = 55$，$r = 2$。

- $x = 2^{55} \bmod 221$。手动计算比较复杂，我们用快速幂（7.1 节有详解），结果是 $x = 128$。
- $x \neq 1$ 且 $x \neq 220$，继续。
- $x = 128^2 \bmod 221 = 30$。$x \neq 220$，继续。
- $x = 30^2 \bmod 221 = 16$。循环结束，$x$ 从未等于 $220$，也从未变成 $1$。

本轮不通过 → $221$ 是合数。事实上 $221 = 13 \times 17$。

## 四、代码实现

### 4.1 试除法（$n \le 10^{12}$，少量查询）

```cpp
using ll = long long;

bool isPrime_trial(ll n) {
    if (n <= 1) return false;
    if (n == 2 || n == 3) return true;
    if (n % 2 == 0 || n % 3 == 0) return false;
    for (ll i = 5; i * i <= n; i += 6)
        if (n % i == 0 || n % (i + 2) == 0)
            return false;
    return true;
}
```

### 4.2 Miller-Rabin 判定（$n \le 10^{18}$，确定性）

算法依赖模意义下的快速幂。以下 `qpow` 使用了**二进制快速幂**（详细原理见第 7 篇）和 `__int128` 避免乘法的 64 位溢出。

```cpp
// 模快速幂：计算 a^b mod m（利用 __int128 防溢出）
ll qpow(ll a, ll b, ll m) {
    ll res = 1;
    while (b) {
        if (b & 1) res = (__int128)res * a % m;
        a = (__int128)a * a % m;
        b >>= 1;
    }
    return res;
}

// Miller-Rabin 单轮测试（底数 a）
bool millerRabin(ll n, ll a) {
    ll d = n - 1, r = 0;
    while (!(d & 1)) d >>= 1, r++;    // n-1 = d * 2^r, d 为奇数
    ll x = qpow(a, d, n);
    if (x == 1 || x == n - 1) return true;
    for (int i = 0; i < r - 1; i++) {
        x = (__int128)x * x % n;
        if (x == n - 1) return true;   // 在平方链中遇到 -1，通过
    }
    return false;                      // 始终未遇到 -1，不通过
}

// 确定性 Miller-Rabin：覆盖整个 long long 范围
bool isPrime(ll n) {
    if (n <= 1) return false;
    // long long 范围内的确定性底数集合
    const ll bases[] = {2, 325, 9375, 28178, 450775, 9780504, 1795265022};
    for (ll a : bases) {
        if (n == a) return true;
        if (n % a == 0) return false;   // 先排除 a 能整除 n 的情况
        if (!millerRabin(n, a)) return false;
    }
    return true;
}
```

使用示例：

```cpp
cout << isPrime(998244353) << endl;        // 1 (true)  — NTT 常用素数
cout << isPrime(1000000000000000003LL) << endl;  // 1 (true)  — 10^18+3 是素数
cout << isPrime(221) << endl;              // 0 (false) — 221 = 13*17
```

## 五、复杂度与正确性分析

| 算法 | 时间复杂度 | 适用 $n$ | 正确性 |
|------|-----------|----------|--------|
| 试除法 | $O(\sqrt{n})$ | $\le 10^{12}$ | 100% 确定 |
| Miller-Rabin | $O(k \log n)$ | $\le 10^{18}$ | 对 long long 范围内 100% 确定（选定底数集合） |

**试除法**：最坏情况下 $i$ 从 $5$ 到 $\sqrt{n}$，步长为 $6$，循环 $\approx \sqrt{n}/6$ 次。正确性由 3.2 节定理保证。

**Miller-Rabin**：每轮测试中，$d$ 的提取是 $O(\log n)$，`qpow` 是 $O(\log d) = O(\log n)$，平方链是 $O(r) = O(\log n)$。总共 $k$ 轮（$k = 7$ 对于 long long 范围），总复杂度 $O(k \log n)$。

正确性方面：对于 $n < 2^{64}$，底数集合 $\{2, 325, 9375, 28178, 450775, 9780504, 1795265022\}$ 已被[严格证明](https://miller-rabin.appspot.com/)不会产生误判——即 $n$ 是素数当且仅当它通过全部 7 个底数的 Miller-Rabin 测试。

## 六、典型应用场景

1. **大整数素性判定**：$n$ 可达 $10^{18}$，Miller-Rabin 是竞赛中的标准做法。第 5 篇筛法处理的是「筛出一批小素数」，而 Miller-Rabin 处理的是「判定一个具体的、可能很大的数」。

2. **质因数分解的前置步骤**：Pollard-Rho 算法分解大合数时，需要先用 Miller-Rabin 判断是否已经得到素因子（第 6 篇）。

3. **密码学背景题**：RSA 等算法依赖大素数生成，这类题目中 Miller-Rabin 是核心组件。

**推荐练习题**（仅涉及已学知识：模运算 + GCD + 裴蜀定理 + 素数判定）：

- [洛谷 P5736](https://www.luogu.com.cn/problem/P5736) — 质数筛（入门版）。用试除法判断 $\le 10^5$ 的数是否素数，$O(\sqrt{n})$ 可过
- [洛谷 B2138](https://www.luogu.com.cn/problem/B2138) — 最大质因子序号。先判定素数，再找最大质因子
- [SPOJ PON](https://www.spoj.com/problems/PON/) — Prime or Not。$n \le 2^{63}-1$，需要 Miller-Rabin

## 七、小结

本篇的核心是从「暴力枚举」到「$\sqrt{n}$ 优化」再到「对数级判定」的完整认知链条：

1. 试除法做了两件事：把范围从 $n$ 降到 $\sqrt{n}$，再用 $6k\pm1$ 形式砍掉 $2/3$ 的检查量。本质上是「排除不可能存在小因子的数」。
2. Miller-Rabin 则完全不同——它不枚举因子，而是通过代数性质（费马+二次探测）反向验证素性。这使得复杂度从指数级跳跃到对数级。

下一篇是**素数（下）：筛法**——我们已经学会了「判定一个数」，接下来要解决的是「一次性找出 $1$ 到 $n$ 中所有的素数」。两种筛法——埃氏筛和欧拉线性筛——将让你以 $O(n \log \log n)$ 甚至 $O(n)$ 的效率批量生成素数表。
