# 裴蜀定理

> Made by Aki.
> 最后更新于 2026.04.29

上一篇我们学会了求 $\gcd(a, b)$。现在我们来回答一个更深的问题：$\gcd(a, b)$ 这个数字，除了"最大公约数"之外，还有什么含义？答案将指引我们走入整数线性组合的世界。

---

## 一、引言

想象你有两个水壶，容量分别是 $3$ 升和 $5$ 升。你可以装满、倒空、互相倒水。请问：能否量出恰好 $4$ 升水？

这个问题本质上是在问：是否存在整数 $x, y$，使得 $3x + 5y = 4$？

这里 $x, y$ 可以是任意整数（正数代表装水的净次数，负数代表倒水的净次数）。如果存在这样的 $x, y$，那我们就说 $4$ 可以用 $3$ 和 $5$「线性组合」出来。

一般地：给定两个整数 $a, b$，哪些整数 $c$ 可以写成 $ax + by = c$（$x, y$ 为整数）？答案由一个极其优美的定理给出——本篇的主角：**裴蜀定理（Bézout's Identity）**。

## 三、核心推导

### 3.1 裴蜀定理——陈述

> **裴蜀定理（Bézout's Identity）**：对于任意不全为零的整数 $a, b$，存在整数 $x, y$，使得
>
> $$ax + by = \gcd(a, b)$$

换句话说，$\gcd(a, b)$ 是 $a$ 和 $b$ 的所有线性组合中能达到的**最小正整数**。

**一个关键推论**：方程 $ax + by = c$ 有整数解的充要条件是 $\gcd(a, b) \mid c$。

为什么这个推论成立？如果 $\gcd(a, b) = g$，那么 $ax + by$ 一定是 $g$ 的倍数（因为 $g \mid a$ 且 $g \mid b$，所以 $g \mid (ax + by)$）。反过来，如果 $g \mid c$，令 $c = g \cdot k$，只需把 $ax_0 + by_0 = g$ 的特解 $(x_0, y_0)$ 乘以 $k$ 即可得到 $c$ 的解。

回到引言的水壶问题：$3$ 和 $5$ 互质，$\gcd(3, 5) = 1$，所以**任意整数**都能被组合出来。当然包括 $4$。具体怎么组合？需要扩展欧几里得算法来算出 $x, y$——那是第 8 篇的内容。

---

### 3.2 证明——最小正元素法

这个证明的核心思想非常漂亮：**不直接构造 $x, y$，而是通过"最小性"来论证。**

**第一步：定义集合。**

考虑 $a$ 和 $b$ 的所有整数线性组合构成的集合：

$$
S = \{ax + by \mid x, y \in \mathbb{Z}\}
$$

显然 $S$ 包含正整数（比如 $a \cdot 1 + b \cdot 0 = a$，如果 $a > 0$；如果 $a < 0$ 则取 $-a$）。所以 $S$ 中存在最小正整数，设为 $d$。

$$
d = \min\{s \in S \mid s > 0\}
$$

因为 $d \in S$，所以存在某组 $x_0, y_0$ 使得 $d = a x_0 + b y_0$。

**第二步：证明 $d$ 是 $a$ 和 $b$ 的公约数。**

以 $a$ 为例（$b$ 的证明完全相同）。用带余除法：

$$
a = dq + r,\quad 0 \le r < d
$$

将 $d = a x_0 + b y_0$ 代入：

$$
\begin{aligned}
r &= a - dq \\
  &= a - (a x_0 + b y_0) q \\
  &= a(1 - x_0 q) + b(-y_0 q)
\end{aligned}
$$

所以 $r$ 也是 $a$ 和 $b$ 的线性组合！即 $r \in S$。

但 $d$ 是 $S$ 中的最小正整数，而 $0 \le r < d$，因此唯一的可能只能是 $r = 0$。

所以 $a = dq$，即 $d \mid a$。同理 $d \mid b$。

**第三步：证明 $d$ 是最大的。**

假设 $d'$ 是 $a$ 和 $b$ 的任意公约数。那么 $d' \mid a$ 且 $d' \mid b$，于是 $d' \mid (a x_0 + b y_0) = d$。所以 $d' \le d$。

这就证明了 $d = \gcd(a, b)$，且存在 $x_0, y_0$ 使得 $a x_0 + b y_0 = d$。

**这个证明的精妙之处在于**：我们从未显式地"算出" $x_0, y_0$，只证明了它们**存在**。这在数学中叫做**非构造性证明**。当然，在实践中我们需要把它们算出来——这是扩展欧几里得算法的任务。

❓ **为什么要学一个"存在但不知道怎么算"的定理？** 因为在竞赛中，大量问题的**判定**只依赖于"有没有解"，而不需要"解是什么"。比如：

- 水壶问题：只问"能不能量出 $z$ 升"，不需要知道具体操作步骤
- 硬币问题：只问"能否恰好凑出 $N$ 元"，不需要知道用几枚硬币
- 同余方程 $ax \equiv b \pmod{m}$ 有没有解？只需要检查 $\gcd(a, m) \mid b$

这些场合，**裴蜀定理一条判断就够了，不需要 exgcd**。当我们确实需要求出 $x, y$ 的具体值时，再用第 8 篇的扩展欧几里得算法。

### 3.3 扩展到 $n$ 个数

裴蜀定理可以自然扩展到 $n$ 个整数：

> 对于不全为零的整数 $a_1, a_2, \dots, a_n$，存在整数 $x_1, x_2, \dots, x_n$，使得
>
> $$ a_1 x_1 + a_2 x_2 + \cdots + a_n x_n = \gcd(a_1, a_2, \dots, a_n)$$

证明思路：对 $n$ 用数学归纳法。设 $g_{n-1} = \gcd(a_1, \dots, a_{n-1})$。由归纳假设，$g_{n-1}$ 是前 $n-1$ 个数的线性组合。再对 $g_{n-1}$ 和 $a_n$ 应用二元裴蜀定理，即得。

这个扩展形式的一个直接应用：

> 给定 $n$ 个整数 $A_1, A_2, \dots, A_n$，求 $S = \sum A_i X_i > 0$ 的最小值（$X_i$ 为任意整数）。

根据多变量裴蜀定理，所有可能的 $S$ 值构成 $\gcd(|A_1|, |A_2|, \dots, |A_n|)$ 的整数倍。所以最小正整数就是这些绝对值的 $\gcd$。——这就是[洛谷 P4549 裴蜀定理](https://www.luogu.com.cn/problem/P4549)模板题。

---

### 3.4 通解结构

如果 $(x_0, y_0)$ 是 $ax + by = c$ 的一个特解，那么所有整数解可以表示为：

$$
\begin{cases}
x = x_0 + \dfrac{b}{g} \cdot t \\[8pt]
y = y_0 - \dfrac{a}{g} \cdot t
\end{cases}
\quad (t \in \mathbb{Z})
$$

其中 $g = \gcd(a, b)$。

**推导**：设 $(x, y)$ 是任意一组解。由 $ax + by = c$ 和 $ax_0 + by_0 = c$ 相减：

$$
a(x - x_0) + b(y - y_0) = 0
$$

即 $a(x - x_0) = -b(y - y_0)$。两边除以 $g$（记 $a' = a/g$，$b' = b/g$，则 $\gcd(a', b') = 1$）：

$$
a'(x - x_0) = -b'(y - y_0)
$$

因为 $\gcd(a', b') = 1$，所以 $b' \mid (x - x_0)$。令 $x - x_0 = b' \cdot t$，代入得 $y - y_0 = -a' \cdot t$。这就是通解公式。

这个公式告诉我们：如果有解，**就有无穷多组解**，它们按模 $b/g$（对 $x$）和模 $a/g$（对 $y$）的周期均匀分布。

---

### 3.5 几何视角——整数格点上的直线

方程 $ax + by = c$ 在二维平面上是一条直线。通解公式说明，这条直线上的整数点（格点）以向量 $(b/g, -a/g)$ 为步长均匀分布：

```
    y
    ↑
    |   ·   ·   ·   ·   ← 这些格点间距为 (b/g, -a/g)
    |     ·   ·   ·   ·
    |       ·   ·   ·   ·
    ———+———————————————→ x
    |
```

这个视角在后续学习扩展欧几里得算法时会非常有用——exgcd 本质上就是在直线上找到离原点最近的那个整数点。

## 四、代码实现

本文提供判断有解性的模板。具体求解 $x, y$ 的算法（扩展欧几里得）属于第 8 篇。

### 4.1 判断 $ax + by = c$ 是否有整数解

```cpp
using ll = long long;

// 判断 ax + by = c 是否有整数解
bool has_solution(ll a, ll b, ll c) {
    ll g = gcd(abs(a), abs(b));
    return c % g == 0;
}
```

### 4.2 二元一次不定方程（基础解法框架）

```cpp
// 求 ax + by = c 的一组整数解（仅当 gcd(a,b) | c 时有效）
// 使用扩展欧几里得算法——具体实现在第 8 篇
// 这里给出调用框架，后续会填充实现细节
ll exgcd(ll a, ll b, ll &x, ll &y);  // 声明，将在第 8 篇定义

bool solve_diophantine(ll a, ll b, ll c, ll &x, ll &y) {
    ll g = gcd(abs(a), abs(b));
    if (c % g != 0) return false;  // 裴蜀定理：无解
    // exgcd 求出 ax' + by' = g 的一组解 (x', y')
    exgcd(abs(a), abs(b), x, y);
    x *= c / g;  // 缩放得到 ax + by = c 的特解
    y *= c / g;
    if (a < 0) x = -x;
    if (b < 0) y = -y;
    return true;
}
```

### 4.3 $n$ 个数的裴蜀定理（洛谷 P4549 模板）

```cpp
// 求满足 Σ A_i X_i > 0 的最小正值
// 答案就是所有 |A_i| 的 gcd
ll bezout_n(vector<ll> &a) {
    ll g = 0;
    for (ll v : a) g = gcd(g, abs(v));
    return g;
}
```

使用示例：

```cpp
ll a = 3, b = 5, c = 4;
cout << has_solution(a, b, c) << endl;  // 1 (true)，因为 gcd(3,5)=1 | 4

a = 6, b = 10, c = 7;
cout << has_solution(a, b, c) << endl;  // 0 (false)，因为 gcd(6,10)=2 ∤ 7
```

## 五、复杂度与正确性分析

`has_solution` 的核心操作是 `gcd(abs(a), abs(b))`，时间复杂度 $O(\log \min(|a|, |b|))$——即辗转相除法的复杂度。

正确性由 3.2 节的最小正元素证明保证。3.1 节的推论（$ax + by = c$ 有解当且仅当 $g \mid c$）是裴蜀定理的直接结论：必要性因为 $g$ 整除任何线性组合，充分性因为特解的 $k$ 倍仍为解。

$n$ 变量版本的 `bezout_n` 对 $n$ 个数依次求 $\gcd$，时间复杂度 $O(n \log M)$，其中 $M = \max |A_i|$。

## 六、典型应用场景

裴蜀定理在竞赛中的作用可以概括为一句话：**"有没有解"的终极判据。**

1. **不定方程判定**：$ax + by = c$ 有没有整数解？不用解出来，只需检查 $\gcd(a, b) \mid c$。这是所有不定方程问题的基础。

2. **同余方程判定**：$ax \equiv b \pmod{m}$ 有没有解？等价于 $ax + my = b$ 有没有整数解，条件为 $\gcd(a, m) \mid b$。这条规则将在第 12 篇完整展开。

3. **可达性问题**：能否从状态 A 经过若干步操作到达状态 B？如果每步操作相当于加或减若干值，那么可达的条件就是 $\gcd$ 整除差值。

4. **$n$ 个数可表示的值**：给定 $A_1, \dots, A_n$，所有形如 $\sum A_i X_i$ 的整数值恰好是 $\gcd(|A_1|, \dots, |A_n|)$ 的所有整数倍。

**推荐练习题**（仅涉及已学知识：模运算 + GCD + 裴蜀定理判定）：

- [洛谷 P4549](https://www.luogu.com.cn/problem/P4549) — 【模板】裴蜀定理，求 $n$ 个数的最小正线性组合值。核心代码仅 4 行
- [Codeforces 1055C](https://codeforces.com/problemset/problem/1055/C) — Lucky Days，用裴蜀定理对齐两个循环区间，求最大重叠。考察 $\gcd(t_a, t_b)$ 在周期对齐中的应用
- [洛谷 P5253](https://www.luogu.com.cn/problem/P5253) — 丢番图，判断 $ax + by = c$ 在 $\ge 0$ 条件下的解的存在性

## 七、小结

裴蜀定理的核心是一句话：**$\gcd(a, b)$ 是 $a$ 和 $b$ 能线性组合出的最小正整数**。

它本身是一个"存在性"定理——它告诉你解存在，但没告诉你解在哪里。在竞赛中，它让你在不需要实际求解的情况下完成判定。当你真正需要求出 $x, y$ 的具体值时——这就是下一篇要铺垫、第 8 篇将完整展开的内容：

$$
ax + by = \gcd(a, b)
$$

的那组 $(x, y)$ 究竟怎么求？我们改日再会。
