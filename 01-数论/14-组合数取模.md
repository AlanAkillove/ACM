# 组合数取模

> Made by Aki.
> 最后更新于 2026.04.30

$\binom{n}{k}$ 是竞赛中最常见的数学对象之一。但 $n, k$ 一大，直接算阶乘就溢出了。本篇系统总结在不同模数条件下求 $\binom{n}{k} \bmod p$ 的方法。

---

## 一、引言

组合数 $\binom{n}{k} = \frac{n!}{k!(n-k)!}$ 在没有取模的情况下，$n=20$ 时 $\binom{20}{10} = 184756$ 还算正常，$n=100$ 时就达到 $10^{29}$ 级别——翻遍 C++ 的任何一个整数类型都放不下。

所以竞赛中的组合数问题几乎总是带模数的。根据 $n, k, p$ 的大小关系和 $p$ 的素性，需要不同的处理策略。

## 三、核心推导

### 3.1 情况分类总览

组合数取模的关键变量是 $n$ 的范围和模数 $p$ 的性质：

| 情况 | $n$ 范围 | $p$ | 方法 |
|------|----------|-----|------|
| A | $\le 10^6$ | 任意素数 | 预处理阶乘 + 逆元 |
| B | $\le 10^{18}$ | $p$ 为小素数 | Lucas 定理 |
| C | $\le 10^{18}$ | $p$ 为素数幂 $p^e$ | 扩展 Lucas |
| D | $\le 10^{18}$ | 模合数 | 分解为素数幂因子 + CRT 合并 |

下面逐一展开。

---

### 3.2 情况 A：预处理阶乘 + 逆元

当 $n \le 10^6$（或 $10^7$）且 $p$ 为素数（如 $10^9+7$ 或 $998244353$）时，可以直接预处理所有阶乘和阶乘的逆元。

**阶乘**：$fact[i] = i! \bmod p$

**阶乘逆元**：$inv\_fact[i] = (i!)^{-1} \bmod p$。可以用费马法求出 $inv\_fact[n]$，然后倒推：

$$
inv\_fact[i-1] = inv\_fact[i] \times i \bmod p
$$

因为 $(i-1)!^{-1} \equiv (i!)^{-1} \times i \pmod{p}$。（这等价于 $\frac{1}{(i-1)!} = \frac{i}{i!}$。）

**单次查询**：

$$
\binom{n}{k} \equiv fact[n] \times inv\_fact[k] \times inv\_fact[n-k] \pmod{p}
$$

每次 $O(1)$。预处理 $O(n)$。

---

### 3.3 情况 B：Lucas 定理——大 $n$、小素数 $p$

当 $n, k$ 达到 $10^{18}$ 级别但 $p$ 较小（如 $p \le 10^5$）时，阶乘数组开不到 $n$。Lucas 定理将大组合数递归地分解为小组合数：

> **Lucas 定理**：
>
> $$
> \binom{n}{k} \equiv \prod_{i=0}^{\infty} \binom{n_i}{k_i} \pmod{p}
> $$
>
> 其中 $n_i, k_i$ 分别是 $n, k$ 在 $p$ 进制下的第 $i$ 位。

**推导思路**（二项式展开法）：在模 $p$ 下，$(1+x)^p \equiv 1 + x^p \pmod{p}$（因为中间的二项式系数 $\binom{p}{i}$ 都是 $p$ 的倍数，对于 $0 < i < p$）。将 $n$ 写成 $p$ 进制 $n = n_0 + n_1 p + n_2 p^2 + \cdots$：

$$
(1+x)^n = \prod_j (1+x)^{n_j p^j} \equiv \prod_j (1+x^{p^j})^{n_j} \pmod{p}
$$

比较 $x^k$ 的系数即得 Lucas 公式。

**实际使用**：递归地拆解：

```cpp
ll lucas(ll n, ll k, ll p) {
    if (k == 0) return 1;
    return C(n % p, k % p, p) * lucas(n / p, k / p, p) % p;
}
```

其中 $C(a, b, p)$ 是 $a, b < p$ 的小组合数，可以直接用阶乘+逆元（情况 A）求解（因为 $a, b < p \le 10^5$）。

---

### 3.4 情况 C：扩展 Lucas——模数为素数幂 $p^e$

当模数 $m = p^e$（$p$ 为素数，$e > 1$）时，阶乘中可能包含因子 $p$，使得 $p$ 成为 $\frac{n!}{k!(n-k)!}$ 的分母——逆元不存在。

思路：**将组合数中的素数因子 $p$ 单独提取**，剩下的部分与 $p$ 互质，可以用逆元处理。

设 $n! = p^{v_p(n!)} \times f(n)$，其中 $f(n)$ 与 $p$ 互质（Legendre 公式给出 $v_p$）。则：

$$
\binom{n}{k} = p^{A} \times \frac{f(n)}{f(k) \cdot f(n-k)}
$$

其中 $A = v_p(n!) - v_p(k!) - v_p((n-k)!)$。

$f(n)$ 有递推公式（基于 $n$ 的 $p$ 进制表示），可以在 $O(p^e)$ 预处理后 $O(\log_p n)$ 查询。详细推导和代码较长，竞赛中常作为黑盒模板使用——这里给出关键思路，完整模板可在学习完本篇后查阅 exLucas 专题资料。

❓ **什么时候需要 exLucas？** 模数可能是 $10^9+7$ 吗？不会——因为 $10^9+7$ 是素数，情况 A 和 B 就够了。exLucas 的典型场景是模数等于 $10^9$ 这种合数，或题目明确给出模数不是素数时。

---

### 3.5 情况 D：模合数——分解 + CRT

当模数 $m$ 是合数（如 $m = 10^9$）时，将 $m$ 分解为素数幂的乘积：

$$
m = p_1^{e_1} p_2^{e_2} \cdots p_t^{e_t}
$$

对每个 $p_i^{e_i}$ 用扩展 Lucas 求 $\binom{n}{k} \bmod p_i^{e_i}$，再用 CRT 合并。

### 3.6 库默尔定理（Kummer's Theorem）

作为一个独立的数学洞察，库默尔定理给出了组合数中素数 $p$ 的幂次的组合解释：

> $v_p\left(\binom{n}{k}\right)$ 等于在 $p$ 进制下做 $n-k$ 与 $k$ 的加法时产生的**进位次数**。

它本质上等价于 Legendre 公式，但提供了「组合数是否被 $p$ 整除」的快速直观判断。例如：$n=10, k=1$ 时 $\binom{10}{1}=10$，$p=5$ 进制下 $10-1=9$ 加 $1$ 不产生进位，所以 $v_5(10) = 1$，有一个因子 $5$——与 $\binom{10}{1}=10$ 吻合。

## 四、代码实现

### 4.1 预处理阶乘 + 逆元（情况 A）

```cpp
struct Combinatorics {
    vector<ll> fact, inv_fact;
    ll p;

    Combinatorics(int n, ll p) : p(p) {
        fact.resize(n + 1);
        inv_fact.resize(n + 1);
        fact[0] = 1;
        for (int i = 1; i <= n; i++)
            fact[i] = fact[i - 1] * i % p;
        inv_fact[n] = qpow(fact[n], p - 2, p);     // 费马求逆
        for (int i = n; i >= 1; i--)
            inv_fact[i - 1] = inv_fact[i] * i % p;  // 倒推
    }

    ll C(int n, int k) {
        if (k < 0 || k > n) return 0;
        return fact[n] * inv_fact[k] % p * inv_fact[n - k] % p;
    }
};
```

### 4.2 Lucas 定理（情况 B）

```cpp
ll C_small(ll n, ll k, ll p) {   // 小组合数（n, k < p），直接公式
    if (k > n) return 0;
    ll num = 1, den = 1;
    for (int i = 1; i <= k; i++) {
        num = num * (n - i + 1) % p;
        den = den * i % p;
    }
    return num * qpow(den, p - 2, p) % p;
}

ll lucas(ll n, ll k, ll p) {
    if (k == 0) return 1;
    return C_small(n % p, k % p, p) * lucas(n / p, k / p, p) % p;
}
```

### 4.3 完整框架

```cpp
// 根据 n 和 p 的关系自动选择方法
ll C(ll n, ll k, ll p) {
    if (k < 0 || k > n) return 0;
    if (n < p) return C_small(n, k, p);   // 情况 A（无需预处理时）
    return lucas(n, k, p);                 // 情况 B
}
```

使用示例：

```cpp
Combinatorics comb(1000000, 1000000007);
cout << comb.C(100, 50) << endl;           // C(100,50) mod 1e9+7

cout << lucas(1000000000000LL, 500000000000LL, 10007) << endl;
// 大 n, k，小素数 p，用 Lucas
```

## 五、复杂度与正确性分析

| 方法 | 预处理 | 单次查询 | $n$ 范围 |
|------|--------|----------|----------|
| 阶乘+逆元 | $O(n)$ | $O(1)$ | $n \le 10^7$ |
| Lucas | $O(p)$ | $O(\log_p n)$ | $n \le 10^{18}$ |
| 扩展 Lucas | $O(p^e)$ | $O(\log_p n)$ | $n \le 10^{18}$ |

Lucas 定理的正确性来自二项式展开 $(1+x)^p \equiv 1+x^p \pmod{p}$ 在模素数下的性质。阶乘逆元倒推的正确性来自 $\frac{1}{(i-1)!} \equiv \frac{i}{i!} \pmod{p}$，即 $(i-1)!^{-1} \equiv (i!)^{-1} \times i \pmod{p}$——验证：两边乘以 $(i-1)!$ 得 $1 \equiv i \times (i!)^{-1} \times (i-1)! \equiv i \times (i! \cdot (i-1)!)^{-1} \equiv 1$。

## 六、典型应用场景

组合数取模是竞赛中的高频操作。除了直接的「求 $\binom{n}{k} \bmod p$」题型，它还广泛隐藏在：

1. **Catalan 数计算**：$C_n = \frac{1}{n+1}\binom{2n}{n}$，需要组合数取模。
2. **概率与期望**：二项分布、超几何分布的概率计算常归约为组合数取模。
3. **DP 优化**：网格路径计数（$\binom{n+m}{n}$）、多重集组合计数。

**推荐练习题**：

- [洛谷 P2822](https://www.luogu.com.cn/problem/P2822) — 组合数问题（NOIP2016）。求 $\binom{i}{j} \equiv 0 \pmod{k}$ 的个数，二维前缀和 + 阶乘预处理
- [洛谷 P3807](https://www.luogu.com.cn/problem/P3807) — 【模板】Lucas 定理。$n,m \le 10^{18}, p \le 10^5$ 的 Lucas 模板
- [洛谷 P4720](https://www.luogu.com.cn/problem/P4720) — 【模板】扩展 Lucas。模合数 $p^e$ 情况的 exLucas

## 七、小结

组合数取模的策略完全由 $n$ 和 $p$ 的大小关系决定。竞赛中最常见的两种是「阶乘预处理 + 逆元」（模 $10^9+7$ 类素数的标配）和「Lucas 定理」（$n$ 极大、$p$ 较小）。exLucas 和模合数分解场景出现较少但作为模板储备是必要的。
