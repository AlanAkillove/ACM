# 类欧几里得算法

> Made by Aki.
> 最后更新于 2026.04.30

算法名带「欧几里得」是有原因的——它和辗转相除法共享同一个递归结构。但它的目标不是求 $\gcd$，而是以 $O(\log n)$ 的速度计算 $\sum \lfloor (ai+b)/c \rfloor$ 这类看似需要 $O(n)$ 暴力的求和式。

---

## 一、引言

来看一道 AtCoder 上的经典题：

> 给定 $n, a, b, c$（$\le 10^9$），求 $\displaystyle\sum_{i=0}^{n-1} \left\lfloor \frac{ai + b}{c} \right\rfloor$。

暴力循环 $n$ 次？$n = 10^9$ 直接超时。但类欧几里得算法可以在 $O(\log \max(a, c))$ 内精确求解——对 $10^9$ 级别的输入只需要大约 $30$ 步。

这个算法在竞赛中虽然不如筛法或快速幂那样无处不在，但它是**亚线性时间处理整除求和**的标准工具，在杜教筛和莫比乌斯反演的整除分块中也有重要的结构对应。

## 三、核心推导

### 3.1 问题定义

定义函数（通常记作 $f$ 或 likegcd）：

$$
f(n, a, b, c) = \sum_{i=0}^{n-1} \left\lfloor \frac{ai + b}{c} \right\rfloor
$$

其中 $a, b, c$ 和 $n$ 均为非负整数，$c > 0$。

**目标**：用与辗转相除法同构的递归结构，在 $O(\log \max(a, c))$ 时间内求出 $f$。

---

### 3.2 递归化简——分情形讨论

**情形 I：$a = 0$**

当 $a=0$ 时，求和式中每一项都是 $\lfloor b/c \rfloor$，共 $n$ 项：

$$
f(n, 0, b, c) = n \cdot \left\lfloor \frac{b}{c} \right\rfloor
$$

这是递归基。

**情形 II：$a \ge c$ 或 $b \ge c$**

将 $\lfloor (ai+b)/c \rfloor$ 分离出整数部分：

$$
\left\lfloor \frac{ai+b}{c} \right\rfloor
= \left\lfloor \frac{(c \cdot \lfloor a/c \rfloor + a \bmod c)i + (c \cdot \lfloor b/c \rfloor + b \bmod c)}{c} \right\rfloor
$$

整数部分的贡献可以直接提出求和号外：

$$
f(n, a, b, c) = \frac{n(n-1)}{2} \left\lfloor \frac{a}{c} \right\rfloor
              + n \left\lfloor \frac{b}{c} \right\rfloor
              + f(n, a \bmod c, b \bmod c, c)
$$

这样就把 $a, b$ 的范围压缩到了 $< c$，向递归基逼近。

**情形 III：$a < c$ 且 $b < c$（核心转化）**

此时每一项 $\lfloor (ai+b)/c \rfloor$ 都 $< 1+i$（因为 $a < c$ 且 $b < c$，所以 $(ai+b)/c < i+1$）。值域很小，但 $n$ 很大——这正是我们要突破的地方。

设 $m = \left\lfloor \frac{a(n-1) + b}{c} \right\rfloor$——这是整个求和式中最大的一项。将求和按「值」（即每一项取到的整数 $k$）重新组织：

$$
\begin{aligned}
f(n, a, b, c)
&= \sum_{i=0}^{n-1} \sum_{k=1}^{\lfloor (ai+b)/c \rfloor} 1 \\
&= \sum_{k=1}^{m} \sum_{\substack{0 \le i < n \\ \lfloor (ai+b)/c \rfloor \ge k}} 1 \\
&= \sum_{k=1}^{m} \left( n - \left\lceil \frac{ck - b}{a} \right\rceil \right)_{\ge 0}
\end{aligned}
$$

其中 $(\cdot)_{\ge 0}$ 表示与 $0$ 取 $\max$。令 $j = m - k$，翻转求和顺序——这个转化不太直观，但最终收敛到一个优雅的递归形式：

$$
f(n, a, b, c) = n \cdot m - f(m, c, c - b - 1, a)
$$

**这是整个算法的核心**——一次参数互换（$a \leftrightarrow c$ 的角色互换），问题规模从 $(n, a, b, c)$ 缩小为 $(m, c, c-b-1, a)$。而 $m \le \frac{a(n-1)+b}{c} < n$（因为 $a < c$ 和 $b < c$），所以新问题的 $n$ 参数**严格减小**。

---

### 3.3 与辗转相除法的同构

将情形 II 和 III 交替使用：

- 情形 II 对应辗转相除法中的取模（$a \to a \bmod c$，$b \to b \bmod c$）
- 情形 III 对应辗转相除法中的参数交换（$a \leftrightarrow c$）

两者交替的效果是：**每一步都使较大的参数被更小的参数替换**，恰好与辗转相除法的递归结构同构。因此总递归深度也是 $O(\log \max(a, c))$。

---

### 3.4 扩展形式

类欧几里得算法不仅可以求解基础的 $\sum \lfloor (ai+b)/c \rfloor$，还可以扩展到更复杂的形式：

- $g(n, a, b, c) = \sum i \cdot \lfloor (ai+b)/c \rfloor$
- $h(n, a, b, c) = \sum \lfloor (ai+b)/c \rfloor^2$

这三种形式共享同一套递归框架——在情形 II 中分别分离出 $i$ 的系数和常数项，在情形 III 中进行类似的求和翻转。代码中通常将三者打包在一个递归函数中，一次调用算出所有结果。

## 四、代码实现

### 4.1 基础版：$\sum \lfloor (ai+b)/c \rfloor$

```cpp
using ll = long long;

ll likegcd(ll n, ll a, ll b, ll c) {
    if (a == 0) return n * (b / c);
    if (a >= c || b >= c)
        return n * (n - 1) / 2 * (a / c)
             + n * (b / c)
             + likegcd(n, a % c, b % c, c);
    ll m = (a * (n - 1) + b) / c;
    return n * m - likegcd(m, c, c - b - 1, a);
}
```

### 4.2 扩展版：三种求和一次求出

```cpp
struct Result { ll f, g, h; };

Result likegcd_ext(ll n, ll a, ll b, ll c) {
    if (a == 0) {
        ll t = b / c;
        return {n * t, n * (n - 1) / 2 * t, n * t * t};
    }
    if (a >= c || b >= c) {
        auto [f, g, h] = likegcd_ext(n, a % c, b % c, c);
        ll a1 = a / c, b1 = b / c;
        return {
            f + n*(n-1)/2 * a1 + n * b1,
            g + n*(n-1)*(2*n-1)/6 * a1 + n*(n-1)/2 * b1,
            h + 2*a1*g + 2*b1*f + a1*a1 * n*(n-1)*(2*n-1)/6
                + a1*b1 * n*(n-1) + b1*b1 * n
        };
    }
    ll m = (a * (n - 1) + b) / c;
    auto [f, g, h] = likegcd_ext(m, c, c - b - 1, a);
    return {
        n * m - f,
        (n * m * (n - 1) - 2 * (g + f)) / 2,
        n * m * m - 2 * g - f
    };
}
```

使用示例：

```cpp
// 计算 ∑_{i=0}^{9} ⌊(3i + 2) / 5⌋
cout << likegcd(10, 3, 2, 5) << endl;   // 30

// 扩展版：同时求 f, g, h
auto [f, g, h] = likegcd_ext(10, 3, 2, 5);
// f = Σ⌊(3i+2)/5⌋, g = Σ i·⌊...⌋, h = Σ⌊...⌋²
```

## 五、复杂度与正确性分析

递归的每一步，情形 II 将 $a, b$ 压缩到 $< c$（与辗转相除法的取模对应），情形 III 将 $n$ 替换为 $m < n$（因为 $m = \lfloor (a(n-1)+b)/c \rfloor < n$ 当 $a < c, b < c$ 时）。递归深度严格 $O(\log \max(a, c))$，每层 $O(1)$。

正确性由情形 II 的代数分离和情形 III 的求和重排保证。情形 III 的推导（双重计数 + 顺序翻转）是全文最困难的部分，但一旦接受 $f(n, a, b, c) = n \cdot m - f(m, c, c-b-1, a)$，后续递归就只是机械套用。

## 六、典型应用场景

1. **整除求和**：直接求 $\sum \lfloor (ai+b)/c \rfloor$——如二维平面上直线下方的整数点计数。
2. **模求和的亚线性计算**：$\sum (ai+b) \bmod c = \sum (ai+b) - c \cdot \sum \lfloor (ai+b)/c \rfloor$，归约到类欧。
3. **杜教筛的辅助计算**：类欧的递归结构与整除分块天然兼容，在第 22 篇杜教筛中会看到两者的协作。

## 七、小结

类欧几里得算法是一个「参数互换」的优美实例。情形 II（分离整数部分）和情形 III（双重计数+翻转）交替使用，恰好对应辗转相除法的取模和参数交换。这一结构的通用性远超本系列的范畴——它最终通向 Stern-Brocot 树和连分数理论（第 20 篇）。
