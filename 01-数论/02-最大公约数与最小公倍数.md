# 最大公约数与最小公倍数

> Made by Aki.
> 最后更新于 2026.04.29

你可能在小学数学课上学过「最大公约数」和「辗转相除法」。但当时学的是怎么算——这一篇要回答的是：**为什么这样算是对的，以及它的效率有多惊人**。

---

## 一、引言

先来看一道题：

> 给定两个正整数 $a, b$（$1 \le a, b \le 10^{18}$），求它们的最大公约数。

如果没有任何算法知识，你可能会这样做：从 $\min(a, b)$ 开始往下枚举，找到第一个能同时整除 $a$ 和 $b$ 的数。这个做法的时间复杂度是 $O(\min(a, b))$——当 $a, b$ 达到 $10^{18}$ 时，即使每秒枚举 $10^9$ 个数，也需要三十多年才能跑完。

而欧几里得在公元前 300 年发明的算法，可以在**不到 100 步**内求出答案——无论数字多大。这正是本文的主角：**辗转相除法（欧几里得算法）**。

本文将从整除的基本性质出发，完整推导辗转相除法的正确性和复杂度，同时介绍它的变种 Stein 算法，以及 GCD 与 LCM 之间的关系。

## 三、核心推导

### 3.1 定义——先说清楚我们要算什么

对于两个不全为零的整数 $a, b$，**最大公约数**（Greatest Common Divisor，GCD）是能同时整除 $a$ 和 $b$ 的最大正整数，记作 $\gcd(a, b)$。

例如：$\gcd(12, 18) = 6$，因为 $6$ 能整除 $12$ 和 $18$，且没有比 $6$ 更大的正整数满足这个条件。

特别地：
- $\gcd(a, 0) = |a|$（任何非零整数都整除 $0$）
- $\gcd(a, 1) = 1$（$1$ 整除一切）

**最小公倍数**（Least Common Multiple，LCM）是能同时被 $a$ 和 $b$ 整除的最小正整数，记作 $\operatorname{lcm}(a, b)$。

例如：$\operatorname{lcm}(12, 18) = 36$。

---

### 3.2 辗转相除法的核心洞察

辗转相除法基于一个极其简洁的性质：

> $$\gcd(a, b) = \gcd(b, a \bmod b)$$

**这是整个算法的基石，我们来做完整的证明。**

设 $d = \gcd(a, b)$。我们要证明两件事：

**(1) $d$ 也是 $b$ 和 $a \bmod b$ 的公约数。**

由带余除法，$a = bq + r$，其中 $r = a \bmod b$，$0 \le r < b$。

因为 $d \mid a$ 且 $d \mid b$，所以 $d \mid (a - bq)$，即 $d \mid r$。

又因为 $d \mid b$，所以 $d$ 同时整除 $b$ 和 $r$——即 $d$ 是 $b$ 和 $r$ 的公约数。

**(2) $b$ 和 $r$ 的任何公约数 $d'$ 不会大于 $d$。**

假设 $d' \mid b$ 且 $d' \mid r$。由于 $a = bq + r$，可得 $d' \mid a$。所以 $d'$ 也是 $a$ 和 $b$ 的公约数。而 $d$ 是 $a$ 和 $b$ 的**最大**公约数，所以 $d' \le d$。

由 (1) 和 (2)：$d$ 是 $b$ 和 $r$ 的最大公约数。证毕。

**直觉上**：如果 $d$ 能整除 $a$ 和 $b$，那它一定能整除 $a - bq$（换句话说，$a$ 减去若干个 $b$ 不影响它是否被 $d$ 整除）。而 $a \bmod b$ 恰好就是 $a$ 减去尽可能多个 $b$ 后剩下的部分。所以 $\gcd(a, b) = \gcd(b, a \bmod b)$ 在直觉上是"显然的"。

---

### 3.3 为什么一定会结束？——归纳原理

反复应用 3.2 的性质：

$$
\gcd(a, b) = \gcd(b, a \bmod b) = \gcd(a \bmod b, b \bmod (a \bmod b)) = \cdots
$$

每经过一步，参与运算的两个数中较大的那个**严格减小**。因为每一步取模后，余数 $r$ 满足 $0 \le r < b$，而下一轮我们用 $r$ 代替了原来的 $b$。

由于正整数不能无限减小，经过有限步之后必然出现：

$$
\gcd(g, 0) = g
$$

此时 $g$ 就是最初的 $\gcd(a, b)$。这个过程就是**辗转相除法**。

---

### 3.4 手算一个例子

以 $\gcd(252, 105)$ 为例：

| 步骤 | $a$ | $b$ | $a \bmod b$ |
|------|-----|-----|--------------|
| 第 1 步 | 252 | 105 | $252 \bmod 105 = 42$ |
| 第 2 步 | 105 | 42 | $105 \bmod 42 = 21$ |
| 第 3 步 | 42 | 21 | $42 \bmod 21 = 0$ |
| 第 4 步 | 21 | 0 | 结束 |

所以 $\gcd(252, 105) = 21$。

只用 3 步就求出了答案。如果从 105 往下枚举，你需要试到 21，也就是 84 步——差了很多。当数字变成 $10^{18}$ 级别时，差距是天文数字。

---

### 3.5 复杂度分析——为什么是 $O(\log n)$？

**这是竞赛中最重要的结论之一：辗转相除法的时间复杂度是 $O(\log \min(a, b))$。**

证明思路：在连续两步中，数字至少减半。

设我们正在计算 $\gcd(a, b)$，其中 $a \ge b > 0$。经过两步：

- 第一步：$\gcd(a, b) \to \gcd(b, a \bmod b)$
- 第二步：$\gcd(b, a \bmod b) \to \gcd(a \bmod b, b \bmod (a \bmod b))$

设 $r = a \bmod b$。如果 $r \le b/2$，那 $r$ 已经不超过 $b$ 的一半。如果 $r > b/2$，那么下一步 $b \bmod r = b - r < b/2$。

所以**每经过两步，较大的数至少减半**。因此总步数不超过 $2 \times \log_2(\min(a, b))$。对于 $10^{18}$ 级别的数，$\log_2 10^{18} \approx 60$，所以最多约 120 步——与实际观察一致。

❓ **问**：如果 $a$ 和 $b$ 是相邻的斐波那契数（比如 $\gcd(F_{n+1}, F_n)$），会发生什么？

这是辗转相除法的**最坏情况**。因为 $F_{n+1} \bmod F_n = F_{n-1}$，每一步只减去一个，需要 $n$ 步。而 $F_n \approx \frac{\varphi^n}{\sqrt{5}}$，所以 $n = O(\log \min(a, b))$ 仍然成立——复杂度上界没有变差。实际上 Fibonacci 相邻项正是这个最坏情况，这也是为什么复杂度分析中用到了 $\varphi$（黄金分割比）。

---

### 3.6 GCD 与 LCM 的关系——一个公式打通

两个正整数的 GCD 和 LCM 之间有一个极其有用的关系：

$$
\gcd(a, b) \times \operatorname{lcm}(a, b) = a \times b
$$

**推导**：设 $g = \gcd(a, b)$，令 $a = g \cdot a'$，$b = g \cdot b'$，其中 $\gcd(a', b') = 1$（$a'$ 和 $b'$ 互质，否则 $g$ 就不是**最大**公约数了）。

那么 $\operatorname{lcm}(a, b) = g \cdot a' \cdot b'$。因为 LCM 必须包含 $a$ 的所有素因子（即 $g$ 和 $a'$）以及 $b$ 的所有素因子（即 $g$ 和 $b'$），且 $a'$ 和 $b'$ 无重叠。所以：

$$
\gcd(a, b) \times \operatorname{lcm}(a, b) = g \times (g \cdot a' \cdot b') = g^2 \cdot a' \cdot b' = (g \cdot a') \times (g \cdot b') = a \times b
$$

这意味着：**只要知道 GCD，LCM 可以直接用公式算出来**，不需要单独设计算法。

---

### 3.7 $n$ 个数的 GCD 和 LCM

对于 $n$ 个数 $a_1, a_2, \dots, a_n$，GCD 和 LCM 满足结合律：

$$
\gcd(a_1, a_2, \dots, a_n) = \gcd(\gcd(a_1, \dots, a_{n-1}), a_n)
$$

$$
\operatorname{lcm}(a_1, a_2, \dots, a_n) = \operatorname{lcm}(\operatorname{lcm}(a_1, \dots, a_{n-1}), a_n)
$$

所以只需从前往后逐个合并，时间复杂度 $O(n \log M)$，其中 $M = \max(a_i)$。

实现时注意：$\operatorname{lcm}$ 的计算可能溢出。正确的做法是先除后乘：

```cpp
ll lcm(ll a, ll b) {
    return a / gcd(a, b) * b;  // 先除后乘，避免溢出
}
```

❓ **为什么先除后乘就安全？** 因为 $\gcd(a, b)$ 整除 $a$，所以 $a / \gcd(a, b)$ 是一个整数，且不超过 $a$。然后乘以 $b$ 时，如果最终 LCM 本身在 `ll` 范围内，就不会溢出。如果 LCM 超出 `ll` 范围，那无论如何都会溢出，这是题目本身数据过大的问题。

---

### 3.8 Stein 算法——不用取模的 GCD

辗转相除法依赖取模运算。虽然取模在现代 CPU 上很快，但在某些场景（如大整数库、不支持除法的环境），可以用 Stein 算法（也叫**二进制 GCD**），只依赖减法、移位和判奇偶：

> 设 $a, b$ 为正整数：
> - 若 $a = b$：返回 $a$
> - 若 $a, b$ 均为偶数：$\gcd(a, b) = 2 \times \gcd(a/2, b/2)$
> - 若 $a$ 为偶数、$b$ 为奇数：$\gcd(a, b) = \gcd(a/2, b)$
> - 若 $a$ 为奇数、$b$ 为偶数：同理
> - 若 $a, b$ 均为奇数（设 $a > b$）：$\gcd(a, b) = \gcd((a-b)/2, b)$

后三种情况都保证了数字在减小（要么除以 2，要么相减后除以 2），复杂度同样是 $O(\log \min(a, b))$。在 ACM 竞赛中，辗转相除法是首选，Stein 算法作为补充了解即可。

## 四、代码实现

### 4.1 辗转相除法（递归版）

```cpp
using ll = long long;

ll gcd(ll a, ll b) {
    return b == 0 ? a : gcd(b, a % b);
}
```

一行核心逻辑。递归深度不超过 $O(\log n)$，不会**爆栈**。

### 4.2 辗转相除法（迭代版）

```cpp
ll gcd_iter(ll a, ll b) {
    while (b != 0) {
        a %= b;
        swap(a, b);
    }
    return a;
}
```

### 4.3 LCM

```cpp
ll lcm(ll a, ll b) {
    return a / gcd(a, b) * b;  // 先除后乘防溢出
}
```

### 4.4 $n$ 个数的 GCD

```cpp
ll gcd_n(vector<ll> &v) {
    ll g = v[0];
    for (int i = 1; i < v.size(); i++)
        g = gcd(g, v[i]);
    return g;
}
```

### 4.5 Stein 算法（二进制 GCD）

```cpp
ll gcd_stein(ll a, ll b) {
    if (a == 0) return b;
    if (b == 0) return a;
    int shift = __builtin_ctzll(a | b);  // 共有因子 2 的个数
    a >>= __builtin_ctzll(a);            // 去掉 a 的所有因子 2
    b >>= __builtin_ctzll(b);            // 去掉 b 的所有因子 2
    while (a != b) {
        if (a > b) swap(a, b);
        b -= a;
        b >>= __builtin_ctzll(b);        // b 变为偶数，去掉因子 2
    }
    return a << shift;                   // 乘回之前提取出来的因子 2
}
```

使用示例：

```cpp
ll a = 252, b = 105;
cout << gcd(a, b) << endl;       // 21
cout << lcm(a, b) << endl;       // 1260
cout << gcd_stein(a, b) << endl; // 21
```

## 五、复杂度与正确性分析

| 算法 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 辗转相除法（递归/迭代） | $O(\log \min(a, b))$ | $O(1)$ |
| Stein 算法 | $O(\log \min(a, b))$ | $O(1)$ |
| $n$ 个数的 GCD | $O(n \log M)$ | $O(1)$ |

辗转相除法的正确性由 3.2 节的证明保证——每一步保持 $\gcd(a, b)$ 不变，且数字严格减小直到一个变成 $0$。复杂度在 3.5 节已证明：每两步至少减半。

Stein 算法的正确性可以逐条验证：公因子 $2$ 被提取，奇数的差是偶数从而可以不断右移，最终必然收敛到 $a = b$。

## 六、典型应用场景

GCD 和 LCM 在竞赛中几乎无处不在。以下是几个最典型的用法：

1. **约分**：将分数 $\frac{a}{b}$ 化为最简分数，分子分母同时除以 $\gcd(a, b)$。

2. **判断两个数是否互质**：$\gcd(a, b) = 1$ 则互质。这是很多数论问题的前置条件（例如逆元的存在性、欧拉函数的计算，将在后续篇章中遇到）。

3. **模运算中除法的基础**：模意义下的除法需要乘法逆元，而逆元存在的条件正是 $\gcd(a, m) = 1$。这个条件贯穿了整个模运算体系，从第 9 篇开始会频繁出现。

4. **周期性问题**：两个运动的周期分别为 $a$ 和 $b$，它们的重合周期是 $\operatorname{lcm}(a, b)$。

**推荐练习题**（仅涉及已学知识：模运算 + GCD/LCM）：

- [洛谷 B3736](https://www.luogu.com.cn/problem/B3736) — 求三个数的最大公约数，本题的代码模板足矣
- [Codeforces 664A](https://codeforces.com/problemset/problem/664/A) — Complicated GCD，考察 $\gcd(a, a+1, \dots, b)$ 的规律，有有意思的思考过程
- [洛谷 P1029](https://www.luogu.com.cn/problem/P1029) — 已知 $\gcd$ 和 $\operatorname{lcm}$，求满足条件的数对个数。核心思想是 $\gcd \times \operatorname{lcm} = a \times b$

## 七、小结

本文的核心是一个性质和一个算法：

**性质**：$\gcd(a, b) = \gcd(b, a \bmod b)$，它让"找最大公约数"从暴力枚举变成了**对数**级别的减半过程。

**算法**：辗转相除法用 3 行代码实现了 $O(\log n)$ 的 GCD 计算，加上 $\gcd \times \operatorname{lcm} = a \times b$ 这个公式，LCM 也可以一并求出。这两个工具将伴随你完成整个 ACM 竞赛数学的学习。

下一篇是**裴蜀定理**——我们将看到 $\gcd(a, b)$ 不仅仅是一个数字，它还能写成 $a$ 和 $b$ 的整数线性组合：$ax + by = \gcd(a, b)$。这个看似简单的事实，是整个不定方程体系的起点。
