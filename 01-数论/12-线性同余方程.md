# 线性同余方程

> Made by Aki.
> 最后更新于 2026.04.30

裴蜀定理告诉你方程有没有解，exgcd 把解算出来。本篇将它们合并成一个完整的工具箱，系统求解 $ax \equiv b \pmod{m}$。

---

## 一、引言

第 8 篇用 exgcd 解了 $ax + my = 1$。但在竞赛中更常见的需求是：

> 解 $ax \equiv b \pmod{m}$。如果有解，输出最小正整数解；如果无解，输出无解。

这就是**线性同余方程**——形式简单，但包含了同余理论最核心的分析方法：存在性判定、解的数量、通解结构。

## 三、核心推导

### 3.1 转化为不定方程

$$
ax \equiv b \pmod{m}
$$

等价于存在整数 $y$ 使得：

$$
ax - b = my \quad\iff\quad ax - my = b
$$

令 $y' = -y$，得标准形式：

$$
ax + my' = b
$$

这正是不定方程 $Ax + By = C$ 的形式。第 3 篇和第 8 篇已经分别解决了它的存在性和求解问题。

---

### 3.2 有解条件与解的数量

由裴蜀定理：$ax + my = b$ 有整数解当且仅当 $\gcd(a, m) \mid b$。

设 $g = \gcd(a, m)$。当 $g \mid b$ 时，方程在模 $m$ 意义下恰好有 **$g$ 个不同的解**。

❓ **为什么恰好 $g$ 个？** 由第 3 篇的通解公式，$x$ 的解形如 $x = x_0 + \frac{m}{g} \cdot t$。在模 $m$ 下，$t = 0, 1, \dots, g-1$ 给出 $g$ 个互不同余的解。第 $g$ 个解 $x_0 + m$ 与 $x_0$ 同余，故不产生新解。

**特例**：当 $\gcd(a, m) = 1$ 时（$a$ 与 $m$ 互质），方程有**唯一解**——这就是第 9 篇中求逆元的场景（$b=1$）。

---

### 3.3 完整求解步骤

1. 计算 $g = \gcd(a, m)$。
2. 若 $b \bmod g \neq 0$，无解，返回。
3. 用 exgcd 解 $a x_0 + m y_0 = g$。
4. 将特解放大：$x_1 = x_0 \cdot (b / g)$。此时 $a x_1 \equiv b \pmod{m}$。
5. 通解：$x_k \equiv x_1 + k \cdot \frac{m}{g} \pmod{m}$，$k = 0, 1, \dots, g-1$。

**找最小非负解**：在通解中取 $k = 0$ 得到 $x_1$，然后用 `(x1 % (m/g) + m/g) % (m/g)` 校正到最小非负剩余。

---

### 3.4 模线性方程组——多个同余方程的初步处理

有时我们需要同时解多个不同的同余方程，例如：

$$
\begin{cases}
a_1 x \equiv b_1 \pmod{m_1} \\
a_2 x \equiv b_2 \pmod{m_2}
\end{cases}
$$

处理的基本策略是：**先各自解成 $x \equiv r \pmod{m}$ 的形式，再合并**。合并的技术就是第 13 篇的 CRT。本篇先完成「转化为最简同余式」这一步——当 $\gcd(a_i, m_i) \mid b_i$ 时，方程 $a_i x \equiv b_i \pmod{m_i}$ 可以转化为 $x \equiv r_i \pmod{m_i'}$（其中 $m_i' = m_i / \gcd(a_i, m_i)$）。多个这样的同余式就可以送入 CRT 处理。

## 四、代码实现

```cpp
using ll = long long;

// 求解 ax ≡ b (mod m)，返回所有模 m 意义下的不同解
// 若无解，返回空 vector
vector<ll> solve_congruence(ll a, ll b, ll m) {
    ll g = gcd(abs(a), abs(m));
    if (b % g != 0) return {};           // 裴蜀定理：无解

    ll x0, y0;
    exgcd(abs(a), abs(m), x0, y0);       // 解 a*x0 + m*y0 = g

    x0 = x0 * (b / g);                   // 缩放
    if (a < 0) x0 = -x0;
    ll step = abs(m) / g;                // x 的周期

    vector<ll> res;
    x0 = (x0 % step + step) % step;      // 最小非负解
    for (int k = 0; k < g; k++)
        res.push_back(x0 + k * step);    // 所有 g 个不同解
    return res;
}

// 更常用的简化版：只返回最小非负解（对 g=1 就是模逆元）
ll solve_congruence_min(ll a, ll b, ll m) {
    auto sols = solve_congruence(a, b, m);
    return sols.empty() ? -1 : sols[0];
}
```

使用示例：

```cpp
// 7x ≡ 5 (mod 12)
auto sols = solve_congruence(7, 5, 12);
// gcd(7,12)=1，只有一个解：x=11
// 验证：7*11 = 77 ≡ 5 (mod 12) ✓

// 6x ≡ 4 (mod 10)
sols = solve_congruence(6, 4, 10);
// gcd(6,10)=2 | 4，有 2 个解：x=4, x=9
// 验证：6*4=24≡4, 6*9=54≡4 (mod 10) ✓

// 6x ≡ 3 (mod 10)
sols = solve_congruence(6, 3, 10);
// gcd(6,10)=2 ∤ 3，无解 → 返回 {}
```

## 五、复杂度与正确性分析

整个求解过程的核心是 exgcd，复杂度 $O(\log m)$。后续的缩放和取模是 $O(1)$。输出所有 $g$ 个解需要 $O(g)$，在最坏情况下 $g$ 可达 $O(m)$（但此时每个解都需要输出，这是无法避免的）。

正确性由裴蜀定理（存在性）和 exgcd（构造性）联立保证，通解结构由第 3 篇 3.4 节给出。

---

## 六、典型应用场景

1. **求模逆元**（特例）：$ax \equiv 1 \pmod{m}$ 的解集。当 $g=1$ 时唯一。
2. **CRT 的前置步骤**：多方程场景中，先将每个 $a_i x \equiv b_i$ 化为 $x \equiv r_i$ 的标准形式。
3. **线性同余方程组**：多个 $a_i x \equiv b_i$ 合并为 CRT 求解。

**推荐练习题**：

- [洛谷 P1082](https://www.luogu.com.cn/problem/P1082) — 同余方程 $ax \equiv 1 \pmod{b}$，本篇最小解的特例
- [洛谷 P1516](https://www.luogu.com.cn/problem/P1516) — 青蛙的约会。转化为 $x(m-n) \equiv (y-x) \pmod{L}$ 后直接套用 `solve_congruence`

---

## 七、小结

线性同余方程的本质是**将同余语言翻译成不定方程，再用 exgcd 求解，然后回译为模同余解**。当 $\gcd(a,m)=1$ 时退化为模逆元；当 $b=0$ 时退化为求解 $ax \equiv 0 \pmod{m}$，解为 $x$ 是 $m/\gcd(a,m)$ 的倍数。

下一篇是**中国剩余定理**——我们将处理「多个同余方程同时成立」的问题，这门技术是韩信点兵的数学本质，也是 CRT 和 exCRT 的核心内容。
