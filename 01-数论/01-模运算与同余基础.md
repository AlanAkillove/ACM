# 模运算与同余基础

> Made by Aki.
> 最后更新于 2026.04.29

这是 ACM 竞赛数学系列的第一篇。我们从最基础也最核心的概念开始——模运算。几乎所有竞赛数学题的代码里都会有 `%` 这个符号，但你真的理解它在做什么吗？

---

## 一、引言

来看一个 Codeforces 上的经典场景：

> 给定三个整数 $a, b, m$（$1 \le a, b \le 10^9$，$1 \le m \le 10^9$），求 $(a \times b) \bmod m$。

你可能会想：直接写 `(a * b) % m` 不就行了？

但如果 $a = 10^9$，$b = 10^9$，那么 $a \times b = 10^{18}$，恰好还在 `long long` 范围内，勉强能过。那如果 $a, b \le 10^{18}$ 呢？直接乘就溢出了。

这个简单的问题引出了 ACM 竞赛中一个贯穿始终的主题：**在有限的整数范围内，如何处理可能无限大的数学结果**。而模运算，就是解决这个问题的核心工具。

本文作为系列的第一篇，将从最基础的概念建起——带余除法、模运算、同余、剩余类——为后续所有数论专题打下地基。即使你之前对模运算只有「取余数」的模糊印象，读完这篇也应该能回答三个问题：

1. 模运算为什么可以"先取模再算"？
2. 负数取模在 C++ 里有什么坑？怎么解决？
3. 「同余」不是「相等」，那它到底有什么用？

---

## 二、前置知识

- C++ 基本语法（会写 `%` 运算符即可）
- 整除概念（$a$ 能被 $b$ 整除）

不需要任何数论基础。

## 三、核心推导

### 3.1 带余除法——所有模运算的起点

在整数范围内，我们熟悉的除法其实是**带余除法**：

> 对于任意整数 $a$ 和正整数 $m$，存在唯一的整数 $q$ 和 $r$，使得
>
> $$a = mq + r,\quad 0 \le r < m$$
>
> 其中 $q$ 叫做**商**，$r$ 叫做**余数**。

这是一个非常重要的定理——它保证了「取模」这个操作的**良定义性**（well-definedness）。无论 $a$ 是多少，$r$ 都是唯一确定的。

我们记：

$$
a \bmod m = r
$$

在 C++ 中就是 `a % m`。

**❓ 初学者容易困惑的点**：$a$ 可以是负数吗？可以。比如 $a = -10$，$m = 3$，按定义：

$$
-10 = 3 \times (-4) + 2
$$

所以余数 $r = 2$，而不是 $-1$。但 C++ 中 `-10 % 3` 的结果是 $-1$。这个坑我们后面会专门讲。

---

### 3.2 模运算的基本性质——为什么可以"先取模再算"

这是竞赛中最常用的技巧：面对一个大数运算，先对所有操作数取模，算完再取模，结果不变。

具体来说，对于加法、减法、乘法，下面的等式成立：

**(1) 模加法**

$$
(a + b) \bmod m = ((a \bmod m) + (b \bmod m)) \bmod m
$$

**(2) 模减法**

$$
(a - b) \bmod m = ((a \bmod m) - (b \bmod m)) \bmod m
$$

**(3) 模乘法**

$$
(a \times b) \bmod m = ((a \bmod m) \times (b \bmod m)) \bmod m
$$

这三个等式是整个模运算体系的基石。我们以加法为例做一次完整推导，另外两个的思想完全一样。

**推导（模加法）**：

设 $a = mq_1 + r_1$，$b = mq_2 + r_2$，其中 $0 \le r_1, r_2 < m$。

那么：

$$
a + b = m(q_1 + q_2) + (r_1 + r_2)
$$

这里 $r_1 + r_2$ 可能 $\ge m$，所以我们需要再做一次带余除法：

$$
r_1 + r_2 = mq_3 + r_3,\quad 0 \le r_3 < m
$$

于是：

$$
a + b = m(q_1 + q_2 + q_3) + r_3
$$

因此 $(a + b) \bmod m = r_3$。

而 $(a \bmod m) + (b \bmod m) = r_1 + r_2$，再取模 $m$ 也是 $r_3$。等式成立。

**直觉理解**：取模相当于把数轴绕在一个周长为 $m$ 的圆上。加法就是在这个圆上沿着顺时针走。你在圆上的位置（余数）只取决于你走了多少步，跟你之前绕了多少圈无关。

**关键推论**：对于加、减、乘，我们可以在运算的**任意时刻**取模，不影响最终结果。这意味着我们可以在每一步都保持数字在 $[0, m)$ 范围内，永远不用担心溢出。

**❓ 除法去哪了？** 注意到我没有写「模除法」。因为**除法不满足这个性质**。例如：

$$
(8 \div 4) \bmod 3 = 2 \bmod 3 = 2
$$

但 $((8 \bmod 3) \div (4 \bmod 3)) \bmod 3 = (2 \div 1) \bmod 3 = 2$，这个恰好对了；换个例子：$(6 \div 2) \bmod 5 = 3$，但 $(1 \div 2) \bmod 5$ 在整数范围内根本没有意义。除法的处理需要引入**乘法逆元**，这是第 9 篇的内容。

### 3.3 同余——"模 $m$ 意义下相等"

有了模运算，我们引入一个更抽象和强大的概念：**同余**。

> 如果 $m \mid (a - b)$，即 $a$ 和 $b$ 除以 $m$ 的余数相同，我们称 $a$ 与 $b$ 模 $m$ **同余**，记作
>
> $$a \equiv b \pmod{m}$$

同余关系有三个基本性质，它们让"模 $m$ 意义下相等"这个说法在数学上是合法的：

**自反性**：$a \equiv a \pmod{m}$。（显然）

**对称性**：若 $a \equiv b \pmod{m}$，则 $b \equiv a \pmod{m}$。（$a - b$ 能被 $m$ 整除，那 $b - a$ 也能）

**传递性**：若 $a \equiv b \pmod{m}$ 且 $b \equiv c \pmod{m}$，则 $a \equiv c \pmod{m}$。（$a - b$ 和 $b - c$ 都能被 $m$ 整除，那它们的和 $(a - c)$ 也能）

这三个性质使同余成为一个**等价关系**。这意味着我们可以把全体整数按照模 $m$ 的同余关系划分成若干个**等价类**——这就是剩余类的来源。

---

### 3.4 负数取模——C++ 的暗坑

这是 ACM 新手最容易踩的坑。在 C++ 中：

```cpp
int a = -10;
int m = 3;
cout << a % m;  // 输出 -1，而不是 2
```

C++ 的 `%` 运算符遵循「向零取整」原则：$-10 \div 3 = -3$（向零方向取整），然后 $-10 - 3 \times (-3) = -1$。

但数学上我们期望的余数应该是非负的。这在我们使用模运算性质（比如算完减法后取模）时会出问题：

```cpp
// 你期望这样不会出错：
int result = (a - b) % m;  // 但如果 a < b，结果是负数！
```

**解决方案**：在每次取模后，如果结果为负，加上 $m$ 即可。

```cpp
int safe_mod(int a, int m) {
    return (a % m + m) % m;
}
```

这个写法保证返回值在 $[0, m)$ 范围内。虽然做了两次取模，但取模运算非常快，可以忽略不计。

**更进一步的理解**：在数论中，我们通常把「$a \bmod m$」定义为满足 $0 \le r < m$ 的那个 $r$，即**最小非负剩余**。C++ 的 `%` 在 $a$ 为负时给出的是**最小绝对值剩余**（向零取整）。两者在 $a$ 为正时一致，在 $a$ 为负时差了一个 $m$。所以 `(a % m + m) % m` 的本质就是把 C++ 的余数校正为数学定义。

---

### 3.5 剩余类与完全剩余系——整数被「分组」了

既然同余是一种等价关系，模 $m$ 就把全体整数分成了恰好 $m$ 个等价类：

$$
\begin{aligned}
[][0]&= \{\dots, -2m, -m, 0, m, 2m, \dots\} \\
[1] &= \{\dots, -2m+1, -m+1, 1, m+1, 2m+1, \dots\} \\
&\;\;\vdots \\
[m-1] &= \{\dots, -m-1, -1, m-1, 2m-1, \dots\}
\end{aligned}
$$

这些都是**模 $m$ 的剩余类**（residue class）。每个剩余类中的任意两个数模 $m$ 同余。

从每个剩余类中各选一个代表，组成的集合叫做**完全剩余系**。最常用的就是 $\{0, 1, 2, \dots, m-1\}$，即**最小非负完全剩余系**。

这个概念有什么用？在竞赛题中，我们经常需要统计「模 $m$ 的余数为 $r$ 的数有多少个」。本质上就是在按剩余类对数据进行分组。例如：

> 给定 $n$ 个数，问有多少对数的**和**能被 $m$ 整除。

把每个数按其模 $m$ 的余数分类。如果两个数分别来自剩余类 $[r]$ 和 $[m-r]$，它们的和就属于剩余类 $[0]$。计数问题瞬间变成了组合问题。

这个视角——**将无限范围内的整数问题转化为有限个剩余类的问题**——是数论处理模相关问题的核心思维。

---

### 3.6 取模时机与溢出防范

在写竞赛代码时，有三个实用的经验法则：

**法则一：加法 / 减法 / 乘法在每一步之后都可以取模。**

```cpp
// 计算 a*b + c*d mod m
ll ans = (a % m) * (b % m) % m;
ans = (ans + (c % m) * (d % m) % m) % m;
```

**法则二：当两个数都在 $[0, m)$ 内时，它们的加法结果在 $[0, 2m-2]$ 内，可能超过 $m$ 但不会超过 $2m$。** 此时如果 $m < 2^{32}$ 或 $m < 2^{63}$，可以直接用 `if` 代替取模来加速：

```cpp
ll add_mod(ll a, ll b, ll m) {
    a += b;
    if (a >= m) a -= m;
    return a;
}
```

这个特化版本在需要极高性能时有用（比如计算 $10^7$ 次模加法），因为它只用了一次分支判断，比 `%` 运算符快。但日常刷题用 `%` 足够了。

**法则三：乘法溢出是真正的杀手。** 当 $m$ 较大（接近 $10^9$）且两个数也接近 $m$ 时，$(a \bmod m) \times (b \bmod m)$ 可能达到 $10^{18}$，超过 32 位 int（$2.1 \times 10^9$）但没有超过 64 位 long long（$9.2 \times 10^{18}$）。所以：

```cpp
// 用 long long 存储中间结果
using ll = long long;
ll a, b, m;
ll result = (a % m) * (b % m) % m;  // 安全，中间结果在 ll 范围内
```

但如果 $m$ 更大（比如 $10^{12}$），两个在 $[0, m)$ 内的数相乘会达到 $10^{24}$，超过 long long。这时需要**慢速乘**（也叫龟速乘），将在第 7 篇快速幂中一并讲解。

## 四、代码实现

以下是本篇所有核心操作的代码模板。

```cpp
#include <bits/stdc++.h>
using ll = long long;

// 安全取模：保证结果在 [0, m) 内（处理负数情况）
ll safe_mod(ll a, ll m) {
    return (a % m + m) % m;
}

// 模加法
ll add_mod(ll a, ll b, ll m) {
    return (a % m + b % m) % m;
}

// 模减法（已处理负数）
ll sub_mod(ll a, ll b, ll m) {
    return ((a % m - b % m) % m + m) % m;
}

// 模乘法
ll mul_mod(ll a, ll b, ll m) {
    return (a % m) * (b % m) % m;
}
```

使用示例：

```cpp
// 计算 (1234567890123 * 9876543210987) % 1000000007
ll a = 1234567890123LL, b = 9876543210987LL, m = 1000000007;
ll ans = mul_mod(a, b, m);  // ans = 484838507
```

## 五、复杂度与正确性分析

取模运算 `%` 在现代 CPU 上是一条除法指令（`idiv`），时间复杂度可以视为 $O(1)$。虽然除法指令比加减乘慢，但在 OI/ACM 竞赛中，除非你在 $10^7$ 级别的大循环中反复取模，否则不需要担心取模本身的开销。

上述四个函数的时间复杂度都是 $O(1)$。正确性由 3.2 节的推导保证——模加法/减法/乘法的「先取模再算」性质已经在数学上被证明。

## 六、典型应用场景

模运算和同余在竞赛中的出现频率极高，几乎所有输出「答案模 $10^9+7$」的题目都依赖本文的基础知识。以下是几个典型的应用模式：

1. **大数组合计数**：计算 $\binom{n}{k} \bmod p$，由于组合数增长极快（$n=100$ 时就能超过 64 位），必须用模运算分段处理。涉及逆元、Lucas 定理等（后文专题讲解）。

2. **哈希与判重**：字符串哈希、集合哈希等算法使用大质数取模来将无限空间映射到有限空间。

3. **周期性问题**：很多数学问题的答案具有周期性，通过模运算可以发现和利用周期。例如：求 $2^n$ 的个位数字，等价于求 $2^n \bmod 10$。

4. **同余约束问题**：题目给出一组同余方程，要求求解。例如「求一个最小的正整数 $x$，使得 $x \bmod 3 = 2$，$x \bmod 5 = 3$，$x \bmod 7 = 2$」——这就是中国剩余定理的经典场景（第 13 篇）。

**推荐练习题**（仅涉及已学知识：模运算、同余、取模操作）：

- [洛谷 B2008](https://www.luogu.com.cn/problem/B2008) — 取模基础练习，直接考察 `%` 运算
- [Codeforces 4C](https://codeforces.com/problemset/problem/4/C) — Registration System，用余数做简单分类统计
- [洛谷 P1427](https://www.luogu.com.cn/problem/P1427) — 小鱼航程，取模处理周期性问题

## 七、小结

本文建立了 ACM 竞赛数学最底层的基础设施：**模运算**把无限整数世界压缩到 $[0, m)$ 的有限圆环上，**同余**让我们在这个圆环上谈论"相等"，**剩余类**则提供了分组计数的视角。

三个核心要点：

1. 加、减、乘的模运算满足「先取模再算，结果不变」，这让我们能安全处理大数。
2. C++ 的 `%` 对负数会返回负值，需要用 `(a % m + m) % m` 校正。
3. 同余将整数分成 $m$ 个剩余类——这是将无限转化为有限的思维利器。

下一篇是**最大公约数与最小公倍数**——我们将学习欧几里得算法，一个两千年前的算法，至今仍是竞赛中最优雅也最常用的工具。
